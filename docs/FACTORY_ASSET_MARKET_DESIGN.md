# Economy 生产设施、生产与资产交易设计

> 状态：核心生产资产专题基线  
> 当前客户端：网页端  
> 服务器约束：2 核 2G  
> 更新时间：2026-07-10

本文件描述生产设施的建造、生产、升级、估值与交易规则。身份、页面结构和总资产规则以 `WEB_MULTIPLAYER_GAME_DESIGN.md` 为准。

## 1. 设计目标

生产设施是玩家生产基础商品的必要资产。玩家不能在没有设施的情况下直接生产商品。

玩家获得设施有两种方式：

1. 使用货币建造新的生产设施。
2. 在共享市场购买其他玩家挂牌出售的现成设施。

```text
自建设施：成本由系统确定，需要等待施工完成
购买设施：价格由卖家确定，成交后立即交割
```

生产设施既是生产工具，也是可以持有、升级、估值和出售的投资资产。

## 2. 核心循环

```text
工作获得启动货币
-> 自建或购买生产设施
-> 设施生产基础商品
-> 玩家领取商品到个人库存
-> 在统一市场出售商品
-> 获得货币并观察价格变化
-> 升级、出售或收购生产设施
-> 调整资产组合
-> 提升总资产与排行榜名次
```

## 3. 统一市场关系

同一个市场入口处理：

- 基础商品买单
- 基础商品卖单
- 生产设施出售挂牌
- 生产设施即时购买
- 人口需求买单

允许通过筛选器查看：

- 全部市场
- 基础商品
- 生产设施
- 人口需求
- 我的订单
- 我的成交

商品和生产设施共用订单、成交、资产流水和市场状态体系，但展示方式不同：

- 标准商品使用按价格聚合的订单簿。
- 唯一设施使用挂牌卡片或列表。

## 4. 两类交易规则

### 4.1 基础商品

基础商品是可分割、可批量交易的标准资产。

采用：

- 限价买单
- 限价卖单
- 价格优先
- 同价时间优先
- 部分成交
- 多次成交
- 未成交部分撤销

### 4.2 生产设施

每座设施都是具有唯一编号的独立资产，数量固定为 1。

MVP 采用：

- 固定价格出售挂牌
- 单个设施对应一笔有效挂牌
- 买家按挂牌价格即时购买
- 不支持部分成交
- 不支持同一设施重复挂牌
- 不支持设施买入委托、竞价或拍卖

## 5. 生产设施基础属性

每座设施至少包含：

- 设施唯一编号
- 设施名称与类型
- 当前所有者
- 等级
- 状态
- 生产周期
- 单周期产量
- 单周期运营费用
- 内部商品容量
- 内部商品数量
- 当前生产进度
- 累计产量
- 建造时间
- 建造成本
- 有效升级投入
- 系统参考估值
- 当前挂牌订单编号

MVP 只开放一种“基础生产设施”和一种暂未命名的基础商品。

## 6. 设施状态

主要状态包括：

- 施工中
- 待启用
- 生产中
- 玩家暂停
- 内部库存已满
- 资金不足
- 升级中
- 挂牌中
- 已出售
- 系统暂停

同一座设施在同一时刻只能处于一个主要状态。辅助原因可以额外记录，例如“暂停：资金不足”。

## 7. 自建设施

### 7.1 建造条件

玩家建造设施前必须满足：

- 拥有足够可用货币。
- 拥有空闲设施槽位。
- 没有超过同时施工数量上限。
- 账号未处于建造或交易限制状态。

### 7.2 建造流程

```text
选择设施类型
-> 查看建造费用与施工时间
-> 确认建造
-> 扣除建造费用
-> 设施进入施工中
-> 等待施工完成
-> 设施变为待启用
-> 玩家启动生产
```

建造费用属于系统货币回收，不进入其他玩家账户。

### 7.3 施工规则

施工期间设施：

- 不能生产。
- 不能挂牌出售。
- 不能升级。
- 占用设施槽位。
- 玩家离线后施工继续。
- 刷新、多标签页和修改本地时间不能提前完工。

### 7.4 MVP 初始参数

| 项目 | 建议值 |
|---|---:|
| 基础设施建造费用 | 60 货币 |
| 基础施工时间 | 5 分钟 |
| 初始设施槽位 | 1 |
| 同时施工数量 | 1 |

数值只用于第一轮测试。

### 7.5 取消施工

MVP 可允许取消施工，但不能全额退款。

建议规则：

- 返还 70% 建造费用。
- 30% 作为施工损耗退出流通。
- 取消后释放设施槽位。

完成施工后不能再取消建造，只能运营、升级或挂牌出售。

## 8. 购买其他玩家的设施

### 8.1 即时交割

```text
买家点击购买
-> 服务器检查资金、槽位与挂牌状态
-> 扣除买家货币
-> 增加卖家货币
-> 转移设施产权
-> 完成挂牌订单
-> 写入成交和产权流水
```

以上步骤必须在同一原子事务中完成。任何一步失败，双方资产保持不变。

### 8.2 购买条件

买家必须：

- 拥有足够可用货币。
- 拥有空闲设施槽位。
- 不是当前卖家。
- 设施挂牌仍然有效。
- 账号没有处于交易限制状态。

### 8.3 保留属性

产权转移不会重置：

- 设施等级
- 生产周期
- 单周期产量
- 运营费用
- 内部容量
- 累计产量
- 已完成升级
- 历史系统估值记录

### 8.4 交易失败

以下情况必须拒绝购买：

- 买家资金不足。
- 买家槽位不足。
- 设施已被其他玩家购买。
- 卖家已撤单。
- 设施状态不允许交割。
- 请求重复提交。

## 9. 设施挂牌出售

### 9.1 订单字段

设施挂牌包含：

- 订单编号
- 资产类型：生产设施
- 设施唯一编号
- 设施名称、类型与等级
- 生产参数摘要
- 系统参考估值
- 挂牌价格
- 卖家昵称
- 创建时间
- 订单状态

数量固定为 1。

### 9.2 挂牌条件

设施挂牌前必须：

- 已完成施工。
- 不处于升级中。
- 已暂停生产。
- 没有未完成生产周期。
- 内部商品已领取或按规则处理。
- 没有未结算运营费用。
- 没有其他有效挂牌。

挂牌期间设施停止生产，直到撤单或成交。

### 9.3 挂牌价格与风险控制

玩家自行设置挂牌价格，但挂牌价格不直接计入总资产。

MVP 可将挂牌价格限制在系统参考估值的合理区间内，例如 50%～200%，用于降低异常转账和操纵风险。具体范围应通过测试调整。

系统估值不是系统回购承诺。

### 9.4 撤单

未成交的设施挂牌可以撤销。撤单后：

- 设施所有权不变。
- 设施回到暂停状态。
- 玩家可以重新启动生产或再次挂牌。

### 9.5 成交结果

成交后：

- 买家货币减少。
- 卖家货币增加。
- 设施所有权转移。
- 挂牌状态变为全部成交。
- 双方生成成交记录。
- 双方生成资金和产权流水。

设施交易只转移已有货币，不增加全服货币总量。

## 10. 生产规则

### 10.1 生产前提

设施必须：

- 已完工。
- 未挂牌。
- 未升级。
- 玩家已启动生产。
- 玩家拥有足够运营资金。
- 内部容量未满。

### 10.2 周期结算

MVP 按生产周期聚合结算。

周期结束时服务器：

1. 检查运营资金和内部容量。
2. 扣除运营费用。
3. 增加设施内部商品。
4. 更新累计产量。
5. 条件满足时开始下一周期。

### 10.3 初始参数

| 项目 | 建议值 |
|---|---:|
| 单周期时间 | 30 秒 |
| 单周期产量 | 1 个基础商品 |
| 单周期运营费用 | 1 货币 |
| 设施内部容量 | 20 个 |

### 10.4 自动暂停

以下情况自动暂停生产：

- 资金不足。
- 内部商品已满。
- 设施挂牌。
- 设施升级。
- 玩家主动暂停。
- 系统进入保护状态。

### 10.5 领取商品

基础商品先进入设施内部库存，再由玩家领取到个人库存。

个人库存空间不足时，未领取商品继续保留在设施内部；内部容量满后设施停止生产。

## 11. 设施升级

升级可以影响：

- 生产周期
- 单周期产量
- 运营费用效率
- 内部容量
- 后续商品解锁能力

升级需要：

- 支付升级费用。
- 等待升级时间。
- 暂停生产。
- 撤销有效挂牌。

有效升级投入可以作为系统估值的一部分，但不能简单按 100% 永久计价。

MVP 可以只实现一级设施，但数据结构和页面必须支持后续升级。

## 12. 设施槽位

- 初始设施槽位为 1。
- 自建和购买都占用槽位。
- 槽位不足时不能建造或购买。
- 后续通过支付系统费用扩充槽位。

设施槽位用于控制玩家生产规模和服务器常驻状态数量。

## 13. 设施系统估值

系统估值可考虑：

```text
基础建造成本
+ 有效升级投入
+ 产能价值
+ 内部商品价值
+ 折旧或使用修正
```

估值可以参考：

- 建造成本
- 设施等级
- 有效升级投入
- 周期产量与生产速度
- 运营费用效率
- 内部容量
- 内部商品参考价值
- 同类设施近期正常成交价

系统估值用于：

- 玩家总资产计算
- 排行榜
- 挂牌价格风险控制
- 收购参考
- 异常交易检查

不得直接使用玩家挂牌价作为设施估值。

## 14. 收益与回本预测

客户端可以根据当前商品参考价和设施参数展示：

```text
每小时理论产量
每小时预计运营费用
每小时预计销售收入
每小时预计净收益
预计回本时间
```

这些结果只用于预测，不属于正式资产，也不保证未来收益。

## 16. 自建与收购的策略差异

### 自建

优势：

- 系统固定成本。
- 不受市场挂牌数量影响。
- 通常初始成本较低。

劣势：

- 必须等待施工。
- 施工期间占用槽位。
- 不能立即扩大产能。

### 收购

优势：

- 立即交割。
- 无施工等待。
- 可以直接购买已升级设施。

劣势：

- 价格由玩家决定。
- 可能高于自建与升级总成本。
- 受市场供应限制。

## 17. 新手流程

1. 在主页面工作获得货币。
2. 查看市场价格和设施挂牌。
3. 选择自建基础设施。
4. 支付建造费用。
5. 等待施工完成。
6. 启动设施生产。
7. 领取第一批基础商品到个人库存。
8. 在市场创建商品卖单。
9. 与人口需求或其他玩家成交。
10. 根据收益选择升级、继续自建或收购设施。
11. 查看总资产和排行榜变化。

## 18. 防作弊与异常规则

- 设施所有权以服务器记录为准。
- 修改客户端倒计时不能提前完工。
- 同一设施不能同时卖给两名玩家。
- 同一购买请求只能成功一次。
- 扣款与产权转移必须同时成功或同时失败。
- 卖家不能购买自己的设施。
- 施工中和升级中的设施不能挂牌。
- 挂牌中的设施不能生产。
- 槽位不足时不能购买。
- 异常高低价交易必须可审计。
- 异常设施成交不得直接抬高系统估值和排行榜资产。

## 19. 2 核 2G 性能约束

- MVP 只开放一种设施类型和一种基础商品。
- 每名玩家初始只有一个设施槽位。
- 设施采用固定价挂牌，不使用实时竞价。
- 设施生产按周期聚合，不模拟机器或工人个体。
- 商品盘口按价格聚合。
- 设施挂牌采用分页列表。
- 全部市场动态限制最近记录数量。
- 排行榜不因每次生产周期实时重算。

## 20. MVP 必须实现

1. 使用货币建造生产设施。
2. 自建设施需要施工等待。
3. 购买其他玩家设施不需要等待。
4. 设施与商品进入同一个市场入口。
5. 设施和商品共用订单、成交和流水体系。
6. 商品采用限价订单簿撮合。
7. 设施采用固定价挂牌即时购买。
8. 设施订单数量固定为 1，不能部分成交。
9. 设施完工后才能生产基础商品。
10. 挂牌设施必须暂停生产。
11. 买家必须拥有空闲槽位。
12. 设施成交后立即转移产权。
13. 设施交易不增加全服货币总量。
14. 同一设施不能重复出售。
15. 刷新和断线不能重置施工时间。
16. 修改客户端时间不能提前完工。
17. 设施商品可以领取到玩家库存。
18. 设施系统估值计入玩家总资产。
19. 市场支持按商品和设施筛选。
20. 所有结果以服务器状态为准。

## 21. 暂不实现

- 设施拍卖
- 设施竞价
- 设施买入委托
- 指定玩家私下交易
- 设施股份和多人共有
- 设施租赁
- 设施抵押与贷款
- 设施破产清算
- 多种设施类型
- 设施地理位置
- 工人和设备个体模拟

## 22. 当前开发决策

- 生产设施是生产基础商品的必要资产。
- 玩家可以自建或购买其他玩家的设施。
- 自建需要等待施工，购买立即交割。
- 设施与商品在同一个共享市场入口交易。
- 两类资产共用订单、成交、流水和市场状态。
- 商品使用价格时间优先订单簿。
- 设施使用固定价格挂牌，数量固定为 1。
- 设施挂牌期间停止生产。
- 设施估值由系统计算，不采用挂牌价。
- MVP 每名玩家初始只有一个设施槽位。
- MVP 只开放一种基础设施与一种基础商品。
