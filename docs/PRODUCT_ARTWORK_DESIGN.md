# Economy 商品图片视觉设计

> 状态：当前商品主视觉、紧凑语义图标与资源验证基线
> 适用项目：`RIVERS0FT/Economy`
> 更新时间：2026-07-22
> 上位设计：`UI_DESIGN_SYSTEM.md`、`INDUSTRY_AND_PRODUCTION_DESIGN.md`

## 1. 目标与边界

商品在需要快速识别和经营游戏质感的主要视觉位置使用本地 PNG 插画；在生产公式、订单表格、资产变动、下拉选择和其他高密度位置继续使用 `ProductIcon` 的本地内联 SVG。

本设计只改变客户端视觉表达，不修改服务器商品目录、商品 ID、市场规则、仓库数量、生产公式、估值、订单或拍卖数据结构。

## 2. 唯一资源目录

正式商品图片统一位于：

```text
src/assets/product-icons/<productId>.png
```

文件名必须与服务器正式商品 ID 完全一致。每张图片必须满足：

- `1024 × 1024`；
- PNG RGBA，具有真实 Alpha 透明通道；
- 四角透明、无白底、无色键残边；
- 单一商品主体，不包含文字、数字、边框、Logo、水印或场景背景；
- 使用 `UI_DESIGN_SYSTEM.md` 规定的三分之四视角、左上暖色主光和柔和接触阴影。

不得在页面目录、`public/`、远程 CDN 或服务器响应中维护第二套商品图片。

## 3. 统一宿主与图片映射

`src/components/icons/ProductIcons.tsx` 的 `ProductIcon` 仍是商品视觉的唯一语义宿主，并通过 `data-product-icon` 暴露正式商品 ID。`src/styles/product-artwork.css` 根据该属性映射本地 PNG。

页面不得直接拼接图片路径，不得在业务组件中维护商品 ID 到图片 URL 的平行映射。未知商品 ID 没有图片映射时必须继续显示 `ProductIcon` 的统一包装箱 SVG 回退，不得隐藏未知商品。

PNG 只覆盖 SVG 的可见描边，不改变原有 DOM、文本标签、点击区域、键盘行为或读屏名称。图片本身属于装饰视觉，商品名称仍由相邻文本提供。

## 4. 正式使用位置

以下位置使用商品 PNG 主视觉：

- 生产页共享仓库商品卡；
- 市场页顶部商品资产目录；
- 概览页当前商品市场标题；
- 拍卖页商品单项主视觉；
- 拍卖资产包编辑行、资产包预览和最近结束记录中的商品缩略视觉。

以下位置必须继续使用紧凑 SVG：

- 工厂生产公式输入和输出；
- 市场订单标题、订单簿、未完成订单和本地成交表；
- 资产变动记录；
- 下拉选择器和其他不具备独立图片空间的标签；
- 所有未知商品回退。

不得为了“全部统一成图片”把 18～24px 的高密度语义位置改成 1024px PNG，也不得把商品插画用于导航、状态、货币、工厂、周期或仓库功能图标。

## 5. 尺寸与布局

- 仓库商品卡：桌面约 `54px`，移动约 `38px`；卡片必须预留稳定高度，图片加载不得造成布局跳动。
- 市场商品目录：桌面约 `40px`，移动约 `34px`；工厂目录继续使用 `FactoryIcon`，不得套用商品图片规则。
- 概览市场标题：桌面约 `42px`，移动约 `34px`。
- 拍卖单项主视觉可在 `7rem`～`11rem` 内自适应；资产包和历史记录使用对应容器内的紧凑缩略尺寸。

图片统一使用 `contain`，不得裁切主体，不得额外添加滤镜、着色、描边、强投影或位移动画。选中、悬浮和状态变化继续由卡片边框、背景和状态标签表达。

## 6. 性能与降级

市场横向商品目录必须使用 `content-visibility: auto` 和稳定的固有尺寸，减少不可见条目的绘制开销。

当浏览器启用 `prefers-reduced-data: reduce` 时，正式图片位置恢复显示原有 SVG，不加载商品 PNG 主视觉。未知商品无论是否启用节省流量模式都必须保留 SVG。

商品图片跟随 Vite 构建并使用内容哈希，不允许运行时请求外部图片服务。

## 7. 自动验证

`scripts/verify-product-artwork.mjs` 是商品图片资源的正式验证入口，并纳入 `verify:architecture` 和正式 `build`。

验证必须覆盖：

1. 31 种正式商品均存在同名 PNG；
2. 每张图片是有效 `1024 × 1024` RGBA PNG；
3. `product-artwork.css` 对每个正式商品 ID 都有唯一映射；
4. 主视觉位置、节省流量降级和 SVG 回退规则仍然存在；
5. 业务页面不得直接引用 `src/assets/product-icons/`；
6. 生产公式继续使用 `ProductIcon` SVG。

新增商品时必须同时扩展服务器目录、`ProductIcons.tsx`、商品 PNG 和验证清单。缺少任一项时构建必须失败。

## 8. 不可回退规则

- 不得恢复 Emoji、字符或字母缩写作为商品图标。
- 不得把全部商品位置无差别替换为 PNG。
- 不得绕过 `ProductIcon` 的 `data-product-icon` 宿主在页面中直接插图。
- 不得删除未知商品 SVG 回退。
- 不得删除 `prefers-reduced-data` 降级。
- 不得移除构建阶段的尺寸、RGBA、目录和映射验证。
