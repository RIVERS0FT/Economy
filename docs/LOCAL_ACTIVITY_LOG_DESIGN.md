# Economy 本地活动日志设计

> 状态：当前浏览器本地资产事件与成交记录基线
> 适用项目：`RIVERS0FT/Economy`
> 更新时间：2026-07-15
> 本地文档版本：v4
> 客户端状态版本：14
> 世界状态版本：10

## 1. 边界

服务器只保存权威经济状态，不保存玩家展示日志。以下内容由当前浏览器根据相邻权威状态生成：

- 资金和冻结资金变化；
- 商品可用与冻结库存变化；
- 仓库等级和容量变化；
- 工厂集群数量、运行意图、状态、配方和施工变化；
- 生产费用、原料消耗和产成品入仓；
- 商品与工厂订单、撤单和逐笔成交。

本地日志不参与资产、仓库、工厂、生产、订单、产权、估值或排名。

## 2. 本地存储

```text
economy.local-activity.v4.<userId>
```

每个账号保存：

- 最后一次服务器权威状态的紧凑快照；
- 最多 480 条资产事件；
- 最多 240 条成交记录。

更换设备、清除站点数据、无痕模式或存储错误时日志允许丢失，不得影响服务器状态。

## 3. 快照

快照只保存生成展示差异所需字段：

```text
credits
frozenCredits
inventories
inventoryCapacity
warehouseLevel
facilityGroups
facilityConstruction
own orders and fills
products
facilityTypes
```

不得保存：

```text
单座 facilities
factory instance id
internalGoods
internalCapacity
Cookie 或会话令牌
服务器世界 JSON
```

首次加载只建立快照，不生成事件。相同状态、幂等重试和页面重复渲染不得重复写日志。

## 4. 资产事件

客户端比较相邻状态：

1. 资金与冻结资金；
2. 商品库存；
3. 仓库等级与容量；
4. 工厂集群数量、状态、当前／待生效配方和 `lifetimeOutput`；
5. 施工任务；
6. 自有统一资产订单；
7. 新增 `fills`。

工厂变化按 `facilityTypeId` 比较，不存在实例 ID。

事件可以记录：

- 工作收入；
- 开始或完成施工；
- 工厂买入、卖出、冻结和解冻；
- 开启、关闭、异常和自动恢复；
- 通用配方切换与下一周期生效；
- 生产完成和产出入仓；
- 仓库扩容；
- 下单、撤单和成交。

## 5. 生产事件

生产通过以下增量识别：

```text
FacilityGroup.lifetimeOutput 增量 > 0
```

事件记录工厂类型、输入商品、输出商品、完成产量、库存变化和资金变化，并使用“工厂集群产出已进入共享仓库”等明确描述。

不得写入 `internalGoodsDelta`，不得创建领取动作。

## 6. 权威逐笔成交

成交记录不根据订单剩余量或市场 `lastPrice` 猜测。

客户端只为相邻快照中新出现的 fill ID 创建记录，并直接使用：

```text
fill.price
fill.quantity
fill.total
fill.counterparty
fill.createdAt
fill.makerOrderId
fill.takerOrderId
```

同一订单扫过多个价格时，每个价格分别生成一条记录，不得合并为平均价或最终市场价。

v1、v2 旧成交可能包含反推价格。升级到 v3 时保留其他资产事件但清空旧成交记录；升级到 v4 时把旧单输入生产记录转换为 `inputs[] + output`，并重新建立包含工厂目录的快照。

## 7. 展示与窗口化

- 市场页显示商品和工厂统一资产的本地成交。
- 资产页显示资金、商品、仓库、工厂和生产变化。
- 概览页最多显示最近 6 条成交。
- 所有界面必须标注“本地记录”。

资产页资产事件和市场页本地成交属于高增长记录列表，必须使用共享 `VirtualList` 窗口化组件：

1. 筛选、排序和总条数统计针对完整本地数组执行；
2. 数据不分页、不截断、不使用 `slice` 隐藏历史记录；
3. DOM 只创建当前滚动视口及少量 `overscan` 范围内的记录；
4. 条目离开视口后卸载，重新进入视口时按稳定记录 ID 恢复；
5. 资产事件允许可变高度测量，不能假定所有事件卡高度相同；
6. 记录视口可独立滚动，必须支持键盘聚焦、触控惯性滚动和移动端横向表格浏览；
7. 窗口化只优化渲染，不改变 480 条资产事件和 240 条成交记录的本地存储上限。

商品目录、工厂目录、5+5 订单簿、未完成订单上限以及概览最近 6 条等明确小规模列表不需要窗口化。

## 8. 清除与错误

清除日志时保留当前权威快照，避免把现有资产误记为新变化。

localStorage 读取或写入失败时：

- 继续接受服务器状态；
- 不阻断工作、下单、撤单、建设、启停、配方切换、扩容或礼品兑换；
- 可以显示本地日志不可用提示；
- 不向服务器上传日志进行恢复。

## 9. 防回退

不得恢复：

- 服务器 `trades`、`ledger` 或 `assetEvents` 数组；
- 单座工厂快照；
- 领取产成品事件；
- 固定价格工厂挂牌成交模型；
- 使用商品或工厂市场 `lastPrice` 反推成交；
- 使用本地日志修改正式资产；
- 对资产事件或本地成交直接使用全量 `.map()` 创建全部 DOM；
- 不得用分页、截断、`slice` 或只显示最近记录替代窗口化；
- 因窗口化而删除记录、改变筛选结果或降低本地存储上限。

未更新本设计和防回退检查的日志修改不应合并。

## 多输入生产事件

生产事件必须记录全部实际输入 `inputs[]` 和单个输出 `output`。本地快照必须包含 `facilityTypes`，并使用周期完成前的 `activeRecipeId` 计算该周期实际消耗；电子工厂记录必须同时显示塑料与铜材，不得只推断库存中第一个增加的商品。
