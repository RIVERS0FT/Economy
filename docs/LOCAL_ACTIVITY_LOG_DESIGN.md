# Economy 本地活动日志设计

> 状态：当前日志存储与展示基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-11

## 1. 核心原则

服务器不得持久化玩家活动日志。

服务器只保存权威经济状态，包括资金、商品库存、仓库、工厂、生产计划、订单、挂牌、市场、统计和幂等数据。

以下记录仅保存在当前浏览器：

- 资金与冻结资金变化；
- 商品可用与冻结库存变化；
- 仓库等级与容量变化；
- 工厂建设、交易、挂牌和状态变化；
- 生产费用、原料消耗和产成品直接入仓；
- 下单、撤单和成交记录。

## 2. 本地存储

```text
economy.local-activity.v1.<userId>
```

每个账号本地保存最后一次权威快照、最多 480 条资产事件和最多 240 条成交记录。本地日志不跨设备同步。

## 3. 生成方式

客户端比较连续两次服务器权威状态：

1. 资金和冻结资金；
2. 商品库存；
3. 仓库等级与容量；
4. 工厂状态、生产计划和 `completedQuantity`；
5. 自有订单与挂牌。

首次加载只建立快照；相同状态不重复生成日志。浏览器存储错误不得阻断游戏操作。

## 4. 直接入仓生产事件

新生产事件不再比较工厂内部商品，因为工厂不拥有内部存储。

生产完成通过以下差异识别：

```text
工厂 completedQuantity 增量 > 0
```

事件记录：

- 工厂名称；
- 输出商品；
- 完成产量；
- 原料商品和推导出的原料消耗量；
- 运营费造成的资金变化；
- 输出商品共享仓库库存变化；
- 描述“生产完成，产成品已直接进入共享仓库”。

新事件不得写入 `internalGoodsDelta`，不得创建 `collectFacility` 动作。

旧浏览器本地历史中的：

```ts
action: 'collected'
internalGoodsDelta
```

可以继续只读展示为“旧版领取”，但不得调用服务器或改变权威状态。

## 5. 仓库扩容事件

扩容事件记录资金扣款、扩容前后等级、扩容前后容量和容量增量。

```ts
warehouseChange?: {
  beforeLevel: number;
  afterLevel: number;
  beforeCapacity: number;
  afterCapacity: number;
  capacityDelta: number;
};
```

仓库占用与剩余空间始终来自服务器状态，不从本地日志推导。

## 6. 服务器边界

SQLite 写入前必须删除：

```text
trades
ledger
assetEvents
```

这些日志不得参与资产估值、仓库占用、订单校验、生产结算或排名。

工厂内部存储字段也不得通过日志恢复或重新提交服务器。

## 7. 清除、丢失与错误

清除日志时保留当前权威快照，避免把全部资产或生产累计量误记为新变化。

更换设备、清除网站数据、无痕模式或存储空间不足时日志允许丢失。日志丢失不得影响服务器资产。

## 8. 展示规则

- 市场页显示本地成交记录。
- 资金页显示资金、商品、仓库、工厂和生产差异。
- 新生产记录显示“产出入仓”。
- 旧领取记录明确标注“旧浏览器本地历史”。
- 概览页最多显示最近 6 条本地成交。
- 所有日志界面必须标注“本地记录”。

## 9. 测试与检查

必须验证：

- API 和 SQLite 不包含日志数组；
- 状态无变化时不生成事件；
- 工厂生产通过 `completedQuantity` 和库存差异生成事件；
- 新事件不包含内部商品变化；
- 不存在新的领取动作；
- 旧领取历史可以安全读取；
- 扩容记录包含等级和容量变化；
- localStorage 异常不阻断游戏。

## 10. 不可回退规则

不得：

- 恢复服务器持久化玩家日志；
- 把日志数组加回 `EconomyState`；
- 让本地日志参与资产、仓库、生产或排名计算；
- 使用本地日志恢复工厂内部存储；
- 恢复 `collectFacility` 本地动作；
- 删除账号隔离或清除功能；
- 把本地日志上传到任何接口。

本地日志不参与资产计算。未更新本设计和防回退检查的日志存储修改不应合并。
