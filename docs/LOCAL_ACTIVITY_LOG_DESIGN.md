# Economy 本地活动日志设计

> 状态：当前日志存储与展示基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-11

## 1. 核心原则

服务器不得持久化玩家活动日志。

服务器只保存完成经济结算所需的权威状态，包括资金、冻结资金、商品库存、工厂、生产计划、订单、挂牌、市场状态、统计和幂等数据。

用户可见的以下记录仅保存在当前浏览器：

- 资金变动；
- 冻结资金变化；
- 商品可用与冻结库存变化；
- 工厂建设、收购、出售、挂牌和状态变化；
- 生产费用、原料和产成品变化；
- 下单与撤单记录；
- 商品和工厂成交记录。

## 2. 本地存储

本地日志使用 `localStorage`，按账号 ID 隔离：

```text
economy.local-activity.v1.<userId>
```

本地文档保存：

- 最后一次服务器权威状态的紧凑快照；
- 最多 480 条资产事件；
- 最多 240 条成交记录。

本地日志不是账号云数据，不在设备间同步。

## 3. 生成方式

客户端每次收到服务器权威状态时：

1. 读取当前账号的上一次本地快照；
2. 比较资金、冻结资金、库存、工厂、自有订单和自有挂牌；
3. 生成本地资产事件和成交记录；
4. 写入 `localStorage`；
5. 更新本地快照。

首次加载只建立快照，不生成虚假变化。

状态没有变化时不生成日志。相同幂等结果重复返回时也不会重复记录。

## 4. 服务器边界

服务器写入 SQLite 之前必须删除玩家对象中的：

```text
trades
ledger
assetEvents
```

这些字段不得出现在：

- `economy_world.state_json`；
- `EconomyState`；
- 排名计算；
- 资产估值；
- 订单校验；
- 生产结算。

领域层在单次请求内临时产生的日志对象只属于内存过程，写入前必须清除。

## 5. 非权威性质

本地日志不参与资产计算。

本地日志不得：

- 修改资金；
- 修改库存；
- 修改工厂；
- 修改订单；
- 修改排名；
- 作为服务器校验依据；
- 作为争议结算或审计凭证。

界面必须明确标注“本地记录”。

## 6. 清除与丢失

玩家可以清除当前账号的本地日志。

清除时必须保留当前权威状态快照，避免下一次刷新把现有全部资产误记为新增变化。

以下情况日志允许丢失：

- 清除网站数据；
- 更换浏览器；
- 更换设备；
- 使用无痕模式；
- 浏览器拒绝本地存储；
- 本地存储空间不足。

日志丢失不得影响游戏状态。

## 7. 错误处理

读取或写入 `localStorage` 失败时：

- 忽略日志错误；
- 继续接受服务器权威状态；
- 不阻断下单、撤单、生产、建设或资金操作；
- 不向服务器重试上传日志。

## 8. 隐私与账号隔离

- 每个用户 ID 使用独立存储键；
- 退出登录不自动删除日志；
- 切换账号时不得混合显示；
- 重置服务器经济状态不会自动上传或恢复旧日志；
- 本地日志不得包含密码、Cookie、会话令牌或主页账号敏感数据。

## 9. 展示规则

市场页显示：

- 当前浏览器本地成交记录；
- 商品或工厂；
- 方向；
- 数量；
- 参考价格；
- 总额；
- 本地记录时间。

资金页显示：

- 本地资金与资产变动；
- 资金、商品、工厂和生产差异；
- 分类筛选；
- 清除本地记录按钮。

概览页最多显示最近 6 条本地成交。

## 10. 测试与检查

必须验证：

- API 状态不包含日志数组；
- SQLite 世界 JSON 不包含日志数组；
- 旧服务器日志在下一次读取后被清除；
- 幂等请求不会改变权威状态或创建服务器日志；
- 本地存储按账号隔离；
- 状态无变化时不生成本地事件；
- 清除日志不改变服务器状态；
- localStorage 异常不阻断游戏。

## 11. 不可回退规则

不得：

- 恢复服务器持久化玩家日志；
- 把 `assetEvents`、`trades` 或 `ledger` 加回 `EconomyState`；
- 让本地日志参与资产或排名计算；
- 删除账号隔离；
- 删除本地日志清除功能；
- 让日志存储错误阻断游戏操作；
- 把本地日志上传到任何接口。

未更新本设计和防回退检查的日志存储修改不应合并。
