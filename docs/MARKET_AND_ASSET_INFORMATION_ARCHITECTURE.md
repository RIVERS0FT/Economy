# Economy 市场订单与资金资产信息架构

> 状态：当前市场、订单、成交、资金、资产记录与页面归属基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-11  
> 客户端状态版本：`version: 7`  
> 世界状态版本：`version: 3`

本文件定义市场订单管理、生产仓库管理、资金资产结果和“服务器权威状态 + 浏览器本地日志”的边界。

页面内容与模块归属以 `docs/PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 为准；商品、工厂和生产计划以 `docs/INDUSTRY_AND_PRODUCTION_DESIGN.md` 为准；仓库等级和扩容以 `docs/WAREHOUSE_EXPANSION_DESIGN.md` 为准；日志存储以 `docs/LOCAL_ACTIVITY_LOG_DESIGN.md` 为准。

## 1. 核心决策

- 市场负责交易行为：商品选择、下单、订单簿、撤单、冻结资产摘要、本地成交记录和工厂挂牌。
- 生产负责共享仓库完整管理、工厂建设、生产计划、启停、领取和自己的工厂挂牌。
- 资金负责资产结果：现金、冻结资金、商品库存与估值、工厂估值、生产变化和本地资产变动。
- 设置只负责玩家资料、客户端偏好、登录会话和重置经济状态，不展示经营摘要。
- 不再提供独立“订单”导航或独立订单页面。
- 未完成订单数量显示在“市场”导航上。
- 玩家下单后可以在当前商品订单区直接撤单。
- 服务器只保存当前权威经济状态，不保存面向玩家的历史日志。
- 用户可见日志只保存在浏览器本地。
- 本地日志不参与资产计算、仓库计算、订单撮合、排名或任何服务器判断。

## 2. 导航与路由

正式导航固定为：

```text
概览
市场
生产
资金
排行
设置
```

稳定路由 ID：

```text
home
market
production
assets
leaderboard
settings
```

不得恢复 `records` 导航或 `records` 路由分支。

## 3. 市场页面职责

市场页必须包含：

1. 商品选择；
2. 行情摘要；
3. 商品限价下单；
4. 当前商品未完成订单和快捷撤单；
5. 商品订单簿；
6. 价格历史；
7. 全部未完成订单；
8. 冻结资金与冻结商品摘要；
9. 当前浏览器本地成交记录；
10. 工厂挂牌、撤销挂牌和收购。

市场页不得包含：

- 共享仓库扩容卡；
- 建设工厂；
- 工厂启停；
- 生产计划编辑；
- 资产配置；
- 账号设置。

市场可以通过状态栏或下单提示显示仓库已用、剩余和买单预占，但不得提供扩容按钮。

### 3.1 当前商品快捷订单

下单区域必须显示当前选中商品的所有未完成订单：

- 方向；
- 限价；
- 剩余数量与原始数量；
- 撤单按钮。

玩家不需要离开市场页即可撤销订单。

### 3.2 全部未完成订单

“我的订单与成交”面板展示全部商品的未完成订单：

| 字段 | 说明 |
|---|---|
| 商品 | 稳定商品 ID 对应的显示名称 |
| 方向 | 买入或卖出 |
| 限价 | 订单限价 |
| 剩余/原始 | 未成交数量与初始数量 |
| 冻结资产 | 买单冻结资金或卖单冻结商品 |
| 状态 | 等待成交或部分成交 |
| 时间 | 服务器创建时间 |
| 操作 | 直接撤单 |

订单继续受每名玩家最多 10 笔未完成订单限制。

### 3.3 本地成交记录

成交记录使用浏览器本地 `TradeRecord`，展示商品或工厂、方向、数量、参考成交价、总额、来源和本地记录时间。

这些记录由两次服务器权威状态的差异生成，不由服务器数据库保存。更换设备、无痕模式或清除网站数据后允许丢失。

## 4. 生产页面中的仓库与工厂交易边界

“生产”导航进入 `ProductionPage`，该页面是共享仓库唯一完整管理入口。

生产页负责：

- `WarehouseUpgradeCard`；
- 仓库等级、容量、实物库存、买单预占和剩余容量；
- 扩容费用和扩容按钮；
- 建设工厂；
- 生产计划；
- 启动和停止；
- 领取产成品；
- 发起自己的工厂挂牌；
- 撤销自己的工厂挂牌。

工厂挂牌的买方浏览和收购仍在市场页。生产页负责卖方资产管理，市场页负责公开交易发现与购买。

## 5. 资金与资产页面职责

“资金”导航进入“资金与资产”页面。页面必须包含：

- 可用资金；
- 冻结资金；
- 当前总资产；
- 商品资产；
- 工厂资产；
- 资产配置；
- 各商品库存和估值；
- 货币发行与系统回收；
- 当前浏览器本地资金与资产变动。

资金页必须提供清除本地记录的操作。清除记录不得改变任何服务器资产。

资金页不得包含：

- `WarehouseUpgradeCard`；
- 仓库扩容按钮；
- 建厂、生产计划、启动、停止或领取操作；
- 商品下单和撤单表单；
- 玩家资料设置。

资金页可以显示仓库扩容的本地历史结果，包括扣款、等级变化和容量变化。这是资产结果，不是仓库管理入口。

## 6. 设置页面边界

设置页只负责：

- 玩家资料和昵称；
- 邮箱、注册时间和角色等资料上下文；
- 客户端音效、数字格式、刷新频率和画面性能；
- 当前浏览器登录会话与退出登录；
- 重置服务器经济状态及风险提示。

设置页不得显示账号与产业摘要或仓库使用摘要。

设置页还不得显示：

- 工厂数量、运行、施工或阻塞摘要；
- 商品种类、库存或估值摘要；
- 未完成订单摘要；
- 当前排名摘要；
- 仓库等级、容量、使用量或买单预占；
- 扩容后容量；
- 下次扩容费用；
- 扩容按钮；
- `WarehouseUpgradeCard`。

这些经营信息分别归概览、市场、生产、资金、排行和全局状态栏。

## 7. 服务器权威状态

服务器持久化：

- 可用资金和冻结资金；
- 各商品可用库存与冻结库存；
- 仓库等级、容量和占用计算所需状态；
- 工厂、生产计划和工厂状态；
- 当前订单与订单状态；
- 工厂挂牌；
- 商品市场状态和价格历史；
- 工作冷却、经济统计和排名计算所需数据；
- 幂等响应缓存。

服务器不得把以下玩家日志数组写入 `economy_world.state_json`：

```text
trades
ledger
assetEvents
```

领域层可以在一次请求的内存处理中临时产生这些数组；写入 SQLite 前必须删除。

## 8. 客户端状态版本 7

`EconomyState` 使用 `version: 7`，状态中不包含：

```ts
trades: TradeRecord[];
ledger: LedgerEntry[];
assetEvents: AssetEvent[];
```

日志类型仍保留在前端类型系统中，但必须标注为浏览器本地模型，不能重新加入 `EconomyState`。

## 9. 本地资产事件

浏览器本地 `AssetEvent` 由两次权威状态快照差异生成，可以记录：

- 工作收入；
- 下单与撤单导致的资金、冻结资金和库存变化；
- 商品与工厂成交；
- 仓库扩容扣款、等级和容量变化；
- 工厂建设、产权和状态变化；
- 生产费用、原料和产成品变化。

服务器不保存玩家成交或资产历史日志，因此本地记录不保证跨设备恢复。

## 10. 本地存储规则

本地日志使用按账号隔离的 `localStorage` 键：

```text
economy.local-activity.v1.<userId>
```

规则：

- 首次加载只建立快照，不创建虚假事件；
- 状态没有变化时不创建事件；
- 相同权威状态重复返回时不重复记录；
- 浏览器存储失败不得阻断游戏操作；
- 清除日志时保留当前状态快照；
- 日志不得上传服务器；
- 日志不得用于计算平均资产、余额、库存、仓库、排名或订单状态。

## 11. 不可回退规则

不得：

- 把 `trades`、`ledger` 或 `assetEvents` 重新加入 `EconomyState`；
- 把玩家日志重新写入 SQLite 世界状态；
- 使用服务器日志计算玩家资产；
- 使用本地日志修改服务器资产；
- 恢复独立订单页面；
- 把撤单操作移出市场页；
- 把共享仓库管理移出生产页；
- 把经营状态摘要恢复到设置页；
- 删除本地日志清除功能；
- 让 localStorage 错误阻断交易、生产或状态刷新；
- 在未更新本设计、测试和架构检查的情况下改变页面边界。
