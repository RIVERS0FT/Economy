# Economy 市场订单与资金资产信息架构

> 状态：当前市场、订单、成交、资金与资产记录的设计基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-11  
> 客户端状态版本：`version: 7`  
> 世界状态版本：`version: 3`

本文件定义市场订单管理、资金资产页面职责，以及“服务器权威状态 + 浏览器本地日志”的边界。

涉及商品目录、工厂目录和生产计划时，以 `docs/INDUSTRY_AND_PRODUCTION_DESIGN.md` 为准；涉及日志保存位置时，以 `docs/LOCAL_ACTIVITY_LOG_DESIGN.md` 为准；涉及视觉规则时，以 `docs/UI_DESIGN_SYSTEM.md` 为准；涉及服务器事务、幂等和部署时，以 `docs/SERVER_ARCHITECTURE_AND_DEPLOYMENT_DESIGN.md` 为准。

## 1. 核心决策

- 市场负责交易行为：商品选择、下单、订单簿、撤单、冻结资产摘要、本地成交记录和工厂挂牌。
- 资金负责资产结果：现金、冻结资金、商品库存、工厂估值、生产变化和本地资产变动。
- 不再提供独立“订单”导航或独立订单页面。
- 未完成订单数量显示在“市场”导航上。
- 玩家下单后可以在当前商品订单区直接撤单。
- 服务器只保存当前权威经济状态，不保存面向玩家的历史日志。
- 用户可见日志只保存在浏览器本地。
- 本地日志不参与资产计算、订单撮合、排名或任何服务器判断。

## 2. 导航与路由

正式导航固定为：

```text
概览
市场
生产
资金
排行
设置
```

对应稳定路由 ID：

```text
home
market
production
assets
leaderboard
settings
```

不得恢复 `records` 导航或 `records` 路由分支。

## 3. 市场页面职责

市场页必须包含：

1. 商品选择；
2. 行情摘要；
3. 商品限价下单；
4. 当前商品未完成订单和快捷撤单；
5. 商品订单簿；
6. 价格历史；
7. 全部未完成订单；
8. 冻结资金与冻结商品摘要；
9. 当前浏览器本地成交记录；
10. 工厂挂牌、撤销挂牌和收购。

### 3.1 当前商品快捷订单

下单区域必须显示当前选中商品的所有未完成订单：

- 方向；
- 限价；
- 剩余数量与原始数量；
- 撤单按钮。

玩家不需要离开市场页即可撤销订单。

### 3.2 全部未完成订单

“我的订单与成交”面板展示全部商品的未完成订单：

| 字段 | 说明 |
|---|---|
| 商品 | 稳定商品 ID 对应的显示名称 |
| 方向 | 买入或卖出 |
| 限价 | 订单限价 |
| 剩余/原始 | 未成交数量与初始数量 |
| 冻结资产 | 买单冻结资金或卖单冻结商品 |
| 状态 | 等待成交或部分成交 |
| 时间 | 服务器创建时间 |
| 操作 | 直接撤单 |

订单继续受每名玩家最多 10 笔未完成订单限制。

### 3.3 本地成交记录

成交记录使用浏览器本地 `TradeRecord`，展示：

- 商品或工厂；
- 买入或卖出；
- 数量；
- 参考成交价；
- 总额；
- 来源；
- 本地记录时间。

这些记录由两次服务器权威状态的差异生成，不由服务器数据库保存。更换设备、无痕模式或清除网站数据后允许丢失。

## 4. 资金与资产页面职责

“资金”导航进入“资金与资产”页面。页面必须包含：

- 可用资金；
- 冻结资金；
- 当前总资产；
- 商品资产；
- 工厂资产；
- 仓库使用；
- 资产配置；
- 各商品库存和估值；
- 货币发行与系统回收；
- 当前浏览器本地资金与资产变动。

资金页必须提供清除本地记录的操作。清除记录不得改变任何服务器资产。

## 5. 服务器权威状态

服务器持久化的内容包括：

- 可用资金和冻结资金；
- 各商品可用库存与冻结库存；
- 仓库容量；
- 工厂、生产计划和工厂状态；
- 当前订单与订单状态；
- 工厂挂牌；
- 商品市场状态和价格历史；
- 工作冷却、经济统计和排名计算所需数据；
- 幂等响应缓存。

服务器不得把以下玩家日志数组写入 `economy_world.state_json`：

```text
trades
ledger
assetEvents
```

领域层可以在一次请求的内存处理中临时产生这些数组，用于完成当前操作；写入 SQLite 前必须删除。

## 6. 客户端状态版本 7

`EconomyState` 升级到 `version: 7`。

状态中不再包含：

```ts
trades: TradeRecord[];
ledger: LedgerEntry[];
assetEvents: AssetEvent[];
```

日志类型仍保留在前端类型系统中，但必须标注为浏览器本地模型，不能重新加入 `EconomyState`。

## 7. 本地资产事件

浏览器本地 `AssetEvent` 由两次权威状态快照差异生成：

```ts
interface AssetEvent {
  id: string;
  category:
    | 'work'
    | 'order'
    | 'trade'
    | 'inventory'
    | 'facility'
    | 'production'
    | 'system';
  createdAt: number;
  description: string;
  cashDelta: number;
  availableCashAfter: number;
  frozenCashDelta: number;
  frozenCashAfter?: number;
  inventoryChanges: AssetInventoryChange[];
  facilityChanges: AssetFacilityChange[];
  productionChanges: AssetProductionChange[];
  sourceType?: string;
  sourceId?: string;
  localOnly: true;
}
```

### 7.1 下单

本地差异可以记录：

- 可用资金减少；
- 冻结资金增加；
- 可用商品减少；
- 冻结商品增加；
- 新订单状态。

### 7.2 撤单

本地差异可以记录：

- 可用资金恢复；
- 冻结资金减少；
- 可用商品恢复；
- 冻结商品减少；
- 订单状态变为已取消。

### 7.3 成交

本地差异通过自有订单剩余数量变化识别成交，并记录商品、方向、数量和当前市场参考价。

由于服务器不保存玩家成交日志，本地记录不保证恢复真实对手方名称；界面统一显示“市场成交”或“设施市场”。

### 7.4 生产

本地差异可以同时记录：

- 运营费变化；
- 原料库存变化；
- 工厂内部产成品变化；
- 定量计划完成量变化；
- 工厂阻塞或计划完成状态。

### 7.5 工厂产权

本地差异可以识别：

- 新建工厂；
- 收购工厂；
- 出售工厂；
- 挂牌和撤销挂牌；
- 启动、停止和计划修改。

## 8. 本地存储规则

本地日志使用按账号隔离的 `localStorage` 键：

```text
economy.local-activity.v1.<userId>
```

每个本地文档包含：

- 最后一次权威状态的紧凑快照；
- 最多 480 条资产事件；
- 最多 240 条成交记录。

规则：

- 首次加载只建立快照，不创建虚假事件；
- 状态没有变化时不创建事件；
- 相同权威状态重复返回时不重复记录；
- 浏览器存储失败不得阻断游戏操作；
- 清除日志时保留当前状态快照，避免下一次刷新把全部资产误记为新变化；
- 日志不得上传服务器；
- 日志不得用于计算平均资产、余额、库存、排名或订单状态。

## 9. 离线与跨设备行为

玩家重新打开同一浏览器时，本地快照可以识别离线期间发生的资产变化。

以下情况日志允许缺失：

- 更换设备；
- 使用另一个浏览器；
- 清除网站数据；
- 无痕窗口关闭；
- 浏览器拒绝或超出本地存储配额。

缺失日志不能影响服务器权威资产。

## 10. 幂等与事务

- 所有写操作继续使用 `Idempotency-Key`；
- 相同幂等键重复请求返回同一权威结果；
- 本地日志根据状态差异生成，同一状态不会重复写入；
- 服务器事务失败时不返回新状态，客户端不得生成成功日志；
- 客户端日志不能反向提交到服务器。

## 11. 旧服务器日志清理

升级后，服务器读取旧世界状态时必须删除：

```text
player.trades
player.ledger
player.assetEvents
```

下一次保存时，旧日志不再存在于 SQLite 世界 JSON 中。

该清理只删除历史展示数据，不改变：

- 资金；
- 库存；
- 工厂；
- 订单；
- 挂牌；
- 市场价格；
- 经济统计。

## 12. 文件职责

| 文件 | 职责 |
|---|---|
| `server/src/storage.js` | 在读写边界删除玩家日志并只返回权威状态 |
| `server/src/asset-events.js` | 瞬时状态差异和写入前日志清理工具，不持久化历史 |
| `src/utils/localActivityStore.ts` | 本地快照比较、日志生成、保存和清除 |
| `src/app/gameViewModel.ts` | 每次接受权威状态时同步本地日志 |
| `src/pages/MarketPage.tsx` | 订单管理和本地成交记录 |
| `src/pages/AssetsPage.tsx` | 本地资金与资产变动 |
| `scripts/verify-market-assets.mjs` | 市场、资产和日志存储边界检查 |

## 13. 不可回退规则

不得：

- 把 `trades`、`ledger` 或 `assetEvents` 重新加入 `EconomyState`；
- 把玩家日志重新写入 SQLite 世界状态；
- 使用服务器日志计算玩家资产；
- 使用本地日志修改服务器资产；
- 恢复独立订单页面；
- 把撤单操作移出市场页；
- 删除本地日志清除功能；
- 让 localStorage 错误阻断交易、生产或状态刷新；
- 在未更新本设计、测试和架构检查的情况下改变日志保存边界。

未更新本设计、迁移和测试的市场或资产信息架构回退不应合并。
