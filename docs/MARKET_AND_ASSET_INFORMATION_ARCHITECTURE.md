# Economy 市场订单与资金资产信息架构

> 状态：当前市场、订单、成交、资金、资产记录与页面归属基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-11  
> 客户端状态版本：`version: 7`  
> 世界状态版本：`version: 3`

本文件定义市场订单管理、资金资产结果和“服务器权威状态 + 浏览器本地日志”的边界。

页面内容与模块归属以 `docs/PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 为准；商品、工厂和生产计划以 `docs/INDUSTRY_AND_PRODUCTION_DESIGN.md` 为准；仓库等级和扩容以 `docs/WAREHOUSE_EXPANSION_DESIGN.md` 为准；日志存储以 `docs/LOCAL_ACTIVITY_LOG_DESIGN.md` 为准。

## 1. 核心决策

- 市场负责交易行为：商品选择、下单、订单簿、撤单、冻结资产摘要、本地成交记录和工厂挂牌。
- 生产负责共享仓库完整管理、工厂建设、生产计划、启停、领取和自己的工厂挂牌。
- 资金负责资产结果：现金、冻结资金、商品库存与估值、工厂估值、生产变化和本地资产变动。
- 不再提供独立“订单”导航或独立订单页面。
- 未完成订单数量显示在“市场”导航上。
- 玩家下单后可以在当前商品订单区直接撤单。
- 服务器只保存当前权威经济状态，不保存面向玩家的历史日志。
- 用户可见日志只保存在浏览器本地。
- 本地日志不参与资产计算、仓库计算、订单撮合、排名或任何服务器判断。

## 2. 导航与路由

正式导航固定为：

```text
概览
市场
生产
资金
排行
设置
```

稳定路由 ID：

```text
home
market
production
assets
leaderboard
settings
```

不得恢复 `records` 导航或 `records` 路由分支。

## 3. 市场页面职责

市场页必须包含：

1. 商品选择；
2. 行情摘要；
3. 商品限价下单；
4. 当前商品未完成订单和快捷撤单；
5. 商品订单簿；
6. 价格历史；
7. 全部未完成订单；
8. 冻结资金与冻结商品摘要；
9. 当前浏览器本地成交记录；
10. 工厂挂牌、撤销挂牌和收购。

市场页不得包含：

- 共享仓库扩容卡；
- 建设工厂；
- 工厂启停；
- 生产计划编辑；
- 资产配置；
- 账号设置。

市场可以通过状态栏或下单提示显示仓库已用、剩余和买单预占，但不得提供扩容按钮。

### 3.1 当前商品快捷订单

下单区域必须显示当前选中商品的所有未完成订单：

- 方向；
- 限价；
- 剩余数量与原始数量；
- 撤单按钮。

玩家不需要离开市场页即可撤销订单。

### 3.2 全部未完成订单

“我的订单与成交”面板展示全部商品的未完成订单：

| 字段 | 说明 |
|---|---|
| 商品 | 稳定商品 ID 对应的显示名称 |
| 方向 | 买入或卖出 |
| 限价 | 订单限价 |
| 剩余/原始 | 未成交数量与初始数量 |
| 冻结资产 | 买单冻结资金或卖单冻结商品 |
| 状态 | 等待成交或部分成交 |
| 时间 | 服务器创建时间 |
| 操作 | 直接撤单 |

订单继续受每名玩家最多 10 笔未完成订单限制。

### 3.3 本地成交记录

成交记录使用浏览器本地 `TradeRecord`，展示：

- 商品或工厂；
- 买入或卖出；
- 数量；
- 参考成交价；
- 总额；
- 来源；
- 本地记录时间。

这些记录由两次服务器权威状态的差异生成，不由服务器数据库保存。更换设备、无痕模式或清除网站数据后允许丢失。

## 4. 生产页面中的仓库与工厂交易边界

“生产”导航进入 `ProductionPage`，该页面是共享仓库唯一完整管理入口。

生产页负责：

- `WarehouseUpgradeCard`；
- 仓库等级、容量、实物库存、买单预占和剩余容量；
- 扩容费用和扩容按钮；
- 建设工厂；
- 生产计划；
- 启动和停止；
- 领取产成品；
- 发起自己的工厂挂牌；
- 撤销自己的工厂挂牌。

工厂挂牌的买方浏览和收购仍在市场页。生产页负责卖方资产管理，市场页负责公开交易发现与购买。

## 5. 资金与资产页面职责

“资金”导航进入“资金与资产”页面。页面必须包含：

- 可用资金；
- 冻结资金；
- 当前总资产；
- 商品资产；
- 工厂资产；
- 资产配置；
- 各商品库存和估值；
- 货币发行与系统回收；
- 当前浏览器本地资金与资产变动。

资金页必须提供清除本地记录的操作。清除记录不得改变任何服务器资产。

资金页不得包含：

- `WarehouseUpgradeCard`；
- 仓库扩容按钮；
- 建厂、生产计划、启动、停止或领取操作；
- 商品下单和撤单表单；
- 玩家资料设置。

资金页可以显示仓库扩容的本地历史结果，包括扣款、等级变化和容量变化。这是资产结果，不是仓库管理入口。

## 6. 设置页面中的资产摘要边界

设置页可以在“账号与产业摘要”中显示一行只读仓库使用量，例如：

```text
仓库使用 120/500
```

设置页不得显示：

- 仓库等级；
- 扩容后容量；
- 下次扩容费用；
- 扩容按钮；
- `WarehouseUpgradeCard`。

## 7. 服务器权威状态

服务器持久化：

- 可用资金和冻结资金；
- 各商品可用库存与冻结库存；
- 仓库等级、容量和占用计算所需状态；
- 工厂、生产计划和工厂状态；
- 当前订单与订单状态；
- 工厂挂牌；
- 商品市场状态和价格历史；
- 工作冷却、经济统计和排名计算所需数据；
- 幂等响应缓存。

服务器不得把以下玩家日志数组写入 `economy_world.state_json`：

```text
trades
ledger
assetEvents
```

领域层可以在一次请求的内存处理中临时产生这些数组；写入 SQLite 前必须删除。

## 8. 客户端状态版本 7

`EconomyState` 使用 `version: 7`，状态中不包含：

```ts
trades: TradeRecord[];
ledger: LedgerEntry[];
assetEvents: AssetEvent[];
```

日志类型仍保留在前端类型系统中，但必须标注为浏览器本地模型，不能重新加入 `EconomyState`。

## 9. 本地资产事件

浏览器本地 `AssetEvent` 由两次权威状态快照差异生成：

```ts
interface AssetEvent {
  id: string;
  category:
    | 'work'
    | 'order'
    | 'trade'
    | 'inventory'
    | 'warehouse'
    | 'facility'
    | 'production'
    | 'system';
  createdAt: number;
  description: string;
  cashDelta: number;
  availableCashAfter: number;
  frozenCashDelta: number;
  frozenCashAfter?: number;
  inventoryChanges: AssetInventoryChange[];
  warehouseChange?: AssetWarehouseChange;
  facilityChanges: AssetFacilityChange[];
  productionChanges: AssetProductionChange[];
  sourceType?: string;
  sourceId?: string;
  localOnly: true;
}
```

### 9.1 下单与撤单

本地差异可以记录：

- 可用资金与冻结资金变化；
- 可用商品与冻结商品变化；
- 订单新增、部分成交或取消。

### 9.2 成交

本地差异通过自有订单剩余数量变化识别成交，并记录商品、方向、数量和当前市场参考价。

服务器不保存玩家成交日志，因此本地记录不保证恢复真实对手方名称；界面统一显示“市场成交”或“设施市场”。

### 9.3 仓库扩容

本地差异可以记录：

- 扩容扣款；
- 扩容前后等级；
- 扩容前后容量；
- 容量增量。

扩容操作在生产页执行，结果在资金页本地事件中展示。

### 9.4 生产与工厂产权

本地差异可以记录：

- 运营费、原料和产成品变化；
- 定量计划完成量；
- 工厂阻塞或计划完成；
- 新建、收购、出售、挂牌和撤销挂牌；
- 启动、停止和计划修改。

## 10. 本地存储规则

本地日志使用按账号隔离的 `localStorage` 键：

```text
economy.local-activity.v1.<userId>
```

每个本地文档包含：

- 最后一次权威状态的紧凑快照；
- 最多 480 条资产事件；
- 最多 240 条成交记录。

规则：

- 首次加载只建立快照，不创建虚假事件；
- 状态没有变化时不创建事件；
- 相同权威状态重复返回时不重复记录；
- 浏览器存储失败不得阻断游戏操作；
- 清除日志时保留当前状态快照；
- 日志不得上传服务器；
- 日志不得用于计算资产、仓库、余额、库存、排名或订单状态。

## 11. 离线与跨设备行为

玩家重新打开同一浏览器时，本地快照可以识别离线期间发生的资产变化。

以下情况日志允许缺失：

- 更换设备；
- 使用另一个浏览器；
- 清除网站数据；
- 无痕窗口关闭；
- 浏览器拒绝或超出本地存储配额。

缺失日志不能影响服务器权威资产。

## 12. 幂等与事务

- 所有写操作继续使用 `Idempotency-Key`；
- 相同幂等键重复请求返回同一权威结果；
- 本地日志根据状态差异生成，同一状态不会重复写入；
- 服务器事务失败时不返回新状态，客户端不得生成成功日志；
- 客户端日志不能反向提交到服务器。

## 13. 文件职责

| 文件 | 职责 |
|---|---|
| `server/src/storage.js` | 在读写边界删除玩家日志并只返回权威状态 |
| `server/src/asset-events.js` | 写入前日志清理工具，不持久化历史 |
| `src/utils/localActivityStore.ts` | 本地快照比较、日志生成、保存和清除 |
| `src/app/gameViewModel.ts` | 每次接受权威状态时同步本地日志 |
| `src/pages/MarketPage.tsx` | 订单管理和本地成交记录 |
| `src/pages/ProductionPage.tsx` | 共享仓库和工厂管理 |
| `src/pages/AssetsPage.tsx` | 资产结果和本地资产变动 |
| `src/pages/SettingsPage.tsx` | 账号、客户端设置和只读产业摘要 |
| `scripts/verify-market-assets.mjs` | 市场、资产和日志存储边界检查 |
| `scripts/verify-page-content.mjs` | 页面模块唯一归属检查 |

## 14. 不可回退规则

不得：

- 把 `trades`、`ledger` 或 `assetEvents` 重新加入 `EconomyState`；
- 把玩家日志重新写入 SQLite 世界状态；
- 使用服务器日志计算玩家资产；
- 使用本地日志修改服务器资产或仓库；
- 恢复独立订单页面；
- 把撤单操作移出市场页；
- 把仓库扩容移出生产页；
- 把 `WarehouseUpgradeCard` 放入资金页或设置页；
- 删除本地日志清除功能；
- 让 localStorage 错误阻断交易、生产或状态刷新；
- 在未更新页面职责设计、专项设计和架构检查的情况下改变页面归属。

未更新设计、迁移和测试的市场或资产信息架构回退不应合并。
