# Economy 市场订单与资金资产信息架构

> 状态：当前商品交易、工厂数量交易、资产结果与本地日志边界  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-11  
> 客户端状态版本：`version: 8`  
> 世界状态版本：`version: 4`

页面职责以 `docs/PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 为准；工厂集群与市场输入细则以 `docs/FACILITY_GROUP_AND_MARKET_V3_DESIGN.md` 为准；生产规则以 `docs/INDUSTRY_AND_PRODUCTION_DESIGN.md` 为准；本地日志以 `docs/LOCAL_ACTIVITY_LOG_DESIGN.md` 为准。

## 1. 核心边界

- 市场负责商品下单、快捷数量、单列订单簿、撤单、本地成交和工厂数量购买。
- 生产负责共享仓库、建设、统一进度、统一计划、统一启停和卖方数量挂牌。
- 资金负责现金、冻结资金、商品资产、工厂数量资产、估值和本地变化。
- 设置只负责资料、客户端偏好、登录会话和重置。
- 不提供独立订单页面。
- 服务器保存当前权威状态，不保存玩家历史日志。
- 浏览器本地日志不参与资产、仓库、生产、订单或排名计算。

## 2. 正式导航

```text
概览｜市场｜生产｜资金｜排行｜设置
```

稳定路由 ID：

```text
home
market
production
assets
leaderboard
settings
```

不得恢复 `records` 路由、独立仓库页或工厂实例页。

## 3. 商品下单区

字段顺序固定为：

```text
买入／卖出
限价
数量
1/4 仓｜1/2 仓｜全仓
订单总额
提交订单
当前商品未完成订单与撤单
```

限价必须位于数量之前。快捷按钮必须直接位于数量输入下方。

### 3.1 买入快捷数量

```text
maxBuy = min(
  warehouseAvailableCapacity,
  floor(credits / price)
)
```

按钮填入：

```text
1/4 仓 = max(1, floor(maxBuy × 0.25))
1/2 仓 = max(1, floor(maxBuy × 0.50))
全仓   = maxBuy
```

`maxBuy = 0` 时按钮禁用。买入数量不得超过资金或共享仓库剩余空间。

### 3.2 卖出快捷数量

```text
maxSell = selectedInventory.available
```

冻结商品不得计入可卖数量。按钮按 `maxSell` 的四分之一、二分之一和全部填入。

正式资金、库存和仓库校验继续由服务器执行。

## 4. 单列订单簿

订单簿不分左右两列，不进行价格档位聚合。

固定顺序：

```text
卖盘标题
价格最低的 5 笔有效卖单（最低价最靠近成交线）
最近成交与价差
价格最高的 5 笔有效买单（最高价最靠近成交线）
买盘标题
```

每行只显示：

```text
方向｜价格｜剩余数量
```

不得显示玩家身份。买卖盘各最多 5 笔。

## 5. 未完成订单与本地成交

市场页必须显示：

- 当前商品未完成订单和快捷撤单；
- 全部商品未完成订单；
- 冻结资金与冻结商品；
- 仓库剩余和买单预占；
- 当前浏览器本地成交记录。

每名玩家最多 10 笔未完成商品订单。

本地成交记录展示资产、方向、数量、成交参考价、总额、来源和本地时间，不上传服务器。

## 6. 工厂数量市场

公开挂牌保存：

```ts
facilityTypeId: string;
quantity: number;
unitPrice: number;
```

市场展示：

- 工厂类型；
- 可购买数量；
- 卖家；
- 单座价格；
- 周期、单座周期产量、单座周期成本和原料；
- 购买数量；
- 总价；
- 数量购买按钮。

允许部分购买。购买后挂牌剩余数量减少；买方同类集群运行中时，购买数量从下一周期加入。

市场不得展示工厂实例 ID、实例名称、内部容量、累计产量、小时指标或系统参考估值。

卖方数量挂牌和撤销挂牌位于生产页；公开浏览和购买位于市场页。

## 7. 生产与仓库边界

`ProductionPage` 是共享仓库唯一完整管理入口，负责：

- `WarehouseUpgradeCard`；
- 建设工厂；
- 同类型统一周期、计划与启停；
- 新数量下一周期加入；
- 数量挂牌和撤销挂牌。

市场只显示仓库剩余和买单预占，不提供扩容。

## 8. 资金与资产页面

资金页必须包含：

- 可用资金、冻结资金和总资产；
- 商品资产；
- 工厂数量资产；
- 资产配置与估值；
- 商品库存与估值；
- 货币发行与系统回收；
- 当前浏览器本地资金、商品、仓库、工厂集群和生产变化；
- 本地事件筛选与清除。

工厂估值：

```text
Σ(类型持有数量 × 该类型单座系统估值)
```

资金页不得提供仓库扩容、建厂、生产控制或下单。

## 9. 设置页面

设置页只负责玩家资料、界面偏好、登录会话和重置经济状态。不得显示工厂、商品、订单、仓库、排名或产业摘要。

## 10. 服务器权威状态

服务器持久化：

- 资金、冻结资金；
- 商品库存和冻结库存；
- 仓库等级和容量；
- `facilityGroups` 和施工任务；
- 商品订单；
- 工厂数量挂牌；
- 市场状态、价格历史和经济统计；
- 幂等响应缓存。

服务器不得持久化：

```text
player.facilities
facilityId（工厂资产）
trades
ledger
assetEvents
```

## 11. 客户端状态版本 8

`EconomyState.version` 固定为 8，包含 `facilityGroups` 和可选 `facilityConstruction`，不包含单座 `facilities` 数组或服务器历史日志。

浏览器本地存储键升级为：

```text
economy.local-activity.v2.<userId>
```

旧 v1 日志可以保留只读记录，但旧实例快照不得用于生成新事件。

## 12. 不可回退规则

不得：

- 把数量输入放在限价之前；
- 把快捷仓位按钮移出数量下方；
- 恢复双列订单簿、价格档位聚合或超过 5 笔买／卖盘；
- 在订单簿显示玩家身份；
- 恢复单座工厂交易；
- 把撤单移出市场页；
- 把共享仓库管理移出生产页；
- 把经营摘要恢复到设置页；
- 把日志或工厂实例重新写入 SQLite；
- 修改边界而不更新设计、迁移、测试和防回退检查。
