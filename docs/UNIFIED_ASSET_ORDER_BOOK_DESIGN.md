# Economy 统一资产订单簿设计

> 状态：商品和工厂交易、成交价格与估值的当前权威设计  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-12  
> 客户端状态版本：11  
> 世界状态版本：7

## 1. 统一资产模型

商品和工厂都使用 `AssetOrder`：

```ts
interface AssetOrder {
  id: string;
  assetKind: 'commodity' | 'facility';
  assetId: string;
  productId?: string;
  facilityTypeId?: string;
  side: 'buy' | 'sell';
  ownerType: 'player' | 'population' | 'market';
  ownerId?: number;
  ownerName: string;
  price: number;
  quantity: number;
  remaining: number;
  fills?: OrderFill[];
  status: 'open' | 'partial' | 'filled' | 'cancelled';
  createdAt: number;
}
```

当前市场不设置商品／工厂二级页面切换，不使用工厂固定价格挂牌卡。商品与工厂资产标签在同一横向目录中连续展示。

商品资产标签使用对应商品的 `ProductIcon`。工厂资产标签使用独立厂房 SVG `FactoryIcon`，不得使用机械商品的齿轮图标，也不得恢复 `⚙` 字符。图标只承担视觉语义，不改变资产 ID、撮合或估值规则。

`FacilityListing` 只允许作为版本 10 迁移兼容空结构存在，不得重新成为业务模型。

## 2. 下单与冻结

所有订单均由服务器校验：

- 价格和数量必须为有效整数范围。
- 单笔数量为 1～10,000。
- 每名玩家最多 10 笔未完成订单。
- 买单冻结对应资金。
- 商品卖单冻结对应可用库存。
- 工厂卖单冻结对应工厂数量。
- 撤单释放未成交部分资产。
- 部分成交只保留剩余数量对应的冻结资产。
- 同一玩家不得自成交。
- 所有写操作使用 `Idempotency-Key`。

商品买单还会预占共享仓库容量；工厂买单不占仓库。

## 3. 工厂运行中卖出

运行中的工厂允许进入卖单。

服务器提交工厂卖单时：

1. 结算已经完成的完整周期。
2. 从待加入数量和当前参与数量中冻结卖出数量。
3. 立即降低当前或下一周期参与数量。
4. 冻结数量停止产生商品和运营成本。
5. 保持剩余未冻结工厂的运行意图和周期语义。

撤销工厂卖单时，未成交数量解冻：

- 集群停止时立即恢复可用数量；
- 集群运行时进入 `pendingJoinCount`，从下一周期加入；
- 不重置当前周期，不生成部分周期产出。

## 4. 撮合

撮合固定遵循：

1. 资产种类和资产 ID 必须相同。
2. 价格优先。
3. 同价时间优先。
4. 最高买价大于或等于最低卖价时成交。
5. 允许部分成交。
6. 未成交部分继续保留。
7. 禁止自成交。
8. 成交、冻结、双方资产和统计在同一事务中更新。

商品订单和工厂订单不得互相撮合；不同商品或不同工厂类型也不得互相撮合。

## 5. maker price 与逐笔成交

每次成交使用撮合前已经位于订单簿中的对手方挂单价格，即 maker price。

服务器在相关订单 `fills` 中逐笔保存：

```ts
interface OrderFill {
  id: string;
  quantity: number;
  price: number;
  total: number;
  counterparty: string;
  createdAt: number;
  makerOrderId: string;
  takerOrderId: string;
  liquidity: 'maker' | 'taker';
}
```

同一订单扫过多个价格时必须生成多条独立成交明细，不得合并为平均价。浏览器不得使用市场 `lastPrice` 反推玩家成交价，也不得以最终市场价覆盖真实逐笔价格。

订单簿和价格曲线属于市场状态；玩家成交记录属于订单级权威明细。两者不得互相替代。

## 6. 单列订单簿

每个选中资产固定显示：

```text
卖盘标题
价格最低的 5 笔有效卖单
无文字中性分隔线
价格最高的 5 笔有效买单
买盘标题
```

每行只显示：

```text
方向｜价格｜剩余数量
```

规则：

- 不显示玩家身份。
- 不按价格档位聚合。
- 买卖盘各最多 5 笔。
- 订单簿内部不显示买一、卖一、价差、最近成交或指标卡式成交线。
- 桌面订单簿拉伸到所在网格行高度；订单不足时不生成虚假空行。
- 移动端按内容自然增长。

## 7. 下单界面

顺序固定为：

```text
方向
限价
数量
1/4 仓｜1/2 仓｜全仓
订单总额
提交订单
当前资产未完成订单与撤单
```

商品买入快捷数量：

```text
maxBuy = min(warehouseAvailableCapacity, floor(credits / price))
```

商品卖出快捷数量使用当前可用库存。冻结商品不能出售。

工厂买卖快捷数量使用当前可用资金或未冻结工厂数量，不使用共享仓库容量。

## 8. 估值与排行榜

每项商品和工厂的估值价格使用：

```text
该资产最高非本人有效买入价
```

- 排除资产所有者自己的买单。
- 没有非本人有效买单时估值为 0。
- 商品估值数量包含可用和冻结库存。
- 工厂估值数量包含玩家拥有的全部数量，包括卖单冻结数量。
- 总资产、资产摘要和排行榜由服务器统一计算。
- 客户端不得使用系统参考价、玩家自己的买单或固定挂牌价替代正式估值。

## 9. 本地成交

浏览器本地成交记录只比较相邻权威状态中新出现的 fill ID，并使用 `fill.price`、`fill.quantity`、`fill.total` 和 `fill.counterparty`。

本地日志 v3 升级时清空由旧 `lastPrice` 推导的成交记录，保留其他资产事件，并先建立新快照。

## 10. 防回退

不得恢复：

- 商品和工厂二级市场切换；
- 工厂固定价格挂牌或即时购买卡；
- 工厂必须停止后才能挂卖单；
- 按价格档位聚合订单簿；
- 双列买卖盘；
- 使用 `lastPrice` 反推成交价格；
- 使用系统参考价计算正式总资产；
- 客户端撮合或客户端解冻资产；
- 工厂资产标签使用机械齿轮图标、`⚙` 字符或与机械商品相同的 SVG 路径。
