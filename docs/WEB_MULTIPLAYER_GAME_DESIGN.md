# Economy 网页端多人经济游戏设计

> 状态：当前产品与功能设计基线  
> 当前交付平台：网页端  
> 后续可扩展平台：手机端、小程序端、电脑端  
> 首发服务器约束：2 核 2G  
> 更新时间：2026-07-10

## 1. 产品定位

Economy 是一款多人在线经济模拟与实时交易游戏。

玩家通过点击“工作”获得基础货币，再选择建造自己的工厂或在共享市场购买其他玩家的现成工厂。工厂持续生产基础商品，玩家将商品放入统一市场，与其他玩家或虚拟人口的消费需求成交，并通过工厂经营、商品交易和资产收购扩大企业规模。

当前只实现网页端。其他客户端暂不开发，但未来必须共享同一账号、资产、工厂、订单、成交、市场和赛季数据。

游戏中的货币、商品、企业和市场均为虚构内容，不对应真实货币、证券或商品。

## 2. 核心玩法循环

```text
点击工作获得货币
-> 自建工厂或购买现成工厂
-> 工厂生产基础商品
-> 商品进入企业仓库
-> 在统一市场提交商品订单或工厂挂牌
-> 与玩家或人口消费需求成交
-> 获得货币或资产
-> 升级、扩建、交易或收购工厂
-> 扩大企业总资产
```

游戏不设置劳动力资源，也不允许玩家在没有工厂时直接生产商品。

## 3. 玩家主要路线

### 3.1 工作启动型

- 通过间歇点击获得启动资金。
- 积累第一座工厂的建造或收购资金。
- 适合新玩家和短时上线玩家。

点击收入只负责启动和兜底，不应成为中后期最高收益来源。

### 3.2 工厂经营型

- 建造或收购工厂。
- 管理运营费用、生产进度和仓储。
- 根据人口需求和市场价格安排生产。
- 通过规模化生产获得利润。

### 3.3 市场交易型

- 观察商品盘口和工厂挂牌。
- 低价买入商品、高价卖出。
- 收购被低估的工厂并继续经营或转售。
- 为统一市场提供流动性。

### 3.4 综合经营型

- 点击获得基础资金。
- 自建与收购工厂并行。
- 同时参与商品和工厂交易。
- 根据市场变化调整产能与资产配置。

## 4. 核心设计原则

### 4.1 服务器结果为准

点击收入、资金、库存、工厂产权、施工、生产、订单、成交、人口预算、奖励和排行榜均以服务器记录为唯一有效结果。

### 4.2 点击收入受冷却控制

每次有效点击直接获得货币。冷却未结束时点击无收益；连续工作会逐步延长冷却。

### 4.3 工厂是生产前提

基础商品只能由已完工、可运行且未挂牌的工厂生产。

### 4.4 自建需要等待，购买立即交割

- 自建工厂：支付系统建造费用并等待施工。
- 购买工厂：按卖家挂牌价付款，成交后立即获得产权，不增加施工时间。

### 4.5 商品与工厂共用统一市场

商品订单、工厂挂牌、人口买单、当前订单、历史订单、成交记录和资产流水都在同一市场体系中管理。

### 4.6 公平且可追溯

所有资金、库存和产权变化都必须有明确原因与关联记录。

### 4.7 复杂展示放在客户端

图表、深度展示、经营预测、工厂收益分析和人口动画尽量由玩家电脑计算；正式经济结果仍由服务器确认。

## 5. 玩家账户与资产

每名玩家拥有：

- 玩家名称与企业名称
- 企业等级与经验
- 可用资金与冻结资金
- 可用库存与冻结库存
- 工厂列表与工厂槽位
- 工厂施工、生产和升级状态
- 企业仓库
- 当前订单与历史订单
- 成交记录与资产流水
- 点击工作状态
- 排名、成就和赛季数据

### 5.1 总资产

```text
总资产 = 可用资金
       + 冻结资金
       + 商品市场估值
       + 工厂系统估值
       + 其他企业资产估值
```

系统估值只用于展示、风控和排行榜，不代表系统承诺回购。

## 6. 点击工作

### 6.1 初始测试规则

| 项目 | 建议值 |
|---|---:|
| 每次有效点击收入 | 1 货币 |
| 基础冷却 | 3 秒 |
| 连续工作冷却档位 | 3 / 5 / 8 / 12 秒 |
| 无操作恢复时间 | 5 分钟 |

数值仅为首轮测试基线。

### 6.2 规则

- 点击收益由服务器确认。
- 修改本地时间不能缩短冷却。
- 刷新和多标签页不能重置冷却。
- 不提供离线点击收入。
- 每次成功点击生成资金流水。
- 冷却期间重复点击不能重复发放货币。

## 7. 工厂获得方式

### 7.1 自建工厂

```text
选择工厂类型
-> 查看费用与施工时间
-> 确认建造并扣除货币
-> 工厂进入施工中
-> 等待施工完成
-> 工厂变为待启用
-> 启动生产
```

初始测试建议：

| 项目 | 建议值 |
|---|---:|
| 基础工厂建造费用 | 60 货币 |
| 基础施工时间 | 5 分钟 |
| 初始工厂槽位 | 1 |
| 同时施工数量 | 1 |

施工期间工厂不能生产、升级或挂牌出售。玩家离线后施工继续。

### 7.2 购买现成工厂

```text
在统一市场选择工厂挂牌
-> 确认价格与工厂属性
-> 检查资金和工厂槽位
-> 扣除买家货币
-> 工厂产权立即转移
-> 卖家获得货币
-> 买家可立即启用工厂
```

购买不需要等待施工，也不重置工厂等级、产能、容量或累计经营记录。

## 8. 工厂状态与属性

工厂属性包括：

- 唯一编号
- 工厂名称与类型
- 当前所有者
- 等级
- 生产速度
- 单周期产量
- 单周期运营费用
- 内部成品容量
- 当前生产进度
- 累计产量
- 建造成本与系统估值
- 当前挂牌订单编号

工厂状态包括：

- 施工中
- 待启用
- 生产中
- 暂停生产
- 成品已满
- 资金不足
- 升级中
- 挂牌中
- 已出售
- 系统暂停

## 9. 工厂生产

MVP 只开放一种基础工厂和一种暂未命名的基础商品。

初始测试建议：

| 项目 | 建议值 |
|---|---:|
| 单周期时间 | 30 秒 |
| 单周期产量 | 1 个基础商品 |
| 单周期运营费用 | 1 货币 |
| 工厂内部成品容量 | 20 个 |

生产周期结束时：

1. 检查运营资金和内部容量。
2. 扣除运营费用。
3. 增加基础商品。
4. 更新生产进度与累计产量。
5. 条件满足时开始下一周期。

资金不足、内部成品已满、挂牌、升级或系统保护时自动暂停。

基础商品先进入工厂内部成品区，再由玩家领取到企业仓库。仓库不足时不能超额领取。

## 10. 工厂升级与扩建

工厂升级可以提高：

- 生产速度
- 单周期产量
- 运营费用效率
- 内部成品容量
- 后续商品解锁能力

升级需要支付费用并等待升级时间，期间工厂停止生产且不能挂牌。

企业扩建可以增加工厂槽位。槽位不足时不能自建或购买工厂。

MVP 可以只开放一级工厂，但页面、规则和资产结构必须支持后续升级。

## 11. 统一市场

市场同时包含：

- 基础商品买单
- 基础商品卖单
- 工厂出售挂牌
- 工厂即时购买
- 人口消费买单

市场页面可以通过筛选查看“全部、基础商品、工厂、人口需求、我的订单、我的成交”，但不能拆成互不关联的市场系统。

### 11.1 共用内容

- 订单编号
- 订单状态
- 当前订单
- 历史订单
- 成交记录
- 资产流水
- 市场动态
- 市场暂停与恢复状态
- 搜索、筛选与分页

## 12. 基础商品限价订单

基础商品是可分割、可批量交易的标准资产。

玩家提交：

- 买入或卖出
- 数量
- 限价

买单限价表示玩家愿意支付的最高单价；卖单限价表示玩家愿意接受的最低单价。

### 12.1 撮合规则

买单：价格从高到低，同价按服务器确认时间从早到晚。

卖单：价格从低到高，同价按服务器确认时间从早到晚。

最高买价大于或等于最低卖价时成交，成交价格采用先进入订单簿一方的挂单价格。

商品订单允许：

- 部分成交
- 多次成交
- 撤销未成交部分
- 未成交部分继续等待

买单冻结对应资金，卖单冻结对应库存。

## 13. 工厂挂牌订单

工厂是唯一资产，每座工厂数量固定为 1。

MVP 工厂交易采用固定价格出售挂牌：

- 卖家选择一座可出售工厂。
- 输入挂牌价格。
- 创建工厂卖单。
- 买家按挂牌价即时购买。
- 工厂订单不支持部分成交。

挂牌前工厂必须：

- 完成施工。
- 停止生产。
- 不处于升级中。
- 没有未完成生产周期。
- 内部成品已领取或清空。
- 没有其他有效挂牌。

挂牌期间工厂不能生产。卖家可以在未成交前撤单。

为防止利用工厂交易进行异常转账，挂牌价格建议限制在系统估值的 50%～200% 之间。

## 14. 统一订单状态

- 等待成交
- 部分成交
- 全部成交
- 已取消
- 已过期
- 系统撤销

商品订单可以部分成交；工厂订单数量固定为 1，不会部分成交。

## 15. 人口消费需求

虚拟城市人口为基础商品提供有限买单。

人口需求由以下因素决定：

- 城市人口
- 消费周期
- 目标需求量
- 居民消费预算
- 当前市场价格
- 居民价格承受能力
- 上一周期需求满足率

人口买单进入基础商品订单簿，遵循相同的价格与时间优先规则。

人口购买后的商品视为被消费并退出流通。人口不是无限回收商店，也不保证玩家以任意价格盈利。

## 16. 货币来源与回收

### 16.1 货币来源

- 点击工作
- 人口消费基础商品
- 有限任务与活动奖励

玩家间商品交易和工厂交易只转移已有货币，不增加全服货币总量。

### 16.2 货币回收

- 工厂建造费用
- 工厂运营费用
- 工厂升级费用
- 企业扩建与工厂槽位费用
- 仓库升级费用
- 后续市场服务费用

测试期间必须分别统计点击发行、人口发行和系统回收量。

## 18. 刷新、断线与多标签页

刷新或重连后恢复：

- 资金和库存
- 点击冷却
- 工厂产权与槽位
- 施工、生产与升级进度
- 当前订单
- 最近成交和市场状态

断线期间：

- 工厂施工继续。
- 工厂生产按正式状态继续。
- 已有订单继续有效并可成交。
- 不产生离线点击收入。

多标签页共享同一账号的冷却、资产、工厂和订单状态。

## 19. 异常与安全规则

- 同一点击请求只能发放一次货币。
- 同一工厂不能同时卖给两名玩家。
- 工厂购买必须同时完成扣款与产权转移。
- 买家工厂槽位不足时不能购买。
- 施工中或升级中的工厂不能挂牌。
- 挂牌中的工厂不能生产。
- 卖家不能购买自己的工厂。
- 商品和工厂订单都不能产生负资金或负库存。
- 客户端本地修改不能改变正式资产、施工或成交结果。
- 所有异常交易必须可通过流水审计。

## 20. 2 核 2G 约束

- 单一共享市场。
- 一种基础商品和一种基础工厂。
- 人口按周期聚合，不模拟单个居民。
- 工厂生产按周期结算，不模拟机器和工人个体。
- 工厂固定价挂牌，不做实时竞价或拍卖。
- 每名玩家初始一个工厂槽位。
- 每名玩家最多 10 笔未完成订单。
- 商品盘口只展示有限深度。
- 工厂挂牌、历史订单和流水全部分页。
- 图表、排行榜和经营分析允许延迟刷新。
- 50 名同时在线为最低稳定压测验收线，100 名为首发设计目标，正式人数以压测为准。

## 21. MVP 必须实现

1. 玩家注册或登录。
2. 点击工作直接获得货币。
3. 服务器确认点击冷却和递增冷却。
4. 使用货币自建基础工厂。
5. 自建工厂具有施工等待。
6. 工厂完工后按周期生产基础商品。
7. 工厂拥有运营费用和内部成品容量。
8. 玩家可以在统一市场购买其他玩家的工厂。
9. 购买工厂即时交割且不需要施工等待。
10. 工厂与商品共用市场、订单、成交和流水页面。
11. 基础商品使用限价订单簿。
12. 工厂使用固定价挂牌订单。
13. 商品订单支持部分成交。
14. 工厂订单数量固定为 1，不支持部分成交。
15. 人口消费生成有限基础商品买单。
16. 人口购买后的商品退出流通。
17. 资金、库存、产权和冻结资产正确结算。
18. 刷新和断线后恢复完整正式状态。
19. 工厂建造、运营和升级形成货币回收。
20. 所有正式结果以服务器状态为准。

## 22. 暂不实现

- 劳动力资源
- 指定名称的正式商品品类
- 多种商品与工厂类型
- 工厂拍卖、竞价和买入委托
- 工厂股份、租赁、抵押和贷款
- 玩家自由转账
- 私聊、好友和公会
- 杠杆、做空和真实货币交易
- 跨服务器市场
- 完整赛季系统
- 手机端、小程序端和独立电脑端

## 23. 当前开发决策

- 当前只实现网页端。
- 首发服务器为 2 核 2G。
- 点击直接获得货币，不设置劳动力。
- 工厂是生产基础商品的必要资产。
- 玩家可以自建工厂或购买玩家工厂。
- 自建需要等待施工；购买立即交割。
- 工厂与商品进入同一个共享市场。
- 商品使用限价订单簿；工厂使用固定价挂牌。
- 人口消费提供基础商品买单。
- 客户端负责展示和预测，服务器负责正式经济结果。
