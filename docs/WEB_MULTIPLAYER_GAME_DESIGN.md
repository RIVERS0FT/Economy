# Economy 网页多人经济游戏设计

> 状态：当前产品与功能总设计基线  
> 当前交付平台：网页端  
> 生产地址：`https://game.riversoft.top/economy/`  
> 首发服务器约束：2 核 2G  
> 更新时间：2026-07-11

本文件定义 Economy 的产品定位、核心循环、页面结构和系统边界。

专题优先级：

- 商品、工厂、配方、生产计划、工厂启停、市场维度和旧状态迁移，以 `docs/INDUSTRY_AND_PRODUCTION_DESIGN.md` 为最高专题基线。
- 服务器权威、SQLite、事务、幂等和生产部署，以 `docs/SERVER_ARCHITECTURE_AND_DEPLOYMENT_DESIGN.md` 为准。
- 颜色、控件、布局和响应式设计，以 `docs/UI_DESIGN_SYSTEM.md` 为准。

旧专题文档中的单一商品、单一设施、设施槽位和工厂自动恢复规则均属于历史设计，不得覆盖当前基线。

## 1. 产品定位

Economy 是一款多人在线经济模拟、产业经营与市场交易游戏。

玩家直接以个人身份拥有：

- 货币；
- 多种商品；
- 工厂；
- 生产计划；
- 商品订单；
- 工厂挂牌；
- 成交、库存、资金和产权记录；
- 总资产与排行榜名次。

游戏不设置企业身份，不依赖角色或城市空间。游戏中的货币、商品、工厂和市场均为虚构内容，不对应真实货币、证券或商品。

核心体验：

> 判断供需与价格，组织产业链，控制工厂生产，配置资产，并让自己的财富持续增长。

## 2. 核心玩法循环

```text
工作获得基础资金
-> 建设或收购工厂
-> 选择持续或定量生产计划
-> 手动启动工厂
-> 消耗资金和原料生产商品
-> 领取产成品
-> 在对应商品订单簿买卖
-> 调整产业链和资产配置
-> 提升总资产与排行榜名次
```

各系统定位：

- 工作：新玩家启动和资金不足时的兜底收入。
- 原料工厂：创造粮食和铁矿石。
- 加工工厂：将原料转化为中间产品和最终产品。
- 商品市场：价格发现、库存流动和玩家竞争。
- 工厂市场：唯一生产资产的即时产权交割。
- 总资产排行榜：长期竞争目标。

## 3. 当前商品与产业链

首批商品：

```text
粮食
铁矿石
面粉
钢材
食品
机械
```

首批产业链：

```text
农场 -> 粮食 -> 面粉厂 -> 面粉 -> 食品厂 -> 食品
矿场 -> 铁矿石 -> 钢铁厂 -> 钢材 -> 机械厂 -> 机械
```

不同商品拥有独立库存、冻结库存、订单簿、行情和需求。不同商品订单不得互相撮合。

正式配方、费用、周期和建造参数由服务器目录控制，详见 `docs/INDUSTRY_AND_PRODUCTION_DESIGN.md`。

## 4. 工厂规则

### 4.1 工厂持有

- 玩家持有工厂数量不设上限。
- 不存在设施槽位、空闲槽位或购买工厂时的槽位检查。
- 玩家可以建设工厂，也可以购买其他玩家挂牌的工厂。
- 同一玩家同时只能施工一座工厂，这是施工节奏限制，不是持有数量限制。

### 4.2 手动控制

- 施工完成的工厂进入停止状态。
- 玩家必须手动启动工厂。
- 玩家可以手动停止运行中的工厂。
- 手动停止前，服务器只结算已经完成的完整周期。
- 未完成周期不扣运营费、不消耗原料、不增加产出。
- 资金、原料或内部容量问题解决后，工厂不会自动重启。

### 4.3 生产计划

每座工厂支持：

- 持续生产：一直运行到玩家停止或工厂阻塞。
- 定量生产：达到目标产量后自动停止。

定量计划必须是单周期产量的整数倍。修改计划前必须先停止工厂。

### 4.4 离线生产

玩家离线前处于运行状态的工厂继续按服务器时间生产。服务器按完整周期批量结算，不逐秒模拟设备或工人。

## 5. 商品市场

每种商品使用独立限价订单簿。

玩家下单时指定：

- 商品；
- 买入或卖出；
- 数量；
- 限价。

撮合规则：

1. 商品必须相同。
2. 价格优先。
3. 同价时间优先。
4. 允许部分成交。
5. 未成交部分继续保留。
6. 未成交订单可以撤销。
7. 最高买价大于或等于最低卖价时成交。
8. 成交价格采用先进入订单簿一方的挂单价格。
9. 同一玩家不得自成交。

资产冻结：

- 买单提交时冻结对应资金。
- 卖单提交时冻结对应商品。
- 冻结商品不能用于工厂原料。
- 撤单时释放未成交部分资产。
- 部分成交后只保留剩余数量对应的冻结资产。

成交必须在同一服务器事务内更新：

- 买卖双方资金；
- 对应商品库存；
- 冻结资产；
- 订单剩余量和状态；
- 成交记录；
- 资金和库存流水；
- 该商品行情。

## 6. 工厂市场

工厂使用固定价格挂牌：

- 每座工厂是唯一资产，数量固定为 1。
- 卖家设置挂牌价。
- 挂牌前必须停止生产并领取内部产成品。
- 买家资金足够即可购买，不检查设施槽位。
- 成交后转移原始工厂 ID 和产权。
- 卖家不能购买自己的工厂。
- 收购后的工厂保持停止状态。
- 收购后的旧生产计划清除，恢复为持续生产模式，由买家重新决定计划。
- 玩家挂牌价不直接计入总资产估值。

## 7. 需求与价格

需求系统采用聚合模型，不模拟具体居民或企业。

- 食品和粮食主要由人口需求购买。
- 中间产品和机械可由企业采购需求购买。
- 玩家加工工厂会自然产生原料需求。
- 系统流动性订单只提供有限深度。
- 系统不保证所有商品都有固定利润或无限收购。

每种商品独立记录：

- 最近成交价；
- 价格历史；
- 买一价；
- 卖一价；
- 买卖价差；
- 成交量；
- 需求满足率。

## 8. 玩家仓库

首版使用共享仓库容量：

```text
所有商品可用数量
+ 所有商品冻结数量
+ 未完成买单预占数量
<= 玩家仓库容量
```

各商品分别记录可用数量和冻结数量。加工工厂只消耗可用原料。

## 9. 工作与货币

工作并入概览页，不作为独立主页面。

当前测试规则：

- 每次有效工作获得 1 货币。
- 连续工作冷却为 3、5、8、12 秒。
- 停止工作 5 分钟后恢复基础冷却。
- 不提供离线工作收入。
- 收益、冷却和服务器时间由游戏 API 判定。

货币发行来源：

- 玩家工作；
- 人口或企业需求消费；
- 后续有限任务与活动。

货币回收来源：

- 工厂建造费；
- 工厂运营费；
- 后续仓库扩容、升级和市场费用。

不得恢复设施槽位扩充收费，因为当前设计不存在设施槽位。

## 10. 总资产与排行榜

```text
总资产 =
可用资金
+ 冻结资金
+ Σ(各商品可用与冻结数量 × 对应参考价)
+ Σ(工厂系统估值 + 内部产成品估值)
```

规则：

- 每种商品使用自己的市场参考价。
- 成交样本不足时使用系统基础参考价。
- 工厂使用系统估值，不采用玩家挂牌价。
- 未成交预期利润不计入资产。
- 排行榜按总资产从高到低排序。
- 公开前 100 名，并始终显示当前玩家名次。
- 排行榜可以延迟刷新，资产正确性不能延迟提交。

## 11. 页面结构

### 11.1 概览

显示：

- 资金、总资产和排名；
- 工作入口；
- 关注商品市场摘要；
- 六种商品快速行情；
- 运行、停止、阻塞和施工工厂数量；
- 定量计划数量和计划剩余量；
- 待领取产成品；
- 最近成交与提醒。

### 11.2 市场

显示：

- 六种商品切换；
- 当前商品持仓和冻结库存；
- 独立买一、卖一、价差和价格历史；
- 商品下单表单；
- 独立订单簿；
- 需求满足率；
- 工厂挂牌与收购。

### 11.3 生产

显示：

- 六类工厂和配方；
- 建造费、施工时间、周期、运营费和容量；
- 工厂状态和停止原因；
- 输入原料、输出产品和原料库存；
- 持续或定量生产计划；
- 目标、完成量和剩余量；
- 保存计划、手动启动、手动停止、领取和挂牌。

生产页面不得显示设施槽位。

### 11.4 资产

显示：

- 现金和冻结资金；
- 各商品库存、冻结数量和估值；
- 工厂资产估值；
- 总仓库使用量；
- 资金发行与回收；
- 最近资产变化。

### 11.5 排行榜

显示总资产、现金、工厂数量、周期资产变化和更新时间。

### 11.6 订单与记录

统一展示：

- 各商品当前订单；
- 商品名称、方向、价格、数量和状态；
- 工厂成交；
- 商品成交；
- 冻结资金和冻结商品；
- 资金、商品、生产计划和产权流水。

### 11.7 设置

显示：

- 玩家资料；
- 界面设置；
- 工厂总数；
- 运行和施工统计；
- 仓库使用量；
- 商品种类；
- 当前排名；
- 退出登录和重置经济状态。

设置页面不得显示设施槽位。

## 12. 服务器与客户端边界

服务器负责：

- 资金、商品库存和冻结资产；
- 商品与工厂目录；
- 工厂产权、施工、配方、生产和计划；
- 手动启停和阻塞状态；
- 离线批量结算；
- 多商品订单撮合；
- 工厂挂牌和产权交割；
- 需求、行情和参考价；
- 总资产、排行榜、成交和流水；
- 旧状态迁移。

客户端负责：

- 页面展示和用户输入；
- 倒计时和进度可视化；
- 盘口、行情图表、筛选和排序；
- 不改变正式资产的预测与分析。

浏览器本地状态、修改本地时间和客户端计算不得改变正式资金、商品、工厂、计划、订单或产权。

## 13. 旧数据迁移

现有玩家数据必须无损迁移：

- 旧基础商品映射为粮食。
- 旧库存和冻结库存映射到粮食库存。
- 旧商品订单增加粮食商品 ID。
- 旧基础设施映射为农场。
- 保留设施 ID、产权、运行状态、累计产量和挂牌。
- 删除 `facilitySlots`，不转化成新限制。
- 迁移在 SQLite 读取层完成并在事务提交时写回。

## 14. 2 核 2G 约束

- 工厂按批量周期结算。
- 单次处理设置安全周期上限。
- 长期离线生产写聚合流水。
- 每名玩家最多 10 笔未完成订单。
- 盘口只展示有限深度。
- 订单、挂牌、成交和流水限制保存数量。
- 行情、排行榜和公开统计可以降低刷新频率。
- 资金、商品、原料、产出、计划、订单和产权正确性不能降级。

## 15. 不可回退项

除非先更新产品设计、迁移和自动化测试，否则不得：

- 恢复设施槽位或工厂持有上限；
- 将市场重新降为单一商品；
- 将所有库存重新合并为一个商品数量；
- 允许不同商品订单互相撮合；
- 删除工厂手动启动和停止；
- 让阻塞工厂自动恢复运行；
- 删除持续和定量生产计划；
- 让定量计划超产；
- 让客户端决定原料消耗、运营费或产出；
- 清零旧玩家数据代替迁移；
- 绕过事务、幂等键或服务器权威验证。

未同步更新 `docs/INDUSTRY_AND_PRODUCTION_DESIGN.md`、迁移逻辑和自动化测试的产业系统回退不应合并。

## 统一资产订单簿与玩家系统（2026-07-12）

- 商品和工厂共用同一限价订单簿，不再使用工厂固定价格挂牌或商品／工厂二级切换。
- 商品与工厂估值使用最高非本人有效买入价，服务器统一计算总资产和排行榜。
- 运行中工厂允许进入卖单，冻结数量立即减少当前参与数量和周期产量。
- 工作冷却固定为 10 秒，连续工作不再提高冷却。
- 玩家资料统计固定为点击工作次数、生产商品总数、买入商品总数、卖出商品总数。
- 状态栏固定显示可用资金、总资产、排行榜和仓库剩余。
- 设置页玩家资料卡包含统计、退出和重置；侧栏退出按钮保留；设置页增加礼品兑换。
- 管理员页面固定为 /economy/admin。
- 详细规则以 docs/UNIFIED_ASSET_ORDER_BOOK_DESIGN.md 和 docs/GIFT_CODE_AND_ADMIN_DESIGN.md 为准。
