# Economy 市场交易版多人游戏设计

> 状态：当前产品与功能设计基线  
> 当前交付平台：网页端  
> 后续可扩展平台：手机端、小程序端、电脑端  
> 首发服务器约束：2 核 2G  
> 更新时间：2026-07-10

本文件是 Economy 当前最高优先级的产品设计基线。其他专题文档与本文件冲突时，以本文件为准。

## 1. 产品定位

Economy 是一款多人在线经济模拟与交易游戏。

玩家通过工作获得基础资金，通过生产制造商品，通过市场交易配置资产，并在总资产排行榜中与其他玩家竞争。

核心体验是：

> 判断价格、管理资金、配置生产资产、参与交易，并让自己的财富持续增长。

游戏不需要具体玩家形象，不表现城市空间，也不设置企业身份。每位用户直接以“玩家”身份拥有货币、商品、生产设施和其他资产。

游戏中的货币、商品、设施和市场均为虚构内容，不对应真实货币、证券或商品。

## 2. 核心玩法循环

```text
工作获得基础资金
-> 建造或购买生产设施
-> 生产商品
-> 在市场买入或卖出商品
-> 利用价格波动与供需关系获利
-> 收购、升级或出售生产资产
-> 调整资产组合
-> 提升玩家总资产
-> 参与财富排行榜竞争
```

各系统的定位：

- 工作负责启动和破产兜底。
- 生产负责创造商品和稳定现金流。
- 交易负责资产流动、价格发现和玩家间竞争。
- 资产投资负责中长期财富增长。

## 3. 玩家资产结构

取消所有企业相关概念，资产直接归属于玩家。

每名玩家拥有：

- 可用货币
- 冻结货币
- 可用商品库存
- 冻结商品库存
- 生产设施
- 正在出售的资产
- 未完成订单
- 成交和资产流水
- 总资产与排行榜名次

玩家资料只需要：

```text
玩家昵称
账号头像
玩家等级或称号
注册时间
总资产
排行榜名次
```

不再使用：

```text
企业名称
企业资料
公司仓库
企业账户
企业总资产
```

原“企业仓库”统一改为“玩家库存”。

│ 左侧导航     │ 资金 / 总资产 / 排名 / 市场价格     │
│              ├─────────────────────────────────────┤
│ 主页面       │                                     │
│ 市场         │            当前功能内容区           │
│ 生产         │                                     │
│ 资产         │                                     │
│ 排行榜       │                                     │

- 顶部资产状态栏持续显示核心财富状态。

- 移动端后续可将左侧导航改为底部导航或侧滑菜单。

## 5. 左侧导航结构

### 5.1 主页面

主页面是玩家进入游戏后的核心操作中心，同时包含工作功能。

显示：

- 当前可用资金
- 当前总资产
- 当前排行榜名次
- 商品库存
- 生产设施状态
- 最近成交
- 市场价格摘要
- 工作按钮
- 待处理提醒

主页面不再以大段说明或企业资料为中心，而是提供可以立即操作的信息。

### 5.2 市场

市场是游戏最重要的页面。

包含：

- 商品买卖订单簿
- 最新成交
- 买卖价差
- 市场深度
- 限价买单
- 限价卖单
- 生产设施挂牌
- 生产设施收购
- 玩家自己的当前订单

### 5.3 生产

用于管理玩家拥有的生产设施。

包含：

- 建造新设施
- 已有设施
- 施工进度
- 生产状态
- 运营费用
- 内部商品数量
- 领取商品
- 暂停或恢复生产
- 设施升级
- 挂牌出售

### 5.4 资产

用于展示玩家财富构成和资产配置。

包含：

- 现金
- 冻结资金
- 商品库存估值

- 收入与支出
- 资产变化趋势
- 各类资产占比

### 5.5 排行榜

- 前 100 名玩家

### 5.6 订单与记录

- 当前买单

- 资产买卖记录
- 资金流水

- 产权变更记录

### 5.7 设置

包含：

- 玩家资料
- 音效设置

- 数字显示方式
- 市场刷新频率
- 退出登录

## 6. 主页面设计

主页面集中展示最关键的财富状态和快捷操作。

### 6.1 顶部状态

```text
可用资金：¤ 1,240
总资产：¤ 3,680
排行榜：第 42 名
商品库存：86
未完成订单：3 / 10
```

### 6.2 工作区

工作不再作为独立页面，而是主页面中的小型功能模块。

```text
基础工作

每次有效工作获得：¤1
当前连续工作档位：2 / 4
下一次可工作：4 秒

[开始工作]
```

MVP 测试参数：

- 每次有效工作获得 1 货币。
- 冷却档位为 3、5、8、12 秒。
- 停止工作 5 分钟后恢复基础冷却。
- 不提供离线工作收益。
- 工作收益和冷却由服务器确认。

工作按钮不应占据主页面的大部分空间。它只是新玩家启动和资金不足时的补充工具。

### 6.3 市场摘要

显示：

- 最近成交价
- 当前最高买价
- 当前最低卖价
- 买卖价差
- 最近成交数量
- 市场短期趋势

点击摘要可直接进入市场页面。

### 6.4 生产摘要

显示：

- 运行中的设施数量
- 施工中的设施数量
- 待领取商品
- 本小时预计产量
- 本小时预计运营费用

### 6.5 财富变化

显示：

- 今日资产变化
- 本周资产变化
- 当前总资产
- 排名变化

### 6.6 最近活动与提醒

显示最近成交、设施完工、生产暂停、订单成交、库存已满和资金不足等事件，并提供对应操作入口。

## 7. 商品市场

基础商品采用统一限价订单簿。

玩家下单时指定：

- 买入或卖出
- 数量
- 限价

### 7.1 撮合规则

1. 价格优先。
2. 同价时间优先。
3. 允许部分成交。
4. 未成交部分继续保留。
5. 未成交订单可以撤销。
6. 最高买价大于或等于最低卖价时成交。
7. 成交价格采用先进入订单簿一方的挂单价格。

### 7.2 资产冻结

- 买单提交时冻结对应资金。
- 卖单提交时冻结对应商品。
- 撤单后释放未成交部分对应的冻结资产。
- 部分成交后只保留剩余数量对应的冻结资产。

### 7.3 原子结算

成交后服务器必须在同一原子事务中更新：

- 买家资金
- 买家库存
- 卖家资金
- 卖家库存
- 订单剩余数量
- 订单状态
- 成交记录
- 资金、库存和资产流水

任何一步失败时，整笔成交不得部分生效。

### 7.4 市场信息

市场页面重点展示：

- 买一价
- 卖一价
- 买卖价差
- 最近成交价
- 最近成交量
- 盘口深度
- 玩家持仓
- 玩家平均成本
- 可用资金与库存

第一阶段不需要复杂 K 线，先提供简化价格曲线和最近成交列表。

## 8. 生产系统

生产设施是玩家创造商品的主要方式。

MVP 保留一种基础设施和一种基础商品。

初始测试参数：

| 项目 | 建议值 |
|---|---:|
| 建造费用 | 60 货币 |
| 施工时间 | 5 分钟 |
| 生产周期 | 30 秒 |
| 周期产量 | 1 个商品 |
| 周期运营费 | 1 货币 |
| 内部容量 | 20 个商品 |
| 初始设施槽位 | 1 |
| 同时施工数量 | 1 |

生产流程：

```text
支付建造费
-> 等待施工完成
-> 启动生产
-> 每周期支付运营费
-> 商品进入设施内部库存
-> 玩家领取到个人库存
-> 商品进入市场出售
```

生产设施在以下情况暂停：

- 资金不足
- 内部商品已满
- 正在出售
- 正在升级
- 玩家主动暂停
- 系统保护或维护

施工和正式生产状态以服务器时间与服务器记录为准。玩家离线后，施工与已启动生产按照正式状态继续。

## 9. 生产资产投资

生产设施不仅是生产工具，也是一类可交易资产。

玩家可以：

- 自建生产设施
- 收购其他玩家的设施
- 升级设施
- 持有设施获取生产收益
- 挂牌出售设施
- 根据回报率调整持仓

设施交易使用固定价格挂牌：

- 每座设施是唯一资产。
- 数量固定为 1。
- 卖家设置价格。
- 买家即时购买。
- 不允许部分成交。
- 成交后直接转移产权。
- 卖家不能购买自己的设施。
- 买家必须拥有足够资金和空闲设施槽位。

设施应展示：

```text
设施等级
生产周期
周期产量
运营费用
内部容量
累计产量
参考估值
挂牌价格
预计回本时间
```

参考估值由系统计算，不能直接使用玩家挂牌价，避免通过虚高挂牌价格制造虚假总资产。

设施挂牌期间停止生产。设施必须完成施工、不在升级中且不存在冲突状态，才可以挂牌出售。

## 10. 总资产计算

排行榜的核心依据是玩家总资产。

```text
总资产 =
可用资金
+ 冻结资金
+ 可用商品 × 商品参考价
+ 冻结商品 × 商品参考价
+ 生产设施系统估值
```

系统估值只用于展示、风险控制和排行榜，不代表系统承诺回购。

### 10.1 商品参考价

商品参考价不能直接采用单笔异常成交价。

建议使用：

- 最近一段时间的成交量加权平均价。
- 成交样本不足时使用系统基础参考价。
- 对异常价格设置上下限保护。
- 过滤明显自成交、关联异常成交和极小数量拉价。

### 10.2 生产设施估值

生产设施估值可以考虑：

```text
基础建造成本
+ 有效升级投入
+ 产能价值
+ 内部商品价值
+ 折旧或使用修正
```

不计入总资产：

- 玩家自行设置的挂牌溢价
- 未成交订单的预期利润
- 预测性收入
- 虚假或异常关联交易产生的估值

## 11. 排行榜设计

主排行榜按总资产从高到低排序。

显示字段：

```text
排名
玩家昵称
总资产
现金资产
生产设施数量
本周资产变化
更新时间
```

排行榜规则：

- 每 1～5 分钟更新一次。
- 只公开前 100 名。
- 始终显示玩家自己的名次。
- 排名相同时，优先达到该资产的玩家靠前。
- 资产计算必须由服务器完成。
- 异常成交不应直接影响排行榜参考价。
- 管理员可冻结异常账号的排名数据。

第一阶段只实现总资产排行榜。

后续可以增加：

- 周财富增长榜
- 商品成交量榜
- 生产总量榜
- 设施数量榜
- 赛季排行榜

## 12. 人口需求

人口系统继续保留，但不表现城市空间，也不模拟具体居民。

它作为系统需求模型存在，为市场生成有限商品买单。

人口需求根据以下数据计算：

- 当前商品价格
- 市场供应量
- 历史需求满足率
- 当前周期预算
- 商品库存或缺口
- 经济系统货币投放目标

人口买单进入与玩家相同的商品订单簿，遵循相同的价格与时间优先规则。人口购买后的商品退出流通。

人口不会：

- 购买生产设施
- 无限购买商品
- 提供永久固定保底价格

人口需求只负责提供基础消费，不保证玩家盈利。

## 13. 货币发行与回收

### 13.1 货币发行来源

- 玩家工作
- 人口消费
- 有限任务奖励
- 后续活动奖励

玩家之间的交易只转移已有货币，不创造新货币。

### 13.2 货币回收渠道

- 设施建造费
- 设施运营费
- 设施升级费
- 设施槽位扩充
- 玩家库存容量升级
- 市场手续费
- 后续资产维护费

主页面和资产页面可以展示：

```text
全服工作发行
全服人口发行
全服系统回收
当前货币净变化
```

## 16. 2 核 2G 首发约束

- 单一共享市场。
- 一种基础商品。
- 一种基础生产设施。
- 一个聚合人口需求模型。
- 生产按周期结算，不模拟工人、机器或居民个体。
- 生产设施使用固定价挂牌，不实现竞价和拍卖。
- 每名玩家初始一个设施槽位。
- 每名玩家最多 10 笔未完成订单。
- 商品盘口只展示有限深度。
- 设施挂牌、历史订单和流水限制展示数量并分页。
- 图表、排行榜和公开统计允许延迟刷新。
- 50 名同时在线为最低稳定压测验收线。
- 100 名同时在线为首发设计目标。
- 正式开放人数以实际压力测试为准。

服务器资源不足时，优先降低图表、排行榜和公开统计刷新频率。货币、库存、设施产权、订单和成交正确性不能降级。

## 17. 刷新、断线与多标签页

刷新或重连后恢复：

- 资金和库存
- 工作冷却
- 生产设施产权与槽位
- 施工、生产与升级进度
- 当前订单
- 最近成交和市场状态
- 总资产与最近排行榜结果

断线期间：

- 设施施工继续。
- 已启动生产按照正式状态继续。
- 已有订单继续有效并可成交。
- 不产生离线工作收入。

多标签页共享同一账号的冷却、资产、设施和订单状态。

## 18. 异常与安全规则

- 同一工作请求只能发放一次货币。
- 同一设施不能同时卖给两名玩家。
- 设施购买必须同时完成扣款与产权转移。
- 买家设施槽位不足时不能购买。
- 施工中或升级中的设施不能挂牌。
- 挂牌中的设施不能生产。
- 卖家不能购买自己的设施。
- 商品和设施订单都不能产生负资金或负库存。
- 客户端本地修改不能改变正式资产、施工或成交结果。
- 所有资产变化必须有可审计流水。
- 异常成交不得直接推高参考价与排行榜资产。

## 19. MVP 必须实现

1. 玩家注册或登录。
2. 玩家资料不包含企业身份。
3. 工作功能并入主页面。
4. 服务器确认工作收益和递增冷却。
5. 使用货币自建基础生产设施。
6. 自建设施具有施工等待。
7. 设施完工后按周期生产基础商品。
8. 设施拥有运营费用和内部商品容量。
9. 玩家可以在市场购买其他玩家的设施。
10. 购买设施即时交割且不需要施工等待。
11. 基础商品使用限价订单簿。
12. 设施使用固定价挂牌。
13. 商品订单支持部分成交。
14. 设施订单数量固定为 1，不支持部分成交。
15. 买单冻结资金，卖单冻结库存。
16. 人口需求生成有限基础商品买单。
17. 人口购买后的商品退出流通。
18. 资金、库存、产权和冻结资产正确结算。
19. 服务端计算商品参考价与生产设施估值。
20. 服务端计算玩家总资产和排行榜。
21. 排行榜公开前 100 名并始终显示玩家本人。
22. 刷新和断线后恢复完整正式状态。
23. 所有正式结果以服务器状态为准。

## 22. 当前开发决策

- 当前只实现网页端。
- 首发服务器为 2 核 2G。
- 不设置企业身份、角色形象或城市场景。
- 所有货币、商品和生产资产直接归属于玩家。
- 工作并入主页面，只负责启动和兜底。
- 市场是核心页面，商品使用限价订单簿。
- 生产设施既是生产工具，也是可交易资产。
- 设施使用固定价格挂牌并即时交割。
- 人口需求只提供有限基础消费。
- 商品参考价、设施估值、总资产和排行榜由服务器计算。
- 客户端负责展示和预测，服务器负责正式经济结果。

## 23. 最终体验目标

新的 Economy 不再像企业管理后台，也不依赖角色或城市场景。

它应呈现为一款清晰、快速、信息密集的市场经济游戏：

```text
左侧切换主要功能
顶部持续显示财富状态
主页面完成工作和查看关键摘要
市场页面进行核心交易
生产页面管理商品供给
资产页面分析财富结构
排行榜页面推动玩家竞争
```

游戏的核心乐趣是：

> 从少量启动资金开始，通过生产、交易和资产配置形成自己的财富策略，并在不断变化的市场中提升总资产排名。
