# Economy 概览布局完整性设计

> 状态：当前正式概览布局防回退基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-17

## 1. 目标

概览页必须在桌面、紧凑桌面和平板宽度下保持完整的经营决策层级，不得因为外层网格继承、侧栏占宽或标题压缩而把核心卡片挤成纵向窄条。

本规则补充 `PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 与 `UI_DESIGN_SYSTEM.md`。发生冲突时，页面内容职责仍以前两份文档为准；概览外层轨道、真实内容宽度响应式和几何回归测试以本文为准。

## 2. 外层轨道唯一性

概览 React 结构包含两条业务行：

```text
第一排：今日经营 + 市场快照
第二排：生产摘要 + 资产构成 + 当前挂单
```

`.home-grid` 只是这两条业务行的纵向容器，必须始终只有一个网格轨道：

```css
.home-grid {
  display: grid;
  grid-template-columns: minmax(0, 1fr);
}
```

不得让全局历史样式中的三列定义成为概览最终计算值。`src/styles/overview.css` 在概览样式职责中必须明确重置该轨道；以后重命名选择器时也必须保持“外层单轨、内部各自分栏”的结构。

## 3. 实际内容宽度响应式

桌面侧栏展开、折叠和自动紧凑都会改变页面真正可用宽度，因此概览桌面布局不得只根据浏览器视口宽度判断。

`.home-grid` 是命名为 `overview` 的内联尺寸容器，内部布局按实际内容宽度切换：

- 大于 `1050px`：第一排按 `5:7` 两列，第二排三等列；
- 不大于 `1050px`：第一排单列，第二排两列，当前挂单跨两列；卡片恢复自然高度；
- 不大于 `580px`：两排全部单列；
- 现有 `1199px` 与 `760px` 视口媒体查询作为不支持容器查询时的回退，并继续承担正式移动外壳的布局保护。

侧栏折叠后如果内容宽度跨过断点，概览必须在同一运行时重新排版，不得要求刷新页面。

## 4. 可读宽度和标题

宽布局下必须满足：

- 今日经营宽度大于 `420px`；
- 市场快照宽度大于 `560px`，且宽于今日经营；
- 三张摘要卡中的最窄卡大于 `280px`；
- 第一排和第二排都占满概览内容区；
- 卡片、业务行和外层容器不得产生水平内容溢出。

概览卡片标题保持单行并在极端情况下省略，不得依赖全局 `overflow-wrap: anywhere` 逐字纵排。标题右侧操作是不可压缩项；空间不足时由响应式布局换行或改为上下排列。

## 5. 高度和内部滚动

大内容宽度继续使用约 `330px` 的第一排和约 `320px` 的第二排。固定高度卡片只有在内容确实超出时才允许内部纵向滚动。

当实际内容宽度不大于 `1050px` 时，卡片恢复自然高度，经营提醒、资产事件和当前挂单列表必须使用页面主滚动区，不得保留嵌套纵向滚动。空状态和短列表不得出现可滚动溢出。

## 6. 浏览器运行时回归测试

Playwright 必须至少覆盖：

1. `1684×931`：外层单轨，第一排两卡同排，第二排三卡同排，两排占满内容区；
2. `1440×900`：第二排标题和核心指标位于首屏；
3. `900×1000`：第一排单列、第二排两列，嵌套列表恢复可见溢出；
4. `1280×900`：展开侧栏时为紧凑布局，折叠侧栏后按实际内容宽度恢复两列和三列；
5. 空市场场景不渲染完整图表，空提醒与空挂单不产生内部滚动溢出；
6. 所有概览核心容器 `scrollWidth` 不得大于 `clientWidth`。

测试必须比较真实 `getBoundingClientRect()`、计算后的网格轨道和滚动尺寸，不能只检查 CSS 源码中是否存在某个字符串。

## 7. 文件职责

- `src/styles/overview.css`：概览外层单轨、内部两排、容器查询、标题和列表滚动规则；
- `tests/browser/runtime.spec.ts`：真实浏览器几何、断点、侧栏折叠和溢出回归；
- `scripts/verify-overview-content.mjs`：确保上述样式与运行时测试不可被静默移除；
- `src/styles/globals.css`：不得成为概览最终分栏规则的唯一来源。
