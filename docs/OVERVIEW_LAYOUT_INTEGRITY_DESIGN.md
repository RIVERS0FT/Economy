# Economy 概览布局完整性设计

> 状态：当前正式概览布局防回退基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-20

## 1. 目标

概览页必须在桌面、紧凑桌面和平板宽度下保持完整的经营决策层级，不得因为外层网格继承、侧栏占宽、标题压缩、图表宽高比或短列表滚动而降低核心信息可读性。

本规则补充 `PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 与 `UI_DESIGN_SYSTEM.md`。发生冲突时，页面内容职责仍以前两份文档为准；概览外层轨道、真实内容宽度响应式、紧凑图表几何、现金变化语义和浏览器几何回归测试以本文为准。

## 2. 外层轨道唯一性

概览 React 结构包含两条业务行：

```text
第一排：今日经营 + 市场快照
第二排：生产摘要 + 资产构成 + 当前挂单
```

`.home-grid` 只是这两条业务行的纵向容器，必须始终只有一个网格轨道：

```css
.home-grid {
  display: grid;
  grid-template-columns: minmax(0, 1fr);
}
```

不得让全局历史样式中的三列定义成为概览最终计算值。`src/styles/overview.css` 在概览样式职责中必须明确重置该轨道；以后重命名选择器时也必须保持“外层单轨、内部各自分栏”的结构。

## 3. 实际内容宽度响应式

桌面侧栏展开、折叠和自动紧凑都会改变页面真正可用宽度，因此概览桌面布局不得只根据浏览器视口宽度判断。

`.home-grid` 是命名为 `overview` 的内联尺寸容器，内部布局按实际内容宽度切换：

- 大于 `1050px`：第一排按 `5:7` 两列，第二排三等列；
- 不大于 `1050px`：第一排单列，第二排两列，当前挂单跨两列；卡片恢复自然高度；
- 不大于 `580px`：两排全部单列；
- 现有 `1199px` 与 `760px` 视口媒体查询作为不支持容器查询时的回退，并继续承担正式移动外壳的布局保护。

侧栏折叠后如果内容宽度跨过断点，概览必须在同一运行时重新排版，不得要求刷新页面。

## 4. 可读宽度和标题

宽布局下必须满足：

- 今日经营宽度大于 `420px`；
- 市场快照宽度大于 `560px`，且宽于今日经营；
- 三张摘要卡中的最窄卡大于 `280px`；
- 第一排和第二排都占满概览内容区；
- 卡片、业务行和外层容器不得产生水平内容溢出。

概览卡片标题保持单行并在极端情况下省略，不得依赖全局 `overflow-wrap: anywhere` 逐字纵排。标题右侧操作是不可压缩项；空间不足时由响应式布局换行或改为上下排列。

## 5. 高度和内部滚动

大内容宽度使用约 `370px` 的第一排和约 `330px` 的第二排，以容纳完整紧凑行情图、状态说明和最多三条现金事件。该规则覆盖早期文档中的 `330px / 320px` 近似值，但不得通过继续增加高度来掩盖图表或列表布局错误。

经营提醒最多显示三条，资产构成最多显示三条真实现金事件，因此两者不得启用内部纵向滚动。当前挂单在不超过三条时也必须自然显示；只有超过三条时才能添加 `overview-open-orders-list--scrollable` 并在固定卡片内滚动。

当实际内容宽度不大于 `1050px` 时，卡片恢复自然高度，经营提醒、资产事件和当前挂单列表必须使用页面主滚动区，不得保留嵌套纵向滚动。空状态和短列表不得出现可滚动溢出。

## 6. 紧凑市场图表

概览紧凑图与完整市场图必须使用独立几何配置。紧凑图固定使用适合宽卡的 `960×228` 视图框，不得重新套用完整图的 `960×540` 视图框后再通过固定高度缩放。

紧凑图必须满足：

- SVG 宽度使用市场卡可用内容宽度的至少 `85%`，不得因保持错误宽高比缩在中央；
- 12 个两小时分段继续生成竖向网格线，但最多显示 6 个均匀分布的 `HH:mm` 文字标签，首尾标签必须保留；
- 价格轴使用 3 个文字刻度，成交量轴使用最大值和 0 两个文字刻度；
- 图例文案固定为“主动买入／主动卖出／均衡”，完整市场图继续使用完整文案；
- 横轴标签保持水平，所有标签、刻度、图例与轴标题不得互相重叠或超出 SVG 边界；
- 紧凑图不得设置 `clamp(168px, 20vw, 210px)` 一类与视图框比例不一致的内联高度。

### 6.1 移动端市场指标

视口宽度不大于 `760px` 时，市场快照中的“最近成交／最高买价／最低卖价／当前持仓”必须保持单行四等列。四个完整标签不得换行、隐藏或缩写，允许适度收紧单元格水平内边距和标签字号，但数值仍须保持主视觉层级。

移动端不得恢复为 `2×2` 指标网格，不得通过横向滚动容纳四项指标，也不得保留前两项的行间底边框。单行布局只保留列间分隔线，最后一列继续不显示右边框。

## 7. 市场空值与订单簿语义

最高买价和最低卖价只有在对应挂单存在时才使用买方或卖方状态色。缺少买价或卖价时，“暂无”必须使用中性色，不得表现为危险、亏损或错误。

有市场活动时，四项指标下方必须显示一条当前订单簿说明：

- 只有买单：说明暂无可供买入的卖单；
- 只有卖单：说明暂无可立即成交的买单；
- 买卖双方均存在：显示非负买卖价差；
- 只有历史成交而无有效挂单：说明当前没有有效买卖挂单。

说明只用于解释当前快照，不新增同等权重的市场入口。

## 8. 本周现金变化

“本周资金变化”只展示最近七天且 `cashDelta !== 0` 的本地现金事件。服务器状态同步、纯库存变化、估值刷新以及其他零现金变化事件不得进入该列表，也不得显示“资产状态更新”占位。

辅助说明固定为“当前设备现金记录”，明确该列表不是服务器完整账本。没有真实现金事件时显示“本周暂无现金收入或支出记录。”。

## 9. 状态栏趋势表达

总资产周变化只能使用一次方向表达：

- 上升：`↑ 本周 数值`；
- 下降：`↓ 本周 数值`；
- 无变化：`→ 本周 0`。

可见数值统一使用绝对值，不得同时显示下降箭头和负号。无障碍名称必须完整说明“本周资产上升／下降／无变化”。

## 10. 浏览器运行时回归测试

Playwright 必须至少覆盖：

1. `1684×931`：外层单轨，第一排两卡同排，第二排三卡同排，两排占满内容区；
2. `1440×900`：第二排标题和核心指标位于首屏；
3. `900×1000`：第一排单列、第二排两列，嵌套列表恢复可见溢出；
4. `1280×900`：展开侧栏时为紧凑布局，折叠侧栏后按实际内容宽度恢复两列和三列；
5. 空市场场景不渲染完整图表，空提醒、空资产事件与空挂单不产生内部滚动溢出；
6. 活动市场图宽度、高度、居中位置、刻度碰撞、图例碰撞和 SVG 边界全部通过真实几何检查；
7. 单边和双边订单簿分别验证中性空值、卖方状态色和市场状态说明；
8. 零现金同步事件不显示，一至三条现金事件不滚动；
9. 一至三条挂单不滚动，超过三条时必须启用内部滚动；
10. 所有概览核心容器 `scrollWidth` 不得大于 `clientWidth`；
11. `390×844`：四项市场指标必须处于同一水平行，标签不换行且市场卡不产生水平滚动。

测试必须比较真实 `getBoundingClientRect()`、计算后的网格轨道和滚动尺寸，不能只检查 CSS 源码中是否存在某个字符串。

## 11. 文件职责

- `src/styles/overview.css`：概览外层单轨、内部两排、容器查询和基础卡片结构；
- `src/styles/overview-polish.css`：紧凑图表比例、移动端四列市场指标、短列表滚动、辅助文字和本轮视觉完整性覆盖；
- `src/components/charts/PriceSparkline.tsx`：完整图与紧凑图的独立几何、刻度密度和图例；
- `src/pages/OverviewPage.tsx`：市场空值语义、订单簿说明、现金事件筛选和挂单滚动状态；
- `tests/browser/runtime.spec.ts`：真实浏览器几何、文字碰撞、断点、短列表和市场语义回归；
- `scripts/verify-overview-content.mjs`：确保上述样式、语义与运行时测试不可被静默移除；
- `src/styles/globals.css`：不得成为概览最终分栏或紧凑图表尺寸的唯一来源。
