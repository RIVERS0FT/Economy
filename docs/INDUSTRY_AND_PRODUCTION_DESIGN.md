> 状态：当前商品目录、工厂数量资产与生产设计基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-15
> 客户端状态版本：12
> 世界状态版本：8

市场交易以 `UNIFIED_ASSET_ORDER_BOOK_DESIGN.md` 为准；共享仓库以 `WAREHOUSE_EXPANSION_DESIGN.md` 为准；页面归属以 `PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 为准。

## 1. 不可回退决策

- 工厂资产按 `facilityTypeId + count` 保存。
- 不存在单座工厂实例、工厂实例 ID、实例列表、设施槽位、内部商品、内部容量或领取操作。
- 每种 `facilityTypeId` 只有一个工厂集群。
- 同类型工厂共享运行意图、三种顶层状态、周期进度、当前配方和启停操作。
- 新建、买入或解冻的数量在运行中集群的下一周期加入，不重置当前周期，也不获得部分周期产出。
- 工厂卖单冻结数量立即不参与生产。
- 同一玩家同时只能施工一座工厂。
- 产成品完成后直接进入共享仓库。
- 所有正式目录和生产参数由服务器提供，客户端不得维护另一套正式数值。

## 4. 商品与工厂目录扩展规则

新增商品和工厂时必须：

1. 保持 ID 唯一。
2. 为新增商品提供生产来源或明确系统来源。
3. 保证工厂输入、输出均引用正式商品目录。
4. 旧存档自动补齐新增商品库存、市场价格历史、基础流动性和工厂集群空状态。
5. 旧存档自动补齐新增商品库存时不得重置资金、既有库存、订单、工厂数量、仓库等级或排行榜数据。
6. 客户端按服务器目录动态渲染，不得写死 6 个商品 ID、6 种工厂或固定商品名称。

## 5. 工厂集群状态

```ts
interface FacilityGroup {
  facilityTypeId: string;
  count: number;
  participatingCount: number;
  pendingJoinCount: number;
  listedCount: number;
  availableCount: number;
  nextCycleCount: number;
  enabled: boolean;
  status: 'running' | 'stopped' | 'error';
  statusReason?: FacilityStatusReason;
  cycleStartedAt?: number;
  lifetimeOutput: number;
  activeRecipeId: string;
  pendingRecipeId?: string;
}
```

三种顶层状态：

- `running`：开启且当前完整周期正在运行。
- `stopped`：玩家关闭运行意图。
- `error`：运行意图仍开启，但资金、原料、仓库或可用数量条件不足。

`enabled` 表示玩家运行意图，不等同于当前是否正在生产。

## 6. 统一周期

```text
周期产量 = 单座周期产量 × participatingCount
周期成本 = 单座周期成本 × participatingCount
周期原料 = 单座周期原料 × participatingCount
周期净仓库增长 = max(0, 周期产量 - 周期原料)
```

每个完整周期在一个 SQLite 事务中：

1. 检查运行意图、可用数量、资金、原料、仓库空间和当前配方。
2. 扣除整个集群成本。
3. 扣除整个集群原料。
4. 产成品直接增加到共享仓库。
5. 增加 `lifetimeOutput` 和玩家生产统计。
6. 推进周期时间。
7. 在周期边界合并 `pendingJoinCount`。
8. 应用 `pendingRecipeId`，然后按新配方进入下一周期。

不得让同一集群的一部分工厂单独生产。

## 7. 下一周期加入

施工完成、工厂买单成交或卖单撤销解冻发生在运行期间时：

```text
当前参与数量保持不变
新增数量进入 pendingJoinCount
当前周期按旧数量结算
下一周期合并新增数量
```

新增数量不得重置 `cycleStartedAt`，不得获得部分周期产出。

运行中提交工厂卖单时，服务器先结算已经完成的完整周期，再从当前参与数量和待加入数量中冻结对应数量。冻结数量立即降低当前或下一周期产能。

## 8. 运行、停止与自动恢复

- 玩家打开开关后，`enabled = true`。
- 条件满足时状态为 `running`。
- 资金、原料、仓库或可用工厂不足时状态为 `error`。
- 条件恢复后自动恢复，从新的完整周期开始，不补算异常期间错过的周期。
- 玩家关闭开关后 `enabled = false` 且状态为 `stopped`，不得自动启动。
- 停止前只结算服务器时间上已经完成的完整周期；未完成周期不扣费、不耗料、不产出。

自动恢复必须由统一协调函数处理，不能由页面按钮或本地倒计时决定。

## 9. 持续生产与通用配方切换

- 工厂只有持续生产模式；开启后一直运行到玩家关闭或出现异常。
- 不存在定量生产、目标产量、生产计划、计划完成状态或计划自动保存。
- 每个工厂集群保存一个 `activeRecipeId`；多配方工厂可以选择其他正式配方，单配方工厂提交唯一配方时幂等成功。
- 停止或异常状态下切换配方立即写入 `activeRecipeId`，随后重新检查自动运行条件。
- 运行中切换配方只写入 `pendingRecipeId`；当前完整周期仍按旧配方结算，周期边界再提升为新的 `activeRecipeId`。
- 同一周期内连续切换只保留最后一次选择，不得重置 `cycleStartedAt`、重复产出或获得部分周期产量。
- 工厂作为资产交易时不携带卖方配方配置；买入数量加入买方既有工厂集群，并使用买方当前配方。
- 所有工厂卡统一显示“生产配方”选择器；多配方工厂可选择，单配方工厂显示唯一选项并禁用。

客户端开关只绑定 `group.enabled`，配方选择器只提交服务器正式配方 ID。旧 `/plan` 接口返回 `410 Gone`，不得静默接受旧生产计划。

## 10. 生产页面布局

页面顺序固定为：

```text
页面标题与运行／停止／异常／施工汇总
→ 共享仓库
→ 生产管理区：左侧建设新工厂，右侧工厂集群列表
```

共享仓库、建设新工厂和每张工厂集群卡属于生产页同一一级平面，统一使用 `.production-surface`：大于 `720px` 时四边内边距统一为 `16px`，不大于 `720px` 时统一为 `12px`。卡片类型、内容数量和容器宽度不得改变这一级外层内边距；容器查询只允许调整内部密度、换行和公式布局。

建设卡：

- 桌面独占生产管理区左列，宽度 `280px–320px`。
- 使用 `position: sticky`，只在生产管理区范围内常驻。
- 低于 960px 恢复普通文档流并位于工厂列表之前。

工厂卡：

- 工厂卡高度由内容自适应；大于 1380px 时右侧固定四列，同一网格行中的卡片等高，不同网格行可以使用不同高度；961px–1380px 使用自然高度单列。
- 移动端恢复自然高度；不得在单张卡内部增加纵向滚动条，也不得只提高某一种工厂的高度。
- 工厂名称和运行开关位于第一行同一水平框内，名称在左、开关在右。
- 运行中、已停止或异常状态独占第二行；异常原因完整写入该行状态标签。
- “运行中／下一周期加入／冻结中”数量摘要独占第三行，使用三列单行布局；移动端不得改为纵向堆叠。
- 运行状态不显示额外说明；剩余周期只在生产公式的进度区出现。
- 标题显示总数量，不显示独立“持有”项、数量卡片或“下一周期按若干座结算”辅助句。
- 工厂卡使用集群生产公式替代周期、产量、成本和原料四格规格区；正式配方仍来自服务器目录。
- 生产公式支持单输入、单输出、多输入和多输出；当前服务器仍使用单输入、单输出目录，客户端通过兼容数组结构动态渲染未来目录。
- 输入位于进度条上方左侧；无输入统一显示“无”；多输入使用独立项目和 `+` 分隔，每个输入项目下方显示对应当前可用库存。
- 输出位于进度条上方右侧；多输出使用独立项目和 `+` 分隔，不显示输出库存。
- 输入与输出之间固定显示周期 SVG 图标、配方周期数值、成本 SVG 图标和当前参与规模的集群运行成本；周期不乘以工厂规模，不得显示总工时。
- 生产进度条位于输入、输出和周期成本下方并横向铺满，作为生产过程唯一连接视觉；不得恢复箭头。
- 生产公式顶部的输入、输出和运行成本都按 `participatingCount` 计算；周期保持配方周期。多输入、多输出必须逐项乘以参与数量。生产进度条是公式容器最后一个可见元素，进度条下方不得显示当前周期、恢复运行、产出、成本或其他说明文字。
- 所有工厂卡在生产公式上方统一显示“生产配方”选择器；单配方工厂显示唯一选项并禁用，多配方工厂允许选择。
- 存在 `pendingRecipeId` 时只在配方选择器标题区显示“下一周期切换为：配方名称”，当前生产公式继续展示本周期配方；生产公式进度条下方不得显示 `nextCycleCount` 预览或其他说明。
- 输入、输出和运行成本按 `participatingCount` 计算；周期不乘以工厂规模。待生效配方和 `nextCycleCount` 只用于服务器状态与无障碍描述，不在进度条下方渲染说明。
- 标题元素盒子的左上锚点与共享仓库、建设卡一致；运行开关的 `44 × 44px` 点击区域不得将标题向下居中。
- 卡片主要内容向上排列，唯一弹性空白位于生产内容和市场入口之间；市场入口强制固定在卡片底部。
- 页面不得显示生产模式、目标产量、保存计划或持续／定量状态卡。
- 底部只显示“前往市场交易该工厂 →”，不得同时显示额外白色说明。

## 12. 防回退

不得恢复：

- 单座工厂实例或设施槽位；
- 工厂内部仓库和领取产成品；
- 固定价格工厂挂牌市场；
- 手动修复异常后必须再次点击启动；
- 恢复定量生产、目标产量、生产计划或计划完成状态；
- 运行中切换配方立即重置周期或中途改变当前周期产出；
- 把一个玩家的同类工厂集群拆分成多种并行配方；
- 新数量中途加入当前周期；
- 让共享仓库、建设卡和工厂集群在同一断点使用不同一级外层内边距，或按工厂卡自身宽度切换一级 `padding`；
- 让同一网格行中的工厂卡高度不一致、恢复瀑布流，或把市场入口从卡片底部移开；
- 把状态标签放回名称和开关所在第一行，或把数量摘要移出第三行；
- 顶部“正常生产中”重复说明；
- 独立持有／参与／待加入／冻结数量卡；
- “下一周期按若干座结算”辅助句；
- 移动端把运行中／下一周期加入／冻结中改为纵向排列；
- 恢复周期、产量、成本和原料四格规格区；
- 在生产公式中恢复箭头、可见“周期”标签或可见“运行成本”标签；
- 把输入、中央周期成本和输出的左右关系改为无固定语义的自由排列；
- 丢失多输入、多输出、逐输入库存、`+` 分隔或按集群规模展示多输出的能力；
- 在生产进度条下方恢复当前周期、恢复运行、产出、成本或其他可见说明文字；
- 静态计划模式文本、手动“保存计划”按钮或持续／定量状态小卡片；
- 隐藏单配方工厂的配方选择器，或允许单配方选择器提交其他配方；
- 客户端维护正式配方、成本或周期；
- 让生产周期乘以工厂规模、显示总工时，或让卡片四边使用不同内边距；
- 恢复白色市场说明“在统一订单簿中买卖该工厂”或旧入口“前往市场 →”。
