# Economy 共享仓库与扩容设计

> 状态：当前共享仓库容量、买单预占、生产空间与扩容规则基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-12  
> 客户端状态版本：10  
> 世界状态版本：6

生产与工厂集群以 `INDUSTRY_AND_PRODUCTION_DESIGN.md` 为准；页面归属以 `PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 为准。

## 1. 核心规则

- 所有商品共用一个玩家仓库。
- 仓库等级、容量、占用和剩余空间均为服务器权威状态。
- 完整仓库管理入口只位于生产页面。
- 工厂产成品直接进入共享仓库。
- 工厂没有内部商品、内部容量或领取操作。
- 商品买单预占容量；工厂买单不预占仓库。
- 仓库空间不足时，开启中的工厂进入异常，空间恢复后自动从新周期运行。
- 扩容费用由系统回收并计入 `stats.systemSinks`。
- 仓库内容只展示可用或冻结数量大于零的商品，不提供全部商品视图。

## 2. 等级、容量与费用

```text
初始等级：1
初始容量：500
每级增加：250
最高等级：12
最高容量：3250
capacity(level) = 500 + (level - 1) × 250
cost(level) = 150 × level²
```

最高等级的 `warehouseUpgradeCost` 为 `null`。扩容必须在 SQLite 事务中完成并使用 `Idempotency-Key`。

## 3. 权威占用

```text
实物库存 = Σ(各商品可用数量 + 冻结数量)
买单预占 = Σ(当前玩家 open/partial 商品买单剩余数量)
已用容量 = 实物库存 + 买单预占
剩余容量 = max(0, 当前容量 - 已用容量)
```

不计入买单预占：

- 工厂买单；
- 商品卖单；
- 已完成或已撤销订单；
- 其他玩家订单。

API 返回：

```ts
warehouseStoredQuantity: number;
warehouseReservedQuantity: number;
warehouseUsedCapacity: number;
warehouseAvailableCapacity: number;
```

客户端不得重新计算正式仓库占用。

## 4. 下单约束

提交商品买单前，服务器必须确认剩余容量可以覆盖订单的未成交数量。

```text
最大商品买入数量 =
min(仓库剩余容量, floor(可用资金 / 限价))
```

部分成交后，已成交商品从预占转为实物库存；未成交部分继续预占。撤单释放剩余预占。

## 5. 生产约束与自动恢复

一个集群完整周期的净增长：

```text
集群周期产量 = 单座产量 × participatingCount
集群周期原料 = 单座原料 × participatingCount
netStoragePerCycle = max(0, 集群周期产量 - 集群周期原料)
```

加工集群可以利用同周期原料消耗释放的空间。服务器必须以整个集群为单位检查，不能只运行部分工厂。

仓库不足时：

```text
enabled = true
status = error
statusReason = warehouse_full
```

扩容、出售商品、撤销商品买单或其他释放空间的操作完成后，统一协调函数重新检查。条件满足时从新的完整周期自动恢复，不补算异常期间错过的周期。玩家关闭开关后不得自动恢复。

## 6. 生产页面布局

共享仓库是生产页顶部的单一面板，位于建设新工厂之前。

桌面内部采用：

```css
grid-template-columns: minmax(220px, 1fr) minmax(0, 3fr);
```

左侧只展示：

- 等级和容量状态；
- 已用容量、总容量和进度；
- 实物库存、买单预占、剩余容量；
- 下一等级容量、容量增量、费用和扩容按钮。

右侧从服务器商品目录中过滤并展示有库存商品：

```text
显示条件 = available > 0 或 frozen > 0
库存总量 = available + frozen
```

仓库不再区分“有库存”和“全部商品”，不得显示筛选按钮，也不得在空仓状态提供“查看全部商品”。

每张商品卡采用与资产摘要卡一致的紧凑三层信息结构，但不显示估值：

1. 统一 `ProductIconLabel` 商品 SVG 与商品名称；
2. 醒目的“库存 N”主值；
3. 简化文本“可用 N · 冻结 N”。

整张卡进入对应商品市场。不得显示估值、买一价、最近成交价、单项容量、买单预占或直接出售表单。

宽屏四列、普通桌面三列、平板和手机两列，低于 420px 才单列。低于 960px 时左右区域改为上下布局。

空仓时只显示：

- `仓库中暂无商品`；
- `生产或买入商品后，商品会显示在这里。`

## 7. 页面职责

- `WarehouseUpgradeCard` 只能由 `ProductionPage` 渲染。
- 状态栏可以显示只读仓库剩余。
- 市场页可以显示仓库剩余和买单预占，但不得扩容。
- 资产页负责资产汇总和本地资产变化，不得重复显示逐商品库存与估值卡。
- 设置页不得显示仓库经营摘要。

## 8. 防回退

不得恢复：

- 每个商品独立容量；
- 工厂内部仓库或领取按钮；
- 客户端容量公式作为权威结果；
- 设置页或市场页扩容入口；
- 六个重复的大型仓库指标卡；
- 仓库满后要求玩家手动重新开启工厂；
- 商品估值或价格进入仓库商品卡；
- 仓库“有库存／全部商品”筛选、全部商品视图或“查看全部商品”空状态按钮；
- 没有商品 SVG 的纯文字仓库卡；
- 在资产页恢复逐商品库存与估值卡。
