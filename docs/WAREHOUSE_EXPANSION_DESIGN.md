# Economy 共享仓库与扩容设计

> 状态：当前共享仓库容量、买单预占、生产空间与扩容规则基线
> 适用项目：`RIVERS0FT/Economy`
> 更新时间：2026-07-17
> 客户端状态版本：14
> 世界状态版本：11

生产与工厂集群以 `INDUSTRY_AND_PRODUCTION_DESIGN.md` 为准；页面归属以 `PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 为准。

## 1. 核心规则

- 所有商品共用一个玩家仓库。
- 仓库等级、容量、占用和剩余空间均为服务器权威状态。
- 完整仓库管理入口只位于生产页面。
- 工厂产成品直接进入共享仓库。
- 工厂没有内部商品、内部容量或领取操作。
- 商品买单预占容量；工厂买单不预占仓库。
- 仓库空间不足时，开启中的工厂进入异常，空间恢复后自动从新周期运行。
- 扩容费用由系统回收并计入 `stats.systemSinks`。
- 仓库内容只展示可用或冻结数量大于零的商品，不提供全部商品视图。
- 仓库没有玩家可见的最高等级，等级 12 后仍可继续扩容。

## 2. 无限等级、容量与费用

```text
初始等级：1
初始容量：500
当前等级 L 升到 L + 1 的容量增量：250 + 50 × (L - 1)
升级费用：150 × L²
```

标准等级容量：

| 等级 | 总容量 | 升到下一级增加 |
|---:|---:|---:|
| 1 | 500 | 250 |
| 2 | 750 | 300 |
| 3 | 1050 | 350 |
| 4 | 1400 | 400 |
| 5 | 1800 | 450 |
| 6 | 2250 | 500 |

闭式公式：

```text
capacity(level) =
500
+ 250 × (level - 1)
+ 25 × (level - 1) × (level - 2)
```

`warehouseUpgradeCost` 只在计算超出安全整数范围时返回 `null`，不代表满级。界面不得显示 `/12`、满级、最高容量或“已达最高等级”。扩容必须在 SQLite 事务中完成并使用 `Idempotency-Key`。

## 3. 旧存档迁移

迁移必须满足：

```text
新等级 = max(旧等级, 能覆盖旧容量的最小公式等级)
新容量 = max(旧容量, 新等级公式容量)
```

不得降低已有容量。下一次扩容必须在当前实际容量上增加该等级对应的完整增量，不能因为旧容量高于公式值而出现增加量为 0。发生非安全整数、费用溢出或容量溢出时，服务器拒绝本次扩容，但不得将其表现为玩家达到最高等级。

## 4. 权威占用

```text
实物库存 = Σ(各商品可用数量 + 冻结数量)
买单预占 = Σ(当前玩家 open/partial 商品买单剩余数量)
已用容量 = 实物库存 + 买单预占
剩余容量 = max(0, 当前容量 - 已用容量)
```

不计入买单预占：

- 工厂买单；
- 商品卖单；
- 已完成或已撤销订单；
- 其他玩家订单。

API 返回：

```ts
warehouseLevel: number;
warehouseUpgradeCost: number | null;
warehouseNextCapacity: number;
warehouseNextCapacityIncrease: number;
warehouseStoredQuantity: number;
warehouseReservedQuantity: number;
warehouseUsedCapacity: number;
warehouseAvailableCapacity: number;
```

客户端不得重新计算正式仓库占用。

## 5. 下单约束

提交商品买单前，服务器必须确认剩余容量可以覆盖订单的未成交数量。

```text
最大商品买入数量 =
min(仓库剩余容量, floor(可用资金 / 限价))
```

部分成交后，已成交商品从预占转为实物库存；未成交部分继续预占。撤单释放剩余预占。

## 6. 生产约束与自动恢复

一个集群完整周期的净增长：

```text
集群周期产量 = 单座产量 × participatingCount
集群周期原料 = 单座原料 × participatingCount
netStoragePerCycle = max(0, 集群周期产量 - 集群周期原料)
```

加工集群可以利用同周期原料消耗释放的空间。服务器必须以整个集群为单位检查，不能只运行部分工厂。

仓库不足时：

```text
enabled = true
status = error
statusReason = warehouse_full
```

扩容、出售商品、撤销商品买单或其他释放空间的操作完成后，统一协调函数重新检查。条件满足时从新的完整周期自动恢复，不补算异常期间错过的周期。玩家关闭开关后不得自动恢复。

## 7. 生产页面布局

共享仓库是生产页顶部的单一面板，位于生产管理区之前。

共享仓库与建设新工厂、工厂集群属于生产页同一一级平面，统一使用 `.production-surface`。大于 `720px` 时四边内边距为 `16px`，不大于 `720px` 时为 `12px`；仓库分栏、商品数量、等级标签和响应式布局不得单独改变这一级外层内边距或标题左上锚点。

桌面内部采用：

```css
grid-template-columns: minmax(220px, 1fr) minmax(0, 3fr);
```

左侧只展示：

- 等级和容量状态；
- 已用容量、总容量和进度；
- 实物库存、买单预占、剩余容量；
- 下一等级容量、容量增量、费用和扩容按钮。

右侧从服务器商品目录中过滤并展示有库存商品：

```text
显示条件 = available > 0 或 frozen > 0
```

“仓库内容”标题下不显示“X 种商品有库存”等商品数量说明。仓库不再区分“有库存”和“全部商品”，不得显示筛选按钮，也不得在空仓状态提供“查看全部商品”。

每张商品卡采用固定三层信息结构，但不显示库存总量或估值：

1. 统一 `ProductIconLabel` 商品 SVG 与商品名称；
2. 醒目的“可用 N”，沿用原库存主值字号；
3. 弱化的“冻结 N”，沿用原第三层辅助文字字号。

冻结数量为零时仍显示“冻结 0”，保证商品卡结构和高度一致。整张卡进入对应商品市场。不得显示“库存 N”总量行、估值、买一价、最近成交价、单项容量、买单预占或直接出售表单。

### 7.1 商品网格密度

仓库商品网格使用 `.warehouse-content` 的容器查询，而不是只按浏览器视口判断。正式断点为：

| 仓库内容区宽度 | 每行商品卡 |
|---:|---:|
| `< 360px` | 2 列 |
| `360px–559px` | 3 列 |
| `560px–759px` | 4 列 |
| `760px–959px` | 5 列 |
| `≥ 960px` | 6 列 |

即正式密度为 2／3／4／5／6 列。商品卡最小高度为 `84px`，卡片内边距为 `8px`，网格间距为 `8px`。商品名称、可用数量和冻结数量均保持单行省略，不得造成横向溢出。

低于页面 `960px` 时仓库管理区和内容区从左右分栏改为上下排列，但内容区仍根据自身容器宽度选择列数。不得恢复“宽屏固定四列、普通桌面三列、移动固定两列”的视口规则，也不得在 `420px` 或其他移动断点强制单列。

空仓时只显示：

- `仓库中暂无商品`；
- `生产或买入商品后，商品会显示在这里。`

## 8. 页面职责

- `WarehouseUpgradeCard` 只能由 `ProductionPage` 渲染。
- 状态栏可以显示只读仓库剩余。
- 市场页可以显示仓库剩余和买单预占，但不得扩容。
- 资产页负责资产汇总和本地资产变化，不得重复显示逐商品库存与估值卡。
- 设置页不得显示仓库经营摘要。

## 9. 防回退

不得恢复：

- 固定最高等级或固定最高容量；
- 每级固定增加 250 的旧公式；
- `warehouseMaxLevel` 客户端字段；
- 每个商品独立容量；
- 工厂内部仓库或领取按钮；
- 客户端容量公式作为权威结果；
- 设置页或市场页扩容入口；
- 六个重复的大型仓库指标卡；
- 仓库满后要求玩家手动重新开启工厂；
- 商品估值或价格进入仓库商品卡；
- 仓库“有库存／全部商品”筛选、全部商品视图或“查看全部商品”空状态按钮；
- “X 种商品有库存”说明、独立“库存 N”总量行或把可用与冻结重新合并到同一行；
- 在 `420px` 或其他移动断点把仓库商品网格恢复为单列；
- 恢复固定桌面四列、普通桌面三列或只由视口宽度决定列数；
- 删除 2／3／4／5／6 列容器查询，或把商品卡恢复到 `92px` 高和 `12px` 内边距；
- 让共享仓库在同一生产页断点使用不同于建设卡和工厂集群的一级外层内边距；
- 没有商品 SVG 的纯文字仓库卡；
- 在资产页恢复逐商品库存与估值卡。
