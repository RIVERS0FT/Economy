# Economy 人口需求与基础商品买单设计

> 状态：多人市场需求侧专题基线  
> 适用范围：网页端多人交易版本  
> 更新时间：2026-07-10

本文件描述系统人口需求模型。人口只作为聚合需求和有限货币投放机制存在，不表现城市空间，不模拟具体居民。

## 1. 设计目的

共享市场不能只依赖玩家相互买卖。玩家数量较少、生产者多于交易者或服务器刚开放时，可能出现买单不足、商品无法出售和经济循环停滞。

系统通过聚合人口需求生成有限基础商品买单，为市场提供基础消费需求。

人口需求不是固定价格商店，也不是无限回收系统。所有买单都必须受到需求量、周期预算、市场价格、供应量和历史满足率限制。

## 2. 核心经济循环

```text
玩家拥有生产设施
-> 设施生产基础商品
-> 玩家向统一市场提交卖单
-> 系统人口模型计算消费需求和预算
-> 人口需求形成基础商品买单
-> 统一订单簿撮合
-> 商品成交后退出流通
-> 卖方获得货币
-> 玩家继续生产、交易或调整资产配置
```

人口需求负责基础需求侧，玩家生产设施负责供给侧。

## 3. 核心原则

### 3.1 有限需求

每个周期都有明确的目标需求和消费预算。需求满足或预算用尽后，不再继续购买。

### 3.2 不保证玩家盈利

人口只会在可承受价格范围内购买。玩家报价过高时，卖单可以长期无法成交。

### 3.3 共用商品订单簿

人口买单进入与玩家相同的基础商品订单簿，遵循价格优先、同价时间优先、部分成交和订单过期规则。

### 3.4 商品退出流通

人口购买的基础商品视为已经消费，不进入系统库存，也不能再次出售。

### 3.5 可解释但不可完全预测

页面可以展示需求强弱、预算使用、满足率和趋势，但不公开下一批每个价格档位的精确数量，避免无风险抢占系统需求。

### 3.6 不参与生产资产交易

人口不会购买、持有或出售生产设施，也不能用于产权转移或玩家间变相转账。

## 4. 聚合需求状态

系统维护的聚合状态包括：

- 有效消费规模
- 周期可用预算
- 基础商品目标需求
- 剩余需求
- 已满足需求
- 需求满足率
- 当前商品参考价
- 正常承受价格区间
- 紧缺承受价格区间
- 市场供应量
- 上一周期成交量与均价
- 当前货币投放目标
- 需求趋势或事件修正

MVP 不逐个模拟居民，不维护居民账户、职业、家庭或购物路径。

## 5. 消费周期

MVP 建议一个需求周期为 5 分钟。

每周期执行：

1. 计算可用消费预算。
2. 计算基础商品目标需求。
3. 读取上一周期需求满足率和当前供应量。
4. 读取当前商品参考价与订单簿状态。
5. 生成少量价格阶梯买单。
6. 与玩家商品卖单实时撮合。
7. 周期结束时过期或重新评估未成交人口订单。
8. 记录消费量、均价、预算使用量和需求满足率。

买单生成后可立即成交，不需要等到周期结束。

## 6. 需求与预算

可以使用以下基础模型：

```text
周期目标需求 = 基础需求规模 × 需求修正系数
```

```text
周期消费预算 = 基础预算 × 预算修正系数
```

需求修正系数可以考虑：

- 上一周期需求满足率
- 当前市场供应量
- 当前参考价
- 连续短缺或连续过剩
- 系统货币投放目标
- 有限活动或事件修正

人口所有成交总量不能超过剩余目标需求，成交总额不能超过剩余预算。

未使用预算和未满足需求可以少量结转，但必须设置上限，避免长期缺货后一次性产生极端需求。

## 7. 价格承受能力

基础商品需求模型维护：

- 系统基础参考价
- 市场参考价
- 正常承受价格
- 紧缺承受价格
- 最高承受价格

价格越高，人口愿意购买的数量越少。

```text
可购买数量 = 周期剩余需求 × 价格需求系数
```

初始测试示例：

| 基础商品价格 | 需求系数 |
|---:|---:|
| 3 | 1.20 |
| 4 | 1.10 |
| 5 | 1.00 |
| 6 | 0.80 |
| 7 | 0.55 |
| 8 | 0.30 |
| 9 及以上 | 0 |

数值仅用于第一轮测试，不代表正式价格范围。

## 8. 人口买单生成

人口每周期只生成少量价格阶梯，不按消费规模生成大量独立订单。

以参考价 5 为例：

| 买价 | 数量 | 说明 |
|---:|---:|---|
| 5 | 8 | 即时需求 |
| 4 | 7 | 价格敏感需求 |
| 3 | 5 | 低价补充需求 |

所有买单共享本周期剩余预算和剩余需求。

人口订单在统一市场中显示为“人口需求”或“系统消费”，资产类型为基础商品。

## 9. 撮合与结算

人口买单成交后：

```text
玩家冻结库存减少
-> 玩家获得成交货币
-> 人口剩余预算减少
-> 人口剩余需求减少
-> 基础商品退出流通
-> 生成成交记录与资金流水
```

人口订单支持部分成交，但不能超额使用同一周期预算或需求。

结算必须由服务器原子执行，避免预算重复使用或商品重复成交。

## 10. 需求满足率

```text
需求满足率 = 已消费数量 / 周期目标需求
```

| 满足率 | 状态 |
|---:|---|
| 90% 以上 | 供应充足 |
| 60%～89% | 供应正常 |
| 30%～59% | 供应紧张 |
| 30% 以下 | 严重短缺 |

满足率可以影响：

- 下一周期目标需求
- 下一周期预算
- 可接受价格区间
- 页面趋势提示
- 玩家生产和资产配置判断

MVP 只用于需求调整和趋势展示，不加入复杂城市事件或惩罚。

## 11. 供给过剩

当设施产量和市场卖单远高于人口需求时：

- 人口不会无限增加购买量。
- 低价卖单优先成交。
- 高价卖单可能无法成交。
- 商品价格可能自然下降。
- 设施内部库存和玩家库存可能积压。
- 玩家需要暂停生产、降价、减少扩张或等待需求恢复。

即使价格很低，额外消费也必须有上限。

## 12. 商品短缺

当人口需求长期无法满足时：

- 少量未满足需求可以结转。
- 最高承受价可以有限提高。
- 页面显示供应紧张或严重短缺。
- 更高利润可能吸引玩家建造、收购或升级设施。

短缺不能直接生成玩家商品或生产设施。

## 13. 人口需求与生产资产价值

人口需求会间接影响生产设施价值：

- 需求旺盛时，商品价格和设施预期收益可能提高。
- 供给过剩时，设施可能被暂停或以较低价格挂牌。
- 设施交易价格由玩家决定。
- 设施系统估值可以参考正常商品参考价和产能，但不能保证利润。
- 单次异常人口成交不能直接推高商品参考价或设施估值。

## 14. 货币发行

人口成交会向经济中注入新货币，因此需要持续监测：

- 人口消费发行货币量
- 玩家工作发行货币量
- 设施建造、运营、升级和槽位扩充回收量
- 市场手续费回收量
- 玩家持有货币增长速度
- 全服货币净变化
- 基础商品参考价与成交价格
- 设施成交价格
- 玩家总资产增长速度

人口预算是有限的系统货币发行来源，不能无限增长。

## 15. 玩家可见信息

主页面或市场页面的人口需求摘要可以展示：

- 本周期预计需求
- 本周期已消费数量
- 剩余需求
- 预算使用进度
- 需求满足率
- 下次需求刷新时间
- 当前价格承受范围
- 供应状态
- 下一周期趋势

不公开下一批每档买单的精确数量。

## 16. 防止套利与操纵

- 不设置固定永久保底价。
- 所有人口订单共享有限周期预算。
- 成交后立即扣减剩余预算和需求。
- 高于最高承受价的卖单不会被人口购买。
- 极低价格也不能产生无限额外需求。
- 人口订单不能购买生产设施。
- 人口成交不能用于玩家间变相转账。
- 明显自成交、关联异常成交和极小数量拉价不应直接影响参考价。
- 用于任务或排行榜的有效人口成交量可以设置周期上限。

## 17. 2 核 2G 约束

- 人口需求按聚合周期计算。
- MVP 只维护一个需求模型和一种基础商品。
- 每周期只生成 3～5 档人口买单。
- 不模拟具体居民、账户、职业、家庭或购物路径。
- 需求图表和趋势预测由客户端计算。
- 服务器只维护正式预算、需求、订单和成交结果。
- 服务器压力较高时可以降低需求刷新频率，但不能重复发行预算。

## 18. MVP 初始建议

| 项目 | 建议值 |
|---|---:|
| 消费周期 | 5 分钟 |
| 基础商品目标需求 | 每周期 20 个 |
| 人口买单档位 | 3 档 |
| 基础参考价 | 5 货币 |
| 未满足需求结转上限 | 1 个周期基础需求 |
| 未使用预算结转上限 | 1 个周期基础预算 |

所有数值只用于第一轮经济测试。

## 19. 验收标准

1. 人口需求不依赖城市页面或城市空间表现。
2. 人口需求以聚合模型运行，不模拟具体居民。
3. 人口买单进入与玩家相同的商品订单簿。
4. 人口买单遵循价格和时间优先规则。
5. 人口预算和需求都具有周期上限。
6. 人口购买后的商品退出流通。
7. 人口不会购买生产设施。
8. 人口不会提供无限数量或永久固定价格的保底收购。
9. 人口成交货币发行量可统计和审计。
10. 人口买单支持部分成交但不能超额使用预算。
11. 高价卖单可以长期不成交。
12. 供给过剩和商品短缺都能形成可解释的市场结果。
13. 人口需求不会直接改变玩家总资产或排行榜。
14. 所有正式需求、预算和成交结果以服务器状态为准。

## 20. 当前开发决策

- 人口系统只作为聚合需求模型存在。
- 不表现城市空间或具体居民。
- 人口需求为基础商品提供有限买单。
- 人口买单进入统一商品订单簿。
- 人口购买后的商品退出流通。
- 人口不会购买生产设施。
- 人口需求不保证玩家盈利。
- 人口预算是受限制、可统计的货币发行来源。
