# Economy 排行榜设计

> 状态：排行榜周期、成绩、排序、奖励、迁移与审计的权威规则
> 适用项目：`RIVERS0FT/Economy`
> 更新时间：2026-07-20
> 排行榜周期状态版本：1
> 交易榜规则版本：2

## 1. 文档优先级

本文件是排行榜功能的权威设计文档。若本文件与 `PRODUCT_AND_GAMEPLAY_DESIGN.md` 中旧的排行榜描述冲突，以本文件为准；旧文档中“交易积分封顶”“同交易对手每日上限”以及 `Asia/Taipei` 周期标识均属于已废止规则，不得恢复。

## 2. 固定榜单结构

排行榜页面固定同时展示四列：

1. 财富榜；
2. 增长榜；
3. 生产榜；
4. 交易榜。

窄屏保持四列并允许横向滚动，不拆成四个路由。每个榜单返回前 10 名和当前玩家；当前玩家不在前 10 名时仍返回真实名次。

## 3. 周期与奖励

- 财富榜实时更新，不参与周结算，也不发放宝石。
- 增长榜、生产榜和交易榜使用北京时间 `Asia/Shanghai`。
- 每个周期从北京时间周一 00:00:00 开始，到下周一 00:00:00 结束。
- 成交记录计入条件为 `startsAt <= fill.createdAt < endsAt`。
- 三个周榜前三名分别获得 **30 / 20 / 10** 宝石。
- 同一玩家可以在多个周榜分别获奖。
- 只有成绩大于 0 的玩家可以获奖，不足三名时剩余奖励不发放。
- 首个不完整周只记录排名，不发放奖励。
- 周历史最多保留 52 期，同一 `periodKey` 只能结算一次。

北京时间和台北时间当前同为 UTC+8，但权威时区标识必须固定为 `Asia/Shanghai`，页面、类型、接口和测试不得继续返回 `Asia/Taipei`。

## 4. 排序规则

所有榜单依次使用以下条件形成唯一名次：

1. 成绩降序；
2. 榜单次级指标降序；
3. 榜单第三级指标降序；
4. 注册时间升序；
5. 内部用户 ID 升序。

内部用户 ID 不对普通玩家公开。

## 5. 财富榜

财富榜使用服务器权威资产和最近一次统一订单簿真实成交价：

```text
现金资产 = 可用资金 + 冻结资金
商品估值 = Σ(商品总数量 × 最近一次商品订单簿真实成交价)
工厂估值 = Σ(工厂总数量 × 最近一次工厂订单簿真实成交价)
总资产 = 现金资产 + 商品估值 + 工厂估值
```

模拟历史、未成交挂单、基础价、工厂 `systemValue` 和拍卖价格不得参与财富榜估值。从未发生真实成交的资产估值为 0。

## 6. 增长榜

增长榜继续使用固定经营资产，不使用市场成交价：

```text
经营资产 = 可用与冻结资金
         + Σ(商品总数量 × 商品基础价)
         + Σ(工厂总数量 × 工厂 systemValue)
本周经营增长 = 当前经营资产 - 周初经营资产 - 本周外部赠送资金
```

礼品兑换、宝石商店兑换和管理员直接发放的普通货币属于外部赠送资金，必须从增长成绩中扣除。

## 7. 生产榜

```text
生产积分 += 实际产出数量 × 商品基础价
```

只有服务器确认完成的生产周期计分。未完成周期、资金不足、原料不足或仓库不足均不计分。次级指标为实际产出数量。

## 8. 交易榜

### 8.1 成绩定义

交易榜只统计卖方在当前周期内通过统一订单簿完成的实际卖出成交额：

```text
实际卖出成交额 += fill.quantity × fill.price
```

- 商品订单和工厂订单均统计。
- 玩家卖给玩家的真实成交统计。
- 玩家卖给市场需求的真实成交统计。
- 使用手续费扣除前的成交总额；手续费仍按市场规则正常扣除。
- 买方不获得同一笔成交的交易榜成绩。
- 交易榜单位为普通货币，不显示为“分”。

### 8.2 不计入项目

以下内容不得计入交易榜：

- 未产生 `fill` 的挂单；
- 撤单的未成交剩余数量；
- 已过期或被系统取消但未成交的数量；
- 买入成交额；
- 初始化模拟行情、参考价格或报价；
- 拍卖成交；
- 自成交或无效成交。

订单在撤销前已经生成的真实 `fill` 仍正常计入。判断依据是服务器成交记录，而不是订单最终状态。因此，卖单 100 件、实际成交 35 件后撤销剩余 65 件时，只计入 35 件对应的实际成交额。

### 8.3 去重与次级指标

- 服务器使用成交记录 ID 去重。
- 同一成交不能因为买卖双方各保存一份记录、轮询、服务重启或重复处理而重复计入。
- 次级指标为实际成交笔数。
- 第三级指标为不同玩家买家数量。
- 市场需求成交不增加不同玩家买家数量。

交易榜不再使用以下旧限制：

- 商品成交价按基础价 3 倍封顶；
- 工厂成交价按 `systemValue` 2 倍封顶；
- 同一卖家和玩家买家每日最多计入 10,000 分。

任何修改不得恢复上述封顶规则。

## 9. 规则迁移

周期状态保存 `tradingRuleVersion`。从旧交易榜规则升级到版本 2 时：

1. 保留当前周期边界、增长榜、生产榜、财富数据和历史结算记录；
2. 从玩家累计交易统计中移除当前周期旧交易积分和旧成交笔数；
3. 清空当前周期旧交易成绩、旧成交去重集合和旧交易对手每日上限状态；
4. 使用服务器仍保存的当前周期卖单 `fills` 按实际成交额重新计算；
5. 完成迁移后写入 `tradingRuleVersion = 2`；
6. 后续轮询和重启不得再次执行迁移或重复计入成交。

迁移只改变排行榜统计口径，不得改变玩家资金、商品、工厂、冻结资产、订单成交结果或手续费。

## 10. 事务与审计

周边界处理、排名固定、宝石发放、历史写入和下一周期初始化必须在服务器 SQLite 世界事务中完成。排行榜可以延迟刷新，但不得使用客户端日志、本地缓存或客户端预测决定成绩和奖励。

服务器内部累计维护：

- 生产积分；
- 实际卖出成交额；
- 实际卖出成交笔数；
- 市场需求发行额；
- 系统回收；
- 商店兑换货币累计值；
- 排行榜奖励宝石累计值。

这些统计用于排行榜和审计，不得替代玩家资产余额。
