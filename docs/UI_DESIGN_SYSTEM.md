# Economy UI 设计系统

> 状态：当前视觉、共享组件、响应式与可访问性实现基线  
> 适用项目：`RIVERS0FT/Economy`  
> 当前平台：网页端  
> 更新时间：2026-07-13

产品和页面职责分别以 `PRODUCT_AND_GAMEPLAY_DESIGN.md`、`PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 为准；应用外壳玻璃材质以 `LIQUID_GLASS_CHROME_DESIGN.md` 为准。

## 1. 界面目标

- 深色、稳定、专业的金融与产业经营氛围；
- 信息密度高但层级清晰；
- 玩家可见固定文案统一使用中文；
- 绿色表示主要操作、增长、买入和正常；
- 红色表示卖出、损失、危险和错误；
- 金色表示价格、等待和施工；
- 蓝色表示信息；
- 桌面适合持续观察，移动端适合单手操作并尊重安全区；
- 标题只保留主标题和必要说明，不显示英文眉题或重复小字。

## 3. 共享 React 组件

业务页面优先使用：

- `PageLayout`
- `Panel`
- `WidgetHeading`
- `Button`
- `StatusTag`
- `MetricCard`
- `DataList`
- `DataRow`
- `SwitchControl`
- `ToggleField`
- `ScrollableTable`
- `EmptyState`

`SwitchControl` 是布尔开关的唯一 React 基础组件，`.ui-switch` 是唯一视觉实现。不得新增工厂开关、音乐开关或设置开关的平行 CSS。

页面标题从 `h1` 开始，卡片标题从 `h2` 开始。`PageLayout` 和 `WidgetHeading` 不提供 `eyebrow` 参数。

## 4. 开关焦点环与点击区域

`SwitchControl` 的可访问点击区域为至少 `44 × 44px`，视觉轨道仍保持紧凑。

焦点环应位于轨道伪元素外侧，并与轨道形状一致。不得恢复围绕整个 44px 点击区域的额外的大圆环。

必须保留：

- `.ui-switch:focus`
- `.ui-switch:focus-visible`
- `.ui-switch:focus-visible::before`
- 明确的 `outline-offset`

工厂开关始终对齐卡片右上角。异常状态通过 `StatusTag` 文字表达，不得创建异常开关变体。

## 5. 统一 SVG 图标体系

应用外壳图标来自 `src/components/icons/GameIcons.tsx`，商品语义图标来自 `src/components/icons/ProductIcons.tsx`：

- 状态栏和桌面／移动导航不得继续使用 Unicode 字符、Emoji 或字体符号作为图标；
- 商品图标不得使用字符、字母缩写或 Emoji 作为占位；
- 工厂资产标签必须使用 `GameIcons.tsx` 的 `FactoryIcon`，不得使用机械商品的齿轮图标或 `⚙` 字符；
- `FactoryIcon` 使用厂房与烟囱轮廓，`machinery` 商品继续使用齿轮机械轮廓，两者不得共用路径；
- 工厂生产公式的周期使用 `GameIcons.tsx` 的 `CycleIcon`，成本使用 `CreditsIcon`，输入库存使用 `WarehouseIcon`；不得用 Emoji 替代；
- 所有图标使用统一 `24 × 24` `viewBox`、`currentColor` 和圆角描边；
- SVG 根节点必须带 `.game-icon`，商品 SVG 额外带 `.product-icon`；
- 图标本身使用 `aria-hidden="true"` 和 `focusable="false"`；
- 导航配置只保存 `id` 与中文 `label`，不得重新加入字符型 `icon` 字段；
- 桌面侧栏和移动底栏复用同一套导航 SVG，不维护两套图标；
- 图标颜色必须继承 `currentColor`；
- `ProductIconLabel` 是商品图标与名称的统一并排结构。

### 5.1 商品 SVG 图标目录

当前 12 种正式商品必须在 `ProductIcons.tsx` 中各有一个独立、可辨识的本地内联 SVG。

- `grain`：麦穗；
- `timber`：原木；
- `ore`：矿石；
- `crude-oil`：油滴；
- `flour`：面粉袋；
- `lumber`：堆叠木板；
- `steel`：工字钢；
- `plastic`：塑料瓶；
- `food`：食物碗；
- `furniture`：椅子；
- `machinery`：齿轮机械；
- `electronics`：芯片。

服务器未来返回未知商品 ID 时必须使用统一包装箱 SVG 回退，页面仍按服务器数组动态渲染，不得隐藏商品。

市场商品标签、概览商品行情、仓库商品卡、商品订单和商品资产变动必须使用相同的 `ProductIcon`／`ProductIconLabel`。

## 6. 设计令牌、按钮与表单

- 基础视觉必须使用 `design-system.css` 中的颜色、文字、间距、圆角、阴影和控件高度令牌。
- 相同语义在所有页面使用相同颜色，颜色不能作为状态的唯一表达。
- 桌面卡片使用统一 `--radius-card`，移动主要卡片使用 `--radius-card-mobile`。
- 业务操作使用 `Button` 的正式变体。
- 危险操作不得使用绿色主要按钮。
- 输入框、选择器和文本域使用统一高度、背景、边框和焦点。
- 所有可点击控件在移动端至少提供 44px 的有效触控高度。
- 禁用状态使用原生 `disabled`。

## 7. 状态、表格与数据

- 状态使用 `StatusTag`，必须同时包含文字。
- 表格外层使用 `ScrollableTable`，窄屏允许横向滚动列。
- 页面主滚动由 `.page-scroll` 承担；业务表格不得抢占页面纵向滚动。
- 本地成交记录面板自然增长，不设置固定高度、分页、折叠或内部垂直滚动。
- 订单簿每行只显示方向、价格和剩余数量，不显示所有者。
- “紧凑数字”是全局客户端显示偏好；`GameApp` 必须在状态栏和页面内容渲染前通过 `setCompactNumbersEnabled` 同步当前偏好。
- 偏好开启时，`formatCurrency` 和 `formatCompactNumber` 对大额数值统一使用 K/M/B/T；关闭时统一恢复带千分位的完整整数。
- 排名、百分比、时间、时长和可编辑数字输入保持精确原值，不得因紧凑偏好改变业务值。
- 页面不得自行维护另一套紧凑开关，也不得绕过统一格式器为同一类金额选择固定的完整或缩写格式。

## 8. 统一资产市场

- 商品和工厂使用同一资产标签和下单区域。
- 订单簿为单列 5+5 笔，不按价格档位聚合。
- 不得恢复商品／工厂二级切换、双列买卖盘或工厂固定价格卡。
- 买入使用成功色，卖出使用危险色，但方向必须有文字。
- 商品标签使用商品 SVG，工厂标签使用独立厂房 SVG。

## 9. 目录型横向导航

商品和工厂标签必须根据服务器目录动态生成：

- 使用可滚动横向区域或自适应自动列；
- 不得使用固定项目数量的 `repeat(6, ...)`；
- 不得在 JSX 中硬编码 6 个商品或工厂；
- 目录为空时显示明确空状态；
- 标签文本过长时保持可访问名称，不得让页面整体横向溢出。

## 10. 生产与仓库布局

- 共享仓库桌面采用 1:3 分栏，低于 960px 上下排列。
- 仓库只显示可用或冻结数量大于零的商品，不显示筛选按钮或全部商品视图。
- “仓库内容”标题不显示商品种类统计说明。
- 仓库商品卡使用 `ProductIconLabel`，固定采用“图标与名称／可用主值／冻结辅助值”的三层紧凑结构；不得显示独立库存总量行。
- “可用 N”沿用原库存主值字号，“冻结 N”沿用原第三层辅助文字字号；冻结为零时仍显示。
- 商品卡宽屏四列、桌面三列，平板、手机和 `320px` 以上极窄屏固定两列，不得恢复移动单列。
- 建设新工厂卡桌面独占 `280px–320px` 左列并使用 `position: sticky`；低于 960px 恢复普通文档流。
- 工厂卡桌面固定高度为 384px，移动端使用自然高度。
- 工厂卡大于 1380px 时固定四列，961px–1380px 单列、顶部对齐。
- 工厂卡顶部第一行显示状态标签、名称和右侧 `SwitchControl`，名称与开关必须保持同一水平行。
- 顶部数量摘要只显示“运行中／下一周期加入／冻结中”，固定为下一行三列单行布局；移动端不得纵向堆叠。
- 异常原因只在顶部状态标签显示一次。
- 工厂生产公式固定采用三列顶层布局：输入在左、周期成本在中、输出在右；多输入或多输出内部允许换行，但中列必须保持视觉居中。
- 输入和输出项目使用“数量 × 商品 SVG”结构，多项目之间使用独立 `+` 元素；输入项目下方使用仓库 SVG 与数值显示该商品当前可用库存。
- 周期成本中列只显示周期 SVG、周期数值、成本 SVG和成本数值，不显示可见的“周期”或“运行成本”标签；图标尺寸保持 `14px–16px` 并与数字基线对齐。
- 生产进度条横向铺满公式容器并作为输入到输出的唯一连接视觉；运行、停止和异常状态都保留同一进度轨道，停止或异常时进度为零并显示明确状态文字。
- 生产公式下方使用紧凑单行或自动换行的集群汇总；多输出逐项显示，成本继续使用成本 SVG，不显示“成本”文字标签。
- `960px` 以下允许输入／输出与中央信息换行；`560px` 以下输入和输出保持左右两区，中央信息独占下一行并居中；不得造成页面横向滚动。
- 生产公式必须提供完整文本无障碍描述，说明单座输入、输出、周期、成本、当前参与数量和进度，不能只依赖图标。
- 计划编辑标题固定为“当前计划”；待生效提示位于标题右侧，使用 `var(--color-text-muted)` 灰色小字并通过 `aria-live="polite"` 更新。
- 模式值只由一个下拉框表达；不得同时显示“当前计划：持续运行”等重复静态文本。
- 定量模式使用两列字段行，模式下拉框在左、带 `placeholder="目标产量"` 的数字输入框在右；持续模式下下拉框横跨两列。
- 计划字段自动保存，不显示“保存计划”按钮；保存失败必须使用明确错误文字，不能仅靠颜色表达。
- 自动保存期间不得阻塞玩家继续修改；同一集群请求必须保持顺序，最后一次有效输入为准。
- 不得恢复独立数量卡、运行描述、下一周期产量辅助句或持续／定量状态小卡片。
- 生产信息、计划控件和统一市场入口不得因紧凑布局被删除。

## 11. 导航颜色与不透明度

桌面侧栏和移动底栏的图标、文字都必须保持 `opacity: 1`：

- 未选中状态使用完全不透明的 `var(--color-text-muted)` 灰色；
- 选中文字使用完全不透明的 `var(--color-text-primary)`；
- 选中图标使用完全不透明的 `var(--color-success)`；
- 图标继承按钮颜色，不得为未选中图标另设半透明 RGBA；
- 背景、边框和玻璃材质仍可使用透明度；
- hover、focus、active 不得降低图标或文字透明度。

## 12. 中文、品牌、响应式与安全区

- 玩家可见界面统一使用中文，不允许固定文案中英文夹杂。
- 技术枚举必须转换为中文。
- 时间使用秒、分钟、小时。
- 桌面侧栏品牌第一行固定为“金融帝国”，第二行显示玩家用户名。
- 正式导航名称固定为“概览｜市场｜生产｜资产｜藏品｜拍卖｜排行｜设置”。
- 生产页 `PageLayout` 主标题固定为“生产”，建设卡和工厂类型业务名称继续使用“工厂”。
- `.page-scroll` 是唯一页面纵向滚动容器。
- 移动状态栏和底栏必须使用 `safe-area-inset-*`。
- 移动底部导航允许横向滚动，首尾保留完整空白。
- 移动端首次创建客户端偏好状态时，“紧凑数字”默认开启；桌面端默认关闭；切换后当前游戏外壳和所有使用统一格式器的页面立即同步。
- 活动导航按钮和图标不得位移、缩放或改变几何尺寸。
- 页面底部预留空间必须保证最后一张卡能滚动到导航栏上方。
- 移动登录页面通过 `100dvh` 和矮屏媒体查询适配软键盘；输入、按钮焦点和提交中的原生 `disabled` 状态不得改变标题字号、区块间距或整体对齐。
- 登录按钮在普通与“正在连接账号服务…”状态下保持至少 48px 高度和单行文案；表单使用 `aria-busy` 表达提交状态，不通过布局变化表达等待。
- 登录表单中的账号和密码必须保留原生未受控表单值，并使用稳定的 `name="email"` 与 `name="password"`；提交时通过 `FormData(event.currentTarget)` 读取浏览器自动填充内容，即使浏览器没有触发 `input` 或 `change` 事件也必须可以登录。

## 13. 防回退

不得：

- 在业务页面复制基础控件视觉；
- 恢复英文眉题；
- 恢复价格档位聚合订单簿；
- 恢复 `records` 导航；
- 新增平行开关；
- 在导航或状态栏中恢复 Unicode 字符、Emoji 或字体符号图标；
- 绕过 `GameIcons.tsx` 新增平行界面图标库；
- 删除当前商品显式图标、未知商品包装箱回退或 `ProductIconLabel`；
- 让工厂标签恢复齿轮、`⚙` 或与机械商品相同的 SVG 路径；
- 让导航图标或文字恢复半透明；
- 恢复仓库商品筛选、全部商品视图、商品种类统计说明、独立库存总量行或资产页逐商品库存与估值卡；
- 把移动仓库商品网格恢复为单列，或把可用与冻结重新合并到同一行；
- 把建设卡恢复为桌面独占整行；
- 把工厂卡恢复为桌面自然高度；
- 把工厂开关移出名称所在的第一行，或把三项数量摘要改为移动端纵向布局；
- 恢复工厂周期、产量、成本、原料四格规格区；
- 在生产公式中使用箭头、Emoji 周期或成本图标、可见“周期”标签或可见“运行成本”标签；
- 删除 `CycleIcon`、`CreditsIcon`、`WarehouseIcon` 或用平行 SVG 文件替代统一图标组件；
- 破坏输入／中央信息／输出三列语义、`+` 分隔、逐输入库存、多输出或集群多输出汇总；
- 恢复静态计划模式文本、手动“保存计划”按钮、定量输入上下堆叠或其他重复计划视觉；
- 把生产页主标题恢复为“工厂”；
- 把移动端“紧凑数字”初始状态恢复为关闭；
- 让“紧凑数字”只影响单个断点、单个状态项或单个页面，或让页面绕过 `formatCurrency`／`formatCompactNumber` 固定显示另一种格式；
- 让移动状态栏或底栏忽略安全区；
- 给导航活动态添加位移或缩放；
- 使用 `.login-shell:focus-within` 或其他焦点选择器改变移动登录页标题字号、区块间距或整体对齐；
- 让提交按钮进入 `disabled`、切换等待文案或失去焦点时改变移动登录页几何布局；
- 不得把账号或密码重新绑定到初始为空的 React `value` 状态，或仅从 React 状态读取凭据而忽略浏览器自动填充后的原生表单值；
- 使用运行时 DOM 扫描、文本匹配或 `MutationObserver` 修补已渲染页面；
- 未更新本文档和验证脚本就改变令牌、断点、共享组件、图标体系或关键布局。
