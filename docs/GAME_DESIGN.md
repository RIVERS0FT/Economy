# Economy 游戏设定记录

## 1. 当前方向

采用 `TheGenius` 的技术栈，开发一款多人联机虚拟市场策略游戏。

游戏不再围绕资源争夺，也不需要创建房间。玩家打开客户端后直接进入统一的共享市场大厅，在同一个虚拟市场中进行生产、兑换、买卖和博弈。

所有劳动力、货物、点数、行情和交易均为游戏内虚拟设定。

玩家持久化状态接入 Steam Inventory Service。劳动力、货物、金融货币和设施等级以 Steam 库存物品的形式保存。

## 2. 游戏关键词

- 多人联机
- 共享市场大厅
- Steam 库存持久化
- 点击产生劳动力
- 劳动力兑换货物
- 货物市场出售
- 虚拟金融货币
- 市场价格波动
- 买单 / 卖单
- 订单撮合
- 玩家间协议交易

## 4. 核心玩法

玩家进入共享市场后，不再通过点击直接获得金融货币。点击行为只产生“劳动力”。玩家使用劳动力兑换或生产货物，再将货物投入市场出售，从而获得游戏内金融货币。

核心循环：

1. 启动客户端并通过 Steam 登录。
2. 读取玩家 Steam 库存。
3. 自动进入共享市场大厅。
4. 点击按钮产生劳动力。
5. 劳动力写入 Steam 库存中的劳动力物品堆叠。
6. 使用劳动力兑换或生产货物。
7. 货物写入 Steam 库存中的货物物品堆叠。
8. 查看货物行情、价格和成交量。
9. 将货物挂单出售，或买入其他货物。
10. 成交后更新 Steam 库存中的货物和金融货币。
11. 玩家调整劳动力、货物库存和金融货币余额。
12. 玩家之间可进行协议交易。
13. 对局结束时，根据资产总值和声望判断胜负。

## 5. 不需要创建房间

游戏取消“创建房间 / 加入房间”流程。

联机方式改为：

- 默认市场服务器
- 自动进入主市场
- 所有玩家共享同一套行情
- 所有玩家共享订单簿
- 所有成交实时广播

客户端只需要提供：

- Steam 登录状态
- 昵称显示
- 连接状态
- 进入市场按钮
- 断线重连提示

## 6. 经济系统

游戏内包含三类核心经济对象：

| 类型 | 标识 | 用途 | Steam 库存形式 |
|---|---|---|---|
| 劳动力 | labor | 点击产生，用于兑换或生产货物 | 可堆叠物品 |
| 货物 | goods | 可由劳动力兑换获得，可在市场出售 | 可堆叠物品 |
| 金融货币 | credits | 通过出售货物获得，用于买入货物、支付手续费、升级设施和协议交易 | 可堆叠物品 |

经济链路：

```text
点击 -> 劳动力 -> 货物 -> 市场出售 -> 金融货币
```

## 7. Steam 库存系统

Steam 库存作为玩家持久化状态来源。

存入 Steam 库存的内容：

- 劳动力余额
- 金融货币余额
- 货物库存
- 设施等级或设施凭证
- 特殊称号或装饰性物品

不直接存入 Steam 库存的内容：

- 实时市场价格
- 订单簿
- 当前挂单状态
- 临时 UI 状态
- 实时行情事件

这些实时状态由市场服务器维护，并通过 WebSocket 同步。

### Steam ItemDef 草案

| ItemDef | 名称 | 类型 | 是否堆叠 | 用途 |
|---:|---|---|---|---|
| 1001 | labor_point | 劳动力 | 是 | 点击产出，兑换货物 |
| 2001 | food_pack | 货物 | 是 | 可出售 |
| 2002 | metal_part | 货物 | 是 | 可出售 |
| 2003 | energy_cell | 货物 | 是 | 可出售 |
| 2004 | precision_part | 货物 | 是 | 可出售 |
| 3001 | credit_token | 金融货币 | 是 | 市场结算与设施升级 |
| 4001 | work_seat_level | 设施 | 否 | 提升劳动力产出 |
| 4002 | storage_level | 设施 | 否 | 提升库存上限 |
| 4003 | market_terminal_level | 设施 | 否 | 提升订单能力 |

### Steam 库存操作原则

- 客户端可以读取 Steam 库存并展示玩家状态。
- 发布版本中，关键资产变更必须由可信市场服务器校验。
- 客户端不能自行决定发放大量货币或货物。
- 劳动力、货物、金融货币都以物品堆叠数量表示。
- 服务器需要校验冷却、库存、订单价格和成交结果。

## 8. 点击产生劳动力

玩家点击按钮时，不直接产出金融货币，而是产出劳动力。

建议规则：

- 每次点击获得 1 点劳动力。
- 点击有短冷却，例如 0.2 秒。
- 工作席位等级可提升单次点击获得的劳动力。
- 生产设施可提升劳动力获取效率。
- 劳动力不能直接作为金融货币使用。
- 劳动力变化需要同步到 Steam 库存。

点击流程：

1. 客户端发送 `click-labor` 指令。
2. 市场服务器校验点击冷却。
3. 校验通过后计算劳动力产出。
4. 服务器更新 Steam 库存中的 `labor_point` 数量。
5. 客户端重新同步库存并刷新 UI。

## 9. 劳动力兑换货物

玩家可以消耗劳动力兑换货物。

示例兑换：

| 货物 | 消耗劳动力 | 特点 |
|---|---:|---|
| 食品包 | 3 | 基础货物，成交稳定 |
| 金属件 | 5 | 波动较高 |
| 能源罐 | 8 | 受事件影响明显 |
| 精密组件 | 12 | 单价较高，生产较慢 |

兑换规则：

1. 玩家选择要生产或兑换的货物。
2. 系统检查 Steam 库存中的劳动力是否足够。
3. 扣除劳动力物品数量。
4. 增加对应货物物品数量。
5. 货物可以保留，也可以进入市场出售。

## 10. 货物市场交易

市场允许玩家买卖货物。

支持订单：

- 市价买入
- 市价卖出
- 限价买入
- 限价卖出
- 撤单

成交流程：

1. 玩家提交货物买入或卖出订单。
2. 系统检查 Steam 库存中的金融货币或货物库存是否足够。
3. 订单进入共享订单簿。
4. 买卖价格匹配时成交。
5. 成交后更新 Steam 库存中的金融货币和货物库存。
6. 成交记录同步给所有玩家。

卖出货物流程：

```text
货物库存减少 -> 金融货币增加 -> Steam 库存同步 -> 市场日志记录
```

买入货物流程：

```text
金融货币减少 -> 货物库存增加 -> Steam 库存同步 -> 市场日志记录
```

## 11. 价格波动

货物价格由以下因素影响：

- 玩家买卖订单
- 当前成交量
- 货物库存压力
- 虚拟市场事件
- 基础趋势
- 随机波动

设计目标：

- 货物价格可以被玩家行为影响。
- 劳动力产出决定货物供给。
- 金融货币主要通过市场出售货物获得。
- 玩家需要判断何时生产、何时出售、何时囤货。
- 市场价格不写入玩家 Steam 库存，而由服务器统一维护。

## 12. 持仓与库存

每个玩家拥有：

- 劳动力余额
- 金融货币余额
- 货物库存
- 设施状态
- 挂单列表
- 成交记录
- 总资产估值
- 声望

其中，劳动力、金融货币、货物库存和设施状态来自 Steam 库存。

总资产估值：

```text
总资产 = 金融货币余额 + 货物库存市值
```

## 13. 玩家间协议交易

玩家可以直接向其他玩家发起协议交易。

协议交易可以包含：

- 给出金融货币
- 给出货物
- 请求金融货币
- 请求货物
- 附加留言

交易状态：

- `pending`：等待处理
- `accepted`：已接受
- `rejected`：已拒绝
- `cancelled`：已取消

协议交易结算时，同样需要更新双方 Steam 库存。

## 14. 设施系统

设施用于提升生产和市场效率。

| 设施 | 功能 | 存储方式 |
|---|---|---|
| 工作席位 | 提升点击获得的劳动力 | Steam 库存设施物品 |
| 仓储中心 | 提升货物库存上限 | Steam 库存设施物品 |
| 加工设施 | 降低货物兑换所需劳动力 | Steam 库存设施物品 |
| 市场终端 | 提升订单数量上限 | Steam 库存设施物品 |
| 清算中心 | 降低游戏内手续费 | Steam 库存设施物品 |

## 15. 实时同步

游戏使用实时 tick 推进。

建议规则：

- 每 1 秒 1 tick。
- 每 tick 更新挂单状态。
- 每 3 tick 尝试撮合订单。
- 每 5 tick 更新货物价格。
- 点击产生劳动力按冷却独立结算。
- 服务器负责权威状态。
- Steam 库存用于持久化结果，不承担高频行情同步。

## 18. 当前开发决策

- 游戏形态：多人联机虚拟市场策略。
- 核心不再是资源争夺。
- 不需要创建房间。
- 不需要玩家列表。
- 玩家直接进入共享市场大厅。
- 点击不直接产出金融货币。
- 点击产出劳动力。
- 劳动力用于兑换货物。
- 货物可以在市场出售以获得金融货币。
- 玩家状态接入 Steam Inventory Service。
- 劳动力、货物、金融货币和设施存储在 Steam 库存中。
- 实时市场价格和订单簿由服务器维护。
- 允许游戏内市场交易。
- 交易范围仅限游戏内虚拟对象。
- 状态管理使用 Zustand。
- 桌面封装使用 Tauri。
