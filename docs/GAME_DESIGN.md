# Economy Arena 游戏设定记录

## 1. 当前方向

采用 `TheGenius` 的技术栈，开发一款多人联机虚拟市场策略游戏。

游戏不再围绕资源争夺，也不需要创建房间。玩家打开客户端后直接进入统一的共享市场大厅，在同一个虚拟市场中进行买卖、竞争和博弈。

所有点数、资产、行情和交易均为游戏内虚拟设定。

## 2. 游戏关键词

- 多人联机
- 共享市场大厅
- 点击产出货币
- 虚拟资产买卖
- 市场价格波动
- 买单 / 卖单
- 订单撮合
- 玩家间协议交易

## 3. 技术栈

沿用 `TheGenius` 的技术路线：

- Tauri 2.x
- Rust
- React
- TypeScript
- Vite
- Zustand
- CSS / Tailwind 风格样式
- Browser Web Audio API
- WebSocket 多人同步

## 4. 核心玩法

玩家进入共享市场后，通过点击产出基础信用点，并使用信用点在市场中买卖虚拟资产。玩家需要观察价格变化、管理持仓、完成交易，并与其他玩家进行协议交易。

核心循环：

1. 启动客户端。
2. 输入玩家昵称。
3. 自动进入共享市场大厅。
4. 点击按钮产出信用点。
5. 查看虚拟资产行情。
6. 提交买入或卖出订单。
7. 系统撮合订单并生成成交记录。
8. 玩家调整信用点余额和持仓。
9. 玩家之间可进行协议交易。
10. 对局结束时，根据资产总值和声望判断胜负。

## 5. 不需要创建房间

游戏取消“创建房间 / 加入房间”流程。

联机方式改为：

- 默认市场服务器
- 自动进入主市场
- 所有玩家共享同一套行情
- 所有玩家共享订单簿
- 所有成交实时广播

客户端只需要提供：

- 昵称输入
- 连接状态
- 进入市场按钮
- 断线重连提示

## 6. 货币系统

基础货币为：信用点 `credits`。

信用点用途：

- 买入虚拟资产
- 支付游戏内手续费
- 升级市场设施
- 玩家间协议交易

## 7. 点击产出货币

玩家可以点击“产出信用点”按钮获得基础货币。

建议规则：

- 每次点击获得 1 信用点。
- 点击有短冷却，例如 0.2 秒。
- 交易席位等级可提升点击收益。
- 市场设施可提升点击收益。
- 点击收益低于市场买卖收益，避免破坏策略性。

## 8. 虚拟资产

MVP 阶段使用虚拟资产，不使用现实资产名称。

示例资产：

| 资产 | 类型 | 特点 |
|---|---|---|
| 星港能源 | 成长型 | 波动较高 |
| 蓝湾票据 | 稳定型 | 波动较低 |
| 铁砂合约 | 事件型 | 受市场事件影响明显 |
| 综合市场篮子 | 组合型 | 代表整体市场方向 |

## 9. 市场交易

市场允许玩家买卖虚拟资产。

支持订单：

- 市价买入
- 市价卖出
- 限价买入
- 限价卖出
- 撤单

成交流程：

1. 玩家提交订单。
2. 系统检查信用点或持仓是否足够。
3. 订单进入共享订单簿。
4. 买卖价格匹配时成交。
5. 成交后更新玩家信用点和持仓。
6. 成交记录同步给所有玩家。

## 10. 玩家间协议交易

玩家可以直接向其他玩家发起协议交易。

协议交易可以包含：

- 给出信用点
- 给出虚拟资产
- 请求信用点
- 请求虚拟资产
- 附加留言

交易状态：

- `pending`：等待处理
- `accepted`：已接受
- `rejected`：已拒绝
- `cancelled`：已取消

## 11. 市场设施

市场设施替代传统 RTS 建筑。

| 设施 | 功能 |
|---|---|
| 交易席位 | 提升点击收益和订单数量上限 |
| 研究终端 | 提供更多行情提示 |
| 银行设施 | 提升信用点产出 |
| 清算中心 | 降低游戏内手续费 |
| 风控中心 | 提供风险提示 |

## 12. 实时同步

游戏使用实时 tick 推进。

建议规则：

- 每 1 秒 1 tick。
- 每 tick 更新挂单状态。
- 每 3 tick 尝试撮合订单。
- 每 5 tick 更新市场价格。
- 点击产出信用点按冷却独立结算。
- 服务器负责权威状态。

| 日志面板 | 成交、事件、同步记录 |

## 15. 当前开发决策

- 游戏形态：多人联机虚拟市场策略。
- 核心不再是资源争夺。
- 不需要创建房间。
- 不需要玩家列表。
- 玩家直接进入共享市场大厅。
- 允许游戏内市场交易。
- 交易范围仅限游戏内虚拟资产。
- 基础货币通过点击产出。
- 状态管理使用 Zustand。
- 桌面封装使用 Tauri。
