# Economy Arena 游戏设定记录

## 1. 当前方向

采用 `TheGenius` 的技术栈，开发一款多人联机即时战略经济游戏。

游戏关键词：

- 多人联机
- 即时战略
- 资源争夺
- 据点控制
- 建筑生产
- 单位移动
- 游戏内交易
- 房间制对战

当前明确移除：

- AI 对手
- 回合制流程
- 单机模拟作为核心玩法

## 2. 技术栈

沿用 `TheGenius` 的技术路线：

- Tauri 2.x
- Rust
- React
- TypeScript
- Vite
- Zustand
- CSS / Tailwind 风格样式
- Browser Web Audio API
- WebSocket 多人同步

## 3. 游戏类型

**类型**：多人联机即时战略 RTS

**主题**：经济势力竞争

**核心目标**：

玩家通过占领地图据点、建设经济设施、生产单位、控制资源流通和与其他玩家交易，获得经济优势并赢得比赛。

## 4. 核心循环

1. 创建或加入房间。
2. 房主开始战局。
3. 玩家实时控制单位移动。
4. 单位前往地图节点并占领据点。
5. 据点与建筑持续产出资源。
6. 玩家建造设施、生产单位、扩张控制范围。
7. 玩家之间发起游戏内交易。
8. 目标玩家手动接受或拒绝交易。
9. 通过据点、建筑、经济规模和交易获得声望。
10. 达到胜利条件后比赛结束。

## 5. 胜利条件

默认胜利条件：

- 声望达到目标值；或
- 对局时间结束时声望最高；或
- 玩家失去主要控制区后被淘汰，最后存活者获胜。

声望来源：

- 控制据点
- 建造或升级建筑
- 保持资源产出
- 完成高价值交易
- 控制关键经济节点

## 6. 资源系统

| 资源 | 标识 | 用途 |
|---|---|---|
| 粮食 | grain | 基础扩张、单位维护 |
| 矿石 | ore | 建筑、升级、防御单位 |
| 织物 | textile | 贸易、运输、经济设施 |
| 能源 | energy | 高级建筑、生产效率 |
| 信用点 | credits | 通用货币，用于建造、生产和交易 |

## 7. 地图节点

地图由多个资源节点组成。每个节点包含：

- 节点 ID
- 名称
- 坐标
- 主产资源
- 当前控制者
- 占领进度

示例节点：

| 节点 | 资源 | 作用 |
|---|---|---|
| 中央交易城 | grain | 中心经济区 |
| 北境粮仓 | grain | 前期扩张点 |
| 铁脊矿区 | ore | 建筑核心点 |
| 日冕电站 | energy | 高级设施核心点 |
| 织造港 | textile | 贸易核心点 |
| 自由市场 | textile | 交易收益点 |

## 8. 单位系统

| 单位 | 标识 | 定位 |
|---|---|---|
| 侦察车 | scout | 快速移动、探路、抢点 |
| 运输车 | carrier | 资源运输、经济支援 |
| 守卫队 | guard | 据点防守、区域控制 |

单位基础行为：

- 移动到指定节点
- 参与占领
- 驻守据点
- 后续可扩展护送、巡逻、战斗

## 9. 建筑系统

| 建筑 | 标识 | 功能 |
|---|---|---|
| 总部 | hq | 初始建筑，提供基础收入 |
| 农场 | farm | 提升粮食产出 |
| 矿井 | mine | 提升矿石产出 |
| 电站 | plant | 提升能源产出 |
| 交易所 | exchange | 提升信用点与交易收益 |

建筑依附于地图节点。玩家需要先控制节点，再建设对应设施。

## 10. 实时 tick 规则

游戏使用实时 tick 推进。

建议规则：

- 每 1 秒 1 tick。
- 每 5 tick 执行资源产出。
- 每 10 tick 执行声望结算。
- 地图节点持续检查占领进度。
- 房主或服务器负责推进权威状态。

## 11. 游戏内交易

交易发生在真实玩家之间。

交易内容：

- 发起方给出资源
- 发起方给出信用点
- 请求对方给出资源
- 请求对方给出信用点
- 附加留言

交易状态：

| 状态 | 含义 |
|---|---|
| pending | 等待处理 |
| accepted | 已接受并结算 |
| rejected | 已拒绝 |
| cancelled | 已取消 |

交易结算规则：

1. 发起交易时检查发起方资产。
2. 接受交易时检查双方资产。
3. 双方资产充足则立即结算。
4. 资产不足则交易失败。
5. 交易结果写入战局日志。
6. 联机房间同步交易结果。

## 12. 多人联机模型

MVP 采用房间制。

房间字段：

- roomCode：房间码
- serverUrl：同步服务地址
- localPlayerId：本机玩家 ID
- hostPlayerId：房主玩家 ID
- status：连接状态

MVP 同步策略：

- 房主作为权威状态。
- 非房主发送操作指令。
- 房主应用指令。
- 房主广播最新状态。
- 所有客户端渲染同一份状态。

  "playerId": "player-1",

```json
{
  "type": "state-sync",
  "roomCode": "AB12CD",
  "senderId": "player-host",
  "state": {},
  "sentAt": 1234567890
}

  "playerId": "player-1",

    "type": "move-unit",

    "targetNodeId": "iron-ridge"
  }
}
```

## 16. 当前开发决策

- 游戏形态：多人联机即时战略。
- 对手来源：全部为真实玩家。
- 核心差异点：经济 RTS + 玩家交易。
- 同步方式：MVP 采用 WebSocket 房间同步。
- 状态管理：前端使用 Zustand。
- 桌面封装：Tauri。
