# Economy Arena 游戏设定记录

## 1. 当前方向

采用 `TheGenius` 的技术栈，开发一款多人联机虚拟市场策略游戏。

游戏不再围绕资源争夺，也不需要创建房间。玩家打开客户端后直接进入统一的共享市场大厅，在同一个虚拟市场中进行生产、兑换、买卖和博弈。

所有劳动力、货物、点数、行情和交易均为游戏内虚拟设定。

## 2. 游戏关键词

- 多人联机
- 共享市场大厅
- 点击产生劳动力
- 劳动力兑换货物
- 货物市场出售
- 虚拟金融货币
- 市场价格波动
- 买单 / 卖单
- 订单撮合
- 玩家间协议交易

## 4. 核心玩法

玩家进入共享市场后，不再通过点击直接获得金融货币。点击行为只产生“劳动力”。玩家使用劳动力兑换或生产货物，再将货物投入市场出售，从而获得游戏内金融货币。

核心循环：

1. 启动客户端。
2. 输入玩家昵称。
3. 自动进入共享市场大厅。
4. 点击按钮产生劳动力。
5. 使用劳动力兑换或生产货物。
6. 查看货物行情、价格和成交量。
7. 将货物挂单出售，或买入其他货物。
8. 系统撮合订单并生成成交记录。
9. 玩家调整劳动力、货物库存和金融货币余额。
10. 玩家之间可进行协议交易。
11. 对局结束时，根据资产总值和声望判断胜负。

## 5. 不需要创建房间

游戏取消“创建房间 / 加入房间”流程。

联机方式改为：

- 默认市场服务器
- 自动进入主市场
- 所有玩家共享同一套行情
- 所有玩家共享订单簿
- 所有成交实时广播

客户端只需要提供：

- 昵称输入
- 连接状态
- 进入市场按钮
- 断线重连提示

## 6. 经济系统

游戏内包含三类核心经济对象：

| 类型 | 标识 | 用途 |
|---|---|---|
| 劳动力 | labor | 点击产生，用于兑换或生产货物 |
| 货物 | goods | 可由劳动力兑换获得，可在市场出售 |
| 金融货币 | credits | 通过出售货物获得，用于买入货物、支付手续费、升级设施和协议交易 |

经济链路：

```text
点击 -> 劳动力 -> 货物 -> 市场出售 -> 金融货币
```

## 7. 点击产生劳动力

玩家点击按钮时，不直接产出金融货币，而是产出劳动力。

建议规则：

- 每次点击获得 1 点劳动力。
- 点击有短冷却，例如 0.2 秒。
- 工作席位等级可提升单次点击获得的劳动力。
- 生产设施可提升劳动力获取效率。
- 劳动力不能直接作为金融货币使用。

## 8. 劳动力兑换货物

玩家可以消耗劳动力兑换货物。

示例兑换：

| 货物 | 消耗劳动力 | 特点 |
|---|---:|---|
| 食品包 | 3 | 基础货物，成交稳定 |
| 金属件 | 5 | 波动较高 |
| 能源罐 | 8 | 受事件影响明显 |
| 精密组件 | 12 | 单价较高，生产较慢 |

兑换规则：

1. 玩家选择要生产或兑换的货物。
2. 系统检查劳动力是否足够。
3. 扣除劳动力。
4. 增加对应货物库存。
5. 货物可以保留，也可以进入市场出售。

## 9. 货物市场交易

市场允许玩家买卖货物。

支持订单：

- 市价买入
- 市价卖出
- 限价买入
- 限价卖出
- 撤单

成交流程：

1. 玩家提交货物买入或卖出订单。
2. 系统检查金融货币或货物库存是否足够。
3. 订单进入共享订单簿。
4. 买卖价格匹配时成交。
5. 成交后更新玩家金融货币和货物库存。
6. 成交记录同步给所有玩家。

## 10. 价格波动

货物价格由以下因素影响：

- 玩家买卖订单
- 当前成交量
- 货物库存压力
- 虚拟市场事件
- 基础趋势
- 随机波动

设计目标：

- 货物价格可以被玩家行为影响。
- 劳动力产出决定货物供给。
- 金融货币主要通过市场出售货物获得。
- 玩家需要判断何时生产、何时出售、何时囤货。

## 11. 持仓与库存

每个玩家拥有：

- 劳动力余额
- 金融货币余额
- 货物库存
- 挂单列表
- 成交记录
- 总资产估值
- 声望

总资产估值：

```text
总资产 = 金融货币余额 + 货物库存市值
```

## 12. 玩家间协议交易

玩家可以直接向其他玩家发起协议交易。

协议交易可以包含：

- 给出金融货币
- 给出货物
- 请求金融货币
- 请求货物
- 附加留言

交易状态：

- `pending`：等待处理
- `accepted`：已接受
- `rejected`：已拒绝
- `cancelled`：已取消

## 13. 设施系统

设施用于提升生产和市场效率。

| 设施 | 功能 |
|---|---|
| 工作席位 | 提升点击获得的劳动力 |
| 仓储中心 | 提升货物库存上限 |
| 加工设施 | 降低货物兑换所需劳动力 |
| 市场终端 | 提升订单数量上限 |
| 清算中心 | 降低游戏内手续费 |

## 14. 实时同步

游戏使用实时 tick 推进。

建议规则：

- 每 1 秒 1 tick。
- 每 tick 更新挂单状态。
- 每 3 tick 尝试撮合订单。
- 每 5 tick 更新货物价格。
- 点击产生劳动力按冷却独立结算。
- 服务器负责权威状态。

## 17. 当前开发决策

- 游戏形态：多人联机虚拟市场策略。
- 核心不再是资源争夺。
- 不需要创建房间。
- 不需要玩家列表。
- 玩家直接进入共享市场大厅。
- 点击不直接产出金融货币。
- 点击产出劳动力。
- 劳动力用于兑换货物。
- 货物可以在市场出售以获得金融货币。
- 允许游戏内市场交易。
- 交易范围仅限游戏内虚拟对象。
- 状态管理使用 Zustand。
- 桌面封装使用 Tauri。
