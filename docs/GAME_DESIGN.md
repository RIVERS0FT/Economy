# Economy 游戏设定记录

## 1. 当前方向

优先开发 **抖音小程序单机版**。

小程序版显示名称：**中国股神**。

Economy 是一款虚拟市场经营策略游戏。第一阶段不做多人联机、不做服务器、不做 Steam 库存、不做共享市场，而是先在抖音小程序中做出完整单机可玩循环。

小程序版只聚焦手机移动端体验，不适配桌面端、大屏端、平板横屏或复杂横向布局。

核心体验：

```text
点击工作 -> 获得劳动力 -> 兑换货物 -> 出售货物 -> 获得金融货币 -> 升级设施 / 座驾 / 房子 -> 提升人生资产
```

所有劳动力、货物、金融货币、价格和交易均为游戏内虚拟设定。

## 4. 最小可玩版本

第一版只做一条完整经济链路。

### 初始状态

| 项目 | 初始值 |
|---|---:|
| 年龄 | 18 |
| 座驾 | 步行通勤 |
| 房子 | 合租小屋 |
| 劳动力 | 0 |
| 食品包 | 0 |
| 金融货币 | 0 |
| 工作席位等级 | 1 |
| 食品包价格 | 5 |

### 点击工作

玩家点击“工作”按钮获得劳动力。

| 项目 | 规则 |
|---|---:|
| 基础点击收益 | 1 劳动力 |
| 点击冷却 | 0.2 秒 |
| 直接获得金融货币 | 不允许 |

工作席位等级会提升点击收益。

| 工作席位等级 | 每次点击获得劳动力 |
|---:|---:|
| 1 | 1 |
| 2 | 2 |
| 3 | 3 |

### 生产食品包

玩家消耗劳动力生产食品包。

```text
3 劳动力 -> 1 食品包
```

### 出售食品包

玩家出售食品包获得金融货币。

```text
1 食品包 -> 5 金融货币
```

### 升级工作席位

玩家使用金融货币升级工作席位。

| 等级 | 点击收益 | 升级费用 |
|---:|---:|---:|
| 1 | 1 劳动力 | 初始 |
| 2 | 2 劳动力 | 50 金融货币 |
| 3 | 3 劳动力 | 150 金融货币 |

### 座驾与房子

玩家可以使用金融货币改善人生资产。

座驾等级：

| 等级 | 名称 | 升级费用 |
|---:|---|---:|
| 0 | 步行通勤 | 初始 |
| 1 | 二手电动车 | 30 |
| 2 | 家用代步车 | 120 |
| 3 | 商务轿车 | 400 |

房子等级：

| 等级 | 名称 | 升级费用 |
|---:|---|---:|
| 0 | 合租小屋 | 初始 |
| 1 | 单身公寓 | 80 |
| 2 | 小区住宅 | 300 |
| 3 | 城市别墅 | 1000 |

年龄可以通过“过一年”按钮增长，用于增强人生经营感。

```ts
interface EconomySaveData {
```

本地存档保存内容：

- 座驾等级
- 房子等级
- 劳动力
- 金融货币
- 食品包库存
- 工作席位等级
- 食品包价格
- 统计数据
- 最后保存时间

## 6. 本地存档

抖音小程序单机版使用本地存储保存玩家状态。

保存时机：

- 点击获得劳动力后
- 生产食品包后
- 出售食品包后
- 升级设施后
- 升级座驾后
- 升级房子后
- 年龄增长后
- 页面隐藏时
- 定时自动保存

读取时机：

- 小程序启动时
- 游戏页面加载时

异常处理：

- 没有存档时创建新游戏
- 存档损坏时重置为初始状态
- 版本变更时执行存档迁移

## 7. 单机市场

第一阶段不做真实订单簿，也不做多人市场。

食品包价格先使用固定价格：

```text
食品包价格 = 5 金融货币
```

后续可加入本地价格波动：

- 每隔一段时间随机小幅变化
- 出售过多食品包时价格下降
- 长时间不出售时价格回升
- 触发本地事件影响价格

## 8. 移动端 UI 规划

抖音小程序单机版只需要一个手机竖屏主页面。

移动端布局原则：

- 页面以单列卡片为主。
- 状态信息最多两列展示。
- 年龄、座驾、房子使用纵向列表。
- 主按钮加大，适合触控操作。
- 不使用桌面端双栏大面板。
- 不适配横屏、大屏和桌面端。

| 区域 | 功能 |
|---|---|
| 顶部栏 | 小程序名、存档状态 |
| 个人状态栏 | 劳动力、金融货币、食品包、总资产 |
| 个人信息区 | 年龄、座驾、房子、过一年按钮 |
| 工作区 | 点击工作按钮、当前点击收益、冷却状态 |
| 生产区 | 食品包生产按钮、劳动力消耗、库存显示 |
| 市场区 | 食品包价格、出售按钮、获得金额 |
| 升级区 | 工作席位、座驾、房子升级 |
| 日志区 | 最近操作记录 |

## 9. 页面操作

### 点击工作

条件：冷却结束。

结果：

```text
劳动力 += 当前点击收益
```

### 生产食品包

条件：劳动力 >= 3。

结果：

```text
劳动力 -= 3
食品包 += 1
```

### 出售食品包

条件：食品包 >= 1。

结果：

```text
食品包 -= 1
金融货币 += 食品包价格
```

### 升级工作席位

条件：金融货币足够，且工作席位未满级。

结果：

```text
金融货币 -= 升级费用
工作席位等级 += 1
```

### 升级座驾

条件：金融货币足够，且座驾未满级。

结果：

```text
金融货币 -= 座驾升级费用
座驾等级 += 1
```

### 升级房子

条件：金融货币足够，且房子未满级。

结果：

```text
金融货币 -= 房子升级费用
房子等级 += 1
```

### 过一年

结果：

```text
年龄 += 1
经营年数 += 1
```

## 10. 验收标准

抖音小程序单机 MVP 完成条件：

1. 可以打开游戏主页面。
2. 可以点击工作获得劳动力。
3. 点击不会直接获得金融货币。
4. 可以用 3 劳动力生产 1 食品包。
5. 可以出售 1 食品包获得 5 金融货币。
6. 可以用金融货币升级工作席位。
7. 工作席位升级后点击收益提高。
8. 可以看到年龄、座驾、房子。
9. 可以升级座驾和房子。
10. 关闭并重新打开后，本地存档不丢失。
11. 页面不显示玩家列表。
12. 页面不需要创建房间。
13. 页面布局只按手机移动端优化。
