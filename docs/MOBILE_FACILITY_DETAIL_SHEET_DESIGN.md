# Economy 移动工厂详情悬浮框设计

> 状态：移动生产详情交互最终基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-21

本文件是移动工厂详情悬浮框的最终设计权威。与 `INDUSTRY_AND_PRODUCTION_DESIGN.md` 既有“单配方工厂显示唯一选项并禁用”规则冲突时，以本文件为准：单配方工厂显示唯一选项并保持启用；后续整理主设计文档时不得恢复禁用行为。

## 1. 结构

不大于 `720px` 时，工厂详情使用三段式 Bottom Sheet：

```text
固定头部
→ 唯一 ScrollArea 正文
→ 固定底部操作区
```

- 固定头部包含拖动把手、工厂名称、运行开关、完整状态、关闭按钮以及“运行中／下一周期加入／冻结中”三项摘要。
- 正文只包含生产配方、集群生产公式、生产进度和玩家可见利润分析。
- 固定底部操作区只包含“前往市场交易该工厂 →”。
- 市场入口不得通过负 `bottom` 或正文内部 `position: sticky` 覆盖利润分析。
- 正文最后一项必须能够完整滚动到底部操作区上方。
- `src/styles/facility-detail-sheet.css` 是移动工厂详情悬浮框布局、拖动反馈和固定头尾的最终 CSS 权威；桌面详情继续由 `facility-group-card-grid.css` 管理。

## 2. 配方选择器

- 所有工厂详情统一显示真实 `SelectInput`。
- 多配方工厂显示全部服务器正式配方并允许切换。
- 单配方工厂显示唯一选项并保持启用，不得使用禁用态暗示权限、状态或数据异常。
- 选择器只允许服务器正式配方 ID。
- 重复选择相同正式配方不得提交经济动作、重置周期、刷新进度或显示成功提示。
- 运行中选择其他配方仍只写入 `pendingRecipeId`；停止或异常状态仍按既有服务器规则立即应用。

## 3. 向下拖动关闭

### 3.1 手势归属

- 从拖动把手或固定头部的非交互空白区域开始时，可以直接向下拖动。
- 从正文开始时，只有正文 `scrollTop = 0` 且手势明显向下，才接管为关闭手势。
- 正文尚未到顶时，纵向手势继续归正文滚动，悬浮框不得移动。
- `button`、`select`、`input`、链接、运行开关、滚动条滑块与轨道不得成为关闭拖动起点。
- 移动距离未达到 `8px` 前不得锁定手势；向下位移必须大于横向位移的 `1.2` 倍。

### 3.2 跟手反馈

- 拖动期间仅通过 DOM ref、CSS 自定义属性和 `requestAnimationFrame` 更新 `translateY`，不得在每个移动事件中写 React state。
- 悬浮框只能向下移动，不得向上越界。
- 遮罩随拖动距离逐渐减弱，但悬浮框内容不得跟随遮罩整体变透明。
- 拖动期间使用 pointer capture；触控取消、窗口变化、组件卸载和横竖屏变化必须清理拖动状态。

### 3.3 释放判定

满足任一条件时关闭：

- 向下距离达到悬浮框高度的 `25%`，并限制在至少 `96px`、至多 `160px`；
- 或向下距离至少 `40px` 且释放速度达到 `0.75px/ms`。

未达到阈值时回弹到原位。关闭完成后必须复用既有关闭流程，解除背景滚动锁，并让关闭后焦点返回触发控件。

`prefers-reduced-motion: reduce` 时保留跟手拖动，但释放后立即关闭或立即复位，不播放惯性、弹簧或淡出过渡。

## 4. 滚动与活动滚动条

- 悬浮框正文必须使用共享 `ScrollArea`，设置 `axis="y"` 与 `scrollbarVisibility="adaptive"`。
- 悬浮框正文是唯一纵向滚动容器；配方、公式、利润卡和底部操作区不得创建嵌套纵向滚动。
- 触控模式下仅在 `scrollTop` 实际变化后显示纵向轨道；触摸按下但没有滚动不得显示。
- 活动期间允许触摸拖动滑块与点击轨道翻页；拖动或轨道操作期间不得隐藏。
- 停止滚动并空闲 `1600ms` 后淡出，隐藏时必须 `pointer-events: none`。
- 触控横向轨道继续始终隐藏。
- 悬浮框打开期间，背景 `.page-scroll-area` 的覆盖式纵向轨道必须被抑制，避免与悬浮框轨道同时出现；关闭后只在页面后续实际滚动时重新显示。
- 不得复制滚动条尺寸、计时器、拖动或活动判断；统一复用 `ScrollArea`、`useOverlayScrollbar` 和 `scrollbars.css`。

## 5. 安全区与固定区域

- 悬浮框最大高度保持 `min(88dvh, 760px)`。
- 固定底部操作区必须包含 `env(safe-area-inset-bottom)`。
- 固定头部和底部使用不透明主表面，正文不得透过固定区域形成文字叠加。
- 底部市场按钮最小高度为 `48px`，占满可用宽度，并保留固定中文文案。
- 关闭按钮实际点击区域不得小于 `44px × 44px`。

## 6. 防回退

不得：

- 恢复单配方选择器禁用；
- 把单配方改成静态文本并移除统一选择器；
- 在相同配方重复选择时提交服务器动作；
- 只允许点击关闭而让拖动把手继续成为假交互；
- 在正文未到顶时抢走正常纵向滚动；
- 从选择器、开关、按钮或滚动条开始关闭拖动；
- 在拖动过程中通过高频 React state 触发整页重渲染；
- 恢复正文内部负偏移 sticky 市场入口；
- 使用原生滚动条代替共享活动纵向轨道；
- 悬浮框打开时继续显示背景页面纵向轨道；
- 使用 `overscroll-behavior-y: contain` 阻断纵向滚动链；
- 忽略底部安全区、焦点返回、Escape 关闭或背景滚动锁。
