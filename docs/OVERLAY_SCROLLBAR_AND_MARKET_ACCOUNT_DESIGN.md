# Economy 覆盖式滚动条与订单成交表设计

> 状态：玩家端、管理员端和共享滚动容器的滚动交互权威基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-20

本文定义全局滚动条视觉、双轴输入优先级、移动页面右侧贴边规则、“我的订单与成交”表格结构和本地成交字段语义。滚动容器、市场表格或虚拟列表发生冲突时，以本文为专题权威；通用视觉仍以 `docs/UI_DESIGN_SYSTEM.md` 为准。

## 1. 唯一实现

- `src/components/ui/ScrollArea.tsx` 是应用内覆盖式滚动区域的唯一共享组件。
- `src/hooks/useOverlayScrollbar.ts` 是滑块尺寸、位置、拖动、轨道翻页、实际滚动活动判断和双轴输入分派的唯一实现。
- `src/styles/scrollbars.css` 是滚动条宽度、颜色、圆角、层级和显隐的唯一视觉来源。
- 页面、侧栏、状态栏、移动导航、共享表格、虚拟列表和其他关键滚动区域不得再复制滚动条计时器或定义平行颜色、宽度。
- 原生滚动容器继续负责可访问滚动和浏览器滚动链；原生滚动条视觉在 `ScrollArea` 视口内隐藏，项目滚动条以绝对定位覆盖在内容上方。

## 2. 全局视觉令牌

固定令牌：

- 视觉宽度：`--scrollbar-visual-size: 6px`；
- 可点击轨道：`--scrollbar-hit-size: 14px`；
- 边缘偏移：`2px`；
- 最小滑块：`28px`；
- 默认滑块：`rgba(105, 220, 145, 0.42)`；
- 悬停滑块：`rgba(125, 238, 163, 0.68)`；
- 拖动滑块：`rgba(125, 238, 163, 0.9)`；
- 轨道透明；
- 纵向空闲延迟：`1200ms`；
- 淡入淡出：`120ms`，减少动态效果时关闭动画。

覆盖式轨道不得使用 `scrollbar-gutter: stable`，不得通过内边距预留滚动条空间；显隐前后页面、卡片、表头和表格列宽必须完全不变。

## 3. 显隐语义

“无活动”只表示没有发生实际滚动位置变化：

- `scrollTop` 发生变化时显示纵向滚动条；
- 停止纵向滚动 `1200ms` 后隐藏纵向滚动条视觉；
- 鼠标移动、指针按下、点击内容、焦点进入、滚轮事件本身或键盘事件本身都不得直接显示滚动条；
- 滚轮或键盘只有在确实改变滚动位置并触发 `scroll` 后，才算滚动活动；
- 拖动纵向滑块、操作纵向轨道时保持显示，操作结束后重新开始空闲计时；
- 有横向溢出时水平滚动条必须常驻可见，不参与空闲隐藏；
- 没有对应方向溢出时不得绘制该方向轨道。

## 4. 输入优先级与滚动链

双轴容器固定遵循：

1. 普通滚轮和以 `deltaY` 为主的触控板输入优先垂直滚动；
2. `Shift + 滚轮`、明确以 `deltaX` 为主的触控板输入、拖动水平滑块或点击水平轨道才执行水平滚动；
3. 普通纵向滚轮不得转换成水平滚动；
4. 内部纵向滚动区未到边界时优先消费纵向输入；到达顶部或底部后允许滚动链传给外层页面；
5. 到达纵向边界不得自动改成水平滚动；
6. 双轴轨道同时显示时纵向轨道 `z-index` 更高，水平轨道在右侧避让纵向点击区域；
7. 不绘制额外右下角滚动条角块。

## 5. 移动页面右侧贴边规则

移动端 `.workspace` 仍是页面和外壳的唯一水平几何边界，页面 viewport 和一级卡片继续受 `--mobile-workspace-gutter` 约束。只有覆盖式纵向轨道允许越过工作区内容框进入右侧 gutter：

- `--mobile-scrollbar-edge-escape` 等于普通工作区 gutter 减去右侧安全区后的非负值；
- `.mobile-page-overlay` 与 `.page-scroll-area` 允许轨道可见溢出，最终由 `.workspace` 裁切；
- `.page-scroll-area > .ui-scrollbar--vertical` 使用负的边缘逃逸量，不得扩大 `.page-scroll`、`.page-content` 或卡片宽度；
- 移动纵向滑块在轨道内改为右侧对齐，并保留 `--scrollbar-edge-offset: 2px`；
- 无右侧安全区时，滑块右边缘距屏幕右边 `2px`；存在安全区时，距安全区内缘 `2px`；
- 滚动条显隐前后，页面 `clientWidth`、一级卡片宽度和状态栏／底栏边界必须不变；
- 该特殊定位只适用于移动页面纵向滚动条，不得改变侧栏、表格、虚拟列表或状态栏滚动条。

## 6. 性能与可访问性

- 滚动过程中不得用 React state 更新滑块位置；使用 ref、CSS transform 和 `requestAnimationFrame`，每帧最多同步一次。
- 容器和内容尺寸变化使用 `ResizeObserver`，禁止定时轮询。
- 虚拟列表继续只挂载可视区和少量 `overscan` 项，不得因自定义滚动条恢复全量 DOM。
- 滑块保留 `role="scrollbar"`、方向和当前范围语义，支持拖动、轨道翻页和键盘控制。
- `prefers-reduced-motion: reduce` 只关闭显隐动画，不取消滚动能力。

## 7. 未完成订单表

最终列顺序固定为：

```text
资产｜方向｜价格｜剩余/原始｜状态｜时间｜操作
```

规则：

- 删除独立“类型”列；商品和工厂由资产图标表达；
- 方向列固定为 `60px`，完整显示“买入／卖出”；
- 操作列固定为 `76px`，使用 `position: sticky; right: 0`；
- 撤单按钮必须始终位于横向滚动视口右侧，不得随表格内容滚出；
- 固定列使用不透明表面背景和左侧轻微分隔阴影，避免文字穿透；
- 空状态 `colSpan` 固定为 `7`；
- 表格横向滚动条有溢出时常驻可见，普通纵向滚轮继续滚动页面。

## 8. 本地成交记录

最终列顺序固定为：

```text
资产｜方向｜数量｜价格｜总额｜手续费/实收｜时间
```

规则：

- 删除独立“类型”列；资产图标表达商品或工厂类型；
- `TradeRecord.type` 继续作为内部图标和类型判断字段，不作为可见列；
- `TradeRecord.description` 的展示语义是纯资产名称；
- 买卖方向只由 `TradeRecord.side` 和方向列表达；资产列不得再显示“买入／卖出”前缀；
- 新记录直接保存纯名称；读取历史记录时兼容移除与 `side` 一致的旧方向前缀；
- 方向列固定为 `60px`；虚拟表格固定为七列，最小宽度约 `760px`；
- 外层只负责横向滚动，水平滚动条常驻；内部 `VirtualList` 只负责纵向滚动，纵向滚动条实际滚动时显示并空闲隐藏；
- 内外滚动区不得造成表头和内容列错位。

## 9. 不可回退规则

除非先更新本文和验证脚本，否则不得：

- 恢复页面、侧栏或业务表格自有滚动条宽度、颜色或计时器；
- 使用指针移动、点击、焦点或未产生位移的滚轮作为滚动活动；
- 隐藏有横向溢出的水平滚动条；
- 把普通纵向滚轮转换为水平滚动；
- 使用 `scrollbar-gutter: stable` 或布局内边距为项目滚动条永久占位；
- 在双轴交叉区域让水平轨道覆盖纵向轨道；
- 让移动页面纵向滑块停留在一级卡片右边缘，或通过改变 viewport／卡片宽度实现贴边；
- 让移动轨道越过右侧安全区，或删除 `2px` 边缘偏移；
- 把移动页面专用边缘逃逸规则扩散到其他 `ScrollArea`；
- 恢复未完成订单或本地成交的“类型”列；
- 让撤单操作列离开可视区域；
- 恢复本地成交资产列中的方向前缀；
- 破坏 `VirtualList` 窗口化、滚动链或 `overscan`；
- 未运行架构检查和 Chromium 浏览器回归就合并滚动交互修改。
