# Economy 劳动力、基础产品与货币分离设计

> 状态：核心经济循环设计基线  
> 当前客户端：网页端  
> 服务器约束：2 核 2G  
> 更新时间：2026-07-10

## 1. 设计目标

玩家通过点击工作获得劳动力，再使用劳动力生产基础产品。点击本身不能直接产生货币。

货币只在商品成交、有限奖励或其他明确经济来源中进入玩家账户，从而避免玩家通过高频点击直接制造大量货币，引发通胀。

核心原则：

```text
点击只创造劳动力
劳动力只创造产品
产品通过市场成交后才可能获得货币
```

## 2. 三层经济结构

游戏经济分为三个相互独立的层级。

### 2.1 劳动力

劳动力是玩家个人的生产投入资源。

规则：

- 通过点击工作获得。
- 不可交易。
- 不可挂单。
- 不可赠送。
- 不计入总资产。
- 不能直接兑换货币。
- 只能用于生产商品或后续劳动型行为。
- 存在持有上限，达到上限后继续点击不再增加。

劳动力的作用是限制商品生产速度，而不是充当第二种货币。

### 2.2 基础产品

基础产品由劳动力生产，是第一种可以进入共享市场的资产。

MVP 基础产品为食品包。

规则：

- 消耗劳动力生产。
- 进入玩家仓库。
- 可以提交卖单。
- 可以被玩家或城市人口购买。
- 被人口购买后退出流通。
- 占用仓库容量。
- 市场价格由供需和订单簿决定。

### 2.3 货币

货币用于交易、升级和长期经营。

货币来源包括：

- 向其他玩家出售商品。
- 向城市人口消费需求出售商品。
- 有限任务或运营奖励。

货币不能通过以下行为直接产生：

- 点击工作。
- 获得劳动力。
- 创建生产任务。
- 完成生产但尚未出售商品。
- 在客户端本地完成动画或预测。

## 3. 核心玩法循环

```text
点击工作
-> 获得劳动力
-> 消耗劳动力生产食品包
-> 食品包进入仓库
-> 在共享市场提交卖单
-> 与玩家买单或人口消费买单成交
-> 获得货币
-> 使用货币升级设施和仓库
-> 提升经营能力
```

该循环将“玩家活跃度”“商品供给”和“货币发行”拆开。

即使玩家大量点击，也只会增加劳动力和潜在商品供给，不会直接增加货币总量。

## 4. 点击工作规则

### 4.1 基础规则

MVP 建议：

- 每次有效点击获得 1 劳动力。
- 单次点击收益不随货币数量变化。
- 点击达到频率上限后，额外点击不计入正式劳动力。
- 劳动力达到持有上限后，点击按钮显示“劳动力已满”。

### 4.2 工作回合

为了兼顾点击体验、反作弊和 2 核 2G 服务器压力，点击工作按短工作回合结算。

MVP 建议：

| 项目 | 初始建议 |
|---|---:|
| 一个工作回合 | 10 秒 |
| 每回合最多有效点击 | 20 次 |
| 每次有效点击 | 1 劳动力 |
| 单回合最高劳动力 | 20 |
| 劳动力持有上限 | 100 |

客户端可以即时播放点击动画并显示“待确认劳动力”，但只有服务器确认后的劳动力才能用于正式生产。

超过回合上限的点击只提供视觉反馈，不产生额外正式资源。

### 4.3 点击节奏

点击应提供短时参与感，但不要求玩家长时间持续机械操作。

设计目标：

- 新玩家通过少量点击即可开始第一次生产。
- 高频点击不会直接产生货币。
- 宏或自动点击不能突破每回合有效上限。
- 后期经营重点逐渐转向生产规划和市场交易，而不是无限点击。

## 5. 劳动力持有上限

劳动力必须有上限，避免玩家长期囤积无限生产能力。

MVP 建议初始上限为 100。

达到上限后：

- 不能继续获得正式劳动力。
- 可以继续查看点击动画，但明确提示不再增加。
- 玩家需要先生产商品消耗劳动力。

后续升级可以适度提高劳动力上限，但不建议无限提高单次点击收益。

推荐升级方向：

- 提高劳动力上限。
- 增加生产队列。
- 提高仓库容量。
- 缩短生产时间。

谨慎使用的升级方向：

- 每次点击直接获得多倍劳动力。
- 自动无限获得劳动力。
- 使用货币直接购买大量劳动力。

## 6. 基础产品兑换规则

MVP 食品包生产规则：

```text
3 劳动力 -> 1 食品包
```

示例：

| 当前劳动力 | 最多可生产食品包 |
|---:|---:|
| 2 | 0 |
| 3 | 1 |
| 15 | 5 |
| 100 | 33，剩余 1 劳动力 |

生产只改变劳动力和商品库存，不产生货币。

```text
生产前：劳动力 15，食品包 0，货币 0
生产 5 个后：劳动力 0，食品包 5，货币仍为 0
```

只有食品包完成市场成交后，玩家才获得货币。

## 7. 货币产生与流转

### 7.1 玩家之间成交

玩家之间的交易只转移现有货币，不增加全服货币总量。

```text
买家货币减少
卖家货币增加
全服货币总量不变
```

### 7.2 人口消费成交

城市人口消费会向经济中注入新货币，但受到每个消费周期预算限制。

```text
人口消费预算减少
卖家货币增加
商品退出流通
全服货币总量有限增加
```

因此，正式货币发行量主要由人口消费预算决定，而不是由玩家点击次数决定。

### 7.3 系统奖励

任务和活动奖励也会增加货币总量，因此必须：

- 数量有限。
- 有明确领取条件。
- 不按无限点击次数重复发放。
- 记录货币来源。

## 8. 通胀控制机制

### 8.1 点击不铸币

无论玩家点击多少次，点击只增加劳动力，不直接增加货币。

### 8.2 生产不铸币

生产行为只把劳动力转化为商品，不直接增加货币。

### 8.3 人口预算限制货币注入

人口每个周期只能使用有限消费预算，不会无限收购玩家商品。

### 8.4 玩家交易不增加货币总量

玩家之间买卖只重新分配现有货币。

### 8.5 商品供给受多重限制

商品供给受到以下限制：

- 劳动力获取速度。
- 劳动力持有上限。
- 单件商品劳动力成本。
- 生产队列。
- 生产时间。
- 仓库容量。

### 8.6 货币回收

货币通过以下玩法退出流通：

- 生产设施升级。
- 仓库升级。
- 后续企业扩建。
- 后续市场服务费。
- 后续城市许可或维护费用。

MVP 可以主要依靠设施和仓库升级回收货币，暂不强制加入复杂税费。

## 9. 避免商品无限过剩

劳动力与货币分离可以避免直接通胀，但大量点击仍可能造成商品供给过剩。

因此还需要：

- 劳动力持有上限。
- 工作回合有效点击上限。
- 仓库容量。
- 生产队列限制。
- 人口消费需求上限。
- 市场价格随供给增加而下降。

当大量玩家生产食品包时：

- 卖单数量增加。
- 低价卖单优先成交。
- 人口不会无限提高需求。
- 高价卖单可能长期无法成交。
- 玩家会自然减少生产或降低价格。

商品过剩应通过市场价格和库存压力反馈给玩家，而不是由系统按固定价格无限回收。

## 10. 玩家策略

该结构允许玩家形成不同策略。

### 10.1 活跃生产者

- 通过点击快速获得劳动力。
- 持续生产食品包。
- 以较低利润快速出售。

### 10.2 市场观察者

- 先积累劳动力。
- 在需求上升时生产。
- 避免供给过剩阶段大量入场。

### 10.3 综合经营者

- 点击获得一部分劳动力。
- 根据人口需求和市场盘口决定生产数量。
- 使用利润升级仓库与设施。

## 11. 客户端与服务器职责

### 11.1 客户端负责

- 点击动画。
- 回合进度显示。
- 待确认劳动力显示。
- 可生产数量预览。
- 生产成本计算。
- 预计销售收入。
- 当前供需与利润提示。

### 11.2 服务器负责

- 有效工作回合。
- 最终确认的劳动力数量。
- 劳动力持有上限。
- 正式生产任务。
- 劳动力扣除。
- 商品库存增加。
- 市场成交。
- 正式货币变化。

客户端不能自行上报任意劳动力数量，也不能把本地点击动画结果直接用于生产。

## 12. 服务器性能约束

在 2 核 2G 条件下，不应把每一次鼠标点击都设计成独立的全服市场事件。

产品层面采用工作回合：

- 客户端本地即时反馈。
- 回合结束后确认一次正式结果。
- 点击不会触发全服广播。
- 其他玩家不需要实时看到某位玩家每次点击。
- 只有劳动力确认、生产和市场成交属于正式状态变化。

工作回合不能影响订单撮合优先级，也不能阻塞交易处理。

## 13. 页面交互

生产页顶部显示三种资源：

| 资源 | 说明 |
|---|---|
| 劳动力 | 点击工作获得，仅用于生产 |
| 食品包 | 劳动力生产，可进入市场交易 |
| 货币 | 商品成交获得，用于交易和升级 |

工作区域显示：

- 当前正式劳动力。
- 劳动力上限。
- 当前工作回合剩余时间。
- 本回合有效点击数。
- 本回合最多有效点击数。
- 待确认劳动力。
- 点击工作按钮。

生产区域显示：

- 3 劳动力生产 1 食品包。
- 当前最多可生产数量。
- 仓库剩余空间。
- 生产数量输入。
- 生产确认按钮。

页面必须明确提示：

```text
点击不会直接获得货币。
出售生产出的商品后才能获得货币。
```

## 14. 新手引导

1. 点击工作 3 次并获得 3 劳动力。
2. 使用 3 劳动力生产 1 个食品包。
3. 查看食品包进入仓库。
4. 确认货币仍未增加。
5. 打开共享市场。
6. 将食品包提交卖单。
7. 与人口或其他玩家成交。
8. 查看货币增加和成交流水。

新手引导必须让玩家理解“劳动不是钱，产品成交后才有钱”。

## 15. MVP 初始数值建议

| 项目 | 初始建议 |
|---|---:|
| 每次有效点击劳动力 | 1 |
| 工作回合 | 10 秒 |
| 每回合最多有效点击 | 20 |
| 劳动力持有上限 | 100 |
| 食品包劳动力成本 | 3 |
| 初始仓库容量 | 20 |
| 食品包参考价 | 5 |
| 人口消费周期 | 5 分钟 |
| 人口食品周期预算 | 100 货币 |

这些数值只用于第一轮测试，应根据以下指标调整：

- 玩家首次生产所需时间。
- 玩家每小时平均商品产量。
- 人口需求满足率。
- 商品平均成交价。
- 全服货币增长速度。
- 商品库存积压速度。
- 设施升级的回本时间。

## 16. 经济监测指标

正式测试阶段需要持续观察：

- 全服劳动力产生量。
- 全服劳动力消耗量。
- 食品包生产量。
- 食品包人口消费量。
- 玩家间成交量。
- 未成交卖单库存。
- 人口消费注入货币量。
- 升级回收货币量。
- 全服货币总量。
- 食品包成交均价。

如果货币增长过快，应优先调整人口预算和货币回收，而不是降低点击乐趣。

如果商品库存增长过快，应优先调整劳动力上限、生产成本、仓库容量或人口需求，而不是允许系统无限购买。

## 17. 验收标准

1. 点击工作只增加劳动力，不增加货币。
2. 劳动力不能交易、赠送或挂单。
3. 劳动力不计入总资产。
4. 劳动力达到上限后不能继续正式增加。
5. 每个工作回合的有效点击不能超过上限。
6. 3 劳动力可以生产 1 个食品包。
7. 生产食品包后货币不增加。
8. 食品包成交后卖方才获得货币。
9. 玩家之间成交不改变全服货币总量。
10. 人口成交产生的货币不超过周期预算。
11. 人口购买后的食品包退出流通。
12. 客户端修改点击显示不能增加正式劳动力。
13. 客户端修改劳动力不能创建正式商品。
14. 高频点击不能突破工作回合上限。
15. 大量生产会形成库存和价格压力，而不是由系统无限回收。
16. 刷新或断线后，正式劳动力、库存和货币以服务器状态恢复。
17. 每次货币增加都有成交或奖励流水。
18. 每次劳动力消耗都有生产记录。
19. 服务器压力下可以暂停工作结算，但不能重复发放劳动力。
20. 点击玩法不会直接推动货币通胀。

## 18. 当前开发决策

- 点击工作只产生劳动力。
- 劳动力和货币完全分离。
- 劳动力不可交易且不计入总资产。
- 劳动力只能用于生产基础产品。
- 基础产品成交后才产生玩家收入。
- 玩家交易只转移货币，人口消费有限注入货币。
- 工作采用短回合和有效点击上限。
- MVP 使用 3 劳动力生产 1 食品包。
- MVP 劳动力上限为 100，工作回合为 10 秒，每回合最多 20 次有效点击。
- 所有数值最终以实际试玩和经济数据调整为准。
