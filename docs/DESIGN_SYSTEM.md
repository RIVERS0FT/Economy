# 金融帝国界面设计规范

本文件记录已经确定的界面规则。涉及相同组件的后续修改应优先更新本文件，再修改实现，避免样式在不同 PR 中反复回退。

## 1. 卡片圆角

- 桌面端卡片统一使用 `1.5rem`。
- 移动端卡片统一使用 `2.5rem`。
- 圆角统一由 `src/styles/card-system.css` 中的 `--card-radius` 控制。
- 输入框、按钮、头像、状态标签等控件不是卡片，不跟随大圆角变量。

## 2. 移动端顶部资产卡片与底部导航卡片

顶部资产卡片指 `.asset-bar.panel`，底部导航卡片指 `.sidebar.panel`。

### 2.1 外边距

- 顶部、左侧、右侧和底部使用同一个变量：`--mobile-chrome-inset`。
- 当前设计值为 `.4rem`。
- 底部导航在有系统安全区时使用：
  `max(--mobile-chrome-inset, env(safe-area-inset-bottom))`。
- 不允许分别写死顶部 `.4rem`、底部 `.35rem` 等近似值。
- 内容底部预留空间必须根据导航高度、统一外边距、安全区和导航间隔计算，不能重新改回固定 `68px`。

### 2.2 卡片本体与外部透明渐变层

- 顶部资产卡片和底部导航卡片本体必须使用纯色、完全不透明的 `--mobile-chrome-surface`。
- 卡片自身的 `background` 不得包含透明度渐变、遮罩淡出或整体 `opacity`。
- 透明渐变必须绘制在卡片边界之外，并位于卡片层级下方：
  - 顶部资产卡片由 `.workspace::before` 绘制外部渐变层；
  - 底部导航卡片由 `.sidebar.panel::after` 绘制外部渐变层。
- 外部渐变层从卡片背景纯色过渡到透明，尺寸统一由 `--mobile-chrome-fade-size` 控制。
- 渐变层只负责卡片边界外的过渡，不得覆盖或改变卡片本体内部的颜色。
- 文本、图标、按钮、导航项目和卡片主体始终保持完全不透明。
- 不得再将渐变写入 `.asset-bar.panel` 或 `.sidebar.panel` 的 `background`。

### 2.3 层级与滚动透视

移动端层级必须保持：

1. 卡片本体；
2. 卡片外部透明渐变层；
3. 页面滚动内容。

- 页面内容应位于外部渐变层后方，滚动时可以透过渐变层被看到。
- 页面内容不得透过不透明的卡片本体显示。
- 顶部资产卡片与页面滚动区之间不得重新加入会隔开内容的实体空白层。
- `.page-scroll` 仍是登录后页面唯一的纵向滚动容器。
- 底部导航不得遮挡页面最后一张内容卡片。
- `scroll-padding-bottom` 必须和工作区底部预留空间使用同一套计算公式。
- 顶部资产栏可横向滚动，但其背景与外部渐变层不得随横向滚动改变。

### 2.4 卡片内部元素

- 顶部资产项目允许使用纯色半透明高亮，但不得使用连续渐变。
- 底部当前导航项使用纯色半透明高亮 `rgba(50, 159, 88, .18)`。
- 不得恢复 `.sidebar-nav-button.active` 的线性渐变背景。
- 顶部和底部卡片不使用阴影或背景模糊，以避免形成遮挡内容的厚重浮层。

## 3. 导航文案

底部导航和桌面侧边导航共用同一套简短中文名称。当前固定文案如下：

- `home`：`概览`
- `market`：`市场`
- `production`：`生产`
- `assets`：`资产`
- `leaderboard`：`排行`
- `records`：`订单`
- `settings`：`设置`

不得恢复为“主页面”“排行榜”“订单与记录”等较长文案。移动端空间有限，导航名称应保持两个汉字以内；需要解释功能时应放在页面标题或说明文本中，而不是扩展导航标签。

## 4. 实现位置

- 圆角系统：`src/styles/card-system.css`
- 移动端上下卡片边距、层级、外部渐变和滚动预留：`src/styles/viewport.css`
- 基础布局与组件样式：`src/styles/globals.css`
- 品牌与导航文案同步：`src/main.tsx`

## 5. 修改检查清单

修改移动端顶部或底部卡片前，必须确认：

1. 上下卡片外边距是否仍由同一变量控制；
2. 卡片本体是否仍为纯色且完全不透明；
3. 透明渐变是否仍位于卡片边界之外和卡片层级下方；
4. 页面内容是否只能透过外部渐变层被看到；
5. 文本和控件是否保持完全不透明；
6. 是否没有把渐变重新写回卡片本体的 `background`；
7. 最后一张内容卡片是否能完整滚动到导航上方；
8. 是否同步更新了本设计文档。

修改导航文案前，必须确认：

1. 是否仍使用上述固定中文名称；
2. 移动端标签是否保持两个汉字以内；
3. 可访问名称是否与可见标签一致；
4. 是否同步更新了本设计文档。
