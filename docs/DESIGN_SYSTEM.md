# 金融帝国界面设计规范

本文件记录已确定的界面规则。后续修改必须先核对本规范，避免视觉和交互在不同提交中反复回退。

## 1. 界面语言

- 玩家可见界面统一使用中文，不允许中英文夹杂。
- 按钮、表单标签、占位提示、状态说明、单位和空状态都必须使用中文。
- 一级页面标题上方不显示绿色小字、眉题或副标签，只保留页面主标题与说明文字。
- 卡片标题上方同样不显示由英文翻译而来的绿色小字或眉题，只保留卡片主标题和必要操作。
- `PageLayout` 与 `WidgetHeading` 均不得渲染 `.eyebrow` 元素。
- 技术字段值不得直接显示为英文，例如账号角色必须转换为“管理员”或“普通用户”。
- 时间单位使用“秒、分钟、小时”，不得显示单独的英文单位字母。
- 大额数字说明使用“万、百万”等中文单位，不得显示英文字母缩写。
- 用户邮箱、玩家自定义名称和外部数据原文不属于固定界面文案，不强制改写。

## 2. 卡片圆角

- 桌面端卡片统一使用 `1.5rem`。
- 移动端卡片统一使用 `2.5rem`。
- 圆角由 `src/styles/card-system.css` 中的 `--card-radius` 控制。
- 输入框、按钮、头像和状态标签不是卡片，不跟随大圆角变量。

## 3. 移动端顶部状态栏与底部导航栏

顶部状态栏指 `.asset-bar.panel`，底部导航栏指 `.mobile-bottom-navigation.panel`。

### 3.1 外边距与安全区

- 上下栏基础外边距由 `--mobile-chrome-inset: 1rem` 控制。
- 页面内容左右间距由 `--mobile-content-inset: .4rem` 独立控制。
- 顶部状态栏顶部位置必须使用基础外边距和 `env(safe-area-inset-top)` 中的较大值。
- 顶部状态栏左右位置必须分别使用基础外边距与 `env(safe-area-inset-left)`、`env(safe-area-inset-right)` 中的较大值。
- 横屏全面屏设备上，状态栏关键信息不得进入摄像头开孔、刘海或圆角安全区。
- 底部导航左右位置使用相同的左右安全区计算，底部位置使用基础外边距和 `env(safe-area-inset-bottom)` 中的较大值。
- 移动端 `.game-shell` 必须完整覆盖视口，不得通过应用外层内边距制造卡片外边距。

### 3.2 液态玻璃背景

- 顶部状态栏和底部导航栏共用同一套液态玻璃背景、边框、模糊、高光和阴影参数。
- 背景使用统一半透明纯色，不得使用从不透明到透明的渐变。
- `backdrop-filter` 与 `-webkit-backdrop-filter` 必须同时保留。
- 不得改为完全不透明背景，否则后方内容无法参与混合，玻璃效果会消失。
- 上下栏必须保留完整四边边框和连续圆角弧线。

### 3.3 顶部状态栏

- 顶部状态栏只有一个液态玻璃外层，内部不得嵌套信息卡片。
- `.asset-bar-item` 不得使用独立背景、边框、圆角或阴影。
- 状态项采用“无底座图标 + 数字或文字”的单行结构。
- 状态名称与辅助信息在移动端隐藏，但必须通过可访问名称保留完整语义。
- 移动端紧凑值不得重复显示已经由图标表达的货币或分类符号。
- 桌面完整值和移动端紧凑值分别由 `value` 与 `compactValue` 提供。
- 状态项图标必须由数据显式传入，不得依赖 CSS 顺序选择器生成。
- 状态栏高度由 `--mobile-asset-bar-height` 控制，当前固定为 `56px`。
- 状态项使用自动列布局：内容能够容纳时，各项均匀填充整张状态栏卡片，而不是堆积在左侧。
- 单个状态项内容居中；空间不足时允许横向滚动，不得压缩或截断核心数字和文字。

### 3.4 底部导航栏

- 底部导航卡片本身只保留上下内边距，不使用固定左右内边距。
- 左右空白属于滚动内容，由 `.sidebar-nav::before` 和 `.sidebar-nav::after` 提供。
- 左右空白由 `--mobile-nav-scroll-gutter: 1.25rem` 控制，并随按钮一起滚动。
- 滚动到最左或最右时，首尾按钮与卡片圆角之间必须仍有完整空白。
- 导航图标不使用独立底座。
- 选中项使用按钮级半透明背景、完整边框和图标高亮。
- 底部导航栏高度由 `--mobile-nav-height` 控制，当前为 `76px`。

### 3.5 滚动规则

- `.page-scroll` 是登录后页面唯一的纵向页面滚动容器。
- `.page-scroll` 使用 `overscroll-behavior: contain`。
- `.asset-bar`、`.sidebar-nav`、`.table-wrap`、`.ledger-list` 和 `.market-stat-strip` 使用 `overscroll-behavior: auto`。
- 非页面滚动容器到达边界后必须释放后续滚动手势。
- 页面底部预留空间必须保证最后一张内容卡片能够完整滚动到导航栏上方。

## 4. 桌面侧边栏身份信息

- 品牌区第一行固定显示“金融帝国”，第二行显示当前玩家用户名。
- 不得继续显示固定副标题“市场交易版”。
- 品牌区下方不得存在独立玩家信息卡片、玩家头像卡片或重复用户名区域。
- 用户排名只在顶部状态栏、排行榜和其他业务区域展示，不在侧边栏品牌区重复显示。
- 用户名为空时显示“玩家”作为兜底名称。
- 用户名过长时使用单行省略号截断，并通过 `title` 保留完整文本。
- 删除独立玩家卡片后，导航区域与品牌区之间保留 `.85rem` 的垂直间距。
- 桌面侧边栏身份样式集中在 `src/styles/desktop-sidebar.css`。

## 5. 市场订单簿

- 商品订单簿按价格档位展示，不按单笔订单逐行展示。
- 同一方向、同一价格的所有未成交订单必须聚合为一个价格档位。
- 档位数量为该价格下所有订单 `remaining` 的总和。
- 每个档位可以显示聚合订单笔数，但不得显示买家、卖家或订单所有者名称。
- 买盘按价格从高到低排列，卖盘按价格从低到高排列。
- 买一价和卖一价仍以聚合前后的最佳有效价格为准。

## 6. 导航文案

桌面侧边导航和移动端底部导航共用 `src/config/navigation.ts` 中的配置：

- `home`：`概览`
- `market`：`市场`
- `production`：`生产`
- `assets`：`资产`
- `leaderboard`：`排行`
- `records`：`订单`
- `settings`：`设置`

可见标签、可访问名称和页面编号必须来自同一配置，不得运行时扫描并修改页面节点。

## 7. 页面架构

- `src/main.tsx` 只负责性能初始化、样式加载和应用挂载。
- 禁止使用 `MutationObserver`、`querySelector` 或文本匹配对已渲染界面进行二次修补。
- 七个一级业务页面分别位于 `src/pages/`，由 `PageRouter` 统一选择。
- 游戏初始化、定时处理、衍生数据和局部界面状态集中在 `useGameViewModel`。
- 桌面侧边栏和移动端底部导航是独立组件，但共用导航配置与选择状态。
- 顶部状态栏由状态项数组渲染，状态编号、图标、名称、数值和强调状态必须显式声明。
- `PageLayout` 只渲染一级页面主标题、说明文字和可选操作区，不渲染页面眉题。
- `WidgetHeading` 只渲染卡片主标题和可选操作，不渲染卡片眉题。
- 页面统一使用 `PageLayout`、`Panel`、`WidgetHeading`、`ScrollableTable` 和 `EmptyState`。
- 格式化、价格曲线和设施进度等跨页面逻辑使用独立模块，不得复制到页面文件中。

## 8. 样式职责

- `src/styles/globals.css`：通用组件和业务卡片视觉。
- `src/styles/desktop-sidebar.css`：桌面侧边栏品牌用户名、文本溢出和导航间距。
- `src/styles/viewport.css`：视口、定位、滚动区和基础安全区预留。
- `src/styles/mobile-status-navigation.css`：移动端设计令牌、液态玻璃、状态栏和导航基础视觉。
- `src/styles/mobile-status-layout.css`：状态栏均匀填充、全面屏安全区和最终紧凑尺寸。
- `src/styles/mobile-pages.css`：移动端业务页面响应式规则。
- `src/styles/card-system.css`：卡片圆角系统。

移动端视觉令牌只能在 `mobile-status-navigation.css` 中定义；最终布局文件只能引用令牌，不得重复定义高度和间距变量。

## 9. 移动端业务页面布局

- 桌面双列或多列业务布局必须在 `720px` 及以下显式改为单列。
- 生产页 `.production-grid` 在移动端使用 `grid-template-columns: minmax(0, 1fr)`。
- `.facility-list` 和 `.facility-card` 必须保持 `width: 100%` 与 `min-width: 0`。
- 移动端 `.build-card` 不得保持桌面端粘性定位。
- 玩家已有设施时，设施卡片必须在建造卡片之后通过页面纵向滚动完整显示。

## 10. 修改检查清单

提交界面修改前必须确认：

1. 玩家可见固定文案是否全部为中文；
2. 页面和卡片标题上方是否都没有绿色小字或眉题；
3. 桌面侧边栏品牌区第二行是否显示当前用户名；
4. 是否已删除独立玩家用户名和头像卡片；
5. 商品订单簿是否按同一价格聚合数量且不显示买卖家；
6. 顶部状态栏高度是否仍为 `56px`；
7. 状态项是否均匀填充状态栏而不是左侧堆积；
8. 状态项内容是否居中且没有内部底座；
9. 顶部和左右安全区是否使用 `safe-area-inset`；
10. 摄像头开孔、刘海和圆角区域是否不会遮挡关键信息；
11. 上下栏是否仍共用液态玻璃参数且没有透明度渐变；
12. 底部导航首尾空白是否随内容滚动；
13. 非页面滚动容器是否在边界释放手势；
14. 移动端生产页是否保持单列且设施卡片可见；
15. TypeScript、生产构建和架构验收是否全部通过。
