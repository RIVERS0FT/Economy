# 金融帝国界面设计规范

本文件记录已经确定的界面规则。涉及相同组件的后续修改应优先更新本文件，再修改实现，避免样式在不同 PR 中反复回退。

## 1. 卡片圆角

- 桌面端卡片统一使用 `1.5rem`。
- 移动端卡片统一使用 `2.5rem`。
- 圆角统一由 `src/styles/card-system.css` 中的 `--card-radius` 控制。
- 输入框、按钮、头像、状态标签等控件不是卡片，不跟随大圆角变量。

## 2. 移动端顶部资产卡片与底部导航卡片

顶部资产卡片指 `.asset-bar.panel`，底部导航卡片指 `.sidebar.panel`。

### 2.1 外边距与应用画布

- 顶部资产卡片和底部导航卡片使用同一个变量：`--mobile-chrome-inset`。
- `--mobile-chrome-inset` 当前固定为 `1rem`，与移动端登录页 `.login-shell` 的 `margin: 1rem auto` 保持一致。
- 页面滚动内容左右外边距使用独立变量：`--mobile-content-inset`。
- `--mobile-content-inset` 当前固定为 `.4rem`，不得跟随卡片外边距增大。
- 底部导航在有系统安全区时使用：
  `max(--mobile-chrome-inset, env(safe-area-inset-bottom))`。
- 移动端 `.game-shell` 必须使用 `padding: 0` 并完整覆盖视口。
- 卡片外边距只能施加在顶部资产卡片和底部导航卡片自身，不能施加在整个应用画布上。
- 页面内容外边距只能通过 `.page-scroll` 的左右 padding 设置，不能复用 `--mobile-chrome-inset`。
- 不允许为顶部资产卡片和底部导航分别写入不同的近似外边距。
- 不得使用应用外层 padding 制造卡片外边距，否则会在视口上下或四周露出贯穿整屏的深色边带。
- 内容底部预留空间必须根据导航高度、卡片外边距、安全区和导航间隔计算，不能重新改回固定 `68px`。

### 2.2 卡片内部透明度渐变

- 顶部资产卡片和底部导航卡片的背景必须在卡片本体内部完成渐变。
- 渐变方向固定为从上到下：
  - `0%` 使用完整的 `--mobile-chrome-surface`；
  - `100%` 使用完全透明的同色背景。
- 渐变必须覆盖卡片内部完整的 `0%–100%` 高度，不再使用独立的外部渐变层。
- 不得重新添加 `.workspace::before` 或 `.sidebar.panel::after` 作为渐变尾部。
- 只能修改卡片 `background` 的透明度，不得对整个卡片使用 `opacity`。
- 文字、图标、按钮、导航项目和其他交互内容必须始终完全不透明。
- 顶部资产卡片和底部导航卡片必须使用同一套渐变色值和方向。

### 2.3 边框

- 顶部资产卡片和底部导航卡片必须保留完整四边边框。
- 边框统一使用 `1px solid rgba(212, 245, 224, .12)`。
- 背景在底部变透明时，边框不得随之变透明。
- 不得使用 `border-bottom-color: transparent` 或其他方式隐藏任意一边。
- 底部左右圆角处的边框弧线必须连续，不能提前中断。

### 2.4 层级与滚动透视

- 移动端 `.workspace` 和 `.page-scroll` 必须完整覆盖应用视口。
- 顶部资产卡片使用绝对定位覆盖在滚动内容上方，并单独使用 `--mobile-chrome-inset` 定位。
- 页面初始内容通过顶部 padding 从资产卡片下方开始；滚动后内容可以进入卡片后方，并透过卡片下半部的透明背景被看到。
- 底部导航使用固定定位悬浮在滚动内容上方，并单独使用相同卡片外边距变量定位。
- `.page-scroll` 仍是登录后页面唯一的纵向滚动容器。
- `.page-scroll` 自身铺满视口，左右内容间距统一使用 `--mobile-content-inset: .4rem`，不能通过 `.game-shell` padding 或 `--mobile-chrome-inset` 实现。
- 底部导航不得遮挡页面最后一张内容卡片。
- `scroll-padding-bottom` 必须和页面底部预留空间使用同一套计算公式。
- 顶部资产栏可横向滚动，但背景渐变不得随横向滚动改变。

### 2.5 卡片内部元素

- 顶部资产项目允许使用纯色半透明高亮，但不得使用连续渐变。
- 底部当前导航项使用纯色半透明高亮 `rgba(50, 159, 88, .18)`。
- 不得恢复 `.sidebar-nav-button.active` 的线性渐变背景。
- 顶部和底部卡片不使用阴影或背景模糊，以避免形成遮挡内容的厚重浮层。

## 3. 导航文案

底部导航和桌面侧边导航共用同一套简短中文名称。当前固定文案如下：

- `home`：`概览`
- `market`：`市场`
- `production`：`生产`
- `assets`：`资产`
- `leaderboard`：`排行`
- `records`：`订单`
- `settings`：`设置`

不得恢复为“主页面”“排行榜”“订单与记录”等较长文案。移动端空间有限，导航名称应保持两个汉字以内；需要解释功能时应放在页面标题或说明文本中，而不是扩展导航标签。

## 4. 实现位置

- 圆角系统：`src/styles/card-system.css`
- 移动端上下卡片边距、页面内容边距、内部渐变、完整边框、层级和滚动预留：`src/styles/viewport.css`
- 基础布局与组件样式：`src/styles/globals.css`
- 品牌与导航文案同步：`src/main.tsx`

## 5. 修改检查清单

修改移动端顶部、底部卡片或页面内容间距前，必须确认：

1. `.game-shell` 是否仍为 `padding: 0` 并完整铺满视口；
2. 上下卡片外边距是否仍由同一个 `--mobile-chrome-inset` 控制；
3. `--mobile-chrome-inset` 是否仍为与登录页一致的 `1rem`；
4. 页面内容左右外边距是否仍由独立的 `--mobile-content-inset` 控制；
5. `--mobile-content-inset` 是否仍为 `.4rem`；
6. 是否没有用卡片外边距变量替代页面内容外边距；
7. 卡片外边距是否只应用于卡片自身，而不是应用画布；
8. 卡片背景是否从 `0%` 到 `100%` 在内部连续降低透明度；
9. 是否没有外部伪元素渐变层；
10. 页面内容是否能够滚动到卡片后方并透过透明区域显示；
11. 文本和控件是否保持完全不透明；
12. 是否没有对整个卡片设置 `opacity`；
13. 四边边框和底部圆角弧线是否完整连续；
14. 是否没有将任意边框颜色设置为透明；
15. 视口顶部和底部是否没有贯穿全宽的深色边带；
16. 最后一张内容卡片是否能完整滚动到导航上方；
17. 是否同步更新了本设计文档。

修改导航文案前，必须确认：

1. 是否仍使用上述固定中文名称；
2. 移动端标签是否保持两个汉字以内；
3. 可访问名称是否与可见标签一致；
4. 是否同步更新了本设计文档。
