# Economy 资产页设计

> 状态：资产结果、冻结口径与本地变化记录的当前实现基线
> 适用项目：`RIVERS0FT/Economy`
> 更新时间：2026-07-19

## 1. 页面职责

资产页只负责展示当前玩家普通经济资产的结果、估值组成和当前浏览器保存的资产变化记录。

页面不得承担仓库管理、逐商品库存浏览、建厂、生产控制、下单、撤单、藏品管理或宝石资产展示。商品具体库存继续只属于生产页共享仓库；宝石不计入普通经济总资产。

## 2. 一级结构

资产页固定只有两个一级 `Panel`：

```text
资产总览
本地资产变动
```

不得把资金摘要、配置圆环和估值明细拆成三组重复一级卡片。

## 3. 资产总览

资产总览必须在同一张卡片中完成三层信息：

1. 唯一主指标：当前总资产；
2. 两个次级指标：可支配资产、冻结资产；
3. 资产配置比例与现金／商品／工厂构成明细。

同一数值口径不得在资产总览中重复出现：

- 当前总资产只显示一次；
- 可支配资产只显示一次；
- 冻结资产合计只显示一次；
- 现金、商品、工厂各自的总计、可用和冻结值只在对应构成行显示一次。

资产配置圆环只表达现金、商品和工厂的估值比例，圆环中心使用“资产配置／按估值占比”，不得再次显示总资产或现金占比。圆环旁的图例负责显示三类比例。

## 4. 资产构成

资产构成固定为现金、商品和工厂三行，列为：

```text
类型｜总计｜可用｜冻结
```

- 现金总计使用现金资产口径，可用与冻结分别使用可用资金和冻结资金；
- 商品总计使用全部商品估值，可用与冻结使用服务器资产摘要拆分，并可辅助显示冻结商品数量；
- 工厂总计使用工厂资产估值，可用与冻结使用服务器资产摘要拆分，并可辅助显示冻结座数／总座数；
- 商品与工厂继续按订单簿最高非本人有效买入价估值；
- 不得恢复逐商品估值卡或逐工厂类型估值卡。

桌面使用四列表格语义。移动端每行改为两行布局：第一行显示类型与总计，第二行显示可用与冻结；不得依赖页面横向滚动。

## 5. 冻结说明

“冻结资产仍归当前玩家所有并计入总资产；冻结只限制交易、生产或使用。”只在资产总览底部显示一次。

不得在总资产、冻结资产、估值明细和页面说明中重复维护多份相同冻结解释。

## 6. 本地资产变动

本地资产变动继续保持独立一级卡片，保留：

- 全部／资金／商品／仓库／工厂／生产／订单冻结筛选；
- `VirtualList` 窗口化渲染；
- 资金、商品、仓库、工厂和生产变化明细；
- 清除本地记录操作；
- “记录不上传服务器”的隐私说明。

本地变化记录不属于服务器权威资产账本，更换设备、无痕模式或清除网站数据后可以丢失。

## 7. 样式职责

`src/styles/assets.css` 负责资产总览、构成表、桌面／移动重排和资产页一级卡片密度。

共享按钮、面板、状态标签、货币图标、虚拟列表和焦点视觉继续由现有设计系统文件负责。资产页样式必须先于 `design-system.css` 与 `form-controls.css` 加载，不得复制共享基础控件视觉。

## 8. 防回退

不得：

- 恢复顶部五张资金／总资产摘要卡；
- 把“资产配置”和“资产估值明细”重新拆为两张一级卡片；
- 在配置圆环标题、圆环中心或其他卡片再次显示总资产；
- 为可用资金、冻结资金、冻结资产或可支配资产分别创建重复指标卡；
- 删除现金、商品、工厂的总计／可用／冻结拆分；
- 把冻结资产从总资产中扣除或暗示冻结等同损失；
- 把宝石计入资产页；
- 恢复逐商品库存与估值卡；
- 合并或删除本地资产变动、筛选、隐私说明或窗口化列表；
- 在移动端依赖横向滚动展示资产构成；
- 未同步更新本文档和验证脚本就改变资产页数据口径或一级结构。
