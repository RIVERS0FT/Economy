# Economy 共享仓库与工厂卡片布局设计

> 状态：生产页面正式布局与防回退基线  
> 更新时间：2026-07-12  
> 客户端状态版本：10  
> 世界状态版本：6

## 页面顺序

页面标题与运行／停止／异常／施工汇总之后，依次为共享仓库、建设新工厂和工厂集群列表。

## 共享仓库

共享仓库是单一顶层面板。桌面内部使用 `minmax(220px, 1fr) minmax(0, 3fr)`。左侧展示等级、容量进度、当前容量、实物库存、买单预占、剩余容量、下一等级容量、升级费用和升级按钮。右侧展示仓库内容。低于 960px 改为上下布局。

商品按服务器目录动态生成卡片。卡片只显示产品名称、可用库存和冻结库存，整卡进入对应商品市场。默认“有库存”仅显示可用或冻结大于零的商品，“全部商品”显示零库存商品。买单预占只保留在左侧汇总，不进入单项卡片。商品卡不得显示估值、价格、单项容量、描述或直接出售。

宽屏四列、普通桌面三列、平板和手机两列，低于 420px 才单列。

## 建设卡

建设新工厂卡独立占一行，不与工厂集群卡等高。所有建造费用、施工时间、周期、单座产量、单座成本、施工任务和建设操作保持不变。

## 工厂卡

宽屏双列、自然高度、顶部对齐；低于 960px 单列。卡片不得减少业务内容，但缩小间距和内边距。

顶部是唯一状态说明区：运行显示当前周期剩余时间；主动停止显示“玩家已关闭生产”；计划完成显示完成原因；异常显示资金、仓库、原料、计划或可用工厂问题及自动恢复语义。正文不得重复 `manual` 或异常原因。

数量区保留持有、参与、待加入和卖单冻结，下一周期变化另以紧凑说明显示。规格区保留周期、产量、成本、原料需求和当前库存。运行时显示进度条；停止和异常只显示紧凑状态行。计划区保留当前计划、完成量、待生效计划和全部编辑控件。底部保留统一订单簿市场入口。

## 样式与版本

仓库布局只由 `src/styles/warehouse-expansion.css` 管理，工厂密度和网格只由 `src/styles/industry-system.css` 管理。基础组件仍由 `src/styles/design-system.css` 管理。商品卡使用已有 `game.products` 与 `game.inventories`，不新增 API 字段。客户端版本保持 10，世界版本保持 6。

## 防回退

必须检查 1:3 分栏、商品卡字段和市场跳转、库存筛选、移动端双列、建设卡独立、工厂双列自然高度、状态原因仅顶部一次、停止和异常无大型空进度条、全部生产字段仍存在，以及统一开关未被复制。
