# Economy 工作货币与冷却经济设计

> 状态：核心货币发行规则专题基线  
> 当前客户端：网页端  
> 服务器约束：2 核 2G  
> 更新时间：2026-07-10

本文件是工作、冷却和基础货币发行的专题设计。身份、页面结构和资产术语以 `WEB_MULTIPLAYER_GAME_DESIGN.md` 为准。

## 1. 规则概述

玩家在主页面的“基础工作”模块中执行工作并直接获得货币，不设置劳动力、体力或能量资源。

```text
执行工作
-> 服务器确认冷却结束
-> 玩家获得货币
-> 建造或购买生产设施
-> 设施生产基础商品
-> 在统一市场交易商品与生产资产
-> 提升玩家总资产和排行榜名次
```

工作收入用于提供启动资金和破产兜底。生产、市场交易与资产投资应成为中后期主要收益来源。

工作不再是独立主导航页面，只作为主页面中的小型功能模块存在。

## 2. 初始测试规则

| 项目 | 建议值 |
|---|---:|
| 每次有效工作收入 | 1 货币 |
| 基础冷却 | 3 秒 |
| 连续工作冷却档位 | 3 / 5 / 8 / 12 秒 |
| 无操作恢复时间 | 5 分钟 |

建议连续工作规则：

- 第 1～20 次有效工作：3 秒。
- 第 21～40 次：5 秒。
- 第 41～60 次：8 秒。
- 第 61 次及以后：12 秒。
- 连续 5 分钟没有有效工作后恢复基础冷却。

所有数值都需要根据试玩和经济数据调整。

## 3. 工作状态

- 可以工作
- 冷却中
- 正在确认
- 网络中断
- 工作暂时暂停

冷却期间再次操作：

- 不增加货币。
- 不重复提交正式收入。
- 不能在客户端伪装为成功。

## 4. 服务器确认规则

服务器负责：

- 判断当前工作请求是否满足冷却。
- 维护连续工作冷却档位。
- 确认本次工作收入。
- 增加正式可用货币。
- 写入资金流水。
- 防止刷新、多标签页和重复请求绕过限制。

客户端只负责按钮状态、倒计时、反馈动画和预估展示。

## 5. 货币来源

MVP 增加全服货币的来源：

1. 玩家工作。
2. 聚合人口需求购买基础商品。
3. 有限的新手任务或运营奖励。

玩家之间的商品交易和生产设施交易只转移已有货币，不增加全服货币总量。

## 6. 货币回收

MVP 货币回收包括：

- 生产设施建造费用
- 生产设施运营费用
- 生产设施升级费用
- 设施槽位扩充费用
- 玩家库存容量升级费用

后续可以增加适度市场手续费和资产维护费用，但不能让新玩家无法启动。

## 7. 工作收入的作用

工作收入主要承担三个作用：

- 帮助新玩家积累第一座生产设施的资金。
- 在经营失败或暂时没有设施时提供恢复途径。
- 为短时上线玩家提供确定性进度。

玩家拥有生产设施后，单位时间内的经营利润应逐渐高于工作收入，使玩家自然转向：

- 建造生产设施
- 收购生产设施
- 生产基础商品
- 商品买卖
- 生产资产交易
- 总资产配置

## 8. 工作收入与设施价格

第一座自建设施的费用和工作收入必须共同测试。

目标：

- 新玩家可以在合理时间内接近第一座设施。
- 不能通过持续工作在短时间内大量收购设施。
- 经营失败后仍能通过工作重新启动。
- 购买现成设施的即时性应由市场价格体现，而不是额外等待。

当前建议基础设施建造费用为 60 货币、施工时间为 5 分钟，仅作为首轮测试基线。

## 9. 通胀控制

工作会直接发行货币，因此需要同时监测：

- 每名玩家每小时有效工作次数
- 每名玩家每小时工作收入
- 全服工作发行货币量
- 人口需求发行货币量
- 设施建造回收量
- 设施运营回收量
- 设施升级和槽位扩充回收量
- 市场手续费回收量
- 全服货币净增长量
- 基础商品参考价与成交价格
- 生产设施成交价格
- 玩家总资产增长速度

货币增长过快时，调整顺序建议：

1. 延长连续工作冷却。
2. 降低单次工作收入。
3. 调整人口需求预算。
4. 调整建造、运营和升级回收。
5. 调整市场手续费或其他可解释回收项。

不得通过删除玩家资产或修改已完成交易处理通胀。

## 10. 防作弊

- 冷却以服务器时间为准。
- 修改客户端时间不能提前工作。
- 同一工作请求最多成功一次。
- 多标签页共享同一冷却。
- 刷新不能重置冷却。
- 断线期间不补发工作收入。
- 不提供离线自动工作。
- 工作异常时可以暂停工作功能，但不能影响已有设施、订单和资产。
- 所有成功工作都必须生成可审计资金流水。

## 11. 2 核 2G 约束

- 工作不是全服广播事件。
- 其他玩家不会看到单次工作操作。
- 工作只更新当前玩家余额、冷却状态和流水。
- 冷却期间重复操作不应形成大量有效请求。
- 工作流水分页加载。
- 服务器压力较高时可以临时延长全服基础冷却，并在页面明确提示。

## 12. 主页面交互

主页面工作模块展示：

- 每次有效工作收入
- 工作按钮
- 冷却进度与剩余时间
- 当前连续工作冷却档位
- 恢复基础冷却的剩余时间
- 今日或本周期工作收入

示例：

```text
基础工作

每次有效工作获得：¤1
当前连续工作档位：2 / 4
下一次可工作：4 秒

[开始工作]
```

成功提示：

```text
工作完成，获得 1 货币
```

冷却提示：

```text
工作冷却中，2.1 秒后可再次工作
```

工作模块不应占据主页面的大部分空间。主页面同时需要展示市场摘要、生产摘要、财富变化和最近活动。

## 13. 新手引导

1. 在主页面执行工作，获得第一笔货币。
2. 查看资金流水和冷却状态。
3. 观察市场价格摘要。
4. 积累第一座生产设施所需资金。
5. 选择自建设施或查看玩家设施挂牌。
6. 自建则等待施工，购买则立即获得设施。
7. 启动设施生产并领取商品。
8. 在统一市场出售基础商品。
9. 使用收益升级、扩充槽位或收购设施。
10. 查看总资产和排行榜名次变化。

## 14. 验收标准

1. 游戏中不存在劳动力、体力或能量资源。
2. 每次有效工作直接增加玩家货币。
3. 冷却未结束时工作不增加货币。
4. 修改本地时间、刷新或多标签页不能重置冷却。
5. 每次工作收入都有资金流水。
6. 工作收入能够支持玩家逐步接近第一座设施。
7. 工作不能直接产生商品或设施。
8. 设施建造、运营和升级能够回收货币。
9. 玩家间商品和设施交易不增加全服货币总量。
10. 人口需求发行量受到周期预算限制。
11. 工作功能暂停不会影响生产、市场和已有资产。
12. 中后期主要收入能够从工作转向生产、交易和资产投资。
13. 工作模块位于主页面，而不是独立主导航页面。
14. 工作收入与总资产变化均由服务器确认。

## 15. 当前开发决策

- 工作直接获得货币。
- 工作收益和冷却由服务器确认。
- 连续工作会提高冷却。
- 不设置劳动力、体力或离线工作收入。
- 工作收入用于启动和兜底。
- 工作功能并入主页面。
- 生产设施建造、运营、升级和槽位扩充是主要货币回收方式。
- 生产经营、市场交易和资产投资是中长期主要成长方式。
