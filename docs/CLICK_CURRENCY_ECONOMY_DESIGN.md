# Economy 点击货币与冷却经济设计

> 状态：核心经济规则基线  
> 当前客户端：网页端  
> 服务器约束：2 核 2G  
> 更新时间：2026-07-10

## 1. 最终规则

玩家点击“工作”后直接获得货币，不再设置劳动力资源。

```text
点击工作
-> 服务器确认冷却结束
-> 玩家获得货币
-> 使用货币生产、购买商品或升级设施
-> 在共享市场交易
-> 通过经营扩大资产
```

点击收入是新玩家的启动资金和低门槛基础收入，不应成为中后期最优赚钱方式。

## 2. 核心原则

### 2.1 不设置劳动力

正式玩家资产中不包含劳动力，不存在劳动力兑换产品、劳动力恢复、劳动力交易或劳动力上限。

### 2.2 点击直接获得货币

每次有效点击直接增加可用货币，并生成可追溯的收入流水。

### 2.3 冷却限制货币发行速度

点击必须经过服务器确认的冷却。冷却未结束时点击不产生正式货币。

### 2.4 市场与经营是主要成长来源

点击只能维持基础收入。生产利润、价格判断、库存管理和玩家交易应逐渐成为主要收益来源。

### 2.5 点击收入必须有货币回收对应

生产投入、设施升级、仓库升级和后续市场服务费用用于回收点击产生的货币。

## 3. MVP 点击规则

第一轮测试建议：

| 项目 | 建议值 |
|---|---:|
| 每次有效点击收入 | 1 货币 |
| 基础冷却 | 3 秒 |
| 连续工作统计窗口 | 5 分钟 |
| 连续点击冷却上限 | 12 秒 |
| 无操作恢复时间 | 5 分钟 |

这些数值是测试起点，不是最终承诺。

## 4. 递增冷却

只使用固定冷却仍可能被脚本全天运行，因此采用“基础冷却 + 连续工作递增冷却”。

建议规则：

- 第 1～20 次连续有效点击：3 秒冷却。
- 第 21～40 次：5 秒冷却。
- 第 41～60 次：8 秒冷却。
- 第 61 次及以后：12 秒冷却。
- 玩家连续 5 分钟没有有效点击后，冷却等级恢复到基础值。

冷却是工作节奏，不是一种可交易资源，也不显示为劳动力或体力。

页面必须显示：

- 当前点击收入。
- 当前冷却时间。
- 冷却剩余时间。
- 连续工作导致冷却延长的提示。
- 距离恢复基础冷却的时间。

## 5. 点击状态

点击按钮包含以下状态：

- 可以工作
- 冷却中
- 正在确认
- 网络中断
- 工作暂时暂停

冷却期间再次点击：

- 不增加货币。
- 不重复提交正式收入请求。
- 可以播放轻量提示，但不能伪装成成功。

## 6. 货币来源

MVP 货币来源包括：

1. 点击工作收入。
2. 向其他玩家出售商品。
3. 向城市人口消费需求出售商品。
4. 有限的新手任务或运营奖励。

其中：

- 玩家间交易只转移已有货币，不增加全服货币总量。
- 点击工作和人口消费会增加全服货币总量。
- 所有系统发行货币必须记录来源。

## 7. 货币回收

MVP 货币回收包括：

- 生产食品包所需的原料或生产费用。
- 生产设施升级。
- 仓库升级。
- 企业升级费用。

后续可增加：

- 市场服务费。
- 上架费用。
- 企业维护费。
- 城市许可证。
- 研究与技术升级。

MVP 不应同时加入过多费用，避免新玩家无法启动，但必须保证点击发行的货币有稳定出口。

## 8. 食品包生产

取消劳动力后，食品包生产改为消耗货币和生产时间。

第一轮测试建议：

| 项目 | 建议值 |
|---|---:|
| 单个食品包生产费用 | 4 货币 |
| 食品包初始参考价 | 5 货币 |
| 单个食品包基础生产时间 | 10 秒 |
| 初始生产队列 | 1 |
| 初始仓库容量 | 20 |

生产流程：

```text
选择生产数量
-> 检查可用货币、队列和仓库空间
-> 扣除生产费用
-> 创建生产任务
-> 等待完成
-> 领取食品包进入仓库
```

生产费用属于货币回收，不进入其他玩家账户。

生产完成不会再次增加货币。玩家需要将食品包出售给玩家或人口消费需求，才能获得销售收入。

## 9. 核心玩法循环

```text
点击工作获得启动货币
-> 支付生产费用生产食品包
-> 食品包进入仓库
-> 观察人口需求与玩家盘口
-> 提交卖单
-> 与人口或玩家成交
-> 获得销售收入
-> 扩大生产、仓库和交易规模
```

新玩家可以依靠点击启动，但持续经营的收益应高于持续点击。

## 10. 点击收入与人口消费的关系

点击工作和人口消费都是货币发行来源，因此需要分别统计：

- 点击发行货币量。
- 人口消费发行货币量。
- 任务奖励发行货币量。
- 生产和升级回收货币量。
- 全服货币净增长量。

如果货币增长过快，调整优先级为：

1. 延长连续工作冷却。
2. 降低点击奖励。
3. 调整人口消费预算。
4. 增加合理生产与升级回收。

不能通过删除玩家资产或修改已完成成交回收通胀。

## 11. 为什么不只使用极长固定冷却

固定冷却过短容易被脚本持续运行；固定冷却过长又会破坏正常玩家的即时反馈。

递增冷却能够实现：

- 新玩家前几次点击响应较快。
- 长时间机械点击收益逐步下降。
- 玩家休息后恢复正常节奏。
- 不引入劳动力、体力或能量等额外资源。

## 12. 服务器与客户端职责

### 12.1 客户端负责

- 点击动画。
- 冷却倒计时展示。
- 收入预览。
- 连续工作等级提示。
- 生产费用与利润预览。
- 本地声音和视觉反馈。

### 12.2 服务器负责

- 判断点击是否满足冷却。
- 确认本次点击收入。
- 维护最近有效点击时间与冷却等级。
- 增加正式可用货币。
- 写入点击收入流水。
- 校验生产费用与扣款。
- 处理市场、人口消费和资产结算。

客户端时间、倒计时和按钮状态只能作为展示，不能决定是否发放货币。

## 13. 2 核 2G 性能约束

点击不能成为全服广播事件。

产品规则要求：

- 其他玩家不会实时看到某位玩家的每次点击。
- 点击只更新该玩家的私人余额和流水。
- 市场盘口不会因点击本身更新。
- 点击流水页面采用分页，不实时加载全部历史。
- 冷却中重复点击不得形成大量正式请求。

如果点击请求造成服务器压力，可临时延长全服基础冷却，但必须在页面明确显示当前规则。

## 14. 防作弊规则

- 冷却以服务器时间为准。
- 修改浏览器倒计时不能提前获得货币。
- 重复请求只能成功一次。
- 多标签页共享同一账号冷却。
- 刷新页面不能重置冷却。
- 断线重连不能补发冷却期间的点击收入。
- 不提供离线自动点击收入。
- 点击收入异常时可以暂停工作功能，但不能影响已有市场订单。

## 15. 页面交互

总览或工作区域显示：

- 可用货币。
- 冻结货币。
- 本次点击收入。
- 工作按钮。
- 冷却进度。
- 当前连续工作冷却。
- 今日或本周期点击收入统计。
- 生产快捷入口。

成功点击反馈：

```text
工作完成，获得 1 货币
```

冷却点击反馈：

```text
工作冷却中，2.1 秒后可再次工作
```

连续工作冷却增加反馈：

```text
连续工作过久，当前冷却延长至 5 秒
```

## 16. 新手引导

1. 点击工作并获得第一笔货币。
2. 查看点击收入流水。
3. 等待冷却结束后再次工作。
4. 积累足够生产费用。
5. 支付费用生产第一个食品包。
6. 查看食品包进入仓库。
7. 打开共享市场。
8. 创建卖单。
9. 与人口或其他玩家成交。
10. 使用收入继续生产或升级。

## 17. 玩家策略

### 17.1 基础工作者

通过间歇点击积累启动资金，适合新玩家和短时上线玩家。

### 17.2 生产经营者

使用点击收入投入生产，通过控制成本、生产时间和售价获取利润。

### 17.3 市场交易者

使用货币购买低价商品，再根据供需变化出售；主要收益来自交易判断，不来自点击。

### 17.4 综合经营者

点击、生产和交易结合，根据人口需求决定生产和库存策略。

## 18. 经济监测指标

测试期间持续观察：

- 每名玩家每小时有效点击数。
- 每名玩家每小时点击收入。
- 全服点击发行货币量。
- 人口消费发行货币量。
- 生产费用回收货币量。
- 升级回收货币量。
- 全服货币净增长量。
- 食品包生产量。
- 食品包成交量与均价。
- 未成交库存与卖单数量。
- 点击收入占玩家总收入比例。

中期目标：随着企业成长，点击收入占玩家总收入的比例逐渐下降。

## 19. 验收标准

1. 游戏中不存在劳动力资源。
2. 每次有效点击直接增加货币。
3. 冷却未结束时点击不增加货币。
4. 冷却由服务器确认，修改客户端时间无效。
5. 连续工作会按规则延长冷却。
6. 停止工作一段时间后冷却恢复。
7. 刷新页面或打开新标签页不能重置冷却。
8. 同一点击请求不会重复发放货币。
9. 点击不会直接产生食品包。
10. 食品包生产需要支付生产费用并等待生产时间。
11. 生产费用能够从经济中回收货币。
12. 商品成交后卖方获得货币。
13. 玩家间成交不增加全服货币总量。
14. 人口成交不超过人口消费周期预算。
15. 所有点击收入都有资产流水。
16. 点击功能暂停时，市场订单和玩家资产不受影响。
17. 高频机械点击的长期收益低于正常经营成长。
18. 客户端本地修改不能改变正式点击收入。
19. 点击请求不会广播给所有在线玩家。
20. 经济统计能够区分点击发行、人口发行和货币回收。

## 20. 当前开发决策

- 取消劳动力资源。
- 点击工作直接获得货币。
- 每次点击收入和冷却由服务器确认。
- MVP 使用基础冷却与连续工作递增冷却。
- 点击收入用于提供启动资金，不作为中后期主要收益。
- 食品包通过支付货币和等待生产时间获得。
- 玩家交易只转移现有货币。
- 点击与人口消费是受控货币发行来源。
- 生产和升级是主要货币回收来源。
- 所有初始数值必须通过试玩和经济数据调整。
