# Economy 页面内容与导航职责设计

> 状态：当前正式页面、模块唯一归属与导航基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-16
> 客户端状态版本：12
> 世界状态版本：8

## 1. 正式导航

```text
概览｜市场｜生产｜资产｜藏品｜拍卖｜排行｜设置
```

| 导航 | 路由 ID | React 页面 | 唯一职责 |
|---|---|---|---|
| 概览 | `home` | `OverviewPage` | 工作和经营摘要 |
| 市场 | `market` | `MarketPage` | 商品与工厂统一交易 |
| 生产 | `production` | `ProductionPage` | 仓库、建设和工厂集群 |
| 资产 | `assets` | `AssetsPage` | 资产结果与本地变化 |
| 藏品 | `collections` | `CollectionsPage` | 当前玩家持有的唯一艺术藏品 |
| 拍卖 | `auction` | `AuctionPage` | 藏品竞价拍卖与结算结果 |
| 排行 | `leaderboard` | `LeaderboardPage` | 服务器总资产排名 |
| 设置 | `settings` | `SettingsPage` | 资料、偏好、礼品、退出和重置 |

显示名称从“资金”统一改为“资产”，但路由 ID 继续使用 `assets`，不得为文案改名新增路由、状态迁移或兼容分支。不得恢复独立订单页、成交记录页、仓库页、工厂实例页、`records` 路由或设施槽位页面。藏品和拍卖是正式独立页面，不得合并进商品与工厂市场。

## 2. 应用外壳

`GameShell` 提供桌面侧栏、移动底部导航、全局状态栏、操作结果提示和唯一页面纵向滚动区域。状态栏固定显示可用资金、总资产、排行榜和仓库剩余，只读且不提供扩容、下单、撤单、建厂、改种、藏品拍卖或设置操作。

移动底部导航允许横向滚动，八个页面不得通过隐藏导航项、缩写中文名称或创建二级菜单规避空间限制。

## 3. 概览

必须包含：

- 根据浏览器本地系统时间显示玩家问候；
- 基础工作和 10 秒冷却；
- 按服务器 `products` 目录生成商品下拉框；
- 当前商品进入市场的入口；
- 工厂运行、停止、异常、施工和待加入摘要；
- 财富摘要；
- 当前玩家所有等待成交或部分成交的商品与工厂挂单摘要；
- 前往市场和生产的入口。

概览商品行情不得恢复全部商品快捷切换条。概览选中的商品属于 `PageRouter` 当前登录会话的页面级 UI 状态，不得写入服务器、`localStorage`、`sessionStorage` 或全局 `LoadedGameViewModel`。点击“进入该商品市场”是概览商品影响市场选择的唯一入口。

桌面宽度大于 960px 时，生产摘要、财富构成和当前挂单固定在同一行，三张卡片统一为 `384px` 高。挂单列表在卡片内部滚动。宽度不超过 960px 时改为单列并恢复自然高度。

## 4. 市场

必须包含商品和工厂统一资产标签、买入／卖出方向、价格、数量、快捷仓位、订单总额和提交、当前资产未完成订单与撤单、全部自有未完成订单、单列最优 5 笔卖单与最优 5 笔买单、价格曲线和当前浏览器本地成交记录。

玩家可见输入字段统一使用“价格”，不显示“限价”。商品标签使用商品 SVG；工厂标签使用独立厂房 SVG。市场不得包含藏品拍卖、工厂固定价格挂牌、玩家身份订单簿、价格档位聚合、仓库扩容、建设或启停。

## 5. 生产

页面主标题固定为“生产”。页面顺序：

```text
标题“生产”与工厂状态汇总
→ 共享仓库
→ 生产管理区
   ├─ 左侧：建设新工厂
   └─ 右侧：工厂类型集群卡
```

共享仓库、建设新工厂和工厂集群是生产页同一一级平面的卡片，统一使用 `.production-surface`。大于 `720px` 时四边内边距统一为 `16px`，不大于 `720px` 时统一为 `12px`；卡片内容、列宽和容器查询不得改变一级外层内边距或标题左上锚点。

### 5.1 共享仓库

显示等级、容量、实物库存、买单预占、剩余容量、无限扩容信息和有库存商品卡。完整规则见 `WAREHOUSE_EXPANSION_DESIGN.md`。

仓库不再提供“有库存／全部商品”筛选，只展示可用或冻结数量大于零的商品。“仓库内容”标题不显示商品种类统计。商品卡必须显示统一商品 SVG、商品名称、醒目的可用数量以及弱化的冻结数量，不显示独立库存总量行。

仓库商品网格按内容区宽度使用容器查询：小于 360px 为 2 列，360px 起 3 列，560px 起 4 列，760px 起 5 列，960px 起 6 列。旧规则“平板、手机和极窄屏保持双列”已废止，不得恢复只由页面视口决定的固定列数。

共享仓库标题使用与建设卡和工厂集群相同的一级卡片标题字号、紧凑行高和顶部对齐；右侧等级标签不得将标题垂直居中或改变标题起点。

### 5.2 建设

建设新工厂卡独占左侧列并在桌面滚动时常驻：

- 桌面宽度为 `280px–320px`，使用 `position: sticky`；
- 常驻范围只限生产管理区，不得使用 `position: fixed`；
- 不超过 960px 时恢复普通文档流并位于工厂列表之前。

必须显示：

- 工厂类型；
- 可选配方或固定配方名称；
- 建造费和施工时间；
- 当前施工任务；
- 产成品直接入仓；
- 新数量下一周期加入；
- 同时只能施工一座。

建设卡不显示生产周期、单座产量和单座成本。正式生产参数属于已拥有工厂的生产配方区域，不得在建设卡重复维护。

### 5.3 工厂集群卡

每种工厂类型最多一张卡。工厂卡高度由内容自适应；大于 1380px 时工厂列表固定四列，同一网格行中的卡片等高，不同网格行可以使用不同高度；961px–1380px 使用自然高度单列。移动端继续使用自然高度。

必须显示：

- 工厂名称与总数量位于第一行左侧，统一运行开关位于第一行右侧；
- 运行中、已停止或异常状态位于第二行，异常原因完整显示在状态标签中；
- “运行中／下一周期加入／冻结中”数量摘要位于第三行并保持三列单行；
- 集群生产公式：输入在左、周期与成本 SVG 居中、输出在右，进度条位于下方；
- 多输入、多输出和逐输入库存兼容展示；进度条下方不显示集群汇总或下一周期说明；
- 所有工厂统一显示“生产配方”选择器；多配方工厂可选择，单配方工厂显示唯一选项并禁用；
- 服务器存在待生效配方时，只在配方标题区显示“下一周期切换为：配方名称”；
- 市场入口强制固定在卡片底部，唯一文案为“前往市场交易该工厂 →”。

生产配方是配置展示，不是运行统计。公式固定显示一座工厂的输入、输出、周期和成本；配方数字不随停止或异常变为零，也不随参与数量扩大。实际服务器结算仍按 `participatingCount` 计算，当前运行规模由第三行数量摘要表达。进度条继续动态表示运行、停止和异常。

不得显示顶部“正常生产中”、独立持有／参与／待加入／冻结数量卡、“下一周期按若干座结算”、周期／产量／成本／原料四格规格区、目标产量或生产计划、工厂实例和内部库存。不得以箭头替代生产进度条。

## 6. 资产

必须包含可用资金、冻结资金和总资产，商品资产汇总，工厂数量资产，服务器估值价格与资产组成，货币发行和系统回收，以及当前浏览器本地资金、商品、仓库、工厂和生产变化。

资产页不得再显示逐商品“商品库存与估值”卡片；具体商品库存只在生产页共享仓库展示。资产页不负责仓库管理、建厂、生产控制、下单或藏品管理。

## 7. 藏品

页面主标题固定为“藏品”，只展示当前账号持有的唯一藏品。使用芝加哥艺术博物馆 IIIF 图片卡，显示作品信息、馆藏编号、当前归属、拍卖状态和原馆藏页面链接。图片使用 `loading="lazy"` 和异步解码。

藏品页不负责出价、资金冻结、结算、商品库存或工厂控制。

## 8. 拍卖

页面主标题固定为“拍卖”。必须包含当前玩家可拍卖藏品选择、起拍价和时长、进行中拍卖、最低有效出价、无出价取消以及最近结束结果。

最高出价资金、退款、拍卖状态和归属转移全部由服务器判定。客户端倒计时只用于展示，显示到零时不得自行宣布成交，必须等待服务器返回结算状态。

拍卖页不得允许卖家竞拍自己的藏品、在已有出价后取消、同时为同一藏品创建多场拍卖、把拍卖保存为浏览器唯一状态，或将拍卖最高价写入商品／工厂估值。

## 9. 排行

只读展示我的排名、总资产、与上一名差距、本周变化和完整服务器排行榜，不得包含交易、生产、扩容、藏品拍卖或设置操作。

## 10. 设置

设置页只允许玩家资料与四项统计、客户端偏好、礼品兑换、管理员入口、退出登录和重置经济状态。状态刷新频率默认 5 秒，可选 3／5／10 秒。不得作为经营仪表盘。

## 11. 商品与工厂目录扩展规则

所有目录型页面必须遍历服务器返回的 `products` 和 `facilityTypes`。

- 不得写死 6 个商品 ID。
- 不得固定六个行情卡、六个商品标签或六张工厂卡。
- 新商品和工厂应在不修改页面结构的情况下自动出现。
- 目录型横向导航可以滚动或自动分配列宽，但不能依赖固定项目数量。
- 空目录必须显示明确空状态。

仓库虽然只显示有库存商品，过滤仍必须基于服务器返回的完整 `products` 数组和对应库存，不得维护第二套商品清单。藏品页面必须遍历服务器返回的 `collectibles` 和 `collectibleAuctions`。

## 12. 模块唯一归属

| 模块 | 唯一主要页面 |
|---|---|
| 工作 | 概览 |
| 统一资产下单、订单簿、撤单和本地成交 | 市场 |
| 仓库管理、商品库存和扩容 | 生产 |
| 建设、周期、农场改种、启停和工厂状态 | 生产 |
| 资产组成与本地资产事件 | 资产 |
| 当前玩家藏品画廊 | 藏品 |
| 藏品发售、竞价和结算结果 | 拍卖 |
| 总资产排名 | 排行 |
| 资料、偏好、礼品、退出和重置 | 设置 |
| 礼品码与藏品导入、归属历史 | 管理员页面 |

## 13. 防回退

不得：

- 跨页复制完整管理模块；
- 恢复独立订单、记录、仓库或工厂实例路由；
- 恢复固定价格工厂市场；
- 恢复六商品写死页面；
- 在概览恢复全部商品快捷切换条或本地成交面板；
- 把仓库扩容移出生产页；
- 恢复仓库筛选、商品种类统计、独立库存总量行或资产页逐商品库存卡；
- 恢复仓库固定两列／三列／四列视口方案；
- 让共享仓库、建设新工厂和工厂集群在同一断点使用不同一级外层内边距；
- 把农场改种或启停移到市场页；
- 在建设卡恢复生产周期、单座产量和单座成本；
- 让生产配方可见数字随状态或参与数量变化；
- 把藏品和拍卖塞入商品／工厂统一订单簿；
- 把藏品归属或拍卖结算改为浏览器本地状态；
- 隐藏移动端的藏品或拍卖导航。
