# Economy 页面内容与导航职责设计

> 状态：当前正式页面内容、操作归属与导航基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-11  
> 客户端状态版本：`version: 7`

## 1. 页面划分原则

```text
概览：了解当前状态并进入主要流程
市场：完成商品和工厂资产交易
生产：管理共享仓库、建设工厂和控制生产
资金：查看资产结果与本地变化记录
排行：比较服务器总资产
设置：管理资料、客户端偏好、登录会话和危险操作
```

核心原则：

1. 每个正式操作必须有唯一主要页面；
2. 完整管理模块不得跨页重复；
3. 全局状态栏只提供只读摘要；
4. 设置页不是经营仪表盘；
5. 页面展示不得改变服务器权威边界；
6. 模块唯一归属必须由自动检查保护。

## 2. 正式导航

```text
概览｜市场｜生产｜资金｜排行｜设置
```

| 导航 | 路由 ID | React 页面 | 职责 |
|---|---|---|---|
| 概览 | `home` | `OverviewPage` | 摘要与快捷入口 |
| 市场 | `market` | `MarketPage` | 商品和工厂交易 |
| 生产 | `production` | `ProductionPage` | 共享仓库与工厂管理 |
| 资金 | `assets` | `AssetsPage` | 资产结果与本地记录 |
| 排行 | `leaderboard` | `LeaderboardPage` | 总资产排名 |
| 设置 | `settings` | `SettingsPage` | 资料、偏好、会话和重置 |

本文中的“工厂页面”指用户从“生产”导航进入的 `ProductionPage`。

不得恢复独立订单页、独立成交记录页、独立仓库页、`records` 路由或设施槽位页面。

## 3. 应用外壳

`GameShell` 统一提供桌面侧栏、移动底部导航、全局状态栏、操作提示和页面滚动区域。

状态栏可以显示资金、总资产、排名、选中商品持仓、共享仓库使用与买单预占、最近成交价。状态栏不得提供扩容、下单、撤单、建厂、生产计划或设置操作。

## 4. 概览页面

必须包含：

- 玩家问候；
- 基础工作；
- 当前商品市场摘要与六种商品快捷行情；
- 生产摘要：运行、停止、阻塞、施工、定量计划和共享仓库剩余空间；
- 财富摘要；
- 当前浏览器最近成交；
- 市场、生产和资金快捷入口。

不得包含完整订单簿、完整工厂卡、`WarehouseUpgradeCard`、设置表单或完整排行榜。

概览不得显示“待领取产成品”，因为所有工厂产成品直接进入共享仓库。

## 5. 市场页面

必须包含：

- 商品选择与行情；
- 限价下单；
- 当前商品订单与快捷撤单；
- 聚合订单簿；
- 价格曲线；
- 全部未完成订单；
- 冻结资产摘要；
- 本地成交记录；
- 工厂挂牌、收购和撤销挂牌。

工厂挂牌规格必须说明产成品直接进入共享仓库，不得显示工厂内部容量。

市场不得包含仓库扩容、建设工厂、工厂启停、生产计划、资产配置或账号设置。

## 6. 生产页面（工厂页面）

### 6.1 页面目的

管理生产所需的共享仓库和全部工厂。共享仓库完整管理入口固定在 `ProductionPage`。

### 6.2 页面顺序

```text
页面标题与工厂状态
→ 共享仓库
→ 建设新工厂
→ 已有工厂列表
```

共享仓库必须位于建设与工厂列表之前。`WarehouseUpgradeCard` 只能由 `ProductionPage` 渲染。

### 6.3 共享仓库内容

必须显示等级、已用容量、实物库存、买单预占、剩余容量、扩容后容量、扩容费用、扩容按钮、资金不足、容量超限和满级状态。

### 6.4 建设新工厂内容

必须显示：

- 工厂类型与配方；
- 建造费用和施工时间；
- 生产周期、周期产出和运营费；
- 产成品去向：直接进入共享仓库；
- 建设按钮；
- 同时只能施工一座工厂的说明。

不得显示内部产成品容量。

### 6.5 每座工厂内容

必须显示：

- 名称、状态和停止原因；
- 配方；
- 输出商品在共享仓库中的当前数量；
- 施工或生产进度；
- 周期、小时产量和小时运营费；
- 原料库存；
- 累计产量；
- 系统估值；
- 持续或定量生产计划；
- 定量目标与完成量；
- 手动启动和停止；
- 产成品自动入仓说明；
- 挂牌出售和撤销挂牌。

不得显示工厂内部商品、内部容量或领取按钮。生产完成后不需要玩家领取。

### 6.6 阻塞说明

`full` 和 `output_full` 表示共享仓库无法容纳下一个完整生产周期的净新增商品。仓库扩容后工厂不会自动重启。

## 7. 资金页面

必须包含：

- 可用资金、冻结资金和总资产；
- 商品资产与工厂资产；
- 资产配置与估值明细；
- 商品库存与估值；
- 货币发行与回收；
- 当前浏览器本地资金、商品、仓库、工厂和生产变化；
- 本地事件筛选与清除。

商品库存必须包含工厂直接入仓的产成品。工厂估值只计算工厂系统价值，不再叠加工厂内部商品。

资金页不负责仓库管理，不得渲染 `WarehouseUpgradeCard`、扩容按钮、建厂、生产控制或下单表单。

## 8. 排行页面

只读展示服务器总资产排名，包含我的排名、总资产、与上一名差距、本周变化和完整排行榜。

不得包含交易、生产、仓库扩容或设置操作。

## 9. 设置页面

设置页只允许：

- 玩家资料与昵称；
- 客户端界面偏好；
- 登录会话与退出登录；
- 重置服务器经济状态。

设置页不得显示账号与产业只读摘要或仓库使用摘要，不得显示工厂、商品、订单、排名或仓库经营状态。

## 10. 模块唯一归属

| 模块 | 唯一主要页面 |
|---|---|
| 基础工作 | 概览 |
| 商品下单与撤单 | 市场 |
| 工厂交易 | 市场 |
| 共享仓库扩容 | 生产 |
| 建厂、生产计划、启停 | 生产 |
| 工厂自动入仓说明 | 生产 |
| 资产配置与本地变化 | 资金 |
| 总资产排名 | 排行 |
| 玩家资料、偏好、会话和重置 | 设置 |

不再存在“领取产成品”模块。

## 11. 防回退规则

不得：

- 在生产页之外渲染完整仓库管理卡；
- 恢复工厂内部容量、内部商品或领取按钮；
- 在概览显示待领取产成品；
- 在市场挂牌卡显示内部容量；
- 在资金页提供扩容或生产操作；
- 在设置页恢复经营摘要；
- 恢复独立订单或仓库页面；
- 修改页面职责但不更新设计和检查。

未更新本设计、相关专项设计和防回退检查的页面结构修改不应合并。
