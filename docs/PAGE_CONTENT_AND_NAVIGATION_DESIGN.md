# Economy 页面内容与导航职责设计

> 状态：当前正式页面、模块唯一归属与导航基线
> 适用项目：`RIVERS0FT/Economy`
> 更新时间：2026-07-18
> 客户端状态版本：15
> 世界状态版本：13

## 1. 正式导航

```text
概览｜市场｜生产｜资产｜藏品｜拍卖｜排行｜商店｜设置
```

| 导航 | 路由 ID | React 页面 | 唯一职责 |
|---|---|---|---|
| 概览 | `home` | `OverviewPage` | 工作和经营摘要 |
| 市场 | `market` | `MarketPage` | 商品与工厂统一交易 |
| 生产 | `production` | `ProductionPage` | 仓库、建设和工厂集群 |
| 资产 | `assets` | `AssetsPage` | 资产结果与本地变化 |
| 藏品 | `collections` | `CollectionsPage` | 当前玩家持有的唯一艺术藏品 |
| 拍卖 | `auction` | `AuctionPage` | 藏品、商品与工厂竞价拍卖与结算结果 |
| 排行 | `leaderboard` | `LeaderboardPage` | 服务器总资产排名 |
| 商店 | `gem-shop` | `GemShopPage` | 宝石单向兑换普通货币 |
| 设置 | `settings` | `SettingsPage` | 资料、偏好、邀请、礼品和退出 |

“商店”是宝石兑换页面的唯一玩家可见名称；路由 ID 继续使用 `gem-shop`，React 组件继续使用 `GemShopPage`，服务器 API 与数据表继续沿用现有技术标识，不得仅为显示名称新增迁移或兼容路由。

显示名称从“资金”统一改为“资产”，但路由 ID 继续使用 `assets`，不得为文案改名新增路由、状态迁移或兼容分支。不得恢复独立订单页、成交记录页、仓库页、工厂实例页、`records` 路由或设施槽位页面。藏品和拍卖是正式独立页面，不得合并进商品与工厂市场。

## 2. 应用外壳

`GameShell` 提供桌面侧栏、移动底部导航、全局状态栏、操作结果提示和唯一页面纵向滚动区域。状态栏固定按“可用资金／总资产／宝石／排行榜／仓库剩余”排序；总资产项可以作为只读导航入口打开资产页，其余状态项只读，任何状态项都不得直接执行扩容、下单、撤单、建厂、改种、藏品拍卖或设置写操作。总资产周变化使用方向箭头和颜色表达方向，数值使用绝对值且不得重复显示正负号；宝石不计入总资产或排行榜。

桌面侧栏在宽屏提供显式折叠按钮，折叠后只保留图标、订单徽标、连接状态和退出入口；所有导航按钮继续使用稳定 `aria-label`，折叠按钮必须可由键盘操作并通过 `aria-expanded` 表达状态。`721px–960px` 继续使用自动紧凑侧栏，不显示手动折叠按钮。移动底部导航允许横向滚动，九个页面不得通过隐藏导航项、缩写中文名称或创建二级菜单规避空间限制。

### 2.1 状态交付容量

- 正式客户端默认每 5 秒按服务器修订号轮询，可选 3／5／10 秒。
- 修订号未变化时只接受轻量确认，不得恢复每 1 秒完整状态刷新。
- 发起权威动作时取消旧轮询并暂停新轮询；按钮必须在同一交互周期立即显示“处理中”并阻止重复提交。
- 客户端只接受不低于当前值的状态修订号，任何低于当前修订号的迟到响应不得覆盖工作响应或其他较新的权威动作结果。

### 2.2 紧凑时间与排名

- 玩家界面中的持续时间统一通过 `formatDuration` 展示，只使用小写 `s`、`m`、`h`，例如 `8s`、`2m 05s`、`1h 20m`。
- 时长最多显示两个非零层级；整分钟省略 `00s`，整小时省略 `00m`。具体日期和时刻继续使用 `formatTime`，不得改成相对时长。
- 状态刷新选项使用 `3s`、`5s`、`10s`，拍卖时长输入单位使用 `h`；页面不得另写“秒／分钟／小时”倒计时算法。
- 所有排名数值统一通过 `formatRank` 显示为 `#N`，例如 `#1`；缺失排名显示 `#--`。中文“我的排名”“排行榜”等标签保留，可访问名称继续说明“排名第 N 名”。

### 2.3 登录与注册入口

未登录外壳必须明确拆分“登录”和“注册”两个模式。登录模式只调用现有统一账号登录，不得在 401 后自动注册。注册模式包含账号邮箱、密码、发送验证码、60s 倒计时和 6 位验证码输入；点击发送验证码后，服务器必须先查询统一账号邮箱是否已存在，已注册时直接提示登录且不启动倒计时、不创建验证码记录、不发送邮件。验证码完成后由 Economy 服务创建或登录统一账号并首次创建 Economy 玩家档案。账号和密码继续使用原生未受控表单与 `FormData`，避免浏览器自动填充被 React 空值覆盖。

带 `?invite=邀请码` 的分享链接进入时默认显示注册模式，并提示分享者将在新玩家首次创建 Economy 档案后立即获得宝石。已登录统一账号也必须先调用 Economy 会话初始化接口处理邀请码，再请求正式游戏状态；处理完成后清除 URL 参数。无效邀请码不得阻止普通注册。

## 3. 概览

概览是经营决策首页，不是模块等权堆叠页。必须包含：

- 根据浏览器本地系统时间显示玩家问候；
- 只保留一个根据当前状态变化的页面主操作：优先处理未完成订单，其次打开经营提醒，无提醒时进入市场；
- “今日经营”卡中的紧凑基础工作、固定 10s 冷却与最多 3 条经营提醒；
- 按服务器 `products` 目录生成的商品下拉框，以及当前商品的市场快照；
- 生产摘要、资产构成和当前挂单三张核心经营卡；
- 前往市场、生产和资产页的明确操作入口。

经营提醒固定按以下优先级生成：仓库已满或容量过低、工厂生产受阻、未完成订单、主动停工、施工状态。每条提醒必须说明原因并提供可执行入口；颜色不得作为唯一状态表达。基础工作是兜底收入，不得使用整卡宽度的高权重渐变按钮，也不得让工作卡通过固定大高度产生空白。

桌面第一排由“今日经营”和“市场快照”组成，宽度比例为 `5:7`，卡片高度约 `330px`。“今日经营”同时承载工作和提醒；市场快照保留最近成交、最高买价、最低卖价和当前持仓四项指标。当前商品存在近期成交或有效挂单时才显示紧凑的近 24h 价格／成交量图；既无近期成交也无有效挂单时必须显示约 `140px` 的紧凑空状态、当前参考价和进入该商品市场的按钮，不得渲染完整坐标系。概览紧凑图横轴标签保持水平，完整市场页仍可使用倾斜标签。

第二排固定为生产摘要、资产构成和当前挂单。宽度不小于 `1200px` 时三卡同排并统一约 `320px` 高；`768px–1199px` 时第一排改为单列，第二排改为两列且挂单卡跨两列；不大于 `760px` 时全部单列并恢复自然高度。`1920×1080` 下应完整显示两排核心内容，`1440×900` 下至少必须显示第二排标题与核心指标。

资产构成只显示现金、商品估值、工厂估值、冻结资金和当前浏览器的本周资金变化，不得重复状态栏已经显示的总资产和排名。当前挂单只显示一次买卖统计、冻结资金和订单列表；列表在固定卡片内滚动，概览不提供撤单按钮。生产摘要优先显示正在运行、生产受阻、主动停工和理论日产量，不以总工厂数掩盖异常。

概览不得包含当前浏览器最近成交；本地成交记录只属于市场页。概览商品行情不得恢复全部商品快捷切换条。标题区域不得与市场卡同时放置两个同等权重的市场入口；页面主操作与当前商品上下文入口职责必须不同。

概览选中的商品属于 `PageRouter` 当前登录会话的页面级 UI 状态。切换到市场、生产、资产或其他正式页面后再返回概览时必须保留选择；市场中的后续商品切换不得反向覆盖概览选择。不得写入服务器、`localStorage`、`sessionStorage` 或全局 `LoadedGameViewModel`。刷新整个页面或重新登录后，从服务器目录首项重新初始化；服务器目录更新且原商品不再存在时也回退到首项。点击当前商品市场入口是概览商品影响市场选择的唯一方式。

## 4. 市场

必须包含商品和工厂统一资产标签、买入／卖出方向、价格、数量、快捷仓位、订单总额和提交、当前资产未完成订单与撤单、全部自有未完成订单、单列最优 5 笔卖单与最优 5 笔买单、近 24h 价格与成交量趋势图和当前浏览器本地成交记录。卖出方向必须在订单总额下显示按整张订单完全成交估算的“预计手续费（1%，最低 1）”和“预计到账”；成交额全部被最低手续费抵扣时显示明确提示。服务器仍按真实逐笔成交和单张卖单累计额结算，客户端估算不参与扣费。

概览页与市场页的商品行情统一统计当前资产最近 24h，按 6 分钟聚合为 240 个数据分段。每个分段的价格使用该段最后一笔成交价；没有成交的分段延续最近有效成交价，成交量为该段全部成交数量之和。价格折线图位于上方，成交量柱状图位于下方，两图纵向堆叠并共享同一横轴。概览页只使用紧凑尺寸，不得恢复“最近 24 笔成交”或仅价格折线的独立数据口径。

每笔新成交必须记录吃单方（taker／incoming order）的方向。分段净主动量固定为“主动买入量 − 主动卖出量”；柱高始终表示总成交量，净主动买入使用成功色，净主动卖出使用危险色，主动买卖均衡或旧历史方向未知使用中性色。旧世界中没有方向字段的成交不得按涨跌猜测方向，统一计入中性成交量。

横轴固定划分为 12 个 2h 分段，刻度标签使用浏览器系统时区的 `HH:mm`；纵轴分别标明“价格”和“成交量”，必须显示刻度、网格、方向图例和可访问名称。横纵坐标刻度与轴标题的屏幕渲染字号必须与图表下方统计栏一致，横轴标题固定为“时间”。市场页 `.chart-footer` 与概览页 `.overview-market-footer` 必须在容器本身统一使用 `--font-size-xs`，图表组件据此反算 SVG 坐标、轴标题和图例字号；不得只给容器内部的 `small` 设置字号。

玩家可见输入字段、订单标题和未完成订单表头统一使用“价格”，不显示“限价”。商品标签使用商品 SVG；工厂标签使用独立厂房 SVG。市场不得包含藏品拍卖、工厂固定价格挂牌、玩家身份订单簿、价格档位聚合、仓库扩容、建设或启停。

从其他页面重新进入市场时，当前资产和方向的默认价格只根据客户端已加载的有效订单簿快照初始化。用户在市场页主动切换资产或买卖方向时可以重新计算默认价格；自动刷新和下单后的状态同步不得覆盖当前输入，成交、撤单或其他权威状态回包也只更新订单簿与资产状态。

### 4.1 市场页桌面布局与反馈

市场页桌面交易区按实际内容宽度使用三档布局：宽内容区为固定下单列、订单簿列和宽行情主列；中等内容区保留下单与订单簿同排并让行情跨整行；窄内容区全部单列。完整行情图高度由自身宽度和 `16:9` 比例决定，不能继续使用基于浏览器 `vw` 的固定高度。

商品和工厂保持同一连续目录，但目录使用两行横向布局、明确分组标记、前后滚动控制和“当前”文字状态。下单禁用必须说明资金、仓库或可售资产不足的具体原因。本地成交表必须显示成交总额以及卖方“手续费／实收”，买方对应位置显示无手续费，不得新增来源或对手列。最近 24h 成交笔数与 240 个六分钟行情分段使用同一时间窗口；零涨跌使用中性色。买盘和卖盘标题必须位于对应订单之前，稀疏订单簿按内容自然高度显示。

## 5. 生产

页面主标题固定为“生产”。页面顺序：

```text
标题“生产”与工厂状态汇总
→ 共享仓库
→ 生产管理区
   ├─ 左侧：建设新工厂
   └─ 右侧：工厂类型集群卡
```

共享仓库、建设新工厂和工厂集群是生产页同一一级平面的卡片，统一使用 `.production-surface`。大于 `720px` 时四边内边距统一为 `16px`，不大于 `720px` 时统一为 `12px`；卡片内容、列宽和容器查询不得改变一级外层内边距或标题左上锚点。

### 5.1 共享仓库

显示等级、容量、实物库存、买单预占、剩余容量、无限扩容信息和有库存商品卡。完整规则见 `WAREHOUSE_EXPANSION_DESIGN.md`。

仓库不再提供“有库存／全部商品”筛选，只展示可用或冻结数量大于零的商品。“仓库内容”标题不显示商品种类统计。商品卡必须显示统一商品 SVG、商品名称、醒目的可用数量以及弱化的冻结数量，不显示独立库存总量行。

仓库商品网格按内容区宽度使用容器查询：小于 360px 为 2 列，360px 起 3 列，560px 起 4 列，760px 起 5 列，960px 起 6 列。旧规则“平板、手机和极窄屏保持双列”已废止，不得恢复只由页面视口决定的固定列数。

共享仓库标题使用与建设卡和工厂集群相同的一级卡片标题字号、紧凑行高和顶部对齐；右侧等级标签不得将标题垂直居中或改变标题起点。

### 5.2 建设

建设新工厂卡独占左侧列并在桌面滚动时常驻：

- 桌面宽度为 `280px–320px`，使用 `position: sticky`；
- 常驻范围只限生产管理区，不得使用 `position: fixed`；
- 不超过 960px 时恢复普通文档流并位于工厂列表之前。

必须显示：

- 工厂类型；
- 可选配方或固定配方名称；
- 建造费和施工时间；
- 当前施工任务；
- 产成品直接入仓；
- 新数量下一周期加入；
- 同时只能施工一座。

建设卡不显示生产周期、单座产量和单座成本。正式生产参数属于已拥有工厂的生产配方区域，不得在建设卡重复维护。

### 5.3 工厂集群卡

每种工厂类型最多一张卡。工厂卡高度由内容自适应；大于 1380px 时工厂列表固定四列，同一网格行中的卡片等高，不同网格行可以使用不同高度；961px–1380px 使用自然高度单列。移动端继续使用自然高度。

必须显示：

- 工厂名称与总数量位于第一行左侧，统一运行开关位于第一行右侧；
- 运行中、已停止或异常状态位于第二行，异常原因完整显示在状态标签中；
- “运行中／下一周期加入／冻结中”数量摘要位于第三行并保持三列单行；
- 集群生产公式：输入在左、周期与成本 SVG 居中、输出在右，进度条位于下方；
- 多输入、多输出和逐输入库存兼容展示；进度条下方不显示集群汇总或下一周期说明；
- 所有工厂统一显示“生产配方”选择器；多配方工厂可选择，单配方工厂显示唯一选项并禁用；
- 服务器存在待生效配方时，只在配方标题区显示“下一周期切换为：配方名称”；
- 市场入口强制固定在卡片底部，唯一文案为“前往市场交易该工厂 →”。

生产配方是配置展示，不是运行统计。公式固定显示单座配方的输入、输出、周期和成本；不得因停止、异常、冻结、参与数量或集群总数量变为零、隐藏或改成集群汇总。开关只改变服务器运行意图；异常时仍保持开启视觉，并等待资金、原料或仓库恢复后自动从完整新周期继续。

不得显示顶部“正常生产中”、独立持有／参与／待加入／冻结数量卡、“下一周期按若干座结算”、周期／产量／成本／原料四格规格区、目标产量或生产计划、工厂实例和内部库存。不得以箭头替代生产进度条。

## 6. 资产

必须包含可用资金、冻结资金、总资产、冻结资产和可支配资产，商品与工厂的可用／冻结估值拆分，服务器估值价格与资产组成，以及当前浏览器本地资金、商品、仓库、工厂和生产变化。冻结资产仍归当前玩家所有并计入总资产，页面必须明确说明冻结只限制交易、生产或使用。

资产页不得再显示逐商品“商品库存与估值”卡片；具体商品库存只在生产页共享仓库展示。资产页不负责仓库管理、建厂、生产控制、下单或藏品管理。宝石不属于普通经济资产，不进入该页总资产组成。

## 7. 藏品

页面主标题固定为“藏品”，只展示当前账号持有的唯一藏品。使用芝加哥艺术博物馆 IIIF 图片卡，显示作品信息、馆藏编号、当前归属、拍卖状态和原馆藏页面链接。图片使用 `loading="lazy"` 和异步解码。

藏品页不负责出价、资金冻结、结算、商品库存或工厂控制。

## 8. 拍卖

页面主标题固定为“拍卖”。发布区使用资产包编辑器，而不是只能选择单一类型的提交表单。玩家可以在藏品／商品／工厂分段选择器间切换并连续加入资产；切换类型不得清空已加入项目。单项拍卖也是只有一项的资产包，规范化后最多 20 项。同种商品或工厂重复加入时合并数量，藏品数量固定为 1 且不显示禁用数量输入框。

资产包列表必须显示类型图标、名称、数量、修改或移除操作、项目计数、整包起拍价和 `1～168h` 时长。发布前明确说明全部资产会同时冻结、整体成交且已有出价后不能取消。商品选项只使用可用库存，工厂选项只使用未被订单簿或拍卖冻结的可用数量；已加入数量必须从可继续加入数量中扣除。

进行中拍卖卡最多直接展示前三项并提示剩余项目数，明确使用“当前总价”“整包出价”和“不可拆分资产包”，并显示出价次数、卖家和最高出价者。商品使用 `ProductIcon`，工厂使用 `FactoryIcon`，藏品使用 IIIF 缩略图。客户端倒计时只用于展示，到零后显示“等待服务器结算”，不得自行宣布成交。

页面标题区不得重复显示与分区标题相同的进行中数量；零场不是警告状态。“进行中的拍卖”空状态不得再嵌套第二层 `Panel`。最近结束区域必须始终存在，没有记录时显示紧凑空状态，不能直接删除整个区域。

“发布资产包拍卖”和“最近结束”属于拍卖页同一背景上的一级 `Panel`，必须复用统一卡片内容内边距：大于 `720px` 时四边为 `16px`，不大于 `720px` 时四边为 `12px`。资产包编辑器中的“添加资产”和“拍卖资产包”使用相同内边距，并按自然内容高度顶部对齐，不得依赖网格默认拉伸制造大面积空白。

藏品、商品和工厂类型选择器必须固定为三等分单行布局，不得继承通用二列分段选择器而导致工厂选项换行。使用网格间距组织内容时，标题和辅助文字不得再叠加重复外边距。

最高出价资金、商品仓库预占、卖方资产冻结、退款、拍卖状态和归属转移全部由服务器判定。捆绑拍卖必须按资产包全部商品合计预占仓库，并以原子方式转移全部归属。客户端不得允许卖家竞拍自己的资产包、已有出价后取消、超过可用数量、拆分购买、把拍卖保存为浏览器唯一状态，或将拍卖最高价写入商品／工厂订单簿行情和估值。

## 9. 排行

只读展示我的排名、总资产、与上一名差距、本周变化和完整服务器排行榜，不得包含交易、生产、扩容、藏品拍卖或设置操作。状态栏、我的排名和排行榜首列统一显示 `#N`，不得显示“第 N 名”或裸排名数字。宝石不得计入总资产、周变化或排名。

## 10. 商店

页面主标题固定为“商店”，作为独立一级页面使用路由 ID `gem-shop`。页面必须展示当前宝石、可用资金、服务器固定汇率、整数兑换数量、快捷数量、预计获得货币、确认按钮、累计兑换和最近 20 笔记录。

汇率固定显示为 1 宝石兑换 10 普通货币；单次输入范围 1～100，不能超过当前宝石余额。客户端只计算展示预览，实际扣除、发行、幂等与记录全部等待服务器响应。页面必须明确说明兑换单向且不可撤销，不得提供普通货币购买宝石或将商店塞回设置页。

## 11. 设置

设置页只允许玩家资料与四项统计、已经实现的客户端偏好、邀请入口、礼品兑换、管理员入口和退出登录。不得提供经济状态重置、清空进度或重新开始入口。当前客户端偏好只有“紧凑数字”和“状态刷新频率”；状态刷新频率默认 `5s`，可选 `3s`／`5s`／`10s`。不得显示没有实际运行效果的“界面音效”或“画面性能”控件，也不得作为经营仪表盘。

桌面设置页固定使用两个互不共享网格行高的纵向内容栈：主列依次放置“玩家资料”和“邀请好友”，侧列依次放置“游戏设置”“礼品兑换”和“账号与管理”。不得恢复三等分共享网格、跨列 `span-2` 或由左侧高卡片迫使右侧短卡片产生大面积空白的布局。内容宽度不足时按“玩家资料／游戏设置／邀请好友／礼品兑换／账号与管理”的顺序降为单列。

玩家资料与邀请统计在空间充足时四项同排，窄屏降为两列。昵称编辑在桌面使用输入框与自然宽度保存按钮同行，移动端才允许按钮占满宽度。账号资料、管理员工具和当前会话必须明确分组。设置页不得显示危险区域、重置经济状态按钮或任何清空服务器经济数据的说明。

邀请卡必须展示服务器返回的宝石余额、专属分享链接、永久邀请码、每次奖励宝石、邀请统计和最近邀请。分享链接注册在首次创建 Economy 玩家档案时自动归因并立即奖励分享者；未绑定玩家可以在注册后 24 小时内手动填写一次邀请码，成功后立即奖励邀请人。两种方式互斥，被邀请人不获得宝石。完整奖励、封禁、事务和管理员规则见 `PRODUCT_AND_GAMEPLAY_DESIGN.md`、`SERVER_ARCHITECTURE_AND_DEPLOYMENT_DESIGN.md` 与 `GIFT_CODE_AND_ADMIN_DESIGN.md`。

管理员资料区同时提供普通后台与账号封禁管理入口。普通玩家设置页不得显示邮箱之外的账号隐私、注册 IP、封禁指纹或被邀请人的用户 ID。

## 12. 商品与工厂目录扩展规则

当前页面必须动态展示服务器返回的 22 种商品和 15 种工厂。农场、矿场、畜牧场、冶炼厂和纺织厂显示多配方切换；电子工厂公式必须显示 `1 塑料 + 1 铜材 → 1 电子产品`，不得只展示首个输入。

所有目录型页面必须遍历服务器返回的 `products` 和 `facilityTypes`。

- 不得写死 6 个商品 ID。
- 不得固定六个行情卡、六个商品标签或六张工厂卡。
- 新商品和工厂应在不修改页面结构的情况下自动出现。
- 目录型横向导航可以滚动或自动分配列宽，但不能依赖固定项目数量。
- 空目录必须显示明确空状态。

仓库虽然只显示有库存商品，过滤仍必须基于服务器返回的完整 `products` 数组和对应库存，不得维护第二套商品清单。藏品页面必须遍历服务器返回的 `collectibles` 和 `collectibleAuctions`。

## 14. 防回退

不得：

- 跨页复制完整管理模块；
- 恢复独立订单、记录、仓库或工厂实例路由；
- 恢复固定价格工厂市场；
- 恢复六商品写死页面；
- 在概览恢复全部商品快捷切换条或本地成交面板；
- 让普通页面切换重置仍有效的概览商品选择；
- 让市场中的商品选择反向覆盖概览页面级选择；
- 让自动刷新、下单或其他状态同步覆盖市场当前价格输入；
- 把市场行情图退回只有价格折线、没有成交量柱状图或没有坐标轴的迷你图；
- 改变市场行情图的 24h 窗口、6m × 240 分段、12 个 2h 横轴分段或浏览器系统时区标签；
- 缩小市场行情图横纵坐标刻度或轴标题至低于图表下方统计栏字号，或把横轴标题改成“时间”以外的文案；
- 把仓库扩容移出生产页；
- 恢复仓库筛选、商品种类统计、独立库存总量行或资产页逐商品库存卡；
- 恢复仓库固定两列／三列／四列视口方案；
- 让共享仓库、建设新工厂和工厂集群在同一断点使用不同一级外层内边距；
- 把农场改种或启停移到市场页；
- 在建设卡恢复生产周期、单座产量和单座成本；
- 让生产配方可见数字随状态或参与数量变化；
- 把藏品和拍卖塞入商品／工厂统一订单簿；
- 把藏品归属或拍卖结算改为浏览器本地状态；
- 让拍卖页一级卡片或资产包编辑器在相同断点使用不同内边距，或把三类资产选择器退回两列换行；
- 恢复中文“秒／分钟／小时”的玩家时长展示或页面自定义倒计时算法；
- 恢复“第 N 名”或裸数字排名展示；
- 隐藏移动端的藏品、拍卖或商店导航;
- 恢复“登录或注册”单按钮、登录失败自动注册或没有倒计时的验证码发送；
- 将邀请入口退回只分享普通固定链接，或由客户端生成邀请码、决定归因和发放宝石；
- 允许分享链接与手动邀请码重复绑定或让被邀请人获得邀请宝石；
- 把商店合并进设置页、允许普通货币购买宝石、由客户端决定实际汇率或允许撤销兑换；
- 从管理员页面移除同 IP 封禁复核和手动解禁入口；
- 在设置页恢复未接入运行时行为和持久化的“界面音效”或“画面性能”占位控件；
- 把设置页恢复为共享三列网格、跨列卡片或由同一网格行耦合左右卡片高度；
- 恢复玩家经济状态重置、清空进度、重新开始按钮或对应客户端调用；
- 把账号资料、管理员工具和退出登录重新混入玩家资料卡的单一操作栈。

### 本地成交匿名展示

市场页“本地成交记录”固定展示类型、资产、方向、数量、价格、总额和时间七列，不得设置“来源”列，不得显示人口需求名称、玩家名称、“匿名玩家”或其他对手类型占位。订单反馈只描述本人的买单或卖单部分成交、全部成交以及剩余数量。
