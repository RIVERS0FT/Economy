# Economy 页面内容与导航职责设计

> 状态：当前正式页面内容、操作归属与导航基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-11  
> 客户端状态版本：`version: 7`

## 1. 目的与适用范围

本文件明确桌面端和移动端六个正式页面分别展示什么、允许执行什么操作，以及哪些模块不得重复放置。

涉及页面新增、页面重命名、模块移动、快捷入口、跨页重复展示或导航调整时，必须先更新本文件，再修改代码与防回退检查。

相关设计基线：

- 生产规则：`docs/INDUSTRY_AND_PRODUCTION_DESIGN.md`
- 仓库规则：`docs/WAREHOUSE_EXPANSION_DESIGN.md`
- 市场与资产信息：`docs/MARKET_AND_ASSET_INFORMATION_ARCHITECTURE.md`
- 本地日志：`docs/LOCAL_ACTIVITY_LOG_DESIGN.md`
- 视觉系统：`docs/UI_DESIGN_SYSTEM.md`
- 服务器与部署：`docs/SERVER_ARCHITECTURE_AND_DEPLOYMENT_DESIGN.md`

若其他旧文档对页面归属的描述与本文件冲突，以本文件为准。

## 2. 页面划分原则

页面按照玩家意图划分：

```text
概览：了解当前情况并进入主要流程
市场：完成交易行为
工厂：管理生产资源、共享仓库和工厂
资金：查看资产结果与本地变化
排行：比较服务器总资产
设置：管理账号展示、界面偏好和危险操作
```

核心规则：

1. 每个正式操作必须有一个主要页面；
2. 完整管理模块不得在多个页面重复；
3. 其他页面可以显示只读摘要或跳转入口，但不得复制完整操作流程；
4. 全局状态栏属于应用外壳，不属于任一页面；
5. 本地历史记录可以在资金页显示其结果，但原始操作仍归属于对应业务页面；
6. 页面内容不得改变服务器权威边界。

## 3. 正式导航

导航固定为：

```text
概览｜市场｜工厂｜资金｜排行｜设置
```

稳定路由 ID 保持：

| 导航名称 | 路由 ID | React 页面 |
|---|---|---|
| 概览 | `home` | `OverviewPage` |
| 市场 | `market` | `MarketPage` |
| 工厂 | `production` | `ProductionPage` |
| 资金 | `assets` | `AssetsPage` |
| 排行 | `leaderboard` | `LeaderboardPage` |
| 设置 | `settings` | `SettingsPage` |

`production` 是兼容保留的稳定内部 ID，用户可见名称固定为“工厂”。不得仅为文案重命名破坏已有路由状态。

不得恢复：

- 独立订单页面；
- 独立成交记录页面；
- 独立仓库页面；
- `records` 路由；
- 设施槽位页面。

## 4. 应用外壳

`GameShell` 在六个页面外统一提供：

- 桌面侧栏；
- 移动端底部导航；
- 全局状态栏；
- 操作结果提示；
- 页面滚动区域。

### 4.1 全局状态栏

状态栏可以在所有页面显示紧凑摘要：

- 可用资金与冻结资金；
- 总资产与本周变化；
- 当前排名；
- 当前选中商品持仓；
- 仓库已用容量、总容量和买单预占；
- 当前选中商品最近成交价和趋势。

状态栏只提供只读摘要，不提供扩容、下单、撤单、建厂、生产计划或设置操作。

## 5. 概览页面

### 5.1 页面目的

让玩家在进入游戏后快速了解当前经济状态，并进入市场、工厂或资金页面。

### 5.2 必须包含

1. 玩家问候和当前经营说明；
2. 未完成订单数量摘要；
3. 基础工作组件：
   - 工作按钮；
   - 收益；
   - 冷却；
   - 连续档位；
   - 累计工作次数；
4. 当前选中商品市场摘要：
   - 最近成交价；
   - 买一价；
   - 卖一价；
   - 当前持仓；
   - 近期价格曲线；
5. 六种商品快捷行情；
6. 工厂摘要：
   - 运行数量；
   - 停止数量；
   - 阻塞数量；
   - 施工数量；
   - 定量计划数量与剩余产量；
   - 待领取产成品；
7. 财富构成：
   - 总资产；
   - 现金资产；
   - 商品估值；
   - 工厂估值；
   - 当前名次；
8. 当前浏览器最近成交，最多显示 6 条；
9. 前往市场、工厂和资金页的快捷入口。

### 5.3 不得包含

- 完整订单簿；
- 完整下单或撤单管理；
- 完整工厂控制卡；
- 共享仓库扩容卡；
- 玩家资料设置；
- 完整排行榜表格。

概览页是摘要页面，不得演变成所有功能的重复集合。

## 6. 市场页面

### 6.1 页面目的

完成商品交易与工厂资产交易。市场负责交易行为，不负责生产管理或仓库升级。

### 6.2 必须包含

1. 商品选择标签；
2. 当前商品行情摘要：
   - 买一价；
   - 卖一价；
   - 价差；
   - 可用与冻结持仓；
   - 当前浏览器本地平均成本；
3. 商品限价订单输入：
   - 买入或卖出；
   - 数量；
   - 限价；
   - 订单总额；
   - 可用资金；
   - 可用商品；
4. 当前商品未完成订单和快捷撤单；
5. 聚合订单簿；
6. 近期成交曲线；
7. 需求满足率和成交样本；
8. 全部商品未完成订单；
9. 冻结资金、冻结商品和未完成订单摘要；
10. 当前浏览器本地成交记录；
11. 工厂挂牌市场：
    - 工厂规格；
    - 卖方；
    - 挂牌价格；
    - 收购；
    - 撤销自己的挂牌。

### 6.3 允许的仓库信息

市场可以通过全局状态栏或下单提示显示：

- 当前仓库已用容量；
- 当前剩余容量；
- 买单预占数量。

这些信息用于解释买单容量约束，但市场页不得显示扩容按钮或完整共享仓库卡。

### 6.4 不得包含

- 仓库扩容操作；
- 建设工厂；
- 工厂启动或停止；
- 生产计划编辑；
- 资产配置图；
- 账号设置。

## 7. 工厂页面

### 7.1 页面目的

管理生产所需的共享仓库和全部工厂。共享仓库与工厂产成品领取直接相关，因此其唯一完整管理入口固定在工厂页面。

### 7.2 页面顺序

工厂页面从上到下固定为：

```text
页面标题与运行状态
→ 共享仓库
→ 建设新工厂
→ 已有工厂列表
```

共享仓库必须位于建设与工厂列表之前，使玩家在建厂、生产和领取前先看到容量状态。

### 7.3 共享仓库必须包含

1. 当前等级与最高等级；
2. 已用容量与总容量；
3. 使用率；
4. 实物库存；
5. 未完成买单预占；
6. 剩余容量；
7. 当前容量；
8. 扩容后容量；
9. 本次扩容费用；
10. 扩容按钮；
11. 资金不足状态；
12. 容量超限状态；
13. 满级状态。

`WarehouseUpgradeCard` 只能由 `ProductionPage` 渲染。资金页、设置页、概览页和市场页不得渲染该组件。

### 7.4 建设新工厂必须包含

- 工厂类型选择；
- 配方；
- 建造费用；
- 施工时间；
- 生产周期；
- 周期产出；
- 运营费用；
- 内部产成品容量；
- 建设按钮；
- 同时只能施工一座工厂的说明。

### 7.5 每座工厂必须包含

- 工厂名称和状态；
- 停止原因；
- 配方；
- 内部产成品数量与内部容量；
- 施工或生产进度；
- 周期、小时产量和小时运营费；
- 原料库存；
- 累计产量；
- 系统参考估值；
- 持续或定量生产计划；
- 定量目标与完成量；
- 保存计划；
- 手动启动；
- 手动停止；
- 领取产成品；
- 挂牌价格；
- 挂牌出售；
- 撤销挂牌。

### 7.6 不得包含

- 商品订单簿；
- 全部订单管理；
- 完整资产配置；
- 玩家资料设置；
- 排行榜表格。

## 8. 资金页面

### 8.1 页面目的

展示服务器权威资产结果和当前浏览器本地变化记录。资金页不负责仓库管理。

### 8.2 必须包含

1. 可用资金；
2. 冻结资金；
3. 当前总资产；
4. 商品资产；
5. 工厂资产；
6. 现金、商品、工厂资产配置；
7. 资产估值明细；
8. 各商品：
   - 可用数量；
   - 冻结数量；
   - 参考价；
   - 估值；
9. 货币发行与系统回收；
10. 当前浏览器本地资金与资产变化；
11. 本地事件分类筛选；
12. 清除当前浏览器本地记录。

### 8.3 仓库历史记录

资金页可以在本地资产事件中显示已经发生的仓库扩容结果：

- 扣款；
- 扩容前后等级；
- 扩容前后容量；
- 容量增量。

这是历史结果展示，不是仓库管理入口。

### 8.4 不得包含

- `WarehouseUpgradeCard`；
- 扩容按钮；
- 下次扩容费用管理；
- 建厂或生产控制；
- 下单和撤单表单；
- 账号设置。

## 9. 排行页面

### 9.1 页面目的

只读展示服务器计算的总资产排名。

### 9.2 必须包含

1. 我的排名；
2. 我的总资产；
3. 与上一名差距；
4. 本周资产变化；
5. 排行榜更新时间；
6. 排行表格：
   - 排名；
   - 玩家；
   - 总资产；
   - 现金资产；
   - 工厂数量；
   - 本周变化；
   - 更新时间；
7. 当前玩家突出显示。

### 9.3 不得包含

- 资产转移；
- 玩家交易；
- 直接修改排名；
- 仓库或工厂管理；
- 账号设置。

## 10. 设置页面

### 10.1 页面目的

管理玩家展示信息、客户端偏好、会话和危险操作。

### 10.2 必须包含

1. 玩家资料：
   - 头像文字；
   - 玩家昵称；
   - 邮箱；
   - 注册时间；
   - 账号角色；
   - 保存昵称；
   - 前往主页账号资料；
2. 客户端偏好：
   - 界面音效；
   - 紧凑数字；
   - 市场刷新频率；
   - 画面性能；
3. 账号与产业只读摘要：
   - 账号编号；
   - 账号角色；
   - 工厂总数；
   - 运行中工厂；
   - 施工中工厂；
   - 阻塞工厂；
   - 商品种类；
   - 未完成订单；
   - 当前排名；
4. 退出登录；
5. 重置服务器经济状态及明确警告。

### 10.3 不得包含

- 共享仓库完整信息；
- 仓库扩容按钮；
- 商品下单；
- 工厂启动、停止或计划编辑；
- 资产配置和本地资产日志；
- 排行榜表格。

## 11. 模块唯一归属表

| 模块或操作 | 唯一主要页面 | 其他页面允许内容 |
|---|---|---|
| 工作 | 概览 | 状态摘要 |
| 商品下单 | 市场 | 概览可跳转 |
| 撤销商品订单 | 市场 | 不得在其他页面操作 |
| 商品订单簿 | 市场 | 概览仅行情摘要 |
| 工厂挂牌收购 | 市场 | 工厂页可发起自己的挂牌 |
| 共享仓库扩容 | 工厂 | 状态栏只读摘要；资金页仅历史事件 |
| 建设工厂 | 工厂 | 概览只读摘要与跳转 |
| 生产计划 | 工厂 | 概览只读摘要 |
| 启动与停止工厂 | 工厂 | 概览只读摘要 |
| 领取产成品 | 工厂 | 资金页只显示结果 |
| 资产配置 | 资金 | 概览只显示财富摘要 |
| 本地资产事件 | 资金 | 概览只显示最近成交 |
| 总资产排行榜 | 排行 | 状态栏和概览只显示当前名次 |
| 玩家昵称 | 设置 | 侧栏只显示当前名称 |
| 退出登录 | 设置与侧栏 | 不得混入业务卡片 |
| 重置经济状态 | 设置 | 不得提供快捷按钮 |

## 12. 空状态与错误状态

每个页面必须提供与自身职责一致的空状态：

- 概览：无本地成交时引导进入市场；
- 市场：无订单、无成交、无挂牌时分别显示空状态；
- 工厂：无工厂时引导先确认共享仓库容量并建设第一座工厂；
- 资金：无本地事件时说明当前浏览器暂无记录；
- 排行：服务器无排行数据时显示只读空状态，不伪造玩家；
- 设置：账号或设置数据不可用时不得展示可执行的错误操作。

服务器连接失败由应用加载层统一处理，不得由页面伪造本地权威状态。

## 13. 桌面与移动端一致性

桌面端和移动端必须具有相同业务能力与页面归属。

允许差异：

- 布局列数；
- 卡片排列；
- 表格横向滚动；
- 状态栏紧凑值；
- 导航呈现为侧栏或底栏。

不得因为移动端空间有限而：

- 把仓库扩容移动到设置页；
- 删除撤单；
- 删除工厂启动或停止；
- 删除本地日志清除；
- 把关键操作隐藏在不可发现的手势中。

按钮和输入目标不得小于 44px。

## 15. 测试与防回退要求

必须检查：

1. 导航顺序为“概览、市场、工厂、资金、排行、设置”；
2. 稳定路由 ID 不变；
3. `ProductionPage` 渲染 `WarehouseUpgradeCard`；
4. `AssetsPage` 不渲染 `WarehouseUpgradeCard`；
5. `SettingsPage` 不渲染 `WarehouseUpgradeCard`；
6. 设置页不显示仓库等级、扩容费用或扩容按钮；
7. 资金页保留仓库扩容本地历史事件；
8. 市场页保留全部订单和撤单操作；
9. 概览页只显示摘要，不复制完整管理模块；
10. 本设计文档被 UI 架构检查读取。

## 16. 不可回退规则

不得：

- 把共享仓库完整管理模块移回资金页或设置页；
- 在多个页面重复渲染 `WarehouseUpgradeCard`；
- 把用户可见“工厂”重新改为含糊的“生产”导航而不更新设计；
- 恢复独立订单、记录或仓库页面；
- 在概览页复制全部市场、工厂或资金操作；
- 把交易操作移出市场页；
- 把生产控制移出工厂页；
- 把资产变化日志移出资金页；
- 把重置经济状态移出设置页；
- 在未更新本文件和架构检查的情况下改变页面职责。

未更新本设计、相关专项设计和防回退检查的页面结构修改不应合并。
