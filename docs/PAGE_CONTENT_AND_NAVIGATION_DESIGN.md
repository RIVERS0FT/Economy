# Economy 页面内容与导航职责设计

> 状态：当前正式页面、模块唯一归属与导航基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-13  
> 客户端状态版本：11  
> 世界状态版本：7

## 1. 正式导航

```text
概览｜市场｜生产｜资产｜藏品｜拍卖｜排行｜设置
```

| 导航 | 路由 ID | React 页面 | 唯一职责 |
|---|---|---|---|
| 概览 | `home` | `OverviewPage` | 工作和经营摘要 |
| 市场 | `market` | `MarketPage` | 商品与工厂统一交易 |
| 生产 | `production` | `ProductionPage` | 仓库、建设和工厂集群 |
| 资产 | `assets` | `AssetsPage` | 资产结果与本地变化 |
| 藏品 | `collections` | `CollectionsPage` | 当前玩家持有的唯一艺术藏品 |
| 拍卖 | `auction` | `AuctionPage` | 藏品竞价拍卖与结算结果 |
| 排行 | `leaderboard` | `LeaderboardPage` | 服务器总资产排名 |
| 设置 | `settings` | `SettingsPage` | 资料、偏好、礼品、退出和重置 |

显示名称从“资金”统一改为“资产”，但路由 ID 继续使用 `assets`，不得为文案改名新增路由、状态迁移或兼容分支。

不得恢复独立订单页、成交记录页、仓库页、工厂实例页、`records` 路由或设施槽位页面。藏品和拍卖是正式独立页面，不得合并进商品与工厂市场。

## 2. 应用外壳

`GameShell` 提供：

- 桌面侧栏；
- 移动底部导航；
- 全局状态栏；
- 操作结果提示；
- 唯一页面纵向滚动区域。

状态栏固定显示可用资金、总资产、排行榜和仓库剩余。状态栏只读，不提供扩容、下单、撤单、建厂、计划、藏品拍卖或设置操作。

移动底部导航允许横向滚动，八个页面不得通过隐藏导航项、缩写中文名称或创建二级菜单规避空间限制。

## 3. 概览

必须包含：

- 根据浏览器本地系统时间显示“凌晨好／早上好／中午好／下午好／晚上好”的玩家问候；
- 基础工作和 10 秒冷却；
- 按服务器 `products` 目录生成商品下拉框，切换后只更新概览商品行情、持仓和价格曲线；
- 当前商品进入市场的入口，进入后必须保持概览下拉框选中的商品；
- 工厂运行、停止、异常、施工和待加入摘要；
- 财富摘要；
- 当前玩家所有等待成交或部分成交的商品与工厂挂单摘要；
- 前往市场和生产的入口。

概览商品行情不得恢复全部商品快捷切换条。商品选择必须使用原生选择器和服务器目录，不得硬编码商品；切换选择器不得自动跳转页面。

概览选中的商品属于 `PageRouter` 当前登录会话的页面级 UI 状态：切换到市场、生产、资产或其他正式页面后再返回概览时必须保留选择；不得写入服务器、`localStorage`、`sessionStorage` 或全局 `LoadedGameViewModel`，不得与市场页 `marketAssetId` 双向同步。自动刷新、工作、下单、撤单或其他权威状态更新不得重置仍存在于服务器目录中的概览商品；仅当当前商品从目录移除时回退到首个商品。刷新整个页面或重新登录后，从服务器目录首项重新初始化。

点击“进入该商品市场”是概览商品影响市场选择的唯一入口。该操作将概览当前商品单向传给统一市场；市场中的后续商品切换不得反向覆盖概览选择。

桌面宽度大于 960px 时，生产摘要、财富构成和当前挂单固定在同一行，三张卡片统一为 `384px` 高。挂单列表在卡片内部滚动，不得改变整行高度。宽度不超过 960px 时改为单列并恢复自然高度。

概览挂单只负责显示资产类型、资产名称、买卖方向、价格、剩余／原始数量、状态和挂单时间，并提供进入市场管理的入口。撤单、完整订单表格和完整订单簿仍只属于市场页。

不得包含当前浏览器最近成交、完整订单簿、完整工厂卡、仓库扩容、藏品画廊、拍卖列表、设置表单或完整排行榜。

## 4. 市场

必须包含：

- 商品和工厂统一资产标签；
- 买入／卖出方向；
- 价格；
- 数量；
- 数量下方快捷仓位；
- 订单总额和提交；
- 当前资产未完成订单与撤单；
- 全部自有未完成订单；
- 单列最优 5 笔卖单、中性分隔线和最优 5 笔买单；
- 价格曲线；
- 当前浏览器本地成交记录。

玩家可见输入字段、订单标题和未完成订单表头统一使用“价格”，不显示“限价”。底层订单仍按限价订单规则撮合。

默认价格只读取当前客户端已加载的订单快照，不主动请求服务器：

```text
买入：最低卖出价 → 无卖单时最高买入价 → 买卖盘均空时 1
卖出：最高买入价 → 无买单时最低卖出价 → 买卖盘均空时 1
```

首次进入市场、从其他页面重新进入市场、切换商品或工厂、切换买入／卖出方向时重新填写默认价格。自动刷新和下单后的状态同步不得覆盖当前输入；下次触发重新填写时使用届时最新的本地订单快照。

商品标签使用商品 SVG；工厂标签使用独立厂房 SVG。工厂图标不得使用机械商品的齿轮图标，也不得恢复 `⚙` 字符。

市场不得包含藏品拍卖。藏品是唯一实例，不能复用商品／工厂按数量成交的订单簿。

不得包含：

- 商品／工厂二级页面切换；
- 工厂固定价格挂牌或购买卡；
- 玩家身份出现在订单簿；
- 价格档位聚合；
- 仓库扩容、建设、启停或计划编辑；
- 内部固定高度、分页或垂直滚动的本地成交面板。

## 5. 生产

页面主标题固定为“生产”。页面顺序：

```text
标题“生产”与工厂状态汇总
→ 共享仓库
→ 生产管理区
   ├─ 左侧：建设新工厂
   └─ 右侧：工厂类型集群卡
```

### 5.1 共享仓库

显示等级、容量、实物库存、买单预占、剩余容量、无限扩容信息和有库存商品卡。完整规则见 `WAREHOUSE_EXPANSION_DESIGN.md`。

仓库不再提供“有库存／全部商品”筛选，只展示可用或冻结数量大于零的商品。“仓库内容”标题不显示商品种类统计。商品卡必须显示统一商品 SVG、商品名称、醒目的可用数量以及弱化的冻结数量，不显示独立库存总量行。平板、手机和极窄屏保持双列。

### 5.2 建设

建设新工厂卡独占左侧列并在桌面滚动时常驻：

- 桌面宽度为 `280px–320px`，使用 `position: sticky`；
- 常驻范围只限生产管理区，不得使用 `position: fixed`；
- 不超过 960px 时恢复普通文档流并位于工厂列表之前。

必须显示：

- 工厂类型和配方；
- 建造费和施工时间；
- 周期、单座产量和成本；
- 当前施工任务；
- 产成品直接入仓；
- 新数量下一周期加入；
- 同时只能施工一座。

### 5.3 工厂集群卡

每种工厂类型最多一张卡。桌面卡片固定高度 384px，移动端使用自然高度。大于 1380px 时工厂列表固定四列，961px–1380px 与移动端均为单列。

必须显示：

- 工厂类型和总数量，数量只在标题中出现；
- 运行、停止或异常状态；异常原因直接合并到顶部状态标签；
- 顶部紧凑摘要“运行中／下一周期加入／冻结中”；
- 与工厂名称同一水平行的统一运行开关；
- 单座生产公式：输入在左、周期与成本 SVG 居中、输出在右，进度条位于下方；
- 多输入、多输出和逐输入库存兼容展示，以及当前参与数量的集群周期产出与成本汇总；
- “当前计划”标题、模式下拉框和可选定量输入；
- 服务器存在待生效计划时，在标题右侧显示“下一周期生效”；
- 计划字段自动保存，不显示“保存计划”按钮；
- 前往统一订单簿的工厂市场入口。

不得显示：

- 顶部“正常生产中”或周期剩余重复说明；
- 独立持有、参与、待加入或冻结数量卡；
- “下一周期按若干座结算”辅助句；
- 周期、产量、成本和原料四格规格区，或以箭头替代生产进度条；
- 重复的持续／定量状态小卡片或静态计划模式文案；
- 工厂实例、实例 ID、内部库存、领取、单座启停、固定价格挂牌、小时指标或系统参考估值。

## 6. 资产

必须包含：

- 可用资金、冻结资金和总资产；
- 商品资产汇总；
- 工厂数量资产；
- 服务器估值价格与资产组成；
- 货币发行和系统回收；
- 当前浏览器本地资金、商品、仓库、工厂和生产变化；
- 本地事件筛选与清除。

资产页不得再显示逐商品“商品库存与估值”卡片；具体商品库存只在生产页共享仓库展示。资产页不负责仓库管理、建厂、生产控制、下单或藏品管理。

藏品目前不计入总资产，因此资产页不得为藏品显示任意估值或把拍卖最高价直接计入总资产。

## 7. 藏品

页面主标题固定为“藏品”，只展示当前账号持有的唯一藏品：

- 使用芝加哥艺术博物馆 IIIF 图片卡；
- 显示标题、艺术家、年代、材质、尺寸、馆藏编号和当前归属；
- 提供芝加哥艺术博物馆原馆藏页面链接；
- 显示是否正在拍卖；
- 没有藏品时提供前往拍卖页的入口。

图片使用 `loading="lazy"` 和异步解码。图片加载失败时显示明确占位提示，不得从浏览器修改图片源、归属或创建本地伪藏品。

藏品页不负责出价、资金冻结、结算、商品库存或工厂控制。

## 8. 拍卖

页面主标题固定为“拍卖”。必须包含：

- 当前玩家未在拍卖中的藏品选择；
- 起拍价和 1～168 小时时长；
- 进行中拍卖图片、作品信息、卖家、当前价、最高出价者、出价次数和剩余时间；
- 非卖家的最低有效出价输入；
- 卖家在没有任何出价时的取消操作；
- 最近结束的成交、流拍或取消结果。

最高出价资金、退款、拍卖状态和归属转移全部由服务器判定。客户端倒计时只用于展示，显示到零时不得自行宣布成交，必须等待服务器返回结算状态。

拍卖页不得：

- 允许卖家竞拍自己的藏品；
- 在已有出价后显示可用的取消按钮；
- 允许同时为同一藏品创建多场拍卖；
- 把拍卖数据保存为浏览器唯一状态；
- 将拍卖最高价写入商品／工厂估值。

## 9. 排行

只读展示：

- 我的排名；
- 总资产；
- 与上一名差距；
- 本周变化；
- 完整服务器排行榜。

不得包含交易、生产、扩容、藏品拍卖或设置操作。

## 10. 设置

设置页只允许：

- 玩家资料与四项统计；
- 客户端偏好；
- 礼品兑换；
- 管理员入口，仅 `role=admin` 显示；
- 退出登录；
- 重置经济状态。

状态刷新属于客户端偏好：默认每 5 秒按服务器修订号轮询，可选 3／5／10 秒。设置页必须使用“状态刷新频率”描述，不得恢复每 1 秒完整状态刷新或将该偏好误写成只影响市场页。

存在进行中的藏品出售或当前账号为最高出价者时，服务器拒绝重置。设置页不得绕过这一限制。

不得作为经营仪表盘，不显示工厂、商品、订单、仓库、藏品、拍卖或排名摘要。

## 11. 商品与工厂目录扩展规则

所有目录型页面必须遍历服务器返回的 `products` 和 `facilityTypes`。

- 不得写死 6 个商品 ID。
- 不得固定六个行情卡、六个商品标签或六张工厂卡。
- 新商品和工厂应在不修改页面结构的情况下自动出现。
- 目录型横向导航可以滚动或自动分配列宽，但不能依赖固定项目数量。
- 空目录必须显示明确空状态。

仓库虽然只显示有库存商品，过滤仍必须基于服务器返回的完整 `products` 数组和对应库存，不得维护第二套商品清单。

藏品不是静态前端目录；页面必须遍历服务器返回的 `collectibles` 和 `collectibleAuctions`，不得在 JSX 中写死具体作品。

## 12. 模块唯一归属

| 模块 | 唯一主要页面 |
|---|---|
| 工作 | 概览 |
| 统一资产下单、订单簿、撤单和本地成交 | 市场 |
| 仓库管理、商品库存和扩容 | 生产 |
| 建设、周期、计划、启停和工厂状态 | 生产 |
| 资产组成与本地资产事件 | 资产 |
| 当前玩家藏品画廊 | 藏品 |
| 藏品发售、竞价和结算结果 | 拍卖 |
| 总资产排名 | 排行 |
| 资料、偏好、礼品、退出和重置 | 设置 |
| 礼品码与藏品导入、归属历史 | 管理员页面 |

## 13. 防回退

不得：

- 跨页复制完整管理模块；
- 恢复独立订单、记录、仓库或工厂实例路由；
- 恢复固定价格工厂市场；
- 恢复六商品写死页面；
- 在概览恢复全部商品快捷切换条、切换商品后自动跳转市场或当前浏览器最近成交面板；
- 将概览商品选择重新写入全局 ViewModel、服务器或浏览器存储，或恢复市场商品向概览商品的反向同步；
- 让普通页面切换重置仍有效的概览商品选择；
- 让自动刷新、工作、下单、撤单或其他状态更新重置仍有效的概览商品选择；
- 将概览当前挂单改为浏览器本地记录，或在概览加入撤单和完整订单簿；
- 删除概览三张摘要卡的桌面同排、统一 `384px` 高度或移动端自然高度规则；
- 把仓库扩容移出生产页；
- 恢复仓库“有库存／全部商品”筛选、商品种类统计、独立库存总量行或在空仓状态提供“查看全部商品”；
- 把移动仓库商品卡恢复为单列，或把可用与冻结合并到同一行；
- 在资产页恢复逐商品库存与估值卡；
- 把计划或启停移到市场页；
- 给本地成交表添加内部纵向滚动、分页或折叠；
- 把藏品和拍卖塞入商品／工厂统一订单簿；
- 把藏品归属或拍卖结算改为浏览器本地状态；
- 隐藏移动端的藏品或拍卖导航；
- 为任意图片外链创建管理员上传通道。
