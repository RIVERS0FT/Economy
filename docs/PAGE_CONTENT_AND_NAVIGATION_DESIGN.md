# Economy 页面内容与导航职责设计

> 状态：当前正式页面内容、操作归属与导航基线  
> 适用项目：`RIVERS0FT/Economy`  
> 更新时间：2026-07-11  
> 客户端状态版本：`version: 7`

## 1. 目的与适用范围

本文件明确桌面端和移动端六个正式页面分别展示什么、允许执行什么操作，以及哪些完整模块不得跨页重复。

涉及页面新增、模块移动、页面重命名、快捷入口、跨页重复展示或导航调整时，必须先更新本文件，再修改代码与防回退检查。

相关设计：

- 生产规则：`docs/INDUSTRY_AND_PRODUCTION_DESIGN.md`
- 仓库规则：`docs/WAREHOUSE_EXPANSION_DESIGN.md`
- 市场与资产：`docs/MARKET_AND_ASSET_INFORMATION_ARCHITECTURE.md`
- 本地日志：`docs/LOCAL_ACTIVITY_LOG_DESIGN.md`
- 视觉系统：`docs/UI_DESIGN_SYSTEM.md`
- 服务器与部署：`docs/SERVER_ARCHITECTURE_AND_DEPLOYMENT_DESIGN.md`

若旧文档的页面归属与本文件冲突，以本文件为准。

## 2. 页面划分原则

```text
概览：了解当前状态并进入主要流程
市场：完成商品和工厂资产交易
生产：管理共享仓库、建设工厂和控制生产
资金：查看资产结果与本地变化记录
排行：比较服务器总资产
设置：管理资料、客户端偏好、登录会话和危险操作
```

核心原则：

1. 每个正式操作必须有唯一主要页面；
2. 完整管理模块不得在多个页面重复；
3. 其他页面只能显示本页面决策所必需的摘要或跳转入口；
4. 全局状态栏属于应用外壳，不属于任一页面；
5. 本地历史记录可以在资金页显示结果，但原始操作仍归对应业务页面；
6. 设置页不是经营仪表盘，不得展示产业、仓库、订单或排名摘要；
7. 页面展示不得改变服务器权威边界。

## 3. 正式导航与路由

导航固定为：

```text
概览｜市场｜生产｜资金｜排行｜设置
```

| 导航 | 路由 ID | React 页面 | 页面业务含义 |
|---|---|---|---|
| 概览 | `home` | `OverviewPage` | 摘要与快捷入口 |
| 市场 | `market` | `MarketPage` | 商品与工厂交易 |
| 生产 | `production` | `ProductionPage` | 共享仓库与工厂管理 |
| 资金 | `assets` | `AssetsPage` | 资产结果与本地记录 |
| 排行 | `leaderboard` | `LeaderboardPage` | 总资产排名 |
| 设置 | `settings` | `SettingsPage` | 资料、偏好、会话与重置 |

本文中的“工厂页面”指用户从“生产”导航进入的 `ProductionPage`。`production` 是稳定内部 ID，不得仅因文案调整破坏已有路由状态。

不得恢复：

- 独立订单页面；
- 独立成交记录页面；
- 独立仓库页面；
- `records` 路由；
- 设施槽位页面。

## 4. 应用外壳

`GameShell` 在六个页面外统一提供：

- 桌面侧栏；
- 移动端底部导航；
- 全局状态栏；
- 操作结果提示；
- 页面滚动区域。

### 4.1 全局状态栏

状态栏可以在所有页面显示只读摘要：

- 可用资金与冻结资金；
- 总资产与本周变化；
- 当前排名；
- 当前选中商品持仓；
- 仓库已用容量、总容量和买单预占；
- 当前选中商品最近成交价和趋势。

状态栏不得提供扩容、下单、撤单、建厂、生产计划或设置操作。

## 5. 概览页面

### 5.1 目的

让玩家快速了解当前经营状态，并进入市场、生产或资金页面。

### 5.2 必须包含

1. 玩家问候和经营说明；
2. 未完成订单数量摘要；
3. 基础工作：收益、按钮、冷却、连续档位和累计次数；
4. 当前商品市场摘要：最近成交、买一、卖一、持仓和价格曲线；
5. 六种商品快捷行情；
6. 生产摘要：运行、停止、阻塞、施工、定量计划和待领取产成品；
7. 财富摘要：总资产、现金、商品、工厂估值和当前名次；
8. 当前浏览器最近成交，最多 6 条；
9. 前往市场、生产和资金页面的快捷入口。

### 5.3 不得包含

- 完整订单簿或完整撤单管理；
- 完整工厂控制卡；
- `WarehouseUpgradeCard`；
- 玩家资料设置；
- 完整排行榜表格。

概览是摘要页面，不得演变成所有功能的重复集合。

## 6. 市场页面

### 6.1 目的

完成商品交易与工厂资产交易。市场负责交易行为，不负责生产管理或仓库扩容。

### 6.2 必须包含

1. 商品选择；
2. 当前商品行情：买一、卖一、价差、持仓和本地平均成本；
3. 限价订单输入：方向、数量、价格、总额、可用资金和可用商品；
4. 当前商品未完成订单与快捷撤单；
5. 聚合订单簿；
6. 近期成交曲线、成交样本和需求满足率；
7. 全部商品未完成订单；
8. 冻结资金、冻结商品和订单数量摘要；
9. 当前浏览器本地成交记录；
10. 工厂挂牌市场：规格、卖方、价格、收购和撤销自己的挂牌。

### 6.3 仓库信息边界

市场可以通过状态栏或订单提示显示：

- 仓库已用容量；
- 剩余容量；
- 买单预占。

这些信息只解释买单容量限制。市场不得显示扩容按钮或完整共享仓库卡。

### 6.4 不得包含

- 仓库扩容；
- 建设工厂；
- 工厂启动、停止或生产计划；
- 资产配置图；
- 账号设置。

## 7. 生产页面（工厂页面）

### 7.1 目的

管理生产所需的共享仓库和全部工厂。共享仓库直接约束工厂产成品领取，因此其唯一完整管理入口固定在 `ProductionPage`。

### 7.2 页面顺序

```text
页面标题与工厂状态
→ 共享仓库
→ 建设新工厂
→ 已有工厂列表
```

共享仓库必须位于建设与工厂列表之前，使玩家在建厂、生产和领取前先看到容量状态。

### 7.3 共享仓库必须包含

- 当前等级与最高等级；
- 已用容量与总容量；
- 使用率；
- 实物库存；
- 未完成买单预占；
- 剩余容量；
- 当前容量；
- 扩容后容量；
- 本次扩容费用；
- 扩容按钮；
- 资金不足、容量超限和满级状态。

`WarehouseUpgradeCard` 只能由 `ProductionPage` 渲染。概览、市场、资金、排行和设置页面不得渲染该组件。

### 7.4 建设新工厂必须包含

- 工厂类型；
- 配方；
- 建造费用；
- 施工时间；
- 生产周期；
- 周期产出；
- 运营费用；
- 内部产成品容量；
- 建设按钮；
- 同时只能施工一座工厂的说明。

### 7.5 每座工厂必须包含

- 名称、状态和停止原因；
- 配方；
- 内部产成品与内部容量；
- 施工或生产进度；
- 周期、小时产量和小时运营费；
- 原料库存；
- 累计产量；
- 系统参考估值；
- 持续或定量生产计划；
- 定量目标与完成量；
- 保存计划；
- 手动启动和停止；
- 领取产成品；
- 挂牌价格、挂牌出售和撤销挂牌。

### 7.6 不得包含

- 商品订单簿；
- 全部订单管理；
- 完整资产配置；
- 玩家资料设置；
- 排行榜表格。

## 8. 资金页面

### 8.1 目的

展示服务器权威资产结果和当前浏览器本地变化记录。资金页不负责仓库管理。

### 8.2 必须包含

- 可用资金；
- 冻结资金；
- 当前总资产；
- 商品资产；
- 工厂资产；
- 现金、商品和工厂资产配置；
- 资产估值明细；
- 各商品可用数量、冻结数量、参考价和估值；
- 货币发行与系统回收；
- 当前浏览器本地资金与资产变化；
- 本地事件分类筛选；
- 清除当前浏览器本地记录。

### 8.3 仓库历史边界

资金页可以在本地资产事件中显示已经发生的仓库扩容结果：

- 扣款；
- 扩容前后等级；
- 扩容前后容量；
- 容量增量。

这是历史结果展示，不是仓库管理入口。

### 8.4 不得包含

- `WarehouseUpgradeCard`；
- 扩容按钮或下次扩容费用控制；
- 建厂和生产控制；
- 下单和撤单表单；
- 账号设置。

## 9. 排行页面

### 9.1 目的

只读展示服务器计算的总资产排名。

### 9.2 必须包含

- 我的排名和总资产；
- 与上一名差距；
- 本周资产变化；
- 排行榜更新时间；
- 排名、玩家、总资产、现金资产、工厂数量、本周变化和更新时间；
- 当前玩家突出显示。

### 9.3 不得包含

- 资产转移；
- 玩家交易；
- 直接修改排名；
- 仓库或工厂管理；
- 账号设置。

## 10. 设置页面

### 10.1 目的

管理玩家展示信息、客户端偏好、登录会话和危险操作。设置页不承担经营状态汇总职责。

### 10.2 必须包含

1. 玩家资料：昵称、邮箱、注册时间、角色、保存昵称和主页资料入口；
2. 客户端偏好：音效、紧凑数字、刷新频率和画面性能；
3. 登录会话：退出当前浏览器登录，并说明不会删除服务器经济状态或本地活动记录；
4. 重置服务器经济状态及明确警告。

### 10.3 不得包含

- 账号与产业只读摘要；
- 仓库使用摘要；
- 工厂数量、运行、施工或阻塞摘要；
- 商品种类或库存摘要；
- 未完成订单摘要；
- 当前排名摘要；
- `WarehouseUpgradeCard`；
- 仓库等级、扩容费用或扩容操作；
- 商品下单；
- 工厂控制；
- 资产配置和本地资产日志；
- 排行榜表格。

设置页不得显示账号与产业只读摘要或仓库使用摘要。经营状态分别归概览、市场、生产、资金和排行页面。

## 11. 模块唯一归属

| 模块或操作 | 唯一主要页面 | 其他页面允许内容 |
|---|---|---|
| 工作 | 概览 | 状态摘要 |
| 商品下单与撤单 | 市场 | 概览可跳转 |
| 商品订单簿 | 市场 | 概览仅行情摘要 |
| 工厂挂牌收购 | 市场 | 生产页可发起自己的挂牌 |
| 共享仓库扩容 | 生产 | 状态栏只读摘要；资金页仅历史事件 |
| 建设工厂 | 生产 | 概览只读摘要与跳转 |
| 生产计划 | 生产 | 概览只读摘要 |
| 启动、停止和领取 | 生产 | 资金页只显示结果 |
| 资产配置 | 资金 | 概览只显示财富摘要 |
| 本地资产事件 | 资金 | 概览只显示最近成交 |
| 总资产排行榜 | 排行 | 状态栏和概览只显示当前名次 |
| 玩家昵称与偏好 | 设置 | 侧栏只显示当前名称 |
| 登录会话 | 设置 | 不得提供跨页退出按钮 |
| 重置经济状态 | 设置 | 不得提供快捷按钮 |

## 12. 空状态与错误状态

- 概览无本地成交时引导进入市场；
- 市场对无订单、无成交、无挂牌分别显示空状态；
- 生产无工厂时引导先确认共享仓库容量，再建设第一座工厂；
- 资金无本地事件时说明当前浏览器暂无记录；
- 排行无数据时显示只读空状态，不伪造玩家；
- 设置数据不可用时不得提供错误的可执行操作。

服务器连接失败由应用加载层统一处理，页面不得伪造本地权威状态。

## 13. 桌面与移动端一致性

桌面端和移动端必须具有相同业务能力与页面归属。

允许差异：布局列数、卡片排列、表格横向滚动、状态栏紧凑值，以及侧栏或底栏导航。

不得因为移动端空间有限而：

- 把仓库扩容移动到设置页；
- 把经营摘要复制到设置页；
- 删除撤单；
- 删除工厂启动或停止；
- 删除本地日志清除；
- 把关键操作隐藏在不可发现的手势中。

按钮和输入目标不得小于 44px。

## 15. 测试与防回退要求

必须检查：

1. 导航顺序为“概览、市场、生产、资金、排行、设置”；
2. 稳定路由 ID 不变；
3. `ProductionPage` 渲染 `WarehouseUpgradeCard`；
4. `AssetsPage`、`SettingsPage`、`OverviewPage` 和 `MarketPage` 不渲染该组件；
5. 设置页包含玩家资料、客户端偏好、登录会话和重置；
6. 设置页不包含账号与产业摘要、仓库使用摘要或其他经营状态摘要；
7. 资金页保留仓库扩容本地历史事件；
8. 市场页保留全部订单和撤单；
9. 概览页只显示摘要；
10. 本文件被架构检查读取。

## 16. 不可回退规则

不得：

- 把共享仓库完整管理模块移回资金页或设置页；
- 在多个页面重复渲染 `WarehouseUpgradeCard`；
- 把账号与产业摘要或仓库使用摘要恢复到设置页；
- 恢复独立订单、记录或仓库页面；
- 在概览复制全部市场、生产或资金操作；
- 把交易操作移出市场；
- 把工厂控制移出生产；
- 把资产变化日志移出资金；
- 把重置经济状态移出设置；
- 在未更新本文件和架构检查的情况下改变页面职责。

未更新本设计、相关专项设计和防回退检查的页面结构修改不应合并。
