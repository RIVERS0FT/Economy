# Economy

Economy 是一款多人在线经济模拟与交易游戏。

- 项目：`RIVERS0FT/Economy`
- 生产网页：`https://game.riversoft.top/economy/`
- 当前方向：网页端多人联机金融与产业经营
- 客户端状态版本：9
- 世界状态版本：5

页面内容与模块归属以 `docs/PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 为准；工厂集群和市场第三版以 `docs/FACILITY_GROUP_AND_MARKET_V3_DESIGN.md` 为准。

## 核心循环

```text
工作获得基础资金
→ 扩容共享仓库
→ 建设同类型工厂数量
→ 统一设置生产计划并启动集群
→ 产成品直接进入共享仓库
→ 使用快捷仓位提交商品订单
→ 在单列订单簿观察买卖盘
→ 按数量挂牌或购买工厂
→ 提升总资产与排名
```

## 扩展产业目录

当前目录共 12 种商品和 12 种工厂类型。新增两条完整产业链：

```text
木材 → 木板 → 家具
原油 → 塑料 → 电子产品
```

旧存档在加载时自动补齐新商品的零库存槽位、新市场价格历史和基础买卖流动性，不重置资金、原有库存、工厂数量、订单或仓库。市场、概览、资金和生产页面都必须按服务器目录动态渲染，不得把 6 种商品或 6 种工厂写死在客户端。

## 工厂类型集群

工厂不再以单座实例保存。玩家资产按类型和数量表示：

```text
农场 × 8
伐木场 × 5
矿场 × 4
油田 × 2
面粉厂 × 3
锯木厂 × 2
```

每种工厂类型只有一个统一状态、一条统一进度、一个统一生产计划和一组统一启停操作。

不存在工厂实例 ID、实例列表、展开实例、单座启停、工厂内部仓库或领取产成品。

### 统一周期

```text
周期产量 = 单座周期产量 × 当前参与数量
周期成本 = 单座周期成本 × 当前参与数量
周期原料 = 单座原料 × 当前参与数量
```

生产完成后产成品直接进入共享仓库。

### 新数量下一周期加入

施工完成或市场收购发生在集群运行期间时：

```text
当前参与数量保持不变
新增数量进入 pendingJoinCount
当前周期按旧数量完成
下一周期开始时加入统一生产
```

新增数量不会重置当前进度，也不会获得部分周期产出。

同类型工厂只能全部启动或全部停止。持续和定量计划作用于整个集群。定量目标必须是集群周期产量的整数倍；数量变化后剩余目标不兼容时暂停并要求重新确认。

## 共享仓库

所有商品共用一个仓库。

```text
实物库存 = Σ(可用商品 + 冻结商品)
买单预占 = Σ(当前玩家未完成买单剩余数量)
仓库使用 = 实物库存 + 买单预占
```

扩容规则：初始 1 级、500 容量；每级增加 250；最高 12 级、3250 容量；费用为 `150 × 当前等级²`。

共享仓库完整管理卡固定在“生产”页面，其他页面不提供扩容。

## 商品市场第三版

下单顺序固定为：

```text
方向
限价
数量
1/4 仓｜1/2 仓｜全仓
订单总额
提交订单
```

买入快捷数量按 `min(仓库剩余容量, floor(可用资金 / 限价))` 计算。卖出快捷数量按当前商品可用库存计算，冻结商品不能卖出。

订单簿固定为单列：卖盘最优 5 笔、中性分隔线、买盘最优 5 笔。每行只显示方向、价格和剩余数量，不显示玩家身份，不做价格档位聚合。桌面订单簿卡片拉伸到所在网格行高度，移动端按内容自然增长。

订单管理已合入市场，玩家可以在当前页直接撤单。

## 工厂数量市场

工厂挂牌按“工厂类型 + 数量 + 单座价格”保存。玩家可以一次挂牌多座，也可以购买挂牌中的部分数量。

服务器在一个事务中更新买卖双方资金、双方工厂数量、挂牌剩余数量，以及买方运行中集群的下一周期待加入数量。

存在挂牌数量时，该类型集群仍可启动未挂牌数量。挂牌工厂不参与生产；运行中撤销挂牌的数量从下一周期加入。

## 页面结构

```text
概览：工作、行情、工厂集群摘要、财富、本地成交
市场：快捷仓位下单、单列 5+5 订单簿、撤单、工厂数量市场
生产：共享仓库、建设任务、每种工厂一张统一集群卡
资金：资金、商品和工厂数量资产、本地变化
排行：服务器总资产排行榜
设置：玩家资料、客户端偏好、登录会话、重置
```

正式导航固定为：

```text
概览｜市场｜生产｜资金｜排行｜设置
```

## 服务器权威边界

服务器保存资金、商品库存、仓库、工厂类型集群、统一周期、统一计划、施工任务、商品订单、工厂数量挂牌、市场、统计、排行榜和幂等响应。

SQLite 世界状态不保存：

```text
player.facilities
factory instance id
internalGoods
internalCapacity
trades
ledger
assetEvents
```

## 本地活动日志

日志使用：

```text
economy.local-activity.v3.<userId>
```

日志按账号保存在当前浏览器，不上传服务器，不参与资产、仓库、工厂数量、生产、订单或排名计算。

市场页本地成交记录不设置内部固定高度、分页或垂直滚动区，面板随当前记录数量自然增长；本地存储仍最多保留最近 240 条成交记录。

## 权威成交价格

每次成交使用撮合前订单簿中的对手方挂单价格。服务器在玩家订单 `fills` 中逐笔保存真实成交价格、数量和总额；同一订单扫过多个价格时分别记录。浏览器成交记录不得使用市场最后成交价反推订单成交价。
