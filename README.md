# Economy

Economy 是一款网页端多人在线经济模拟、产业经营、统一资产交易与藏品／商品／工厂单项或捆绑竞价拍卖游戏。

- 项目：`RIVERS0FT/Economy`
- 生产网页：`https://game.riversoft.top/economy/`
- 管理员页面：`https://game.riversoft.top/economy/admin`
- 客户端状态版本：`15`
- 世界状态版本：`13`
- 市场需求模型版本：`1`

## 当前核心循环

```text
工作获得基础资金
→ 建设同类型工厂数量
→ 选择工厂生产配方并开启持续生产
→ 产成品直接进入共享仓库
→ 在统一资产订单簿交易商品和工厂
→ 市场需求形成最终品订单并沿生产链派生上游流动性
→ 通过不可拆分资产包拍卖交易藏品、商品和工厂
→ 邀请新玩家获得独立宝石
→ 在商店按固定汇率兑换普通货币
→ 调整产业链、库存、资金与收藏
→ 提升服务器计算的总资产和排行榜名次
```

## 当前关键规则

- 当前目录共 22 种商品和 15 种工厂类型；商品 ID、工厂 ID 和配方 ID 已进入存档后不得重命名或复用。
- 产业链包括 `小麦 → 面粉 → 食品`、`棉花／毛 → 纺织品 → 服装`、`木材 → 木板 → 家具`、`铁矿石 → 钢材 → 机械` 和 `原油 → 塑料 + 铜矿石 → 铜材 → 电子产品`。
- 商品订单只允许玩家订单和市场需求订单。市场需求只创建买单，不创建系统卖单、企业采购或固定价格流动性。
- 市场需求固定分为 `food` 食品市场和 `household` 家庭消费市场；基础预算分别为 1,000／5 分钟和 900／5 分钟。
- 每组预算按近 7 天经济活跃玩家平方根增长，最多 6 倍；玩家库存数量和库存价值不得扩大市场需求总预算，也不得直接提高某商品的需求份额。
- 每组周期预算中 85% 用于最终消费的直接需求，15% 用于沿正式配方反向推导的派生流动性；两者共享同一总预算，未使用预算不结转。
- 直接需求采用需求类别和替代关系。小麦与水稻、肉与蛋与奶会按相对有效价格、可成交深度和替代弹性平滑调整份额；一个商品涨价不会使替代品份额在单周期内瞬间跳变。
- 同一输出商品存在多条正式配方时形成生产路径替代，例如棉花和毛按等价投入成本竞争纺织品派生需求；每条有效路径保留最低份额并平滑变化。
- 同一正式配方包含多个输入时形成互补关系，例如电子产品同时需要塑料和铜材；派生需求通过互补门控优先补足瓶颈输入，不能继续无限推高已经充足的互补品。
- 最终产品上涨通过下游净回值向上游逐生产边滞后传导；每条边至少延迟一个 5 分钟周期，使用最近三个周期的 60%／30%／10% 分布式滞后信号。
- 上游输入涨价通过成本锚点滞后影响下游。所有价格传播只读取上一周期快照，不得在同一周期跨越多条生产边。
- 有效价格由订单簿深度报价、参考价和过去 30 分钟真实成交 VWAP 组合；未成交挂单、取消订单、自成交和拍卖成交不得作为真实成交信号。
- 市场预算按 65% 旧值与 35% 目标值平滑，单周期涨跌不超过 20%；价格参考值按商品层级分别限幅，并限制在基础价 50%～300%。
- 新世界和旧人口需求模型升级通过 `marketDemand.modelVersion = 1` 一次性重建市场需求状态；世界版本继续保持 13。迁移保留玩家资金、库存、工厂、玩家订单、冻结资产和成交历史。
- 服务器持久化层暂时继续使用 `ownerType: 'population'` 作为系统买单兼容标识，业务语义、显示名称和新状态均为市场需求；不得把该兼容标识重新解释为旧人口模型。
- 商品初始参考价、生产数量、周期秒数和周期成本全部保持整数；当前平衡目标为基础原料 1／分钟、中间产品 3／分钟、最终产品 6／分钟。
- 农场的小麦、水稻和棉花配方保持同参数；矿场同时支持铁矿石和铜矿石；畜牧场可切换肉、蛋、奶和毛。
- `steelworks` ID 永久保留，正式名称为冶炼厂，同时支持铁矿石炼钢和铜矿石炼铜；不新增铜冶炼厂。
- 纺织厂可使用棉花或毛；电子工厂唯一正式配方必须同时消耗塑料和铜材。
- 正式配方统一使用 `inputs[]`；同一周期所有输入必须同时满足并原子扣除，任一输入不足时不得扣除其他原料。
- 工厂按 `facilityTypeId + count` 保存，不存在单座工厂实例、实例 ID、内部仓库或领取产成品。
- 所有工厂集群统一使用服务器正式配方；同类型工厂共用运行意图、三种顶层状态、进度、当前配方和周期结算，运行中切换配方在下一完整周期生效且不重置当前进度。
- 商品和工厂共用统一限价订单结构，支持价格优先、同价时间优先、部分成交、撤单和禁止自成交。
- 玩家通过统一订单簿卖出商品或工厂时，按单张卖单累计成交总额收取 1% 手续费，向上取整且最低为 1；卖给市场需求同样收费，拍卖不收费。
- 工厂运行中可以提交卖单；冻结数量立即退出当前参与数量，并停止产生商品和运营成本。
- 商品和工厂估值均使用最高非本人有效买入价；没有非本人买单时估值为 0。
- 工作冷却固定为 10 秒，不随连续工作次数增加。
- 共享仓库允许无限扩容；容量、扩容费用和冻结预占均由服务器权威计算。
- 服务器只保存权威经济状态；玩家活动和成交展示日志保存在当前浏览器。
- 游戏状态使用世界修订号轮询；客户端默认每 5 秒轮询，可选 3／5／10 秒；空闲读取不得写回未变化世界。
- 普通玩家订单簿和状态接口不得暴露订单所有者、市场需求组、需求角色、对手身份或对手订单 ID。
- 宝石独立于普通货币，不参与市场、生产、总资产或排行榜；邀请、兑换、封禁和管理员规则继续以对应权威文档为准。
- 藏品是服务器记录归属的唯一资产实例；藏品、商品和工厂可以组成不可拆分资产包竞价，拍卖成交不得进入订单簿行情或估值。

## 页面

```text
概览｜市场｜生产｜资产｜藏品｜拍卖｜排行｜商店｜设置
```

- 概览：工作、行情、生产与财富摘要。
- 市场：商品和工厂连续资产目录、行情、自然高度订单簿、下单与撤单。
- 生产：共享仓库、建设卡和工厂类型集群。
- 资产：现金、商品和工厂资产汇总，以及当前浏览器中的资产变化记录。
- 藏品：当前账号持有的艺术藏品、图片、来源与归属。
- 拍卖：组合藏品、商品和工厂，整包竞价和自动结算。
- 排行：服务器财富、增长、生产和交易排行榜。
- 商店：按服务器固定汇率将宝石单向兑换为普通货币。
- 设置：玩家资料、偏好、宝石、邀请、礼品兑换和退出。

```bash

npm run dev
npm run build
```
