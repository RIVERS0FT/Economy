# Economy

Economy 是一款多人在线经济模拟与实时交易游戏。

当前主产品方向：**网页端多人联机版本**。

玩家通过点击工作直接获得货币，使用货币生产商品、升级设施并参与共享市场交易。商品价格主要由玩家挂单、人口消费和市场供需关系形成。

## 当前开发方向

- 当前只实现网页端
- 后端运行在 2 核 2G 服务器上
- 游戏中不设置劳动力资源
- 点击工作直接获得货币
- 点击收入受服务器确认的冷却限制
- 连续工作会逐步延长冷却
- 食品包通过支付生产费用和等待生产时间获得
- 玩家资产、库存、订单和成交由服务器统一保存与判定
- 复杂图表、预测和方案分析尽量由玩家电脑计算
- 所有正式玩家进入同一个共享市场
- 虚拟城市人口通过消费需求提供基础买单
- MVP 只实现食品包和限价单
- 撮合采用价格优先、时间优先
- 手机端、小程序端和电脑端暂不实现，但作为后续互通客户端预留

## 核心玩法

```text
点击工作获得货币
-> 支付生产费用生产食品包
-> 商品进入仓库
-> 观察共享市场与人口需求
-> 提交买单或卖单
-> 与其他玩家或人口消费需求成交
-> 获得商品或销售收入
-> 扩大生产和仓储
-> 提升企业等级与市场排名
```

## 点击工作

MVP 第一轮测试建议：

- 每次有效点击获得 1 货币
- 基础冷却 3 秒
- 连续工作会将冷却逐步提高到 5 秒、8 秒和 12 秒
- 停止工作 5 分钟后恢复基础冷却
- 刷新页面、多标签页或修改客户端时间不能重置冷却
- 不提供离线自动点击收入

所有数值都需要通过试玩和经济数据调整。

## 货币发行与回收

增加全服货币的来源：

- 点击工作
- 城市人口消费
- 有限任务或活动奖励

玩家之间交易只转移现有货币，不增加货币总量。

货币回收方式：

- 商品生产费用
- 生产设施升级
- 仓库升级
- 企业扩建
- 后续市场服务费用

测试期间需要分别统计点击发行、人口发行和系统回收量。

## 服务器与客户端分工

服务器负责正式经济结果：

- 点击冷却与点击收入
- 资金和库存
- 生产费用与生产任务
- 订单创建与撤销
- 订单撮合和成交结算
- 人口周期预算与正式人口买单
- 资产流水、奖励和排行榜正式结果

玩家电脑负责非权威复杂计算：

- 市场价格和成交量图表
- 订单簿深度展示
- 下单和生产预览
- 利润与经营方案分析
- 人口动画和需求曲线
- 页面搜索、筛选和排序

客户端计算只能用于展示和预测，不能直接改变正式资产或市场结果。

第一阶段主要包含：

- 玩家注册或登录
- 玩家与企业资料
- 点击工作与冷却
- 点击收入流水
- 食品包生产费用和生产时间
- 商品库存和仓库
- 单一共享市场
- 虚拟城市人口消费
- 限价买单与卖单
- 资金冻结与库存冻结

- 资产流水
- 网页刷新恢复
- 断线重连状态恢复
- 点击发行、人口发行和货币回收统计
- 基础新手引导

## 2 核 2G 首发约束

- 单一共享市场
- 单一城市人口模型
- 单一商品
- 人口按周期聚合，不逐个模拟居民
- 每名玩家同时最多 10 笔未完成订单
- 排行榜、统计和图表允许延迟刷新
- 首发设计目标为 100 名同时在线玩家
- 50 名同时在线稳定运行是最低压力测试验收线
- 最终正式开放人数以实际压力测试结果为准

服务器资源不足时，应优先关闭或延迟图表、排行榜和非关键统计。资金、库存、点击收入、订单和成交的一致性不能降级。

## 人口消费需求

共享市场的基础买单由虚拟城市人口的消费需求产生。

人口买单受以下因素限制：

- 城市人口
- 消费周期
- 商品目标需求
- 居民消费预算
- 商品市场价格
- 居民价格承受能力
- 上一周期需求满足率

人口买单进入与玩家相同的订单簿。人口购买的商品成交后退出流通。

人口系统不保证玩家以任意价格盈利，也不存在永久固定、无限数量的保底收购价。

## 后续平台方向

未来计划支持：

- 手机端
- 小程序端
- 网页端
- 电脑端

未来客户端共享：

- 同一游戏账号
- 同一玩家资产
- 同一商品库存
- 同一生产进度
- 同一订单与成交记录
- 同一共享市场
- 同一点击冷却
- 同一排行榜与赛季进度

不同客户端只改变布局、信息密度和输入方式，不改变经济规则、点击收益或撮合优先级。

## 现有单机原型

仓库中的 `miniprogram/` 是此前完成的抖音小程序单机 MVP，用于验证早期基础循环。

该版本作为历史原型保留，不代表当前多人版本规则，也不会作为正式多人市场的资产来源。

## 当前开发决策

- 游戏名称：Economy
- 游戏类型：多人在线经济模拟与实时交易
- 当前客户端：网页端
- 当前后端：2 核 2G 服务器权威状态
- 当前工作规则：点击直接获得货币
- 当前限制规则：服务器冷却与连续工作递增冷却
- 当前生产规则：支付货币并等待生产时间
- 当前市场：单一共享市场
- 当前商品：食品包
- 当前订单类型：限价单
- 当前基础需求：虚拟城市人口消费
- 当前计算原则：客户端负责展示与预测，服务器负责正式经济结果
- 当前阶段：以设计文档指导后续开发
- 当前不实现：劳动力、手机端、小程序端和独立电脑端
- 后续扩展：四端共享账号、资产、点击冷却、订单、市场和赛季数据
