# Economy

Economy 是一款多人在线经济模拟与实时交易游戏。

当前主产品方向：**网页端多人联机版本**。

玩家通过点击工作直接获得货币，再选择建造自己的工厂或在统一市场购买其他玩家的现成工厂。工厂持续生产基础商品，玩家通过商品交易、工厂交易和人口消费需求扩大企业资产。

## 当前核心玩法

```text
点击工作获得货币
-> 自建工厂或购买现成工厂
-> 工厂生产基础商品
-> 商品进入企业仓库
-> 在统一市场交易商品或工厂
-> 与玩家或人口消费需求成交
-> 升级、扩建、交易或收购工厂
-> 扩大企业总资产
```

## 当前规则

- 当前只实现网页端
- 后端运行在 2 核 2G 服务器上
- 不设置劳动力、体力或能量资源
- 点击工作直接获得货币
- 点击收益受服务器冷却和连续工作递增冷却限制
- 玩家不能直接生产商品，必须先拥有工厂
- 玩家可以自建工厂，也可以购买其他玩家的工厂
- 自建工厂需要等待施工完成
- 购买现成工厂成交后立即交割，不需要施工等待
- 工厂按周期生产暂未命名的基础商品
- 工厂运行需要支付运营费用
- 商品与工厂进入同一个共享市场
- 基础商品使用限价买卖订单
- 工厂使用固定价格出售挂牌，数量固定为 1
- 虚拟城市人口为基础商品提供有限买单
- 玩家资产、库存、工厂产权、订单和成交由服务器统一保存与判定
- 图表、收益预测和方案分析尽量由玩家电脑计算
- 手机端、小程序端和独立电脑端作为后续互通扩展

## 点击工作

MVP 第一轮测试建议：

- 每次有效点击获得 1 货币
- 基础冷却 3 秒
- 连续工作后冷却逐步提高至 5 秒、8 秒和 12 秒
- 停止工作 5 分钟后恢复基础冷却
- 刷新、多标签页和修改本地时间不能重置冷却
- 不提供离线点击收入

点击负责启动和兜底，工厂经营与市场交易负责中长期成长。

## 工厂

### 自建工厂

玩家支付系统建造费用后，工厂进入施工状态。施工完成前不能生产、升级或挂牌出售。

第一轮测试建议：

- 基础工厂建造费用：60 货币
- 基础施工时间：5 分钟
- 初始工厂槽位：1
- 同时施工数量：1

### 购买工厂

其他玩家可以将已完工的工厂在统一市场按固定价格挂牌。买家拥有足够资金和空闲槽位时，可以立即购买并完成产权转移，不需要等待建设。

工厂购买不会重置等级、产能、内部容量和累计经营记录。

## 工厂生产

MVP 只开放一种基础工厂和一种暂未命名的基础商品。

第一轮测试建议：

- 单周期时间：30 秒
- 单周期产量：1 个基础商品
- 单周期运营费用：1 货币
- 工厂内部成品容量：20 个

基础商品先进入工厂内部成品区，再由玩家领取到企业仓库。资金不足、内部成品已满、挂牌或升级时，工厂自动暂停生产。

## 统一市场

同一市场包含：

- 基础商品买单
- 基础商品卖单
- 工厂出售挂牌
- 工厂即时购买
- 人口消费买单

商品和工厂共用：

- 市场入口
- 当前订单
- 历史订单
- 成交记录
- 资产流水
- 市场状态
- 搜索、筛选和分页

### 基础商品订单

基础商品可以批量交易。玩家指定数量和限价，按照价格优先、同价时间优先撮合，允许部分成交和撤销未成交部分。

### 工厂订单

每座工厂都是唯一资产，数量固定为 1。卖家按固定价格挂牌，买家按挂牌价即时购买。工厂订单不支持部分成交。

## 人口消费

虚拟城市人口根据人口规模、消费周期、预算、商品价格和需求满足率生成有限基础商品买单。

人口买单进入与玩家相同的基础商品订单簿。人口购买后的商品退出流通。

人口不会购买工厂，也不存在永久固定、无限数量的保底收购价。

## 货币发行与回收

增加全服货币的来源：

- 点击工作
- 人口消费
- 有限任务或活动奖励

玩家间商品和工厂交易只转移已有货币。

货币回收方式：

- 工厂建造费用
- 工厂运营费用
- 工厂升级费用
- 企业扩建和工厂槽位费用
- 仓库升级费用
- 后续市场服务费用

## 2 核 2G 首发约束

- 单一共享市场
- 一种基础商品
- 一种基础工厂
- 一个聚合人口模型
- 工厂按周期结算，不模拟工人与机器个体
- 工厂使用固定价挂牌，不实现竞价和拍卖
- 每名玩家初始一个工厂槽位
- 每名玩家最多 10 笔未完成订单
- 商品盘口、工厂挂牌、历史订单和流水限制展示数量并分页
- 排行榜、图表和统计允许延迟刷新
- 50 名同时在线为最低稳定压测验收线
- 100 名同时在线为首发设计目标
- 正式开放人数以实际压力测试为准

服务器资源不足时，优先降低图表、排行榜和公开统计刷新频率。货币、库存、工厂产权、订单和成交正确性不能降级。

## 服务器与客户端分工

服务器负责：

- 点击收入和冷却
- 资金与库存
- 工厂产权、施工、生产和升级
- 商品订单撮合
- 工厂即时交割
- 人口周期预算与正式买单
- 成交、流水、奖励和排行榜正式结果

玩家电脑负责：

- 点击、施工和生产倒计时展示
- 商品市场图表和盘口深度
- 工厂收益与回本预测
- 自建与收购方案比较
- 人口动画和需求曲线
- 页面搜索、筛选、排序和个人统计

客户端计算只能用于展示和预测，不能改变正式经济结果。

## 多人 MVP 范围

- 玩家注册或登录
- 玩家与企业资料
- 点击工作和冷却
- 点击收入流水
- 自建工厂和施工等待
- 工厂周期生产、运营费用和内部容量
- 玩家工厂固定价挂牌和即时购买
- 基础商品库存和企业仓库
- 单一统一市场
- 基础商品限价买卖订单
- 资金冻结与库存冻结
- 商品部分成交和撤单
- 人口消费基础买单
- 当前订单、历史订单和成交记录
- 资产与产权流水
- 网页刷新和断线恢复
- 点击发行、人口发行和货币回收统计
- 基础新手引导

## 后续平台方向

未来手机端、小程序端、网页端和电脑端共享：

- 同一游戏账号
- 同一资金与库存
- 同一工厂产权与生产进度
- 同一商品与工厂订单
- 同一共享市场
- 同一点击冷却
- 同一排行榜与赛季进度

不同客户端只改变布局、信息密度和输入方式，不改变点击收益、施工时间、经济规则或市场优先级。

## 当前开发决策

- 当前客户端：网页端
- 当前服务器：2 核 2G
- 当前工作规则：点击直接获得货币并受冷却限制
- 当前生产规则：必须由玩家工厂生产基础商品
- 当前工厂获得方式：自建或购买玩家工厂
- 当前建造规则：自建需要等待，购买立即交割
- 当前市场：商品和工厂共用统一市场
- 当前商品交易：限价订单簿
- 当前工厂交易：固定价挂牌
- 当前基础需求：虚拟人口消费
- 当前阶段：以设计文档指导后续开发，不规定具体技术实现
