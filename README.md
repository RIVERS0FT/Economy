# Economy

Economy 是一款多人在线经济模拟与实时交易游戏。

当前主产品方向：**网页端多人联机版本**。

玩家通过工作、生产和经营获得商品，再进入所有玩家共享的市场进行实时买卖。商品价格主要由玩家挂单、成交、人口消费和市场供需关系形成。

## 当前开发方向

- 当前只实现网页端
- 后端运行在 2 核 2G 服务器上
- 玩家资产、库存、订单和成交由服务器统一保存与判定
- 复杂展示、图表、预测和方案分析尽量由玩家电脑计算
- 所有正式玩家进入共享市场
- MVP 先实现食品包一种商品
- MVP 先实现限价买单和限价卖单
- 撮合规则采用价格优先、时间优先
- 虚拟城市人口通过消费需求提供基础买单
- 手机端、小程序端和电脑端暂不实现，但作为后续可互通客户端预留

## 核心玩法

```text
获得劳动力与生产能力
-> 生产商品
-> 观察共享市场与人口需求
-> 提交买单或卖单
-> 与其他玩家或人口消费需求成交
-> 获得商品或资金
-> 扩大生产和仓储
-> 提升企业等级与市场排名
```

## 服务器与客户端分工

服务器负责正式经济结果：

- 资金和库存
- 生产任务
- 订单创建与撤销
- 订单撮合和成交结算
- 人口周期预算与正式人口买单
- 资产流水、奖励和排行榜正式结果

玩家电脑负责非权威复杂计算：

- 市场和价格图表
- 订单簿深度展示
- 下单与生产预览
- 利润和经营方案分析
- 人口动画与需求曲线
- 页面搜索、筛选和排序

客户端计算结果只用于展示、预测和输入辅助，不能直接改变正式资产或市场结果。

## 多人 MVP 范围

第一阶段主要包含：

- 玩家注册或登录
- 玩家与企业基础资料
- 初始资金与劳动力
- 食品包生产
- 商品库存
- 单一共享市场
- 限价买单与卖单
- 资金冻结与库存冻结
- 订单簿与自动撮合
- 部分成交与全部成交
- 撤销订单
- 当前订单、历史订单与成交记录
- 资产流水
- 网页刷新恢复
- 断线重连后的状态恢复
- 固定城市人口
- 周期性食品消费需求
- 受预算和价格承受能力限制的人口基础买单
- 人口购买后商品退出流通
- 基础新手引导

## 2 核 2G 首发约束

- 单一共享市场
- 单一城市人口模型
- 单一商品
- 人口按周期聚合，不逐个模拟居民
- 每名玩家同时最多 10 笔未完成订单
- 排行榜、统计和图表允许延迟刷新
- 首发设计目标为 100 名同时在线玩家
- 50 名同时在线稳定运行是最低压力测试验收线
- 最终正式开放人数以实际压力测试结果为准

服务器资源不足时，应优先关闭或延迟排行榜、图表和非关键统计。资金、库存、订单、成交和资产流水的一致性不能降级。

## 人口消费需求

共享市场的基础买单不由系统无限生成，而由虚拟城市人口的消费需求产生。

人口买单受以下因素限制：

- 城市人口
- 消费周期
- 商品目标需求
- 居民消费预算
- 商品市场价格
- 居民价格承受能力
- 上一周期需求满足率

人口买单进入与玩家相同的订单簿并遵守相同撮合规则。人口购买的商品视为已经消费，成交后退出流通。

人口系统用于提供基础需求，但不保证玩家以任意价格盈利，也不存在永久固定、无限数量的保底收购价。

## 后续平台方向

未来计划支持：

- 手机端
- 小程序端
- 网页端
- 电脑端

未来客户端共享：

- 同一游戏账号
- 同一玩家资产
- 同一商品库存
- 同一生产进度
- 同一订单与成交记录
- 同一共享市场
- 同一排行榜与赛季进度

不同客户端只改变布局、信息密度和输入方式，不改变经济规则或撮合优先级。

## 现有单机原型

仓库中的 `miniprogram/` 是此前完成的抖音小程序单机 MVP，用于验证基础经营循环：

```text
点击工作 -> 获得劳动力 -> 生产食品包 -> 固定价格出售 -> 升级设施
```

该版本作为历史原型保留，不代表当前主产品架构，也不会作为正式多人市场的资产来源。

## 当前开发决策

- 游戏名称：Economy
- 游戏类型：多人在线经济模拟与实时交易
- 当前客户端：网页端
- 当前后端：2 核 2G 服务器权威状态
- 当前市场：共享市场
- 当前商品：食品包
- 当前订单类型：限价单
- 当前基础需求：虚拟城市人口消费
- 当前计算原则：客户端负责展示与预测，服务器负责正式经济结果
- 当前阶段：以设计文档指导后续开发
- 当前不实现：手机端、小程序端和电脑端
- 后续扩展：四端共享账号、资产、订单、市场和赛季数据
