# Economy

Economy 是一款多人在线经济模拟与交易游戏。

- 项目：`RIVERS0FT/Economy`
- 生产网页：`https://game.riversoft.top/economy/`
- 当前方向：网页端多人联机金融与产业经营

页面内容与模块归属以 `docs/PAGE_CONTENT_AND_NAVIGATION_DESIGN.md` 为准。

## 核心循环

```text
工作获得基础资金
→ 在生产页扩容共享仓库
→ 建设或收购工厂
→ 设置持续或定量生产计划
→ 手动启动工厂
→ 消耗资金与原料生产商品
→ 产成品直接进入共享仓库
→ 在商品订单簿交易
→ 在市场直接撤单和交易工厂
→ 在资金页查看资产结果与本地变化
→ 提升总资产与排名
```

## 商品与工厂

首批商品：粮食、铁矿石、面粉、钢材、食品、机械。

| 工厂 | 配方 |
|---|---|
| 农场 | 无原料 → 2 粮食 |
| 矿场 | 无原料 → 2 铁矿石 |
| 面粉厂 | 2 粮食 → 1 面粉 |
| 钢铁厂 | 3 铁矿石 → 1 钢材 |
| 食品厂 | 2 面粉 → 3 食品 |
| 机械厂 | 2 钢材 → 1 机械 |

玩家持有工厂数量不设上限，不存在设施槽位。同一玩家同时只能施工一座工厂。

## 工厂生产

- 工厂施工完成后保持停止状态，必须手动启动。
- 玩家可以手动停止运行中的工厂。
- 每座工厂支持持续生产和定量生产。
- 定量计划达到目标后自动停止，不允许超产。
- 阻塞条件解除后不会自动重启。
- 离线前处于运行状态的工厂按服务器时间批量结算。
- **工厂不拥有内部商品或内部容量。**
- **每个完整生产周期完成后，产成品直接进入共享仓库。**
- 不存在“领取产成品”操作、按钮或 API。
- 共享仓库无法容纳下一个周期的净新增商品时，工厂停止并显示“共享仓库已满”。

加工工厂按净库存变化检查空间。例如面粉厂消耗 2 粮食并产出 1 面粉，即使仓库原本满载，也能利用原料消耗释放的空间完成生产。

## 共享仓库与扩容

```text
实物库存 = 各商品可用数量 + 各商品冻结数量
买单预占 = 当前玩家未完成买单剩余数量
仓库使用 = 实物库存 + 买单预占
```

等级规则：

- 初始 1 级、500 容量；
- 每级增加 250；
- 最高 12 级、3250 容量；
- 扩容费用为 `150 × 当前等级²`；
- 费用由服务器事务扣除并计入系统回收；
- 旧玩家容量迁移不会缩减已有容量。

共享仓库完整管理卡固定在“生产”页面，位于建设新工厂和工厂列表之前。共享仓库完整管理入口只存在于此页面。

工厂产出、商品持仓和未完成买单共同使用同一仓库容量。多座工厂不能重复使用同一剩余空间。

## 市场与订单

商品使用限价订单簿：价格优先、同价时间优先、允许部分成交、未完成订单可撤销。

- 买单冻结资金并预占仓库容量；
- 卖单冻结对应商品；
- 成交由服务器原子更新双方资产和订单状态；
- 订单管理和撤单都位于市场页；
- 工厂使用固定价格挂牌并直接转移原始产权；
- 工厂挂牌不包含内部商品或内部容量。

## 资金与资产

资金页展示：

- 可用与冻结资金；
- 当前总资产；
- 商品资产和工厂资产；
- 资产配置；
- 各商品库存与估值；
- 货币发行与系统回收；
- 当前浏览器本地资产变化。

资金页不提供仓库扩容、建厂、生产控制或下单。

```text
总资产 =
可用资金
+ 冻结资金
+ Σ(各商品可用与冻结数量 × 参考价)
+ Σ(工厂系统估值)
```

工厂产成品已经计入共享仓库商品资产，不再重复计入工厂价值。

## 页面职责

正式导航：

```text
概览｜市场｜生产｜资金｜排行｜设置
```

### 概览

基础工作、市场摘要、生产摘要、财富摘要、最近本地成交和快捷入口。概览显示共享仓库剩余空间，不显示待领取产成品。

### 市场

商品行情、下单、订单簿、撤单、本地成交和工厂挂牌交易。不得提供生产控制或仓库扩容。

### 生产

共享仓库完整管理、建设工厂、生产计划、手动启停、产成品自动入仓说明和工厂挂牌管理。共享仓库完整管理入口只存在于此页面。

### 资金

资产配置、商品库存、估值、货币流和本地资产变化。资金页不提供仓库扩容。

### 排行

服务器总资产排行榜和当前玩家排名。

### 设置

玩家资料、客户端偏好、登录会话、退出登录和重置经济状态。设置页不显示账号与产业摘要、仓库使用摘要、仓库等级、扩容费用或扩容按钮。

## 本地活动日志

成交、资金、商品、仓库、工厂和生产变化只保存在当前浏览器：

- 按账号隔离；
- 不上传服务器；
- 不参与正式计算；
- 可以在资金页清除；
- 本地存储失败不会阻断游戏。

新生产日志显示“产出入仓”，依据工厂完成量和商品库存差异生成。旧浏览器保存的领取历史只读保留。

服务器 SQLite 不保存：

```text
trades
ledger
assetEvents
```

## 服务器权威边界

服务器负责资金、商品库存、仓库、工厂、生产计划、生产结算、订单、挂牌、市场、总资产和排名。

客户端只负责展示、输入、图表和本地活动日志。浏览器状态不能改变正式资产。

工厂直接入仓必须在服务器事务中同时完成运营费、原料消耗、产出入仓和计划进度更新。

## 2 核 2G 约束

- 工厂按批量周期结算；
- 每座工厂单次处理周期有安全上限；
- 仓库占用按当前库存和订单聚合；
- 用户日志不写入服务器数据库；
- 每名玩家最多 10 笔未完成订单。
