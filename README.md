# Economy

Economy 是一款多人在线经济模拟与实时交易游戏。

当前主产品方向：**网页端多人联机版本**。

玩家通过工作、生产和经营获得商品，再进入所有玩家共享的市场进行实时买卖。商品价格主要由玩家挂单、成交和供需关系形成。

## 当前开发方向

- 当前只实现网页端
- 后端运行在服务器上
- 玩家资产、库存、订单和成交由服务器统一保存与判定
- 所有正式玩家进入共享市场
- MVP 先实现食品包一种商品
- MVP 先实现限价买单和限价卖单
- 撮合规则采用价格优先、时间优先
- 手机端、小程序端和电脑端暂不实现，但作为后续可互通客户端预留

完整功能设计见：

```text
docs/WEB_MULTIPLAYER_GAME_DESIGN.md
```

## 核心玩法

```text
获得劳动力与生产能力
-> 生产商品
-> 观察共享市场
-> 提交买单或卖单
-> 与其他玩家成交
-> 获得商品或资金
-> 扩大生产和仓储
-> 提升企业等级与市场排名
```

## 多人 MVP 范围

第一阶段主要包含：

- 玩家注册或登录
- 玩家与企业基础资料
- 初始资金与劳动力
- 食品包生产
- 商品库存
- 单一共享市场
- 限价买单与卖单
- 资金冻结与库存冻结
- 订单簿与自动撮合
- 部分成交与全部成交
- 撤销订单
- 当前订单、历史订单与成交记录
- 资产流水
- 网页刷新恢复
- 断线重连后的状态恢复
- 有限系统基础流动性
- 基础新手引导

## 后续平台方向

未来计划支持：

- 手机端
- 小程序端
- 网页端
- 电脑端

未来客户端共享：

- 同一游戏账号
- 同一玩家资产
- 同一商品库存
- 同一生产进度
- 同一订单与成交记录
- 同一共享市场
- 同一排行榜与赛季进度

不同客户端只改变布局、信息密度和输入方式，不改变经济规则或撮合优先级。

## 现有单机原型

仓库中的 `miniprogram/` 是此前完成的抖音小程序单机 MVP，用于验证基础经营循环：

```text
点击工作 -> 获得劳动力 -> 生产食品包 -> 固定价格出售 -> 升级设施
```

该版本作为历史原型保留，不代表当前主产品架构，也不会作为正式多人市场的资产来源。
